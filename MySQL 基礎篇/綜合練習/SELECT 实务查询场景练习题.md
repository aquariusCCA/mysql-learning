# SELECT 实务查询场景练习题

这份题单不再限制使用 `資料/atguigudb.sql`，而是改用一组更接近业务场景的练习数据。目标不是继续背语法，而是把你已经学过的 `SELECT` 相关能力，放进更像真实系统的数据结构里练熟。

这份练习题默认覆盖你目前已经学过的范围：

- 基础 `SELECT`
- `WHERE`
- 运算符
- `ORDER BY`
- `LIMIT`
- 多表查询
- 单行函数
- 聚合函数
- `GROUP BY`
- `HAVING`
- 子查询

## 使用说明

- 练习数据请使用 [资料脚本](../../資料/select_practice_scenarios.sql)。
- 导入后，默认使用数据库 `select_practice_db`。
- 除题目特别说明外，优先使用你目前已经学过的写法完成，不需要刻意使用 `CTE`、视窗函数或更后面的语法。
- 为了让“最近 30 天”“最近 90 天”这类题目结果稳定，这份题单统一约定：

> 请把 `2025-04-15` 当作“当前日期”。

## 练习用表

- `app_users(user_id, name, city, register_date, status)`
- `categories(category_id, category_name)`
- `products(product_id, product_name, category_id, price, stock, is_active)`
- `customer_orders(order_id, user_id, order_date, status, total_amount)`
- `order_items(order_item_id, order_id, product_id, quantity, unit_price)`
- `payments(payment_id, order_id, pay_time, amount, pay_status)`
- `support_tickets(ticket_id, user_id, created_at, closed_at, status, priority)`

## 基础题

### 1. 活跃用户查询

查询所有 `status = 'active'` 的用户姓名、城市与注册日期，并按注册日期从新到旧排序。

### 2. 缺货但仍上架的商品

查询 `stock = 0` 且 `is_active = 1` 的商品名称、价格与分类编号。

### 3. 最近 10 笔已付款订单

查询最近 `10` 笔 `status = 'paid'` 的订单编号、用户编号、订单时间与总金额。

### 4. 高价商品

查询价格大于 `1000` 的商品名称与价格，并按价格从高到低排序。

### 5. 商品名模糊匹配

查询名称中包含 `Pro` 的商品名称与价格。

### 6. 指定城市的活跃用户

查询 `city` 在 `Taipei`、`Taichung`、`Kaohsiung` 的活跃用户。

### 7. 每个分类的商品数

查询每个分类下有多少商品，显示分类编号与商品数。

### 8. 指定金额区间的订单

查询订单总金额在 `500` 到 `2000` 之间的订单编号与用户编号。

## 进阶题

### 9. 从未下单的用户

查询从未下过订单的用户编号与姓名。

### 10. 下单次数不少于 3 次的用户

查询至少下过 `3` 笔已付款订单的用户编号、姓名与订单数。

### 11. 每位用户最近一笔订单

查询每位用户最近一笔订单的订单编号、用户编号、订单日期与总金额。

### 12. 从未卖出过的商品

查询从未被卖出过的商品编号与商品名称。

### 13. 平均价格较高的分类

查询每个分类的平均商品价格，只保留平均价格大于 `800` 的分类。

### 14. 付款金额不足的订单

查询付款成功金额总和小于订单总金额的订单编号与差额。

提示：只统计 `pay_status = 'success'` 的付款记录。

### 15. 买过 Laptop 商品的用户

查询买过 `Laptop` 分类任一商品的用户编号与姓名。

### 16. 同时买过两类商品的用户

查询同时买过 `Laptop` 和 `Accessory` 两种分类商品的用户编号与姓名。

### 17. 每位用户金额最高的一笔订单

查询每位用户金额最高的一笔订单编号、用户编号与订单总金额。

### 18. 订单内高于平均购买数量的明细

查询订单商品明细中，购买数量大于该订单平均购买数量的记录。

要求至少显示：

- `order_id`
- `product_id`
- `quantity`

### 19. 平均订单金额最低的用户

查询平均订单金额最低的用户编号。

要求：

- 至少写出一版子查询解法。
- 如果你愿意，可以再补一版使用 `ALL` 的写法。

### 20. `NOT IN` 空值问题

阅读下面 SQL，说明为什么它可能无法得到你直觉以为的结果：

```sql
SELECT name
FROM app_users
WHERE user_id NOT IN (
    SELECT user_id
    FROM support_tickets
    WHERE status = 'open'
);
```

请重点解释：如果子查询结果里出现 `NULL`，为什么会影响判断。

## 挑战题

### 21. 高于自己历史平均订单金额的订单

查询订单金额高于“自己历史平均订单金额”的订单编号、用户编号与订单总金额。

### 22. 没有提交过工单的活跃用户

查询活跃用户中，从未提交过客服工单的用户编号与姓名。

### 23. 每个分类中价格最高的商品

查询每个分类中价格最高的商品名称、分类编号与价格。

### 24. 订单数高于全站平均订单数的用户

查询下单用户中，订单数高于全站平均订单数的用户编号、姓名与订单数。

### 25. 连续下单间隔不超过 7 天的用户

查询存在至少一组“前后两笔订单间隔不超过 `7` 天”的用户编号与姓名。

提示：

- 不一定要取出所有订单。
- 重点是判断同一用户是否存在这种相邻订单对。

### 26. 有订单且有未关闭工单的用户

查询既有订单、又至少有一张未关闭工单的用户编号与姓名。

### 27. 没有任何上架商品的分类

查询没有任何 `is_active = 1` 商品的分类编号与分类名称。

### 28. 买齐某分类全部上架商品的用户

查询买过 `Laptop` 分类全部上架商品的用户编号与姓名。

提示：

- 这里的“全部”是指该分类下所有 `is_active = 1` 的商品。

## 综合题

### 29. 购买总件数最多的前 5 张已付款订单

查询已付款订单中，购买总件数最多的前 `5` 张订单。

至少显示：

- `order_id`
- `user_id`
- `total_quantity`

### 30. 注册后 30 天内从未下单的用户

查询注册后 `30` 天内从未下过订单的用户编号、姓名与注册日期。

### 31. 高价值用户名单

做一份“高价值用户名单”：

显示用户编号、姓名、已付款订单数、已付款总金额、最近一次下单时间。

要求：

- 只保留活跃用户；
- 只统计 `status = 'paid'` 的订单；
- 已付款总金额大于 `10000`。

### 32. 异常订单名单

做一份“异常订单名单”，找出满足下列任一条件的订单：

- 订单状态是 `paid`，但付款成功金额总和不等于订单总金额；
- 订单状态是 `paid`，但没有任何订单明细；
- 订单状态是 `paid`，但对应用户不是活跃状态。

至少显示：

- `order_id`
- `user_id`
- `status`
- `total_amount`

### 33. 滞销商品名单

做一份“滞销商品名单”：

查询仍在上架、库存大于 `20`，且在最近 `90` 天内没有被卖出过的商品编号、商品名称与库存。

### 34. 客服风险名单

做一份“客服风险名单”：

查询未关闭工单数不少于 `2`，且最近 `30` 天内至少下过 `1` 笔订单的用户编号与姓名。
