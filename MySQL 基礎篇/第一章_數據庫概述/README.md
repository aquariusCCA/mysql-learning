# 第一章_數據庫概述

這一章主要建立資料庫學習的基礎觀念，包含為什麼要使用資料庫、資料庫與 DBMS 的差別、MySQL 的定位，以及關係型資料庫的基本設計思路。

## 建議閱讀順序

1. 先理解為什麼應用需要資料庫，而不是只靠記憶體或檔案保存資料。
2. 再區分 `DB`、`DBMS`、`SQL` 這三個最核心的基礎名詞。
3. 接著認識 MySQL 本身，以及它在關係型資料庫中的位置。
4. 最後再進入 RDBMS 與表關係設計，替後面的建表與查詢打底。
