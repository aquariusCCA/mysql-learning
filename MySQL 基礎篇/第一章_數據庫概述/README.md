# 第一章_數據庫概述

这一章主要建立数据库学习的起点：先理解为什么要用数据库，再分清 `数据库`、`DBMS`、`SQL` 的角色，接着认识 MySQL、关系型数据库与非关系型数据库的差异，最后进入关系型数据库的基本设计规则。

## 本章在整体学习地图中的位置

- 这一章解决的是“数据库是什么、为什么要学、几个基础名词怎么区分”。
- 如果这章没有读稳，后面进入 MySQL 安装、SQL 语句和建表时，很容易只记操作，不理解背后的结构。
- 读完本章后，通常会接着进入 [第二章_MySQL環境搭建](../第二章_MySQL環境搭建/README.md)，开始准备实际操作环境。

## 本章小节

- [1 为什么要使用数据库](./1%20为什么要使用数据库.md)：从“数据为什么不能只放在内存里”切入，建立持久化与数据库价值的基础认识。
- [2 数据库与数据库管理系统](./2%20数据库与数据库管理系统.md)：区分 `DB`、`DBMS`、`SQL` 三个最容易混淆的基础概念。
- [3 MySQL 介绍](./3%20MySQL%20介绍.md)：认识 MySQL 是什么、为什么常被用来做数据库入门，以及它在常见数据库中的定位。
- [4 RDBMS 与非 RDBMS](./4%20RDBMS%20与非%20RDBMS.md)：比较关系型数据库与 NoSQL 的设计思路、适用场景与取舍。
- [5 关系型数据库设计规则](./5%20关系型数据库设计规则.md)：建立表、记录、字段以及一对一、一对多、多对多、自我引用等表关系的设计基础。

- 需要回顾表关系和外键设计方向：看 [5 关系型数据库设计规则](./5%20关系型数据库设计规则.md)
