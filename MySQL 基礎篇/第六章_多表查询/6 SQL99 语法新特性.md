## 本节导读

这一节补充的是 SQL99 在普通 `JOIN ... ON` 之外提供的两种简化写法：`NATURAL JOIN` 和 `USING`。它们都和“同名字段上的等值连接”有关，但控制粒度并不一样。

第一次阅读时，建议先看 `快速回查表`，先分清 `NATURAL JOIN` 和 `USING` 的区别，再看例子。复习时如果你只想确认“自动匹配全部同名字段”还是“只指定某一个同名字段”，直接回到这篇即可。

## 你会在这篇学到什么

- SQL99 提供了 `NATURAL JOIN` 和 `USING` 这类更简化的连接写法。
- `NATURAL JOIN` 会基于两张表中所有同名字段自动做等值连接。
- `USING` 允许你明确指定某个同名字段作为等值连接依据。
- 这两种写法本质上都属于等值连接的简化形式。
- 相比 `NATURAL JOIN`，`USING` 对连接字段的控制更明确。

## 6.1 自然连接

SQL99 在 SQL92 的基础上提供了一些特殊语法，其中 `NATURAL JOIN` 用来表示自然连接。

可以把自然连接理解为：数据库会自动找出两张连接表中所有同名字段，并基于这些同名字段进行等值连接。

也就是说，`NATURAL JOIN` 的核心特点是：

- 不需要手动写 `ON` 条件；
- 会自动使用所有同名字段参与连接；
- 本质上仍然是等值连接。

### SQL92 写法对照

在普通写法中，你需要自己把连接条件写出来，例如：

```sql
SELECT
    employee_id,
    last_name,
    department_name
FROM
    employees e
JOIN departments d
ON
    e.department_id = d.department_id
    AND e.manager_id = d.manager_id;
```

在 SQL99 中，可以写成：

```sql
SELECT
    employee_id,
    last_name,
    department_name
FROM
    employees e
NATURAL JOIN departments d;
```

这表示数据库会自动寻找 `employees` 和 `departments` 中所有同名字段，并对这些字段做等值连接。

### 自然连接的理解重点

- `NATURAL JOIN` 强调“自动”。
- 自动连接的依据不是任意字段，而是两张表里名字相同的字段。
- 如果同名字段不止一个，那么这些字段都会参与连接。

## 6.2 USING 连接

当我们进行连接时，SQL99 还支持使用 `USING` 指定某个同名字段进行等值连接。

它只能和 `JOIN` 一起使用，适合这样的场景：

- 两张表确实存在同名字段；
- 你想简化 `ON a.id = b.id` 这类等值连接；
- 但你又不想像 `NATURAL JOIN` 那样把所有同名字段都自动纳入连接条件。

例如：

```sql
SELECT
    employee_id,
    last_name,
    department_name
FROM
    employees e
JOIN departments d
USING (department_id);
```

这里的意思是：明确指定使用 `department_id` 这个同名字段来完成等值连接。

### 与普通等值连接的对应关系

`JOIN ... USING (department_id)` 与下面这条 SQL 的查询结果相同：

```sql
SELECT
    employee_id,
    last_name,
    department_name
FROM
    employees e,
    departments d
WHERE
    e.department_id = d.department_id;
```

因此，`USING` 可以看成是对“同名字段等值连接”的一种简化写法。

## USING 与 NATURAL JOIN 的区别

两者都能简化等值连接，但差别在于“连接字段由谁决定”：

- `NATURAL JOIN`：自动使用所有同名字段。
- `USING (字段名)`：由你明确指定某个同名字段。

所以从可控性来看：

- `JOIN ... ON ...` 最明确；
- `USING` 次之；
- `NATURAL JOIN` 自动化程度最高，但也最依赖表结构本身。

## 常见混淆点

- `NATURAL JOIN` 和 `USING` 都不是新的连接类型，本质上仍然是在做等值连接。
- `NATURAL JOIN` 不是只连接一个同名字段，而是会自动考虑所有同名字段。
- `USING` 只能用于同名字段，不能拿不同名字的字段直接写进去。
- 如果你希望连接条件完全可见、可控，`JOIN ... ON ...` 仍然是最直观的写法。

## 小结

这一节需要记住：

- SQL99 提供了 `NATURAL JOIN` 和 `USING` 这两种简化连接写法。
- `NATURAL JOIN` 会自动按所有同名字段做等值连接。
- `USING` 会按你指定的同名字段做等值连接。
- 两者本质上都属于等值连接的语法简化。
- 如果更强调清晰和可控，`JOIN ... ON ...` 通常仍然是更稳妥的写法。
