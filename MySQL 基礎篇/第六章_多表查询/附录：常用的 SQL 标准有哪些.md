# 附录：常用的 SQL 标准有哪些

> 所属章节：第六章_多表查询

## 本节导读

这一节是进入多表连接语法前的背景说明，目的是先建立一个前提：SQL 并不是只有一种固定写法，不同标准下的表连接写法会有差异。

第一次阅读时，先理解为什么这里会提到 `SQL92` 和 `SQL99`，再记住课程后续最需要关注的是哪两个标准，以及它们在语法风格上的大致差别。复习时如果只想抓重点，可以直接看 `SQL92`、`SQL99` 和学习建议这几段。

## 你会在这篇学到什么

- SQL 存在多个版本的标准规范。
- 学习多表查询前，需要先知道不同标准下的连接写法会有差异。
- 在常见 SQL 标准中，最值得优先掌握的是 `SQL92` 和 `SQL99`。
- `SQL92` 写法相对简单，但语句通常更长、可读性较差。
- `SQL99` 语法更复杂，但整体可读性更强。

## 快速定位

- `为什么先了解 SQL 标准`：看多表连接为什么会牵涉标准差异。
- `常见 SQL 标准`：看除了 `SQL92`、`SQL99` 之外还有哪些版本。
- `重点掌握的两个标准`：看课程真正关心的是哪两个标准。
- `SQL92 与 SQL99 的区别`：看两者在复杂度与可读性上的对比。
- `学习建议`：看面对众多标准时，实际应该怎么学。

## 为什么先了解 SQL 标准

在正式开始讲连接表的种类时，需要先知道 SQL 存在不同版本的标准规范，因为不同规范下的表连接操作是有区别的。

## 常见 SQL 标准

SQL 有两个主要的标准，分别是 `SQL92` 和 `SQL99`。

这里的 `92` 和 `99` 代表标准提出的时间，也就是 1992 年和 1999 年。当然，除了 `SQL92` 和 `SQL99` 以外，还存在 `SQL-86`、`SQL-89`、`SQL:2003`、`SQL:2008`、`SQL:2011` 和 `SQL:2016` 等其他标准。

## 重点掌握的两个标准

标准虽然很多，但实际学习时最重要的 SQL 标准就是 `SQL92` 和 `SQL99`。

`SQL92` 和 `SQL99` 也是经典的 SQL 标准，分别叫做 `SQL-2` 和 `SQL-3`。后续学习多表查询时，最常需要对照和理解的也主要是这两个标准。

## SQL92 与 SQL99 的区别

一般来说，`SQL92` 的形式更简单，但是写出来的 SQL 语句通常会比较长，可读性相对较差。

相比之下，`SQL99` 的语法更加复杂，但可读性更强。从标准文档的体量也能看出这一点：`SQL92` 的标准大约有 500 页，而 `SQL99` 超过了 1000 页。

## 学习建议

面对这么多 SQL 标准，并不需要试图一次掌握全部内容。实际上，从 `SQL99` 之后，标准内容已经非常庞大，很少有人会完整掌握所有细节。

更实际的做法是先掌握核心标准和常用功能，优先满足日常学习与工作的需求。就这门课的上下文来说，先把 `SQL92` 和 `SQL99` 理解清楚就足够了。

## 补充说明

也正是在 `SQL92` 和 `SQL99` 发布之后，SQL 的影响力越来越大，甚至超越了传统数据库领域。现在 SQL 不仅是数据库领域的主流语言，也广泛出现在信息处理相关场景中，例如图形检索、图像检索和语音检索等领域。
