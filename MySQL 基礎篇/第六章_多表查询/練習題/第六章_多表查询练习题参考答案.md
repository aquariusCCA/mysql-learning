# 第六章_多表查询练习题参考答案

这份参考答案对应 [第六章_多表查询练习题](./第六章_多表查询练习题.md)。

- 练习数据默认使用 `資料/atguigudb.sql`
- 除题目明确要求外，优先使用 `SQL99`
- 字段前统一加表别名，避免歧义

## 基础题

### 1. 关联前提判断

答案：

- `employees` 和 `departments` 可以做多表查询，因为两张表之间存在逻辑关联。
- 最常用的关联字段是：

```text
employees.department_id = departments.department_id
```

- 这个字段在概念上不一定非要已经建成外键，只要业务语义上能正确对应，就能做连接查询。
- 但如果数据库层面真的建了外键，数据一致性会更有保障。

### 2. SQL92 基础等值连接

```sql
SELECT e.last_name, d.department_name
FROM employees e, departments d
WHERE e.department_id = d.department_id;
```

### 3. SQL99 基础等值连接

```sql
SELECT e.last_name, d.department_name
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id;
```

### 4. 三表等值连接

```sql
SELECT e.last_name, d.department_name, l.city
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id
JOIN locations l
  ON d.location_id = l.location_id;
```

说明：这里至少需要两个连接条件，连接链路是 `employees -> departments -> locations`。

### 5. 非等值连接

```sql
SELECT e.last_name, e.salary, jg.grade_level
FROM employees e
JOIN job_grades jg
  ON e.salary BETWEEN jg.lowest_sal AND jg.highest_sal;
```

### 6. 笛卡尔积排错

问题：原 SQL 没有连接条件，会把两张表所有行两两组合，形成笛卡尔积。

正确写法：

```sql
SELECT e.last_name, d.department_name
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id;
```

## 进阶题

### 7. 重复列名与表别名

```sql
SELECT
    e.employee_id,
    e.last_name,
    e.department_id,
    d.department_id,
    d.department_name
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id;
```

### 8. 三表连接加普通筛选条件

```sql
SELECT e.last_name, d.department_name, l.city
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id
JOIN locations l
  ON d.location_id = l.location_id
WHERE l.city = 'Seattle';
```

### 9. 员工、部门、岗位三表内连接

```sql
SELECT e.last_name, d.department_name, j.job_title
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id
JOIN jobs j
  ON e.job_id = j.job_id;
```

### 10. 左外连接

```sql
SELECT e.employee_id, e.last_name, d.department_name
FROM employees e
LEFT JOIN departments d
  ON e.department_id = d.department_id;
```

### 11. 右外连接

```sql
SELECT e.employee_id, e.last_name, d.department_name
FROM employees e
RIGHT JOIN departments d
  ON e.department_id = d.department_id;
```

### 12. 左表独有数据

```sql
SELECT e.employee_id, e.last_name, d.department_name
FROM employees e
LEFT JOIN departments d
  ON e.department_id = d.department_id
WHERE d.department_id IS NULL;
```

### 13. 右表独有数据

```sql
SELECT e.employee_id, e.last_name, d.department_name
FROM employees e
RIGHT JOIN departments d
  ON e.department_id = d.department_id
WHERE e.employee_id IS NULL;
```

### 14. 自连接基础题

```sql
SELECT CONCAT(e.last_name, ' works for ', IFNULL(m.last_name, 'No Manager')) AS relation_info
FROM employees e
LEFT JOIN employees m
  ON e.manager_id = m.employee_id;
```

说明：同一张 `employees` 表在这里被当成“员工表”和“经理表”两次使用。

### 15. 自连接定位经理信息

```sql
SELECT
    e.last_name AS employee_name,
    e.manager_id,
    m.last_name AS manager_name
FROM employees e
LEFT JOIN employees m
  ON e.manager_id = m.employee_id
WHERE e.last_name = 'Chen';
```

## 挑战题

### 16. USING 写法

```sql
SELECT e.last_name, j.job_title
FROM employees e
JOIN jobs j
USING (job_id);
```

### 17. NATURAL JOIN 思考题

答案：

1. `employees` 和 `departments` 的同名字段至少有 `department_id`、`manager_id`。
2. 直接 `NATURAL JOIN` 有风险，因为数据库会对所有同名列自动做连接，而这里真正想连的通常只有 `department_id`。
3. 更稳妥的写法是显式指定连接条件：

```sql
SELECT e.last_name, d.department_name
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id;
```

### 18. UNION 与 UNION ALL

`UNION` 版本：

```sql
SELECT employee_id, last_name
FROM employees
WHERE department_id > 90
UNION
SELECT employee_id, last_name
FROM employees
WHERE email LIKE '%a%';
```

`UNION ALL` 版本：

```sql
SELECT employee_id, last_name
FROM employees
WHERE department_id > 90
UNION ALL
SELECT employee_id, last_name
FROM employees
WHERE email LIKE '%a%';
```

差别：

- `UNION` 会去重
- `UNION ALL` 不去重，重复行会保留，通常也更省一步去重开销

### 19. 模拟满外连接

```sql
SELECT e.employee_id, e.last_name, d.department_name
FROM employees e
LEFT JOIN departments d
  ON e.department_id = d.department_id
UNION
SELECT e.employee_id, e.last_name, d.department_name
FROM employees e
RIGHT JOIN departments d
  ON e.department_id = d.department_id
WHERE e.employee_id IS NULL;
```

说明：第一段拿到“交集 + 左边独有”，第二段补上“右边独有”。

### 20. 只保留左右两边各自独有的数据

```sql
SELECT e.employee_id, e.last_name, d.department_name
FROM employees e
LEFT JOIN departments d
  ON e.department_id = d.department_id
WHERE d.department_id IS NULL
UNION
SELECT e.employee_id, e.last_name, d.department_name
FROM employees e
RIGHT JOIN departments d
  ON e.department_id = d.department_id
WHERE e.employee_id IS NULL;
```

说明：

- 第一段是 `A - A∩B`
- 第二段是 `B - A∩B`
- 两段再 `UNION`，就是“只要两边各自独有，不要交集”

### 21. SQL92 改写为 SQL99

```sql
SELECT e.last_name, d.department_name, l.city
FROM employees e
JOIN departments d
  ON e.department_id = d.department_id
JOIN locations l
  ON d.location_id = l.location_id
WHERE l.city LIKE 'S%';
```

说明：

- 连接条件放在 `ON`
- 普通筛选条件放在 `WHERE`
