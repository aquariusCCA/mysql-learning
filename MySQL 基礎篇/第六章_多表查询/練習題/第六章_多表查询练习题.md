# 第六章_多表查询练习题

这份题单只基于第六章现有笔记内容设计，重点练习：

- 笛卡尔积与连接条件
- 等值连接、非等值连接、自连接
- `SQL92` 与 `SQL99` 两种多表查询写法
- 内连接、左外连接、右外连接
- `UNION`、`UNION ALL`
- `USING`、`NATURAL JOIN`
- MySQL 中模拟满外连接

## 使用说明

- 练习数据默认使用仓库中的 `資料/atguigudb.sql`。
- 除题目特别说明外，优先使用 `SQL99` 的 `JOIN ... ON ...` 写法。
- 涉及多个表时，字段前请加表别名，避免歧义。
- 这一份只放题目，不放答案；建议你先独立完成，再回到第六章各小节对照。

## 本章最常用的关联关系

- `employees.department_id = departments.department_id`
- `departments.location_id = locations.location_id`
- `employees.job_id = jobs.job_id`
- `employees.manager_id = employees.employee_id`
- `employees.salary BETWEEN job_grades.lowest_sal AND job_grades.highest_sal`

## 基础题

### 1. 关联前提判断

说明 `employees` 表和 `departments` 表为什么可以做多表查询，并指出它们最常用的关联字段是什么。再补充说明：这个关联字段在概念上是否一定要已经建成外键。

### 2. SQL92 基础等值连接

使用 `employees` 和 `departments` 两张表，查询员工的 `last_name` 和所属部门的 `department_name`。要求使用 `SQL92` 写法。

### 3. SQL99 基础等值连接

把上一题改写成 `SQL99` 的 `JOIN ... ON ...` 写法。

### 4. 三表等值连接

使用 `employees`、`departments`、`locations` 三张表，查询员工 `last_name`、部门名 `department_name`、城市 `city`。

要求你自己判断这三张表之间至少需要几个连接条件，并把连接链路写完整。

### 5. 非等值连接

使用 `employees` 和 `job_grades` 两张表，查询员工 `last_name`、`salary` 以及对应的工资等级 `grade_level`。

要求使用非等值连接完成。

### 6. 笛卡尔积排错

下面这条 SQL 会产生错误结果，请说明为什么，并改写成正确 SQL：

```sql
SELECT
    e.last_name,
    d.department_name
FROM
    employees e,
    departments d;
```

## 进阶题

### 7. 重复列名与表别名

使用 `employees` 和 `departments`，查询：

- `e.employee_id`
- `e.last_name`
- `e.department_id`
- `d.department_id`
- `d.department_name`

要求：

- 必须使用表别名；
- 结果中保留两张表的 `department_id`，不要省略其中任意一个。

### 8. 三表连接加普通筛选条件

使用 `employees`、`departments`、`locations` 三张表，查询工作地点在 `Seattle` 的员工姓名、部门名称和城市。

要求：

- 连接条件写在 `ON` 中；
- 普通筛选条件写在 `WHERE` 中。

### 9. 员工、部门、岗位三表内连接

使用 `employees`、`departments`、`jobs` 三张表，查询员工 `last_name`、部门名 `department_name`、岗位名 `job_title`。

### 10. 左外连接

使用 `employees` 左连接 `departments`，查询所有员工的 `employee_id`、`last_name`、`department_name`。

要求即使员工没有部门，也要显示出来。

### 11. 右外连接

使用 `employees` 右连接 `departments`，查询 `employee_id`、`last_name`、`department_name`。

要求即使某个部门当前没有员工，也要显示该部门。

### 12. 左表独有数据

基于 `employees` 和 `departments`，查询“没有匹配到部门的员工”。

输出字段：

- `employee_id`
- `last_name`
- `department_name`

### 13. 右表独有数据

基于 `employees` 和 `departments`，查询“当前没有员工的部门”。

输出字段：

- `employee_id`
- `last_name`
- `department_name`

### 14. 自连接基础题

只使用 `employees` 一张表完成自连接，返回如下格式的信息：

`Xxx works for Yyy`

其中 `Xxx` 是员工姓名，`Yyy` 是该员工经理姓名。

### 15. 自连接定位经理信息

查询 `last_name = 'Chen'` 的员工对应的经理信息。

至少返回：

- 员工姓名
- 经理编号
- 经理姓名

## 挑战题

### 16. USING 写法

使用 `employees` 和 `jobs`，查询员工 `last_name` 与岗位名称 `job_title`。

要求：

- 使用 `JOIN ... USING (...)`；
- 不要改写成 `ON`。

### 17. NATURAL JOIN 思考题

`employees` 和 `departments` 中不只一个同名字段。请你思考：

1. 如果直接写 `employees NATURAL JOIN departments`，数据库可能会按哪些同名字段自动连接？
2. 为什么这种写法在这两张表上有风险？
3. 请把它改写成更稳妥、更明确的 SQL。

### 18. UNION 与 UNION ALL

参考第六章示例，查询：

- `department_id > 90` 的员工
- 或者 `email` 中包含字母 `a` 的员工

要求：

- 先写一版 `UNION`；
- 再写一版 `UNION ALL`；
- 最后说明两者在结果语义上的差别。

### 19. 模拟满外连接

基于 `employees` 和 `departments`，在 MySQL 中模拟满外连接，返回：

- 匹配成功的数据
- 没有部门的员工
- 没有员工的部门

要求：

- 不能直接写 `FULL OUTER JOIN`；
- 请使用第六章笔记中的思路完成。

### 20. 只保留左右两边各自独有的数据

基于 `employees` 和 `departments`，查询：

- 左表独有数据
- 加上右表独有数据
- 但不要交集

要求：

- 使用两段 SQL 组合完成；
- 明确体现你对 `A - A∩B`、`B - A∩B`、`A ∪ B - A∩B` 的理解。

### 21. SQL92 改写为 SQL99

把下面这条 SQL92 写法改写成 SQL99 写法，并把连接条件和普通筛选条件分开：

```sql
SELECT
    e.last_name,
    d.department_name,
    l.city
FROM
    employees e,
    departments d,
    locations l
WHERE
    e.department_id = d.department_id
    AND d.location_id = l.location_id
    AND l.city LIKE 'S%';
```
