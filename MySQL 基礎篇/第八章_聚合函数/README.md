# 第八章_聚合函数

这一章开始把“单行数据怎么处理”推进到“多行数据怎么汇总与筛选”。重点不只是记住 `AVG()`、`SUM()`、`COUNT()` 这些函数名称，更重要的是建立一套完整的聚合查询框架：什么是聚合函数、如何分组、如何对分组结果筛选，以及 `SELECT` 查询在执行时到底按什么顺序处理。

目前这一章已经整理出 `4` 个小节，内容从聚合函数基础一路推进到 `GROUP BY`、`HAVING` 和 `SELECT` 的实际执行顺序，适合按顺序学习，也适合在写统计查询时快速回查。

## 本章在整体学习地图中的位置

- 前七章已经建立了数据库概念、MySQL 环境、基础查询、运算符、排序分页、多表查询与单行函数。
- 这一章开始正式进入“多行数据汇总”的阶段，是从基础查询过渡到统计查询的关键节点。
- 如果这一章没有建立好，后面在写报表、分组统计、筛选聚合结果或理解 SQL 执行顺序时会很容易混淆。

- 接着读 [2 GROUP BY](./2%20GROUP%20BY.md) 和 [3 HAVING](./3%20HAVING.md)，把“怎么分组”与“怎么筛选分组结果”连起来理解。
- 如果你常常分不清 SQL 是按什么顺序执行的，再读 [4 SELECT 的执行过程](./4%20SELECT%20的执行过程.md)，把 `WHERE`、`GROUP BY`、`HAVING`、`ORDER BY` 的关系串起来。

## 本章小节

- [1 聚合函数](./1%20聚合函数.md)：先建立聚合函数的概念，整理 `AVG()`、`SUM()`、`MIN()`、`MAX()`、`COUNT()` 的适用场景，并补上 `COUNT(*)` 与 `COUNT(列名)` 的差异。
- [2 GROUP BY](./2%20GROUP%20BY.md)：整理分组查询的基本写法、多列分组与 `WITH ROLLUP`，建立 `SELECT` 字段与分组字段之间的对应关系。
- [3 HAVING](./3%20HAVING.md)：整理如何对分组结果继续筛选，并对比 `WHERE` 与 `HAVING` 的执行阶段、适用条件与性能差异。
- [4 SELECT 的执行过程](./4%20SELECT%20的执行过程.md)：从查询结构、书写顺序、实际执行顺序一路整理到虚拟表演变，帮助理解聚合查询为什么要这样写。
