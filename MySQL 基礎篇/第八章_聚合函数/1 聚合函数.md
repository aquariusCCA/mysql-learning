## 本节导读

本节介绍 SQL 中最常用的一组聚合函数。聚合函数用于对**一组记录**进行汇总计算，输入通常是多行数据，输出通常是一个统计值。常见用途包括：统计记录数、求和、求平均值、找最大值、找最小值，以及统计不重复值的数量。

学习本节时，不只要记住 `AVG()`、`SUM()`、`MIN()`、`MAX()`、`COUNT()` 这些函数名称，更重要的是理解下面几个核心问题：

1. 聚合函数和单行函数有什么区别？
2. 聚合函数如何处理 `NULL`？
3. `COUNT(*)`、`COUNT(1)`、`COUNT(列名)`、`COUNT(DISTINCT 列名)` 有什么差异？
4. 聚合函数为什么不能直接写在 `WHERE` 中？
5. 没有写 `GROUP BY` 时，聚合函数到底是在统计什么？

建议阅读顺序：

## 1.1 聚合函数介绍

### 什么是聚合函数

聚合函数也叫**聚集函数**、**分组函数**。它会对一组数据进行汇总计算，并返回一个统计结果。

简单来说：

- 单行函数：一行输入，通常一行输出。
- 聚合函数：多行输入，通常一个结果输出。

例如：

```sql
SELECT UPPER(first_name)
FROM employees;
```

上面的 `UPPER(first_name)` 是单行函数，它会对每一行的 `first_name` 分别处理。

```sql
SELECT AVG(salary)
FROM employees;
```

上面的 `AVG(salary)` 是聚合函数，它会把多行 `salary` 汇总成一个平均值。

![聚合函数示意图](./images/1554980924940.png)

### 常见聚合函数

| 函数 | 作用 | 常见用途 |
| --- | --- | --- |
| `AVG()` | 求平均值 | 平均薪资、平均成绩、平均订单金额 |
| `SUM()` | 求总和 | 总销售额、总库存、总积分 |
| `MAX()` | 求最大值 | 最高薪资、最新日期、最高分 |
| `MIN()` | 求最小值 | 最低薪资、最早日期、最低分 |
| `COUNT()` | 统计数量 | 总笔数、非空值数量、不重复值数量 |

### 没有 GROUP BY 时会怎样

如果 SQL 中使用了聚合函数，但没有写 `GROUP BY`，那么数据库会把符合条件的所有记录视为**同一组**，然后返回这一组的汇总结果。

```sql
SELECT AVG(salary)
FROM employees;
```

含义是：把 `employees` 表中所有员工视为一组，计算整体平均薪资。

如果搭配 `GROUP BY`，则会按照指定字段分成多组，每一组各自计算一次聚合结果。

```sql
SELECT department_id, AVG(salary)
FROM employees
GROUP BY department_id;
```

含义是：先按照 `department_id` 分组，再分别计算每个部门的平均薪资。

## 1.2 聚合函数的通用语法与使用规则

### 基本语法

常见聚合函数语法如下：

```sql
聚合函数([DISTINCT] 表达式)
```

常见写法：

```sql
SELECT AVG(salary)
FROM employees;

SELECT SUM(salary)
FROM employees;

SELECT MAX(hire_date)
FROM employees;

SELECT MIN(salary)
FROM employees;

SELECT COUNT(*)
FROM employees;

SELECT COUNT(commission_pct)
FROM employees;

SELECT COUNT(DISTINCT department_id)
FROM employees;
```

### 聚合函数可以出现的位置

聚合函数常见于：

1. `SELECT` 子句：显示统计结果。
2. `HAVING` 子句：筛选分组后的统计结果。
3. `ORDER BY` 子句：按照统计结果排序。

例如：

```sql
SELECT department_id, AVG(salary) AS avg_salary
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 8000
ORDER BY avg_salary DESC;
```

### 聚合函数不能直接写在 WHERE 中

`WHERE` 是在分组和聚合之前执行的，它处理的是原始数据行。此时聚合结果还没有产生，所以不能在 `WHERE` 中直接使用聚合函数。

错误示例：

```sql
SELECT department_id, AVG(salary)
FROM employees
WHERE AVG(salary) > 8000
GROUP BY department_id;
```

正确示例：

```sql
SELECT department_id, AVG(salary)
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 8000;
```

### 聚合函数不能在同一层查询中直接嵌套

下面这种写法是不合法或不推荐的：

```sql
SELECT AVG(SUM(salary))
FROM employees
GROUP BY department_id;
```

如果想先计算每个部门的薪资总和，再计算这些部门总和的平均值，应该使用子查询。

```sql
SELECT AVG(dept_total_salary)
FROM (
    SELECT department_id, SUM(salary) AS dept_total_salary
    FROM employees
    GROUP BY department_id
) AS t;
```

理解方式：

1. 内层查询：先算出每个部门的薪资总和。
2. 外层查询：再对这些部门总和求平均值。

## 1.3 聚合函数与 NULL

聚合函数学习时，必须特别注意 `NULL`。

多数聚合函数会忽略 `NULL`，但 `COUNT(*)` 是例外，因为它统计的是**行数**，不是某个字段的值。

| 写法 | 是否忽略 NULL | 没有匹配记录时的结果 | 说明 |
| --- | --- | --- | --- |
| `AVG(expr)` | 忽略 `NULL` | `NULL` | 只计算非 `NULL` 值的平均值 |
| `SUM(expr)` | 忽略 `NULL` | `NULL` | 只累加非 `NULL` 值 |
| `MAX(expr)` | 忽略 `NULL` | `NULL` | 从非 `NULL` 值中取最大值 |
| `MIN(expr)` | 忽略 `NULL` | `NULL` | 从非 `NULL` 值中取最小值 |
| `COUNT(*)` | 不看字段值 | `0` | 统计符合条件的行数 |
| `COUNT(expr)` | 忽略 `NULL` | `0` | 统计 `expr` 不为 `NULL` 的行数 |
| `COUNT(DISTINCT expr)` | 忽略 `NULL` | `0` | 统计不重复且非 `NULL` 的值 |

### 示例：AVG 会忽略 NULL

假设 `commission_pct` 有以下数据：

| employee_id | commission_pct |
| --- | --- |
| 1 | 0.1 |
| 2 | 0.2 |
| 3 | NULL |

执行：

```sql
SELECT AVG(commission_pct)
FROM employees;
```

计算方式是：

```text
(0.1 + 0.2) / 2
```

不是：

```text
(0.1 + 0.2 + NULL) / 3
```

也就是说，`AVG(expr)` 的本质可以理解成：

```text
SUM(expr 的非 NULL 值) / COUNT(expr)
```

### 示例：COUNT(*) 和 COUNT(列名) 不一样

假设数据如下：

| employee_id | commission_pct |
| --- | --- |
| 1 | 0.1 |
| 2 | 0.2 |
| 3 | NULL |

执行：

```sql
SELECT
    COUNT(*) AS row_count,
    COUNT(commission_pct) AS commission_count
FROM employees;
```

结果含义：

| 表达式 | 结果 | 原因 |
| --- | --- | --- |
| `COUNT(*)` | 3 | 有 3 行记录 |
| `COUNT(commission_pct)` | 2 | 只有 2 行的 `commission_pct` 不是 `NULL` |

## 1.4 AVG 和 SUM 函数

`AVG()` 和 `SUM()` 主要用于**数值型数据**。

```sql
SELECT
    AVG(salary) AS avg_salary,
    SUM(salary) AS total_salary
FROM employees
WHERE job_id LIKE '%REP%';
```

![AVG 与 SUM 示例结果](./images/1554981279723.png)

### AVG 函数

`AVG(expr)` 用来计算平均值。

```sql
SELECT AVG(salary)
FROM employees;
```

常见用途：

- 平均薪资
- 平均分数
- 平均订单金额
- 平均库存数量

需要注意：

- `AVG()` 会忽略 `NULL`。
- 如果没有匹配记录，结果是 `NULL`。
- 如果某个字段可能为 `NULL`，平均值的分母不是总行数，而是非 `NULL` 行数。

### SUM 函数

`SUM(expr)` 用来计算总和。

```sql
SELECT SUM(salary)
FROM employees;
```

常见用途：

- 总薪资
- 总销售额
- 总库存
- 总积分

需要注意：

- `SUM()` 会忽略 `NULL`。
- 如果没有匹配记录，结果是 `NULL`。
- 如果所有参与计算的值都是 `NULL`，结果也是 `NULL`。

### DISTINCT 写法

`AVG()` 和 `SUM()` 可以搭配 `DISTINCT`，表示先去重再计算。

```sql
SELECT AVG(DISTINCT salary)
FROM employees;

SELECT SUM(DISTINCT salary)
FROM employees;
```

不过实际开发中，`AVG(DISTINCT ...)` 和 `SUM(DISTINCT ...)` 要谨慎使用，因为它统计的是“不重复值”的平均或总和，不一定符合业务语义。

例如：

```sql
SELECT SUM(DISTINCT salary)
FROM employees;
```

这不是统计所有员工的薪资总额，而是先把重复薪资去掉，再求和。

## 1.5 MIN 和 MAX 函数

`MIN()` 和 `MAX()` 可以用于数值、日期、时间、字符串等可比较的数据类型。

```sql
SELECT
    MIN(hire_date) AS earliest_hire_date,
    MAX(hire_date) AS latest_hire_date
FROM employees;
```

![MIN 与 MAX 示例结果](./images/1554981253194.png)

### MAX 函数

`MAX(expr)` 用来取得最大值。

常见用途：

- 最高薪资
- 最新入职日期
- 最大订单金额
- 字典序最大的字符串

```sql
SELECT MAX(salary)
FROM employees;
```

```sql
SELECT MAX(hire_date)
FROM employees;
```

### MIN 函数

`MIN(expr)` 用来取得最小值。

常见用途：

- 最低薪资
- 最早入职日期
- 最小订单金额
- 字典序最小的字符串

```sql
SELECT MIN(salary)
FROM employees;
```

```sql
SELECT MIN(hire_date)
FROM employees;
```

### 字符串的 MIN 和 MAX

`MIN()` 和 `MAX()` 也可以用于字符串。此时比较规则会受到字符集和排序规则影响。

```sql
SELECT MIN(last_name), MAX(last_name)
FROM employees;
```

理解重点：

- 对数字：比较数值大小。
- 对日期：比较时间先后。
- 对字符串：按照排序规则比较先后顺序。

## 1.6 COUNT 函数

`COUNT()` 是最容易混淆、也最常使用的聚合函数。

它主要有四种常见写法：

```sql
COUNT(*)
COUNT(1)
COUNT(列名)
COUNT(DISTINCT 列名)
```

### 1.6.1 COUNT(*)

`COUNT(*)` 用来统计符合条件的**记录行数**。

```sql
SELECT COUNT(*)
FROM employees
WHERE department_id = 50;
```

![COUNT(*) 示例结果](./images/1554981241299.png)

重点：

- `COUNT(*)` 统计行数。
- 不关心任何具体列的值。
- 即使某些列是 `NULL`，该行仍然会被统计。
- 当你想统计“总共有几笔资料”时，优先使用 `COUNT(*)`。

### 1.6.2 COUNT(1)

`COUNT(1)` 表示对每一行都计算常数 `1`，因为 `1` 永远不是 `NULL`，所以通常也等同于统计符合条件的行数。

```sql
SELECT COUNT(1)
FROM employees;
```

从语义上看：

```text
每一行都产生一个非 NULL 的 1，然后统计这些 1 的数量。
```

在现代 MySQL 中，`COUNT(1)` 通常不会比 `COUNT(*)` 更有优势。写统计行数时，推荐使用语义更清楚的 `COUNT(*)`。

### 1.6.3 COUNT(列名)

`COUNT(列名)` 用来统计该列中**不为 NULL** 的记录数。

```sql
SELECT COUNT(commission_pct)
FROM employees
WHERE department_id = 50;
```

![COUNT(expr) 示例结果](./images/1554981328678.png)

重点：

- `COUNT(列名)` 不统计该列为 `NULL` 的行。
- 它不是单纯统计表中有几行。
- 它适合用来统计某个字段有填写值的数量。

例如：

```sql
SELECT COUNT(commission_pct)
FROM employees;
```

含义是：统计有奖金比例的员工数量。

不是：统计员工总数。

### 1.6.4 COUNT(DISTINCT 列名)

`COUNT(DISTINCT 列名)` 用来统计某列中**不重复且非 NULL** 的值数量。

```sql
SELECT COUNT(DISTINCT department_id)
FROM employees;
```

含义是：统计员工表中出现过多少个不同的部门编号。

需要注意：

- 重复值只计算一次。
- `NULL` 不会被统计。

### 1.6.5 COUNT 多列 DISTINCT

MySQL 支持在 `COUNT(DISTINCT ...)` 中放多个表达式，用来统计不重复的组合。

```sql
SELECT COUNT(DISTINCT department_id, job_id)
FROM employees;
```

含义是：统计不同的 `(department_id, job_id)` 组合数量。

注意：如果组合中的表达式包含 `NULL`，该组合通常不会被统计进 `COUNT(DISTINCT ...)` 的结果中。

### 1.6.6 COUNT 写法对比表

| 写法 | 统计对象 | 是否统计 NULL | 适用场景 | 推荐程度 |
| --- | --- | --- | --- | --- |
| `COUNT(*)` | 行数 | 会统计该行 | 统计总记录数 | 最推荐 |
| `COUNT(1)` | 常数 `1` 的数量 | 等同统计行数 | 统计总记录数，但语义不如 `COUNT(*)` 清楚 | 可用但不优先 |
| `COUNT(列名)` | 某列非 `NULL` 的数量 | 不统计该列为 `NULL` 的行 | 统计某字段有值的笔数 | 按业务需求使用 |
| `COUNT(DISTINCT 列名)` | 某列不重复且非 `NULL` 的值数量 | 不统计 `NULL` | 统计不重复值数量 | 按业务需求使用 |

### 1.6.7 MyISAM 与 InnoDB 下的 COUNT

不同存储引擎对 `COUNT(*)` 的处理方式不同。

#### MyISAM

对于 MyISAM 表，如果是单表且没有 `WHERE` 条件的 `COUNT(*)`，通常可以直接读取表维护的行数，因此速度很快。

```sql
SELECT COUNT(*)
FROM myisam_table;
```

#### InnoDB

对于 InnoDB 表，情况不同。InnoDB 因为支持事务和 MVCC，不会简单维护一个对所有事务都准确可用的总行数。因此 `COUNT(*)` 往往需要扫描索引或数据来得到准确结果。

```sql
SELECT COUNT(*)
FROM innodb_table;
```

实务理解：

- InnoDB 表的精确总行数统计通常比 MyISAM 成本高。
- 统计总行数时仍然推荐写 `COUNT(*)`，因为语义最清楚，优化器也能针对它选择合适的执行方式。
- 不要为了“性能传说”把 `COUNT(*)` 改成 `COUNT(1)`。
- 如果业务只需要估算值，可以考虑使用统计信息；如果业务频繁需要精确计数，可以考虑额外维护计数表或汇总表。

## 1.7 聚合函数和 WHERE、GROUP BY、HAVING 的关系

这一节先建立整体观念，详细内容会在后续的 `GROUP BY` 和 `HAVING` 笔记中继续展开。

### WHERE：分组前过滤原始行

`WHERE` 发生在聚合之前，用来过滤原始资料行。

```sql
SELECT AVG(salary)
FROM employees
WHERE department_id = 50;
```

含义是：

1. 先筛选出 `department_id = 50` 的员工。
2. 再对这些员工的薪资求平均值。

### GROUP BY：决定分组维度

`GROUP BY` 用来决定按照什么字段分组。

```sql
SELECT department_id, AVG(salary)
FROM employees
GROUP BY department_id;
```

含义是：每个部门各自计算平均薪资。

### HAVING：分组后过滤聚合结果

`HAVING` 用来过滤分组后的聚合结果。

```sql
SELECT department_id, AVG(salary)
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 8000;
```

含义是：

1. 先按部门分组。
2. 再计算每个部门的平均薪资。
3. 最后只保留平均薪资大于 8000 的部门。

### WHERE 和 HAVING 的选择原则

| 条件类型 | 应该放哪里 | 示例 |
| --- | --- | --- |
| 原始数据行条件 | `WHERE` | `WHERE salary > 5000` |
| 分组后的聚合条件 | `HAVING` | `HAVING AVG(salary) > 8000` |
| 两者都有 | `WHERE` + `HAVING` | 先过滤原始行，再过滤分组结果 |

推荐写法：

```sql
SELECT department_id, AVG(salary) AS avg_salary
FROM employees
WHERE department_id IS NOT NULL
GROUP BY department_id
HAVING AVG(salary) > 8000;
```

理解：

- `WHERE department_id IS NOT NULL`：分组前先排除没有部门的员工。
- `GROUP BY department_id`：按部门分组。
- `HAVING AVG(salary) > 8000`：只保留平均薪资大于 8000 的部门。

## 1.8 常见错误与推荐写法

### 错误 1：用 COUNT(列名) 统计总行数

不推荐：

```sql
SELECT COUNT(commission_pct)
FROM employees;
```

如果你的目的是统计员工总数，这样写是错的，因为 `commission_pct` 为 `NULL` 的员工不会被统计。

推荐：

```sql
SELECT COUNT(*)
FROM employees;
```

### 错误 2：以为 COUNT(1) 一定比 COUNT(*) 快

不推荐把下面写法当成性能优化：

```sql
SELECT COUNT(1)
FROM employees;
```

推荐：

```sql
SELECT COUNT(*)
FROM employees;
```

原因：

- `COUNT(*)` 表达的是“统计行数”，语义最明确。
- MySQL 优化器会处理 `COUNT(*)` 的执行方式。
- `COUNT(1)` 通常没有必要作为替代写法。

### 错误 3：在 WHERE 中使用聚合函数

错误：

```sql
SELECT department_id, COUNT(*)
FROM employees
WHERE COUNT(*) > 5
GROUP BY department_id;
```

正确：

```sql
SELECT department_id, COUNT(*)
FROM employees
GROUP BY department_id
HAVING COUNT(*) > 5;
```

### 错误 4：搞混 COUNT(*) 和 SUM(某字段)

下面两种统计意义完全不同：

```sql
SELECT COUNT(*)
FROM orders;
```

含义是：订单笔数。

```sql
SELECT SUM(total_amount)
FROM orders;
```

含义是：订单总金额。

### 错误 5：忽略聚合函数对 NULL 的处理

假设某些员工的 `commission_pct` 是 `NULL`：

```sql
SELECT AVG(commission_pct)
FROM employees;
```

这不是用所有员工当分母，而是只用 `commission_pct` 不为 `NULL` 的员工当分母。

如果业务上希望把 `NULL` 当成 `0`，要明确使用 `IFNULL()` 或 `COALESCE()`。

```sql
SELECT AVG(IFNULL(commission_pct, 0))
FROM employees;
```

## 1.9 本节小结

### 核心观念

1. 聚合函数用于对多行数据做汇总计算。
2. 没有 `GROUP BY` 时，所有符合条件的行会被视为同一组。
3. 有 `GROUP BY` 时，每个分组都会各自计算聚合结果。
4. 多数聚合函数会忽略 `NULL`，但 `COUNT(*)` 统计的是行数，不看字段值。
5. 聚合函数不能直接写在 `WHERE` 中；要筛选聚合结果，应该使用 `HAVING`。
6. 同一层查询中不要直接嵌套聚合函数，应该使用子查询拆成两步。

### 常用函数速查

| 函数 | 作用 | 是否忽略 NULL | 常见用途 |
| --- | --- | --- | --- |
| `AVG(expr)` | 平均值 | 是 | 平均薪资、平均成绩 |
| `SUM(expr)` | 总和 | 是 | 总金额、总库存 |
| `MAX(expr)` | 最大值 | 是 | 最高薪资、最新日期 |
| `MIN(expr)` | 最小值 | 是 | 最低薪资、最早日期 |
| `COUNT(*)` | 总行数 | 不看字段值 | 统计总记录数 |
| `COUNT(expr)` | 非空值数量 | 是 | 统计某字段有值的记录数 |
| `COUNT(DISTINCT expr)` | 不重复且非空的值数量 | 是 | 统计不同部门、不同类别 |

### COUNT 选择原则

| 需求 | 推荐写法 |
| --- | --- |
| 统计表中总共有几行 | `COUNT(*)` |
| 统计符合条件的记录数 | `COUNT(*)` |
| 统计某字段有填写值的笔数 | `COUNT(列名)` |
| 统计某字段不重复值数量 | `COUNT(DISTINCT 列名)` |
| 统计不重复组合数量 | `COUNT(DISTINCT 列1, 列2)` |

### 推荐模板

```sql
SELECT
    分组字段,
    聚合函数(统计字段) AS 统计结果
FROM 表名
WHERE 分组前条件
GROUP BY 分组字段
HAVING 分组后条件
ORDER BY 统计结果 DESC;
```

例如：

```sql
SELECT
    department_id,
    COUNT(*) AS employee_count,
    AVG(salary) AS avg_salary,
    MAX(salary) AS max_salary,
    MIN(salary) AS min_salary
FROM employees
WHERE department_id IS NOT NULL
GROUP BY department_id
HAVING COUNT(*) >= 3
ORDER BY avg_salary DESC;
```

这条 SQL 的含义是：

1. 先排除没有部门的员工。
2. 按部门分组。
3. 统计每个部门的员工数、平均薪资、最高薪资、最低薪资。
4. 只保留员工数大于等于 3 的部门。
5. 按平均薪资由高到低排序。
