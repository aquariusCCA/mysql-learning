# 第八章_聚合函数练习题

这份题单只基于第八章现有笔记内容设计，重点练习：

- 聚合函数和单行函数的区别
- `AVG()`、`SUM()`、`MIN()`、`MAX()`、`COUNT()` 的使用边界
- `GROUP BY` 的分组规则与多列分组
- `HAVING` 与 `WHERE` 的分工
- `SELECT` 的书写顺序与实际执行顺序

## 使用说明

- 练习数据默认使用仓库中的 `資料/atguigudb.sql`。
- 除题目特别说明外，默认基于 `employees` 表完成练习。
- 这一份只放题目，不放答案；建议先独立完成，再回到第八章对应小节对照。
- 涉及“判断正误”或“改错”的题目，建议你先说出原因，再改 SQL。

## 基础题

### 1. 聚合函数的定义

说明什么是聚合函数，并对比它和单行函数在“输入数据范围”与“输出结果个数”上的差别。

### 2. 聚合函数识别

判断下面这些函数中，哪些属于聚合函数，哪些不属于：

- `AVG()`
- `SUM()`
- `LENGTH()`
- `COUNT()`
- `CONCAT()`
- `MAX()`

### 3. 数据类型判断

回答下面两个问题：

1. `AVG()` 和 `SUM()` 通常用于什么类型的数据？
2. `MIN()` 和 `MAX()` 是否只能用于数值类型？请说明原因。

### 4. 统计行数的差别

说明下面三种写法在统计语义上的差别：

- `COUNT(*)`
- `COUNT(1)`
- `COUNT(commission_pct)`

请特别说明它们对 `NULL` 的处理差异。

### 5. 基础聚合查询

查询 `employees` 表中所有员工的：

- 平均工资
- 最高工资
- 最低工资
- 工资总和

### 6. 条件过滤后再聚合

查询岗位编号中包含 `REP` 的员工的平均工资与工资总和。

要求：

- 普通筛选条件写在 `WHERE` 中；
- 聚合结果放在 `SELECT` 中。

### 7. `MIN()` / `MAX()` 非数值练习

查询 `employees` 表中最早入职日期与最晚入职日期。

### 8. 判断非法写法

下面哪条 SQL 是非法的？请说明原因。

```sql
SELECT AVG(salary) FROM employees;
```

```sql
SELECT AVG(SUM(salary)) FROM employees;
```

```sql
SELECT COUNT(*) FROM employees;
```

## 进阶题

### 9. 单列分组

按 `department_id` 分组，查询每个部门的平均工资。

### 10. 分组字段与 `SELECT` 字段关系

判断下面 SQL 是否合理；如果不合理，请改写：

```sql
SELECT department_id, last_name, AVG(salary)
FROM employees
GROUP BY department_id;
```

### 11. 分组字段不出现在 `SELECT`

写一条 SQL，按 `department_id` 分组，但结果中只显示每组的平均工资，不显示 `department_id`。

### 12. 多列分组

按 `department_id` 和 `job_id` 分组，查询每个“部门 + 岗位”组合的工资总和。

### 13. `WITH ROLLUP`

查询 `department_id > 80` 的部门平均工资，并在结果最后追加整体汇总行。

要求使用 `WITH ROLLUP`。

### 14. `ROLLUP` 限制判断

说明为什么下面这条 SQL 在本章语境下不推荐或不成立：

```sql
SELECT department_id, AVG(salary)
FROM employees
GROUP BY department_id WITH ROLLUP
ORDER BY department_id;
```

### 15. `HAVING` 基础题

按 `department_id` 分组，查询各部门最高工资大于 `10000` 的部门编号与最高工资。

### 16. `WHERE` 与 `HAVING` 分工

查询 `department_id > 50` 的部门中，平均工资大于 `8000` 的部门编号与平均工资。

要求：

- 行级条件写在 `WHERE`；
- 分组后的筛选写在 `HAVING`。

### 17. 改错题：聚合函数写进 `WHERE`

下面 SQL 有问题，请说明原因并改写成正确版本：

```sql
SELECT department_id, AVG(salary)
FROM employees
WHERE AVG(salary) > 8000
GROUP BY department_id;
```

### 18. `WHERE` 能做什么，`HAVING` 能做什么

分别举出一个例子说明：

2. 哪类条件只能写在 `HAVING`。
3. 哪类查询会同时用到 `WHERE` 和 `HAVING`。

要求你从“执行顺序”角度回答，而不是只背结论。

### 22. 结果推演题

阅读下面 SQL，不需要真的运行，请按执行顺序说明每一步在做什么：

```sql
SELECT department_id, COUNT(*) AS emp_count
FROM employees
WHERE salary > 6000
GROUP BY department_id
HAVING COUNT(*) >= 3
ORDER BY emp_count DESC
LIMIT 5;
```

至少说明：

- 哪一步先筛掉工资不满足条件的行；
- 哪一步开始形成“每组一行”的结果；
- 哪一步再把分组结果过滤一次；
- 哪一步才真正决定最终显示哪些字段。

### 23. 改写题：按执行过程拆解

把下面这条 SQL 拆成“书写顺序”和“实际执行顺序”两种说明：

```sql
SELECT department_id, AVG(salary) AS avg_sal
FROM employees
WHERE department_id IS NOT NULL
GROUP BY department_id
HAVING AVG(salary) > 7000
ORDER BY avg_sal DESC;
```

要求：

- 先写出它的书写顺序；
- 再写出它的实际执行顺序；
- 最后说明 `avg_sal` 这个别名在哪个阶段才真正可用。

### 24. 综合题：从需求到 SQL

完成下面需求：

查询 `employees` 表中，`department_id` 不为空的部门里，平均工资大于 `9000` 的部门编号、平均工资与员工人数，并按照平均工资从高到低排序。

要求：

- 必须同时使用 `WHERE`、`GROUP BY`、`HAVING`、`ORDER BY`；
- 聚合函数至少使用两个；
- 字段别名命名清楚。
