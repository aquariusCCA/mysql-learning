# 第八章_聚合函数练习题参考答案

这份参考答案对应 [第八章_聚合函数练习题](./第八章_聚合函数练习题.md)。

- 查询题默认基于 `資料/atguigudb.sql`
- 除题目特别说明外，默认基于 `employees` 表
- 下文同时区分“书写顺序”和“实际执行顺序”

## 基础题

### 1. 聚合函数的定义

答案：

- 聚合函数是对一组数据做统计汇总后返回结果的函数。
- 它和单行函数的差别在于：
  - 输入范围：聚合函数面向多行，单行函数面向单行
  - 输出个数：聚合函数通常每组一个结果，单行函数通常每行一个结果

### 2. 聚合函数识别

答案：

- 属于聚合函数：`AVG()`、`SUM()`、`COUNT()`、`MAX()`
- 不属于聚合函数：`LENGTH()`、`CONCAT()`

### 3. 数据类型判断

答案：

1. `AVG()` 和 `SUM()` 通常用于数值类型。
2. `MIN()` 和 `MAX()` 不只能用于数值，也可以用于日期、字符串等可比较类型。

### 4. 统计行数的差别

答案：

- `COUNT(*)`：统计总行数，不忽略行
- `COUNT(1)`：统计总行数，语义上也不受某一列 `NULL` 影响
- `COUNT(commission_pct)`：只统计 `commission_pct` 非 `NULL` 的行数

核心差异：前两者统计“行”，第三个统计“该列有值的行”。

### 5. 基础聚合查询

```sql
SELECT
    AVG(salary) AS avg_salary,
    MAX(salary) AS max_salary,
    MIN(salary) AS min_salary,
    SUM(salary) AS sum_salary
FROM employees;
```

### 6. 条件过滤后再聚合

```sql
SELECT
    AVG(salary) AS avg_salary,
    SUM(salary) AS sum_salary
FROM employees
WHERE job_id LIKE '%REP%';
```

### 7. `MIN()` / `MAX()` 非数值练习

```sql
SELECT
    MIN(hire_date) AS earliest_hire_date,
    MAX(hire_date) AS latest_hire_date
FROM employees;
```

### 8. 判断非法写法

答案：

- `SELECT AVG(salary) FROM employees;`：合法
- `SELECT AVG(SUM(salary)) FROM employees;`：非法
- `SELECT COUNT(*) FROM employees;`：合法

原因：聚合函数不能直接再嵌套聚合函数。

## 进阶题

### 9. 单列分组

```sql
SELECT department_id, AVG(salary) AS avg_salary
FROM employees
GROUP BY department_id;
```

### 10. 分组字段与 `SELECT` 字段关系

答案：这条 SQL 不合理，因为 `last_name` 既没有出现在 `GROUP BY` 中，也不是聚合结果。

改写：

```sql
SELECT department_id, AVG(salary) AS avg_salary
FROM employees
GROUP BY department_id;
```

### 11. 分组字段不出现在 `SELECT`

```sql
SELECT AVG(salary) AS avg_salary
FROM employees
GROUP BY department_id;
```

### 12. 多列分组

```sql
SELECT department_id, job_id, SUM(salary) AS sum_salary
FROM employees
GROUP BY department_id, job_id;
```

### 13. `WITH ROLLUP`

```sql
SELECT department_id, AVG(salary) AS avg_salary
FROM employees
WHERE department_id > 80
GROUP BY department_id WITH ROLLUP;
```

### 14. `ROLLUP` 限制判断

答案：

- `WITH ROLLUP` 会额外追加汇总行。
- 在本章语境下再直接 `ORDER BY department_id`，容易让总计行的位置和阅读语义变得不清楚。
- 课程里通常把 `ROLLUP` 当成“结果末尾补一行总计”，不再额外排序。
- 如果确实要复杂排序，通常改成外层再包一层查询更稳妥。

### 15. `HAVING` 基础题

```sql
SELECT department_id, MAX(salary) AS max_salary
FROM employees
GROUP BY department_id
HAVING MAX(salary) > 10000;
```

### 16. `WHERE` 与 `HAVING` 分工

```sql
SELECT department_id, AVG(salary) AS avg_salary
FROM employees
WHERE department_id > 50
GROUP BY department_id
HAVING AVG(salary) > 8000;
```

### 17. 改错题：聚合函数写进 `WHERE`

错误原因：`WHERE` 在分组前执行，不能直接用聚合结果。

正确写法：

```sql
SELECT department_id, AVG(salary) AS avg_salary
FROM employees
GROUP BY department_id
HAVING AVG(salary) > 8000;
```

### 18. `WHERE` 能做什么，`HAVING` 能做什么

答案：

1. 只能写在 `WHERE` 的条件：
   例如 `salary > 5000`
2. 只能写在 `HAVING` 的条件：
   例如 `AVG(salary) > 8000`
3. 同时用到 `WHERE` 和 `HAVING` 的例子：

```sql
SELECT department_id, AVG(salary) AS avg_salary
FROM employees
WHERE department_id IS NOT NULL
GROUP BY department_id
HAVING AVG(salary) > 7000;
```

## 挑战题

### 19. 书写顺序排序题

答案：

```text
SELECT -> FROM -> WHERE -> GROUP BY -> HAVING -> ORDER BY -> LIMIT
```

### 20. 实际执行顺序排序题

答案：

```text
FROM -> WHERE -> GROUP BY -> HAVING -> SELECT -> DISTINCT -> ORDER BY -> LIMIT
```

### 21. 执行顺序判断

答案：

- `WHERE` 早于 `SELECT` 执行，所以它通常还看不到 `SELECT` 里刚起好的别名。
- `ORDER BY` 晚于 `SELECT` 执行，所以它通常可以引用 `SELECT` 中定义的别名。

### 22. 结果推演题

SQL：

```sql
SELECT department_id, COUNT(*) AS emp_count
FROM employees
WHERE salary > 6000
GROUP BY department_id
HAVING COUNT(*) >= 3
ORDER BY emp_count DESC
LIMIT 5;
```

执行说明：

1. `FROM employees`：先拿到原始员工数据
2. `WHERE salary > 6000`：先筛掉工资不满足条件的行
3. `GROUP BY department_id`：开始按部门分组，形成“每组一行”的聚合基础
4. `HAVING COUNT(*) >= 3`：把分组结果再过滤一次
5. `SELECT department_id, COUNT(*) AS emp_count`：决定最终显示哪些列
6. `ORDER BY emp_count DESC`：按人数降序排序
7. `LIMIT 5`：只保留前 `5` 行

### 23. 改写题：按执行过程拆解

SQL：

```sql
SELECT department_id, AVG(salary) AS avg_sal
FROM employees
WHERE department_id IS NOT NULL
GROUP BY department_id
HAVING AVG(salary) > 7000
ORDER BY avg_sal DESC;
```

书写顺序：

```text
SELECT -> FROM -> WHERE -> GROUP BY -> HAVING -> ORDER BY
```

实际执行顺序：

```text
FROM -> WHERE -> GROUP BY -> HAVING -> SELECT -> ORDER BY
```

补充：

- `avg_sal` 这个别名是在 `SELECT` 阶段才真正形成
- 所以 `ORDER BY` 可以用它，`WHERE` 不行

### 24. 综合题：从需求到 SQL

```sql
SELECT
    department_id,
    AVG(salary) AS avg_salary,
    COUNT(*) AS emp_count
FROM employees
WHERE department_id IS NOT NULL
GROUP BY department_id
HAVING AVG(salary) > 9000
ORDER BY avg_salary DESC;
```
