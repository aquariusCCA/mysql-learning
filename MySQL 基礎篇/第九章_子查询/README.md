# 第九章_子查询

这一章整理 `SELECT` 查询中的子查询。

前面几章已经建立了基础查询、排序分页、多表查询、单行函数、聚合函数与 `SELECT` 执行过程。到了这一章，重点会从「如何写出一条完整查询」进一步推进到：

> 查询里面还可以再放查询。  
> 也就是先得到一个中间结果，再让外层查询继续判断、筛选、比较、连接或返回数据。

子查询不是单纯多背一种 SQL 语法，而是在训练一种更重要的查询思维：

1. 最终要查什么？
2. 中间需要先得到什么？
3. 中间结果返回几行几列？
4. 外层查询要如何使用这个结果？
5. 子查询是否依赖外层当前行？
6. 如果结果为空、为 `NULL`、多行或重复，会发生什么？
7. 如果同一个需求有多种写法，该如何选择？

目前这一章已整理出 `7` 篇笔记，涵盖：

- 子查询整体概念；
- 需求分析与问题拆解；
- 单行子查询；
- 多行子查询；
- 相关子查询；
- 自连接与子查询的写法取舍；
- 写子查询时的下手技巧。

新版内容不只整理语法，也加入了：

- 子查询返回结果形态判断；
- `0` 行、`NULL`、多行、空集合的影响；
- `IN`、`ANY`、`ALL`、`SOME` 的语意差异；
- `NOT IN` 遇到 `NULL` 的陷阱与改写方式；
- `EXISTS` / `NOT EXISTS` 的存在性判断；
- 相关子查询与派生表 JOIN 的取舍；
- 自连接、子查询、聚合写法的选择；
- 书写顺序、逻辑理解顺序、实际执行计划的区别。

学习这一章时，建议不要只背 SQL 长相，而要练习从需求一步一步推导出写法。

## 本章在整体学习地图中的位置

- 前面的基础查询、排序分页与多表查询，先建立了 `SELECT` 的主体结构。
- 第八章进一步说明聚合查询与 `SELECT` 执行过程，帮助理解查询各子句之间的先后关系。
- 这一章开始进入复杂查询：查询条件、查询结果或临时资料表本身，也可以来自另一条查询。
- 学完这一章后，会更容易理解后续的综合查询、报表查询、业务统计 SQL、SQL 优化与执行计划分析。

可以把这一章理解成：

> 从「会写单层 SELECT」过渡到「能拆解复杂查询需求」的重要章节。

## 本章学习目标

学完这一章后，应该能够做到：

- 看懂什么是子查询、内层查询、外层查询。
- 判断子查询返回的是一个值、一组值、一组字段组合，还是一张临时表。
- 根据子查询返回结果，选择正确的比较方式。
- 区分单行子查询、多行子查询、行子查询、表子查询。
- 区分不相关子查询与相关子查询。
- 正确使用 `=`、`IN`、`ANY`、`ALL`、`EXISTS`、`NOT EXISTS`。
- 看懂 `NOT IN` 遇到 `NULL` 为什么容易出错。
- 看懂相关子查询如何引用外层当前行。
- 比较子查询、自连接、派生表 JOIN 的写法差异。
- 写子查询时知道该先写内层还是外层。
- 对 SQL 执行顺序保持正确理解：逻辑理解顺序不等于实际执行计划。

- 为什么学习阶段可以先用「内层结果给外层使用」来理解，但不能误以为 MySQL 实际执行时永远按照这个顺序执行。

适合第一次进入子查询主题时阅读。

### 2. [1 需求分析与问题解决](./1%20需求分析与问题解决.md)

- `ANY` / `ALL` 如何用 `MIN()`、`MAX()` 辅助理解；

适合想彻底分清「其中一个」和「全部都满足」时阅读。

### 5. [4 相关子查询](./4%20相关子查询.md)

## 子查询类型快速对照表

| 类型 | 返回结果 | 常见搭配 | 典型场景 |
| --- | --- | --- | --- |
| 标量子查询 / 单行子查询 | `1` 行 `1` 列 | `=`、`>`、`<`、`>=`、`<=`、`<>` | 高于平均工资、等于最高工资 |
| 列子查询 / 多行子查询 | 多行 `1` 列 | `IN`、`NOT IN`、`ANY`、`ALL` | 属于某些部门、比一组值中的任意一个或所有值比较 |
| 行子查询 | 一行多列，或多行多列组合 | `(col1, col2) IN (...)` | 多字段成组匹配 |
| 表子查询 / 派生表 | 多行多列 | `FROM (...) AS alias`、`JOIN` | 先统计每组资料，再和原表连接 |
| 不相关子查询 | 不依赖外层当前行 | 可单独执行 | 公司平均工资、某固定员工资料 |
| 相关子查询 | 依赖外层当前行 | 引用外层字段 | 本部门平均工资、是否存在订单 |
| `EXISTS` 子查询 | 是否返回行 | `EXISTS`、`NOT EXISTS` | 有无对应记录 |

## 操作符快速对照表

| 操作符 | 语意 | 常见用途 |
| --- | --- | --- |
| `=` | 等于单一值 | 等于某员工工资、等于最低工资 |
| `>` / `<` | 和单一值比较大小 | 高于平均工资、低于某员工工资 |
| `IN` | 等于集合中的任意一个值 | 属于某些部门 |
| `NOT IN` | 不属于集合中的任何一个值 | 但要特别注意 `NULL` |
| `ANY` | 至少和集合中的一个值比较成立 | 比其中至少一个大 / 小 |
| `SOME` | `ANY` 的同义词 | 看到即可理解成 `ANY` |
| `ALL` | 必须和集合中的所有值比较成立 | 比所有值都大 / 小 |
| `EXISTS` | 子查询有返回行就成立 | 查询有对应记录的资料 |
| `NOT EXISTS` | 子查询没有返回行才成立 | 查询没有对应记录的资料 |

## 写题判断流程

遇到子查询题时，可以按下面流程判断。

### 第一步：确认最终要查什么

也就是外层查询的主体。

例如：

- 查员工；
- 查部门；
- 查客户；
- 查订单；
- 查分组后的统计结果。

### 第二步：确认中间要先得到什么

也就是内层查询的任务。

例如：

- 公司平均工资；
- 某位员工的工资；
- 某些部门编号；
- 每个部门的平均工资；
- 是否存在订单；
- 是否不存在对应员工。

### 第三步：确认中间结果返回几行几列

| 返回结果 | 可能写法 |
| --- | --- |
| `1` 行 `1` 列 | 单行子查询 |
| 多行 `1` 列 | 多行子查询 |
| 一行多列 / 多行多列组合 | 行子查询 |
| 多行多列临时表 | 表子查询 / 派生表 |
| 只关心有没有资料 | `EXISTS` / `NOT EXISTS` |

### 第四步：选择外层操作符

| 中间结果 | 外层操作符 |
| --- | --- |
| 一个值 | `=、>、<、>=、<=、<>` |
| 一组值 | `IN、NOT IN、ANY、ALL` |
| 一组字段组合 | `(col1, col2) IN (...)` |
| 一张临时表 | `JOIN (子查询) AS alias` |
| 是否存在资料 | `EXISTS` / `NOT EXISTS` |

### 第五步：检查是否依赖外层当前行

如果子查询引用了外层字段，例如：

```sql
e2.department_id = e1.department_id
```

就是相关子查询。

### 第六步：检查风险

写完后检查：

- 单行子查询是否可能返回多行；
- 子查询是否可能返回 `0` 行；
- 子查询结果是否可能为 `NULL`;
- `NOT IN` 是否可能遇到 `NULL`;
- `ANY` / `ALL` 是否可能遇到空集合；
- 自连接是否可能造成重复资料；
- 派生表是否有别名；
- 相关子查询是否写对关联条件；
- 性能判断是否需要看 `EXPLAIN`。

## 本章学习建议

### 第一次学习

建议按照阅读顺序完整读一遍：

1. 先建立整体概念；
2. 再看实际需求；
3. 接着学单行、多行、相关子查询；
4. 最后看写法取舍与小技巧。

第一次不用急着背所有边界情况，但要先把下面三个问题练熟：

```text
子查询返回什么？
返回几行几列？
外层怎么用这个结果？
```

### 第二次复习

第二次复习时，重点放在：

- 单行子查询的多行错误；
- `NULL` 问题；
- `NOT IN` 陷阱；
- `ANY` / `ALL` 差异；
- 相关子查询的外层字段引用；
- `EXISTS` / `NOT EXISTS`；
- 自连接与子查询取舍。

### 刷题时

刷题时建议搭配 [小技巧](./小技巧.md) 使用。

每一题都先写下：

```text
外层要查什么？
内层要查什么？
内层返回几行几列？
外层用什么操作符？
是否依赖外层当前行？
```

这样可以避免只靠感觉写 SQL。

## 一句话抓核心

子查询的核心不是把一条 SQL 硬塞进另一条 SQL，而是先判断内层查询要提供什么中间结果，再根据这个结果的形态选择合适的外层写法；学习时用逻辑顺序理解语意，实务上再结合资料特性、可读性、维护性与执行计划选择最合适的 SQL 表达方式。
