# 第九章_子查询

这一章开始整理 `SELECT` 查询中的子查询。前面几章已经建立了基础查询、排序分页、多表查询、单行函数、聚合函数与 `SELECT` 执行过程；到了这一章，重点会转向“查询里还能再放查询”这件事，也就是如何先得到一个中间结果，再让外层查询继续判断、筛选或返回数据。

目前这一章已整理出 `7` 篇笔记，已经涵盖子查询前言、需求引入、单行子查询、多行子查询、相关子查询、思考题与书写技巧补充。后续如果继续补齐，可以再往不同子句中的限制、性能比较、`EXISTS` / `IN` 的取舍与更多综合题型推进。

## 本章在整体学习地图中的位置

- 前面的基础查询、排序分页与多表查询，先建立了 `SELECT` 的主体结构。
- 第八章进一步说明了聚合查询与 `SELECT` 的执行过程，帮助理解查询各子句之间的先后关系。
- 这一章开始把“一个查询如何写完整”推进到“查询内部还可以嵌套另一个查询”，是从基础查询迈向复杂查询的重要一步。

1. [前言](./前言.md)：先建立子查询的整体概念，理解为什么查询中还需要再嵌套查询。

3. [2 单行子查询](./2%20单行子查询.md)：继续看返回单个值时，子查询如何搭配单行比较符、`HAVING`、`CASE` 与空值问题。
4. [3 多行子查询](./3%20多行子查询.md)：再看返回多行结果时，`IN`、`ANY`、`ALL`、`SOME` 和 `NOT IN` 的差别。
5. [4 相关子查询](./4%20相关子查询.md)：最后看子查询如何依赖外层当前行，并进一步整理 `EXISTS`、`NOT EXISTS`、相关更新与相关删除。

- [前言](./前言.md)：说明什么是子查询、为什么需要子查询，以及学习这一章时应先抓住的整体结论。

- [3 多行子查询](./3%20多行子查询.md)：整理 `IN`、`ANY`、`ALL`、`SOME` 的语意、典型题型，以及 `NOT IN` 的空值陷阱。

- [5 抛一个思考题](./5%20抛一个思考题.md)：用“谁的工资比 Abel 高”这道题比较自连接与子查询，建立写法选择的思考方式。
- [小技巧](./小技巧.md)：整理子查询常见的编写顺序与判断方法，帮助写题时更快下手。
