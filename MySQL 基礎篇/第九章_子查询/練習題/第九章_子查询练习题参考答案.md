# 第九章_子查询练习题参考答案

这份参考答案对应 [第九章_子查询练习题](./第九章_子查询练习题.md)。

- 查询题默认基于 `資料/atguigudb.sql`
- 主要围绕 `employees`、`departments`、`job_history`
- 同一题不只一种写法，下面优先给出和本章笔记贴近的版本

## 基础题

### 1. 子查询的定义

答案：

- 子查询就是嵌套在另一条 SQL 里的查询。
- 外层查询：最终负责输出结果或继续筛选的那层查询。
- 内层查询：先执行、先产生中间结果的那层查询。
- 所以子查询常被理解为：先求出一个中间结果，再让外层继续拿来比较、筛选或计算。

### 2. 可用位置判断

答案：

- 子查询常见可出现在：
  - `WHERE`
  - `HAVING`
  - `FROM`
  - `SELECT`
  - `ORDER BY`
- 通常不直接出现在“表名应该出现的位置”那种不符合语法的地方。

### 3. 分类题：按返回结果分类

答案：

- 标量子查询：返回单个值
- 列子查询：返回一列多行
- 行子查询：返回一行多列
- 表子查询：返回临时结果集，可在 `FROM` 中当派生表使用

典型场景：

- 标量子查询：拿一个工资值给外层比较
- 列子查询：给外层 `IN (...)` 使用
- 行子查询：做成对比较
- 表子查询：先算部门平均工资，再回表连接

### 4. 分类题：按是否依赖外层分类

答案：

- 不相关子查询：内层查询自己就能独立跑完，不依赖外层当前行
- 相关子查询：内层查询要引用外层当前行的值

关键差异：

- 不相关子查询通常先执行一次
- 相关子查询通常会随着外层行逐行参与判断

### 5. 需求判断题

答案：

- “查询工资高于 `Abel` 的员工”这类题，天然适合拆成两步：
  1. 先查出 `Abel` 的工资
  2. 再找工资高于这个值的人
- 子查询刚好能把这两个步骤写进一条 SQL 里

### 6. 基础单行子查询

```sql
SELECT last_name, salary
FROM employees
WHERE salary > (
    SELECT salary
    FROM employees
    WHERE employee_id = 149
);
```

### 7. 双条件单行子查询

```sql
SELECT last_name, job_id, salary
FROM employees
WHERE job_id = (
    SELECT job_id
    FROM employees
    WHERE employee_id = 141
)
AND salary > (
    SELECT salary
    FROM employees
    WHERE employee_id = 143
);
```

### 8. 最低工资员工

```sql
SELECT last_name, job_id, salary
FROM employees
WHERE salary = (
    SELECT MIN(salary)
    FROM employees
);
```

## 进阶题

### 9. 单行比较操作符判断

答案：

- `=`、`>`、`>=`、`<`、`<=`、`<>` 这类操作符一次只适合和一个值比较
- 所以它们天然适合搭配“只返回一个值”的单行子查询

### 10. 改错题：多行结果误用单行比较

错误原因：子查询按部门分组后会返回多行，不能再用 `=` 直接和单值比较。

可改写为：

```sql
SELECT employee_id, last_name
FROM employees
WHERE salary IN (
    SELECT MIN(salary)
    FROM employees
    GROUP BY department_id
);
```

### 11. 子查询为空时的判断

答案：

- 这条 SQL 语法可能没错，但如果内层查不到 `last_name = 'Haas'` 的员工，标量子查询结果就会变成 `NULL`
- 外层条件等价于：

```sql
job_id = NULL
```

- 这个比较结果是未知，不会得到 `TRUE`
- 所以最终可能一行都查不到

### 12. 成对比较与不成对比较

不成对比较：

```sql
SELECT employee_id, last_name, manager_id, department_id
FROM employees
WHERE manager_id IN (
        SELECT manager_id
        FROM employees
        WHERE employee_id IN (141, 174)
    )
  AND department_id IN (
        SELECT department_id
        FROM employees
        WHERE employee_id IN (141, 174)
    )
  AND employee_id NOT IN (141, 174);
```

成对比较：

```sql
SELECT employee_id, last_name, manager_id, department_id
FROM employees
WHERE (manager_id, department_id) IN (
        SELECT manager_id, department_id
        FROM employees
        WHERE employee_id IN (141, 174)
    )
  AND employee_id NOT IN (141, 174);
```

语义差异：

- 不成对比较：经理和部门分别落在集合里即可，组合关系可能被打散
- 成对比较：必须整对 `(manager_id, department_id)` 同时匹配

### 13. `HAVING` 中的单行子查询

```sql
SELECT department_id, MIN(salary) AS min_salary
FROM employees
GROUP BY department_id
HAVING MIN(salary) > (
    SELECT MIN(salary)
    FROM employees
    WHERE department_id = 50
);
```

### 14. `CASE` 中的子查询

```sql
SELECT
    employee_id,
    last_name,
    CASE
        WHEN department_id = (
            SELECT department_id
            FROM departments
            WHERE location_id = 1800
        ) THEN 'Canada'
        ELSE 'USA'
    END AS location_tag
FROM employees;
```

### 15. `IN` 基础题

```sql
SELECT last_name, department_id
FROM employees
WHERE department_id IN (
    SELECT department_id
    FROM departments
    WHERE department_name IN ('IT', 'Human Resources')
);
```

### 16. `ANY` 题型

```sql
SELECT employee_id, last_name, job_id, salary
FROM employees
WHERE job_id <> 'IT_PROG'
  AND salary < ANY (
        SELECT salary
        FROM employees
        WHERE job_id = 'IT_PROG'
    );
```

### 17. `ALL` 题型

```sql
SELECT employee_id, last_name, job_id, salary
FROM employees
WHERE job_id <> 'IT_PROG'
  AND salary < ALL (
        SELECT salary
        FROM employees
        WHERE job_id = 'IT_PROG'
    );
```

### 18. `ANY` 与 `ALL` 比较题

答案：

- `salary < ANY (子查询)`：只要比结果集里的某一个值小就行
- `salary < ALL (子查询)`：必须比结果集里的每一个值都小

自然语言翻译：

- `< ANY`：比其中至少一个更低
- `< ALL`：比里面所有值都低

### 19. 最低平均工资部门

子查询解法：

```sql
SELECT t.department_id
FROM (
    SELECT department_id, AVG(salary) AS avg_salary
    FROM employees
    GROUP BY department_id
) t
WHERE t.avg_salary = (
    SELECT MIN(t2.avg_salary)
    FROM (
        SELECT AVG(salary) AS avg_salary
        FROM employees
        GROUP BY department_id
    ) t2
);
```

可选 `ALL` 写法：

```sql
SELECT department_id
FROM employees
GROUP BY department_id
HAVING AVG(salary) <= ALL (
    SELECT AVG(salary)
    FROM employees
    GROUP BY department_id
);
```

### 20. `NOT IN` 空值问题

答案：

- `NOT IN` 可以理解成对集合里的每个值都做“不等于”比较
- 如果子查询结果里出现 `NULL`，其中一项比较就会变成未知
- 例如：

```sql
employee_id NOT IN (100, 101, NULL)
```

- 可以理解成：

```sql
employee_id <> 100
AND employee_id <> 101
AND employee_id <> NULL
```

- 最后一项是未知，整条条件就可能无法变成 `TRUE`

## 挑战题

### 21. 相关子查询识别

这题适合相关子查询，因为“本部门平均工资”依赖于外层当前员工所在的部门。

```sql
SELECT last_name, salary, department_id
FROM employees e
WHERE salary > (
    SELECT AVG(salary)
    FROM employees
    WHERE department_id = e.department_id
);
```

### 22. 相关子查询与派生表改写

```sql
SELECT e.last_name, e.salary, e.department_id
FROM employees e
JOIN (
    SELECT department_id, AVG(salary) AS avg_salary
    FROM employees
    GROUP BY department_id
) t
  ON e.department_id = t.department_id
WHERE e.salary > t.avg_salary;
```

说明：

- 上一题是相关子查询
- 这一题是 `FROM` 子查询 / 派生表
- 相关子查询是“外层一行，内层跟着判断”
- 派生表是“先整批算出中间表，再回头连接”

### 23. `ORDER BY` 中的相关子查询

```sql
SELECT employee_id, salary
FROM employees e
ORDER BY (
    SELECT d.department_name
    FROM departments d
    WHERE d.department_id = e.department_id
);
```

### 24. 相关计数过滤

```sql
SELECT employee_id, last_name, job_id
FROM employees e
WHERE (
    SELECT COUNT(*)
    FROM job_history jh
    WHERE jh.employee_id = e.employee_id
) >= 2;
```

### 25. `EXISTS` 基础题

```sql
SELECT employee_id, last_name, job_id, department_id
FROM employees e
WHERE EXISTS (
    SELECT 1
    FROM employees m
    WHERE m.manager_id = e.employee_id
);
```

### 26. `NOT EXISTS` 基础题

```sql
SELECT department_id, department_name
FROM departments d
WHERE NOT EXISTS (
    SELECT 1
    FROM employees e
    WHERE e.department_id = d.department_id
);
```

### 27. 写法比较题

子查询版本：

```sql
SELECT last_name, salary
FROM employees
WHERE salary > (
    SELECT salary
    FROM employees
    WHERE last_name = 'Abel'
);
```

自连接版本：

```sql
SELECT e1.last_name, e1.salary
FROM employees e1
JOIN employees e2
  ON e1.salary > e2.salary
WHERE e2.last_name = 'Abel';
```

思路差异：

- 子查询版：先求 `Abel` 的工资，再比较
- 自连接版：把 `Abel` 当成同表中的另一行来连接比较

### 28. 下手顺序判断

答案：

1. 先求一个明确单值，再给外层比较：
   更适合从里往外写
2. 外层输出列很明确，只差补一个条件：
   更适合从外往里写
3. 相关子查询题：
   通常先把外层主查询框架写出来，再补内层

### 29. 综合改错题

错误原因：子查询按部门分组后返回多行，外层却用单值比较 `>`。

正确写法：

```sql
SELECT last_name, salary, department_id
FROM employees e
WHERE salary > (
    SELECT AVG(salary)
    FROM employees
    WHERE department_id = e.department_id
);
```

### 30. 综合题：从需求到 SQL

为什么 `NOT IN (子查询)` 有风险：

- 因为子查询里可能出现 `NULL`
- 一旦有 `NULL`，`NOT IN` 的判断可能整体失效

更稳妥的写法：

```sql
SELECT e.last_name
FROM employees e
WHERE NOT EXISTS (
    SELECT 1
    FROM employees m
    WHERE m.manager_id = e.employee_id
);
```

说明：

- 这题本质上是查“不是任何人经理的人”
- `NOT EXISTS` 不会像 `NOT IN` 那样被 `NULL` 误伤
