# 第九章_子查询练习题

这份题单只基于第九章现有笔记内容设计，重点练习：

- 子查询的基本概念与常见分类
- 单行子查询与单行比较操作符
- 多行子查询与 `IN`、`ANY`、`ALL`、`SOME`
- 相关子查询、`EXISTS` 与 `NOT EXISTS`
- 子查询、自连接与派生表之间的改写思路

## 使用说明

- 练习数据默认使用仓库中的 `資料/atguigudb.sql`。
- 除题目特别说明外，优先基于 `employees`、`departments`、`job_history` 这些课程表完成练习。
- 这一份只放题目，不放答案；建议先独立完成，再回到第九章对应小节对照。
- 遇到“改错题”时，建议先说明错在哪里，再写出正确 SQL。

## 基础题

### 1. 子查询的定义

说明什么是子查询，并分别解释：

- 什么是外层查询；
- 什么是内层查询；
- 为什么说子查询通常是在“先得到一个中间结果，再交给外层继续使用”。

### 2. 可用位置判断

根据本章前言，说明子查询通常可以出现在 `SELECT` 的哪些常见位置，又通常不出现在什么位置。

### 3. 分类题：按返回结果分类

请分别说明下面四类子查询的返回结果形态：

- 标量子查询
- 列子查询
- 行子查询
- 表子查询

并各举一个典型适用场景。

### 4. 分类题：按是否依赖外层分类

说明什么是不相关子查询，什么是相关子查询，并指出两者在执行方式上的关键差异。

### 5. 需求判断题

“查询工资高于 `Abel` 的员工”这类题目，为什么适合用子查询来写？请你先用自然语言拆解这道题，再说明子查询解决了什么问题。

### 6. 基础单行子查询

查询工资高于 `149` 号员工工资的员工姓名与工资。

要求使用单行子查询完成。

### 7. 双条件单行子查询

查询 `job_id` 与 `141` 号员工相同，且工资高于 `143` 号员工的员工 `last_name`、`job_id`、`salary`。

### 8. 最低工资员工

查询公司工资最低的员工姓名、岗位编号与工资。

要求使用子查询，不要直接只写排序分页。

## 进阶题

### 9. 单行比较操作符判断

说明为什么下面这些操作符通常适合搭配单行子查询：

- `=`
- `>`
- `>=`
- `<`
- `<=`
- `<>`

### 10. 改错题：多行结果误用单行比较

下面 SQL 有问题，请说明原因并改写：

```sql
SELECT employee_id, last_name
FROM employees
WHERE salary = (
    SELECT MIN(salary)
    FROM employees
    GROUP BY department_id
);
```

### 11. 子查询为空时的判断

阅读下面 SQL，说明为什么它可能“语法没错，但查不到结果”：

```sql
SELECT last_name, job_id
FROM employees
WHERE job_id = (
    SELECT job_id
    FROM employees
    WHERE last_name = 'Haas'
);
```

### 12. 成对比较与不成对比较

请分别写出两版 SQL，查询与 `141` 号或 `174` 号员工的 `manager_id`、`department_id` 相同的其他员工：

1. 使用不成对比较；
2. 使用成对比较。

并说明两者语义差异。

### 13. `HAVING` 中的单行子查询

查询最低工资大于 `50` 号部门最低工资的部门编号与该部门最低工资。

要求：

- 必须使用 `GROUP BY`；
- 过滤条件写在 `HAVING` 中；
- `HAVING` 中必须包含子查询。

### 14. `CASE` 中的子查询

根据第九章笔记思路，写一条 SQL：

显示员工的 `employee_id`、`last_name` 和一个自定义字段 `location_tag`。如果员工的 `department_id` 与 `location_id = 1800` 对应部门的 `department_id` 相同，则显示 `Canada`，否则显示 `USA`。

### 15. `IN` 基础题

查询所有在 `IT` 或 `HR` 部门的员工姓名与部门编号。

要求：

- 先用子查询找出部门编号；
- 外层查询使用 `IN`。

### 16. `ANY` 题型

查询其它 `job_id` 中，比 `job_id = 'IT_PROG'` 的员工里任意一个工资都低的员工 `employee_id`、`last_name`、`job_id`、`salary`。

### 17. `ALL` 题型

把上一题改成：

查询其它 `job_id` 中，比 `job_id = 'IT_PROG'` 的员工里所有工资都低的员工 `employee_id`、`last_name`、`job_id`、`salary`。

### 18. `ANY` 与 `ALL` 比较题

说明下面两种条件在语义上有什么差别：

- `salary < ANY (子查询)`
- `salary < ALL (子查询)`

并分别用自然语言翻译这两种条件。

### 19. 最低平均工资部门

查询平均工资最低的部门 `department_id`。

要求：

- 至少写出一种子查询解法；
- 如果你愿意，可以再补一版使用 `ALL` 的写法。

### 20. `NOT IN` 空值问题

阅读下面 SQL，说明为什么它可能无法得到你直觉以为的结果：

```sql
SELECT last_name
FROM employees
WHERE employee_id NOT IN (
    SELECT manager_id
    FROM employees
);
```

请重点解释：子查询结果里如果出现 `NULL`，为什么会影响判断。
