## 本节导读

这一节主要说明如何使用 `DESCRIBE` 或 `DESC` 查看一张表的结构。它不会查询表中的业务数据，而是返回字段名称、字段类型、是否允许为空、键约束、默认值和额外信息等元数据。

第一次阅读时，建议先记住 `DESCRIBE 表名;` 这一种完整写法，再认识 `DESC 表名;` 这个简写。之后如果你在写 `SELECT` 语句前不确定表中有哪些列，就可以先回到这里查看表结构。

## 你会在这篇学到什么

- 如何使用 `DESCRIBE` 查看表结构。
- `DESC` 和 `DESCRIBE` 在查看表结构时的关系。
- `Field`、`Type`、`Null`、`Key`、`Default`、`Extra` 各列分别表示什么。
- 如何根据表结构结果判断字段类型、主键、唯一索引、普通索引和默认值。

## 快速定位

- `4.1 使用 DESCRIBE 或 DESC`：看查看表结构的基本命令和输出示例。
- `4.2 表结构结果说明`：看 `Field`、`Type`、`Null`、`Key`、`Default`、`Extra` 的含义。
- `Key 列的常见取值`：看 `PRI`、`UNI`、`MUL` 分别表示什么。
- `常见混淆点`：看 `DESC` 在不同语境里的不同含义，以及 `NULL`、默认值、索引相关误区。

## 建议阅读顺序

- 第一次学习时，先看 `4.1 使用 DESCRIBE 或 DESC`，记住命令本身，再看 `4.2 表结构结果说明` 理解输出字段。
- 如果你只是想确认某张表有哪些列，先直接执行 `DESC 表名;`，再回来对照 `4.2` 解释结果。
- 如果你在写查询、筛选或排序前不确定字段类型和主键信息，优先回查这篇。
- 如果你看到 `DESC` 不确定到底是在“看表结构”还是“降序排序”，直接看 `常见混淆点`。

## 关键字

- `DESCRIBE`：查看表结构的完整命令。
- `DESC`：`DESCRIBE` 的简写形式。
- `Field`：字段名称。
- `Type`：字段类型。
- `Null`：该字段是否允许存储 `NULL`。
- `Key`：该字段是否参与索引或键约束。
- `Default`：字段默认值。
- `Extra`：字段的额外属性，例如 `AUTO_INCREMENT`。

## 快速回查表

| 需求 | 写法 | 说明 |
| --- | --- | --- |
| 查看表结构 | `DESCRIBE employees;` | 完整写法，语义清楚 |
| 查看表结构 | `DESC employees;` | 简写写法，效果等同于 `DESCRIBE` |
| 查看字段类型 | 看结果中的 `Type` 列 | 例如 `int(6)`、`varchar(20)`、`date` |
| 查看是否允许空值 | 看结果中的 `Null` 列 | `YES` 表示允许，`NO` 表示不允许 |
| 查看主键或索引 | 看结果中的 `Key` 列 | 常见值包括 `PRI`、`UNI`、`MUL` |

## 4.1 使用 DESCRIBE 或 DESC

这一小节先解决最实际的问题：当你拿到一张表、却不知道里面有哪些字段时，第一步该怎么查。

使用 `DESCRIBE` 或 `DESC` 命令，可以查看表结构。

```sql
DESCRIBE employees;
-- 或
DESC employees;
```

示例：

```sql
mysql> DESC employees;
+----------------+-------------+------+-----+---------+-------+
| Field          | Type        | Null | Key | Default | Extra |
+----------------+-------------+------+-----+---------+-------+
| employee_id    | int(6)      | NO   | PRI | 0       |       |
| first_name     | varchar(20) | YES  |     | NULL    |       |
| last_name      | varchar(25) | NO   |     | NULL    |       |
| email          | varchar(25) | NO   | UNI | NULL    |       |
| phone_number   | varchar(20) | YES  |     | NULL    |       |
| hire_date      | date        | NO   |     | NULL    |       |
| job_id         | varchar(10) | NO   | MUL | NULL    |       |
| salary         | double(8,2) | YES  |     | NULL    |       |
| commission_pct | double(2,2) | YES  |     | NULL    |       |
| manager_id     | int(6)      | YES  | MUL | NULL    |       |
| department_id  | int(4)      | YES  | MUL | NULL    |       |
+----------------+-------------+------+-----+---------+-------+
11 rows in set (0.00 sec)
```

### 回查提示

写查询前如果你忘了列名，不要猜。先 `DESC 表名;`，再写 `SELECT`，出错会少很多。

## 4.2 表结构结果说明

`DESCRIBE` 返回的是表的结构信息，不是表中的实际业务记录。以上输出中，各列含义如下：

| 列名 | 含义 | 示例说明 |
| --- | --- | --- |
| `Field` | 字段名称 | `employee_id`、`first_name`、`salary` |
| `Type` | 字段类型 | `int(6)`、`varchar(20)`、`date`、`double(8,2)` |
| `Null` | 该列是否可以存储 `NULL` 值 | `YES` 表示允许，`NO` 表示不允许 |
| `Key` | 该列是否已编制索引或参与键约束 | `PRI`、`UNI`、`MUL` |
| `Default` | 该列是否有默认值，如果有，显示默认值是多少 | `NULL`、`0` 等 |
| `Extra` | 与给定列有关的附加信息 | 例如 `AUTO_INCREMENT` |

### `Key` 列的常见取值

- `PRI`：表示该列是表主键的一部分。
- `UNI`：表示该列是唯一索引的一部分。
- `MUL`：表示该列是普通索引的一部分，列中某个给定值允许出现多次。

### 回查提示

看到 `DESCRIBE` 输出时，先按这个顺序读最省力：`Field` 看字段名，`Type` 看数据类型，`Null` 看能不能存空值，`Key` 看索引或主键，最后再看 `Default` 和 `Extra`。

## 常见混淆点

- `DESC employees;` 在这里是 `DESCRIBE employees;` 的简写，用来显示表结构；但在 `ORDER BY salary DESC` 中，`DESC` 表示降序排序，含义不同。
- `Null` 列显示 `YES` 不代表字段当前值一定是 `NULL`，只代表这个字段允许存储 `NULL`。
- `Default` 列显示 `NULL` 时，表示该字段的默认值是 `NULL`，不等于空字符串。
- `Key` 列为空，不代表字段不能查询，只代表它没有在该结果中显示为主键、唯一索引或普通索引。

## 常见回查问题

- 想查看一张表有哪些字段时应该用什么命令？
- `DESCRIBE` 和 `DESC` 有什么关系？
- `Field`、`Type`、`Null`、`Key`、`Default`、`Extra` 分别代表什么？
- `PRI`、`UNI`、`MUL` 分别表示什么？
- 为什么 `DESC` 有时表示查看表结构，有时表示降序？

## 一句话抓核心

显示表结构的核心是用 `DESCRIBE 表名;` 或 `DESC 表名;` 先查看表的元数据，再根据字段名、类型、是否允许 `NULL`、键约束和默认值来判断后续查询语句应该怎么写。

## 小结

这一节你需要记住：

- `DESCRIBE employees;` 和 `DESC employees;` 都可以查看 `employees` 表结构。
- `Field` 表示字段名，`Type` 表示字段类型。
- `Null` 用来判断字段是否允许存储 `NULL`。
- `Key` 中的 `PRI`、`UNI`、`MUL` 分别表示主键、唯一索引和普通索引相关信息。
- `Default` 表示默认值，`Extra` 表示额外属性，例如 `AUTO_INCREMENT`。
