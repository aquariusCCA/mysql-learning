## 快速回查表

| 排序方式 | 示例语法 | 说明 |
| --- | --- | --- |
| 字段名称排序 | `ORDER BY hire_date ASC` | 常见、直观，适合直接按表字段排序 |
| 列别名排序 | `ORDER BY annsal DESC` | 适合按计算字段或表达式结果排序 |
| 字段位置排序 | `ORDER BY 3 DESC` | `3` 表示查询结果中的第三列；简短但维护风险较高 |

## 1. 按列别名排序

在 `SELECT` 中使用运算或函数时，可以通过 `AS` 为结果列定义别名。定义别名后，MySQL 的 `ORDER BY` 可以直接使用这个别名排序。

```sql
SELECT employee_id, last_name, salary * 12 AS annsal
FROM   employees
ORDER BY annsal DESC;
```

这里的 `annsal` 表示年薪，也就是 `salary * 12` 的计算结果。`ORDER BY annsal DESC` 会根据这个计算结果降序排序。

按别名排序的好处是可读性更好，也能避免在 `ORDER BY` 中重复写较长的表达式：

```sql
SELECT employee_id, last_name, salary * 12 AS annsal
FROM   employees
ORDER BY salary * 12 DESC;
```

上面这种写法也能表达相同排序意图，但当表达式变长时，维护成本会更高。

## 2. 按字段位置排序（不推荐）

除了字段名称和列别名，`ORDER BY` 也可以使用查询结果中的字段位置编号：

```sql
SELECT employee_id, last_name, salary * 12 AS annsal
FROM   employees
ORDER BY 3 DESC;
```

这里的 `ORDER BY 3` 表示按照 `SELECT` 列表中的第三列排序，也就是 `salary * 12 AS annsal` 这一列。

这种写法虽然简洁，但不建议作为常规写法。它依赖 `SELECT` 列表的字段顺序，一旦查询列调整位置，排序依据就可能跟着改变。实际写 SQL 时，优先使用字段名称或列别名会更清楚。

## 常见混淆点

- `ORDER BY annsal`：按列别名排序，语意清楚，适合排序计算字段。
- `ORDER BY 3`：按查询结果中的第三列排序，依赖字段顺序，维护风险较高。
- `salary * 12 AS annsal`：`AS` 是定义别名，`annsal` 是别名本身。

## 一句话抓核心

根据别名排序的核心是：给计算字段取一个清楚的别名，再在 `ORDER BY` 中使用它；字段位置排序虽然能写，但不如字段名或别名稳定。
