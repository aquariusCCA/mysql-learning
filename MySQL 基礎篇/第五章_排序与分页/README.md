# 第五章_排序与分页

这一章整理 MySQL 查询结果的顺序控制与分页处理。排序用于让查询结果按指定字段与方向稳定展示；分页用于控制一次返回多少数据，以及从结果集的哪一段开始返回。

本章不只整理 `ORDER BY` 与 `LIMIT` 的基础语法，也会补充实务中常见的稳定排序、稳定分页、`NULL` 排序、大偏移量分页效能、游标分页，以及不同数据库分页写法的差异。

## 本章在整体学习地图中的位置

- 第三章主要建立了 `SELECT`、`WHERE` 和表结构查看的基础查询骨架。
- 第四章补上了表达式、比较判断、逻辑判断等条件查询所需的运算符。
- 这一章开始处理查询结果“怎么展示、怎么截取”的问题，也就是排序和分页，把“查什么”进一步推进到“怎么稳定展示、怎么分段返回”。
- 后续学习索引、执行计划和 SQL 优化时，排序字段、分页条件、偏移量大小都会影响查询性能，因此本章也是进入 SQL 实务优化前的重要基础。

## 本章学习目标

学完本章后，应该能够做到：

- 知道为什么只要业务关心顺序，就不能依赖数据库默认返回顺序。
- 掌握 `ORDER BY` 的基本位置、升序、降序、单列排序和多列排序。
- 看懂 `ORDER BY` 使用列别名、查询列位置编号的写法，并知道哪些写法不适合长期维护。
- 理解多列排序的优先级规则，以及为什么实务上常加唯一字段让排序结果更稳定。
- 知道 MySQL 中 `NULL` 在升序、降序排序时的默认位置。
- 掌握 `LIMIT` 的偏移量、行数、`OFFSET` 写法和分页公式。
- 理解分页前为什么通常要先排序，以及不稳定排序会导致分页结果重复或遗漏的问题。
- 初步认识大偏移量分页的效能问题，以及 Keyset Pagination / Seek Pagination 的基本思路。
- 能分辨 MySQL、PostgreSQL、SQL Server、DB2、Oracle 等数据库常见分页语法差异。

## 本章常见 SQL 骨架

排序和分页通常出现在 `SELECT` 语句靠后的位置，常见顺序如下：

```sql
SELECT   查询字段
FROM     表名
WHERE    过滤条件
GROUP BY 分组字段
HAVING   分组后过滤条件
ORDER BY 排序字段
LIMIT    位置偏移量, 行数;
```

需要特别记住：

- `ORDER BY` 用来决定结果顺序。
- `LIMIT` 用来截取结果范围。
- 同时使用排序和分页时，通常是先 `ORDER BY`，再 `LIMIT`。

## 本章常见错误

- 业务需要固定顺序，却没有写 `ORDER BY`。
- 只写 `LIMIT` 做分页，却没有先指定稳定排序规则。
- 多列排序时误以为所有排序字段会“同时平均生效”，没有理解从左到右逐级排序。
- 长期使用 `ORDER BY 3` 这类位置排序，导致 `SELECT` 字段顺序调整后排序逻辑被意外改变。
- 把 `LIMIT 10, 20` 误解成“第 10 页，每页 20 条”，实际上它表示“从偏移量 10 开始，最多取 20 条”。
- 在大量数据场景下长期使用很大的 offset，例如 `LIMIT 100000, 10`，没有意识到可能存在效能问题。
- 看到其他数据库没有 `LIMIT` 就以为写错，实际上分页语法会因数据库而异。
- 使用 Oracle 传统 `ROWNUM` 写法时，没有注意“先排序再取前几笔”和“先取前几笔再排序”的差异。

## 本章一句话总结

排序与分页的核心是：先用 `ORDER BY` 明确决定稳定顺序，再用 `LIMIT` 指定从哪一条开始、取多少条；当数据量变大时，还要进一步考虑排序字段、唯一兜底字段、索引和大偏移量分页的效能问题。
