# 第五章_排序与分页

这一章开始整理 MySQL 查询结果的顺序控制与分页处理。排序用于让查询结果按指定字段稳定展示；分页则用于控制一次返回多少数据，以及从哪一段结果开始返回。

目前已整理第 1 节，先从 `ORDER BY` 的排序规则开始，理解单列排序、别名排序、字段位置排序和多列排序。后续补齐分页内容后，再把排序与分页放在一起理解。

## 本章主要内容

- `ORDER BY` 子句的基本写法。
- `ASC` 与 `DESC` 的含义。
- 单列排序与多列排序。
- 使用 `SELECT` 中的列别名排序。
- 使用查询列位置编号排序，以及这种写法的维护风险。
- 查询结果默认顺序不稳定时，为什么需要显式排序。
- 后续将继续整理分页查询相关内容。

## 建议阅读顺序

1. 先看：[1 排序数据](./1%20排序数据.md)，掌握 `ORDER BY` 的基本写法与排序方向。

4. 回到第三章的 `SELECT` 基础，把 `ORDER BY` 接在完整查询语句中理解。
