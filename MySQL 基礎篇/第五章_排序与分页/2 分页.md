## 本节导读

这一节主要说明如何在 MySQL 中使用分页查询。分页的核心是：不要一次把所有记录都显示出来，而是只取当前需要查看的那一小段结果。

第一次阅读时，建议先理解“偏移量 + 行数”这组规则，再看常见写法、分页公式和 `LIMIT ... OFFSET ...` 的等价写法。复习时可以重点回查 `LIMIT` 的语法格式、第一页为什么从 `0` 开始，以及 `ORDER BY` 和 `LIMIT` 的先后位置。

- `LIMIT 4, 3` 和 `LIMIT 3 OFFSET 4` 为什么等价。
- 如何用公式快速写出“第 N 页、每页 M 条”的分页 SQL。
- 不同数据库在分页关键字上的常见差异。

## 关键字

- `分页`：把一整批查询结果分成多段显示。
- `LIMIT`：MySQL 中限制返回行数的分页子句。
- `位置偏移量`：从第几条记录开始取数据，第一条记录的偏移量是 `0`。
- `行数`：本次查询要返回多少条记录。
- `OFFSET`：另一种显式表示偏移量的写法。
- `PageNo`：当前页码。
- `PageSize`：每页显示的记录数。

## 快速回查表

| 场景 | 写法 | 说明 |
| --- | --- | --- |
| 取前 10 条记录 | `LIMIT 10` | 等价于 `LIMIT 0, 10` |
| 取第 11 至 20 条记录 | `LIMIT 10, 10` | 从偏移量 `10` 开始，再取 `10` 条 |
| 取第 5 条之后的 3 条记录 | `LIMIT 4, 3` | 偏移量从 `0` 开始计数 |
| 用显式偏移量写法取数据 | `LIMIT 3 OFFSET 4` | 与 `LIMIT 4, 3` 等价 |
| 写第 `PageNo` 页数据 | `LIMIT (PageNo - 1) * PageSize, PageSize` | 常见分页公式 |
| 只需要一条记录 | `LIMIT 1` | 可减少返回量，也常用于“只取一条”场景 |

## 2.1 分页背景

分页常见于两类场景：

- 查询返回的记录太多，一次全部显示出来不方便查看。
- 明明表里有很多数据，但当前页面只想显示其中一小段，例如第 `2`、`3` 条记录，或者第 `11` 至 `20` 条记录。

所谓分页显示，就是将数据库中的结果集按段切开，每次只显示当前需要的那一部分。

### 回查提示

当你已经会写查询，但结果太多不方便展示时，就应该想到分页，而不是把所有数据一次查出来再手动挑选。

## 2.2 MySQL 中使用 LIMIT 实现分页

MySQL 中常用 `LIMIT` 子句实现分页，基本格式如下：

```sql
LIMIT [位置偏移量,] 行数
```

- 第一个参数“位置偏移量”表示从哪一行开始显示，是可选参数。
- 如果不指定偏移量，则默认从第一条记录开始返回。
- 第一条记录的位置偏移量是 `0`，第二条记录的偏移量是 `1`，依此类推。
- 第二个参数“行数”表示本次最多返回多少条记录。

> **注意：** `LIMIT` 子句必须放在整个 `SELECT` 语句的最后；如果同时使用 `ORDER BY`，应先排序，再分页。

### 回查提示

只要你分不清 `LIMIT 10, 20` 里的两个数字分别代表什么，就回到这里先确认“偏移量在前，行数在后”。

## 2.3 常见分页写法

下面是几种最常见的分页写法：

```sql
-- 前 10 条记录
SELECT * FROM 表名 LIMIT 0, 10;

-- 前 10 条记录的简写
SELECT * FROM 表名 LIMIT 10;

-- 第 11 至 20 条记录
SELECT * FROM 表名 LIMIT 10, 10;

-- 第 21 至 30 条记录
SELECT * FROM 表名 LIMIT 20, 10;
```

在 MySQL 8.0 中，还可以写成更直观的 `OFFSET` 形式：

```sql
SELECT * FROM 表名 LIMIT 3 OFFSET 4;
```

这表示从第 `5` 条记录开始，返回后面的 `3` 条记录，和下面这条语句等价：

```sql
SELECT * FROM 表名 LIMIT 4, 3;
```

### 回查提示

如果你只是在做“前几条”“某一段区间”这类分页，直接套这里的写法通常就够了。

## 2.4 分页公式

分页最常用的公式是：

```sql
SELECT *
FROM table
LIMIT (PageNo - 1) * PageSize, PageSize;
```

这里：

- `PageNo` 表示当前页码。
- `PageSize` 表示每页显示多少条记录。
- `(PageNo - 1) * PageSize` 用来算出当前页第一条记录的偏移量。

例如：

- 第 `1` 页、每页 `10` 条，对应 `LIMIT 0, 10`
- 第 `2` 页、每页 `10` 条，对应 `LIMIT 10, 10`
- 第 `3` 页、每页 `10` 条，对应 `LIMIT 20, 10`

### 回查提示

当业务需求不是“前 10 条”，而是“第 N 页，每页 M 条”时，直接套公式比手算更稳。

## 2.5 其他数据库的分页写法

不同数据库管理系统使用的分页关键字可能不同。本节重点是 MySQL 的 `LIMIT`，但下面这些写法也值得知道：

- MySQL、PostgreSQL、MariaDB 和 SQLite 常见使用 `LIMIT`，而且通常放在 `SELECT` 语句最后。
- SQL Server 和 Access 常见使用 `TOP`：

```sql
SELECT TOP 5 name, hp_max
FROM heros
ORDER BY hp_max DESC;
```

- DB2 常见使用 `FETCH FIRST ... ROWS ONLY`：

```sql
SELECT name, hp_max
FROM heros
ORDER BY hp_max DESC
FETCH FIRST 5 ROWS ONLY;
```

- Oracle 传统写法常结合 `ROWNUM`：

```sql
SELECT rownum, last_name, salary
FROM employees
WHERE rownum < 5
ORDER BY salary DESC;
```

### 回查提示

如果你发现别人的分页 SQL 里根本没有 `LIMIT`，先别急着判断写错了，先确认对方用的是哪一种数据库。

## 使用 LIMIT 的价值

合理使用 `LIMIT` 有两个直接好处：

- 可以减少一次返回的数据量，降低结果传输成本。
- 如果业务上只需要少量记录，例如只取一条结果，`LIMIT 1` 往往能减少不必要的扫描和返回。

不过需要注意，分页只是在“结果展示”层面控制返回数量；如果你还关心结果的顺序是否稳定，仍然应该搭配 `ORDER BY` 一起使用。

## 常见混淆点

- `LIMIT` 的第一个数字不是“第几条记录”，而是偏移量；它从 `0` 开始计数。
- `LIMIT 10` 和 `LIMIT 0, 10` 是等价写法。
- 如果查询结果需要稳定顺序，分页前最好先写 `ORDER BY`，否则不同页之间的结果可能不稳定。
- `LIMIT` 在 MySQL 中通常写在 `SELECT` 语句最后。
- 不同数据库的分页语法不统一，跨数据库迁移时不要直接照抄。

## 一句话抓核心

分页的核心是：先确定结果顺序，再用 `LIMIT` 指定“从哪一条开始、取多少条”，把大结果集切成可控制的小段返回。

## 小结

- MySQL 中分页通常使用 `LIMIT [位置偏移量,] 行数`。
- 偏移量从 `0` 开始，第一页通常对应 `LIMIT 0, PageSize`。
- `LIMIT n OFFSET m` 只是 `LIMIT m, n` 的另一种写法。
- 业务上按页查询时，可以直接套用 `(PageNo - 1) * PageSize` 的公式。
- 如果分页结果需要稳定顺序，应和 `ORDER BY` 一起使用。
