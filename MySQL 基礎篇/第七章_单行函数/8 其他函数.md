# 8 其他函数

> 所属章节：[第七章_单行函数](./README.md)
> 关键字：其他函数、`FORMAT()`、`CONV()`、`INET_ATON()`、`INET_NTOA()`、`BENCHMARK()`、`CONVERT()`
> 建议回查情境：想快速确认数字格式化、进制转换、IP 与数字互转、SQL 表达式性能测试，或字符串字符编码转换的函数写法时

## 本节导读

这一节整理一些不容易直接归到数值、字符串、日期或信息函数里的常用函数。虽然它们类别分散，但在实际开发和运维中都很常见，例如格式化数字、做不同进制间转换、处理 IP 地址、粗略测试表达式执行成本，或把字符串转换成指定字符集。

第一次阅读时，建议先按“格式与转换”“网络地址处理”“性能测试”三类来记，再用示例建立印象。复习时如果只是忘了“某个特殊函数该去哪里查”，可以直接看“快速定位”和函数总表。

## 你会在这篇学到什么

- `FORMAT()`、`CONV()`、`INET_ATON()`、`INET_NTOA()`、`BENCHMARK()`、`CONVERT()` 分别解决什么问题。
- 哪些函数适合处理显示格式，哪些适合做底层值转换。
- `INET_ATON()` 与 `INET_NTOA()` 为什么会互为反函数。
- `BENCHMARK()` 在 MySQL 中的基本用途，以及它的结果该怎么理解。

## 快速定位

- 想格式化数字并控制小数位：看 `FORMAT()`
- 想在不同进制之间转换数值：看 `CONV()`
- 想把点分十进制 IP 转成数字：看 `INET_ATON()`
- 想把数字形式 IP 还原成点分十进制：看 `INET_NTOA()`
- 想重复执行表达式来观察耗时：看 `BENCHMARK()`
- 想把字符串转换成指定字符编码：看 `CONVERT(value USING char_code)`

## 先分清三类用途

### 格式与值转换

`FORMAT()`、`CONV()` 和 `CONVERT()` 都和“值的表现形式改变”有关，但方向不同：有的是让显示结果更易读，有的是在不同进制之间切换，有的是调整字符串的字符编码。

### 网络地址处理

`INET_ATON()` 与 `INET_NTOA()` 用于在 IP 地址的两种常见表示形式之间来回转换，一种是人容易阅读的点分十进制形式，一种是方便计算和存储的数字形式。

### 性能测试

`BENCHMARK()` 的作用不是返回表达式本身的值，而是把同一表达式重复执行很多次，用来观察 MySQL 处理该表达式大致需要多少时间。

## 其他函数总表

| 函数 | 用法 |
| --- | --- |
| `FORMAT(value, n)` | 返回对数字 `value` 进行格式化后的结果；`n` 表示四舍五入后保留到小数点后 `n` 位 |
| `CONV(value, from, to)` | 将 `value` 在不同进制之间进行转换 |
| `INET_ATON(ipvalue)` | 将以点分隔的 IP 地址转化为一个数字 |
| `INET_NTOA(value)` | 将数字形式的 IP 地址转化为以点分隔的 IP 地址 |
| `BENCHMARK(n, expr)` | 将表达式 `expr` 重复执行 `n` 次，用于测试 MySQL 处理 `expr` 所耗费的时间 |
| `CONVERT(value USING char_code)` | 将 `value` 所使用的字符编码修改为 `char_code` |

## 使用提醒

- `FORMAT()` 主要用于控制显示结果，不适合拿来替代严谨的数值计算逻辑。
- `INET_ATON()` 与 `INET_NTOA()` 是一组配对函数，适合在 IP 文本值与数字值之间来回转换。
- `BENCHMARK()` 的返回值通常是 `0`，重点不在结果本身，而在客户端显示的耗时。
- `CONVERT(value USING char_code)` 处理的是字符串编码转换，常和 `CHARSET()` 一起配合查看效果。

## 示例

### `FORMAT()`

```sql
# 如果 n 的值小于或者等于 0，则只保留整数部分
SELECT FORMAT(123.123, 2), FORMAT(123.523, 0), FORMAT(123.123, -2);

+--------------------+--------------------+---------------------+
| FORMAT(123.123, 2) | FORMAT(123.523, 0) | FORMAT(123.123, -2) |
+--------------------+--------------------+---------------------+
| 123.12             | 124                | 123                 |
+--------------------+--------------------+---------------------+
1 row in set (0.00 sec)
```

### `CONV()`

```sql
SELECT CONV(16, 10, 2), CONV(8888, 10, 16), CONV(NULL, 10, 2);

+-----------------+------------------+-------------------+
| CONV(16, 10, 2) | CONV(8888, 10, 16) | CONV(NULL, 10, 2) |
+-----------------+------------------+-------------------+
| 10000           | 22B8             | NULL              |
+-----------------+------------------+-------------------+
1 row in set (0.00 sec)
```

### `INET_ATON()`

```sql
SELECT INET_ATON('192.168.1.100');

+----------------------------+
| INET_ATON('192.168.1.100') |
+----------------------------+
|                 3232235876 |
+----------------------------+
1 row in set (0.00 sec)
```

以 `192.168.1.100` 为例，计算方式是：

- `192 * 256^3`
- `168 * 256^2`
- `1 * 256`
- `100`

最后再把这些结果相加。

### `INET_NTOA()`

```sql
SELECT INET_NTOA(3232235876);

+-----------------------+
| INET_NTOA(3232235876) |
+-----------------------+
| 192.168.1.100         |
+-----------------------+
1 row in set (0.00 sec)
```

### `BENCHMARK()`

```sql
SELECT BENCHMARK(1, MD5('mysql'));

+----------------------------+
| BENCHMARK(1, MD5('mysql')) |
+----------------------------+
|                          0 |
+----------------------------+
1 row in set (0.00 sec)

SELECT BENCHMARK(1000000, MD5('mysql'));

+----------------------------------+
| BENCHMARK(1000000, MD5('mysql')) |
+----------------------------------+
|                                0 |
+----------------------------------+
1 row in set (0.20 sec)
```

这个例子可以看出：`BENCHMARK()` 的结果值本身仍然是 `0`，真正有参考价值的是后面显示的执行时间。

### `CONVERT()`

```sql
SELECT CHARSET('mysql'), CHARSET(CONVERT('mysql' USING 'utf8'));

+------------------+----------------------------------------+
| CHARSET('mysql') | CHARSET(CONVERT('mysql' USING 'utf8')) |
+------------------+----------------------------------------+
| utf8mb4          | utf8                                   |
+------------------+----------------------------------------+
1 row in set, 1 warning (0.00 sec)
```

## 典型回查场景

- 报表或展示结果想控制小数位：先查 `FORMAT()`
- 需要处理二进制、十进制、十六进制之间的转换：先查 `CONV()`
- 想把 IP 地址存成数字，或把数字还原成可读 IP：看 `INET_ATON()`、`INET_NTOA()`
- 想粗略比较两个表达式的执行成本：看 `BENCHMARK()`
- 怀疑字符串编码有问题，想临时转换后观察结果：看 `CONVERT()` 配合 `CHARSET()`

## 常见混淆点

- `FORMAT()` 会改变显示形式，不等于单纯做数值运算。
- `CONVERT(value USING char_code)` 在这里处理的是字符编码，不要和类型转换的语境混在一起。
- `BENCHMARK()` 不是用来取得表达式计算结果，而是用来观察重复执行的耗时。
- `INET_ATON()` 和 `INET_NTOA()` 是互逆关系，一个转成数字，一个转回 IP 字符串。
