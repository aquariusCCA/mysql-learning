# 6 加密与解密函数

> 所属章节：[第七章_单行函数](./README.md)
> 关键字：加密函数、解密函数、`PASSWORD()`、`MD5()`、`SHA()`、`ENCODE()`、`DECODE()`、单向处理、可逆处理
> 建议回查情境：忘了 `PASSWORD()`、`MD5()`、`SHA()`、`ENCODE()`、`DECODE()` 的区别，不确定某个结果能不能还原原文，或想快速确认函数写法时

## 本节导读

这一节整理 MySQL 中常见的加密与解密函数。学习重点不是只记函数名，而是先分清两类用途：一类是把原值处理成不可直接还原的结果，例如 `PASSWORD()`、`MD5()`、`SHA()`；另一类是成对出现、可以用同一密钥再解回来的 `ENCODE()` 与 `DECODE()`。

第一次阅读时，建议先看“先分清两类处理方式”和下方函数总表，再看示例感受各函数返回结果的差异。复习时如果只是忘了“这个函数能不能还原原文”，可以直接看“快速定位”和“使用提醒”。

## 你会在这篇学到什么

- `PASSWORD()`、`MD5()`、`SHA()` 和 `ENCODE()`、`DECODE()` 分别适合什么类型的问题。
- 为什么有些函数的处理结果不能直接还原原文，有些则可以。
- `ENCODE()` 和 `DECODE()` 为什么会互为反函数。
- 这些函数在 SQL 中的基本调用方式与输出示例。

## 快速定位

- 想看不可逆的字符串处理函数：看 `PASSWORD()`、`MD5()`、`SHA()`
- 想看可逆的加解密配对函数：看 `ENCODE()`、`DECODE()`
- 想快速确认哪一个相对更安全：看 `SHA()` 与 `MD5()` 的对比说明
- 想直接看函数执行结果：看“示例”

## 先分清两类处理方式

### 不可逆处理

`PASSWORD()`、`MD5()`、`SHA()` 都是把原字符串处理成另一种结果，重点在于结果通常不是拿来再还原成原文，而是用于保存处理后的值，再做比对或校验。

其中教材中的结论是：`SHA()` 相比 `MD5()` 更安全。

### 可逆处理

`ENCODE(value, password_seed)` 与 `DECODE(value, password_seed)` 是一组配对函数。它们需要使用同一个密码种子 `password_seed`，先编码，再用相同密码种子解回原值，因此两者互为反函数。

## 加密与解密函数总表

| 函数 | 用法 |
| --- | --- |
| `PASSWORD(str)` | 返回字符串 `str` 的加密版本，结果为 `41` 位长字符串；加密结果不可逆，常用于用户密码加密 |
| `MD5(str)` | 返回字符串 `str` 的 `MD5` 加密结果；若参数为 `NULL`，则返回 `NULL` |
| `SHA(str)` | 从原明文密码 `str` 计算并返回加密后的密码字符串；若参数为 `NULL`，则返回 `NULL`，并且 `SHA` 比 `MD5` 更安全 |
| `ENCODE(value, password_seed)` | 返回使用 `password_seed` 作为加密密码后的 `value` |
| `DECODE(value, password_seed)` | 返回使用 `password_seed` 作为解密密码后的 `value` |

## 使用提醒

- `PASSWORD()`、`MD5()`、`SHA()` 这类函数更适合用来“处理后保存结果”，而不是期待之后再把结果直接还原回原字符串。
- `ENCODE()` 与 `DECODE()` 必须搭配相同的 `password_seed` 才能正常还原。
- 当参数是 `NULL` 时，`MD5()` 与 `SHA()` 会返回 `NULL`。

## 示例

### `PASSWORD()`

```sql
SELECT PASSWORD('mysql'), PASSWORD(NULL);

+-------------------------------------------+----------------+
| PASSWORD('mysql')                         | PASSWORD(NULL) |
+-------------------------------------------+----------------+
| *E74858DB86EBA20BC33D0AECAE8A8108C56B17FA |                |
+-------------------------------------------+----------------+
1 row in set, 1 warning (0.00 sec)
```

### `MD5()`

```sql
SELECT MD5('123');

-> 202cb962ac59075b964b07152d234b70
```

### `SHA()`

```sql
SELECT SHA('Tom123');

-> c7c506980abc31cc390a2438c90861d0f1216d50
```

### `ENCODE()`

```sql
SELECT ENCODE('mysql', 'mysql');

+--------------------------+
| ENCODE('mysql', 'mysql') |
+--------------------------+
| íg　¼　ìÉ                  |
+--------------------------+
1 row in set, 1 warning (0.01 sec)
```

### `DECODE()`

```sql
SELECT DECODE(ENCODE('mysql', 'mysql'), 'mysql');

+-----------------------------------------+
| DECODE(ENCODE('mysql', 'mysql'), 'mysql') |
+-----------------------------------------+
| mysql                                   |
+-----------------------------------------+
1 row in set, 2 warnings (0.00 sec)
```

这个例子可以看出：先使用 `ENCODE()` 处理原值，再用相同的密码种子调用 `DECODE()`，就能把结果还原回 `mysql`。

## 常见混淆点

- `MD5()` 和 `SHA()` 都是对字符串做不可逆处理，差别不在“能不能解回去”，而在处理算法不同。
- `ENCODE()` / `DECODE()` 和 `MD5()` / `SHA()` 不是同一类用途，前者是可逆配对，后者更偏向单向处理。
- `PASSWORD()`、`MD5()`、`SHA()` 的调用方式看起来都很像，但用途上要先区分“保存处理结果”和“之后需要解回原文”。
