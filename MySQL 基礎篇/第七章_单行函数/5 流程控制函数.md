## 本节导读

流程控制函数的核心作用，是让 SQL 查询不只是「把字段原样查出来」，而是可以根据不同条件返回不同结果。

例如：

- 如果金额为 `NULL`，显示为 `0`。
- 如果订单状态是 `1`，显示为「未付款」。
- 如果薪资大于等于 `15000`，显示为「高薪」。
- 如果部门是 `10`，薪资乘以 `1.1`；如果部门是 `20`，薪资乘以 `1.2`。
- 如果某个栏位是空字符串，统一当成 `NULL`。
- 如果要统计不同状态的订单数量，可以在聚合函数中搭配 `CASE WHEN`。

因此，这一节不是单纯背 `IF()`、`IFNULL()`、`CASE` 的语法，而是要学会判断：

> 当前需求到底是「简单二选一」、「NULL 兜底」、「多条件分类」、「固定代码映射」，还是「条件聚合统计」。

第一次学习时，建议先分清三件事：

1. `IF()`：适合简单二选一。
2. `IFNULL()`：适合处理 `NULL` 兜底。
3. `CASE`：适合多条件分类或固定值映射。

## 5.1 流程控制函数是什么

流程控制函数可以理解为 SQL 查询中的「条件判断工具」。

一般查询是直接取字段：

```sql
SELECT
    employee_id,
    salary
FROM employees;
```

如果加上流程控制函数，就可以根据条件返回不同结果：

```sql
SELECT
    employee_id,
    salary,
    CASE
        WHEN salary >= 15000 THEN '高薪'
        WHEN salary >= 10000 THEN '中高薪'
        ELSE '一般'
    END AS salary_level
FROM employees;
```

这段 SQL 的意思是：

```text
如果 salary >= 15000，显示「高薪」；
否则如果 salary >= 10000，显示「中高薪」；
否则显示「一般」。
```

所以流程控制函数最常见的用途有四类：

| 类型 | 说明 | 示例 |
| --- | --- | --- |
| 二选一 | 条件成立回传 A，否则回传 B | `IF(salary >= 10000, '高', '低')` |
| 空值兜底 | `NULL` 改成默认值 | `IFNULL(commission_pct, 0)` |
| 多条件分类 | 根据多个条件返回不同结果 | `CASE WHEN salary >= 15000 THEN ...` |
| 固定值映射 | 把代码值转成文字说明 | `CASE status WHEN 1 THEN '未付款' ...` |

---

## 5.2 流程控制函数总表

| 函数 / 写法 | 用法 | 适合场景 |
| --- | --- | --- |
| `IF(condition, value_if_true, value_if_false)` | 如果条件成立，返回第二个参数；否则返回第三个参数 | 简单二选一 |
| `IFNULL(value, default_value)` | 如果 `value` 不是 `NULL`，返回 `value`；否则返回默认值 | 空值兜底 |
| `NULLIF(value1, value2)` | 如果 `value1 = value2`，返回 `NULL`；否则返回 `value1` | 把特定值转成空值、避免除以零 |
| `CASE WHEN 条件1 THEN 结果1 WHEN 条件2 THEN 结果2 ... ELSE 结果n END` | 按条件依次判断，命中后返回对应结果 | 多条件分类、区间判断 |
| `CASE expr WHEN 值1 THEN 结果1 WHEN 值2 THEN 结果2 ... ELSE 结果n END` | 用同一个表达式依次比较不同固定值 | 状态码、等级代码、部门编号映射 |

### 记忆方式

```text
IF()：简单二选一。
IFNULL()：NULL 兜底。
NULLIF()：相同转 NULL。
CASE WHEN：多条件判断。
CASE expr WHEN：固定值映射。
```

## 5.3 IF()

`IF()` 适合最简单的二选一场景。

### 5.3.1 语法

```sql
IF(condition, value_if_true, value_if_false)
```

含义：

```text
如果 condition 成立，返回 value_if_true；
如果 condition 不成立，返回 value_if_false。
```

---

### 5.3.2 基本示例

```sql
SELECT IF(1 > 0, '正确', '错误') AS result;
```

结果：

```text
正确
```

再看一个字段示例：

```sql
SELECT
    employee_id,
    salary,
    IF(salary >= 10000, '高薪', '一般') AS salary_type
FROM employees;
```

这段 SQL 会根据每一位员工的薪资，输出「高薪」或「一般」。

---

### 5.3.3 IF() 适合什么场景

`IF()` 最适合只有两个分支的判断：

```text
是 / 否
通过 / 不通过
有效 / 无效
高 / 低
有资料 / 无资料
```

例如：

```sql
SELECT
    order_id,
    total_amount,
    IF(total_amount >= 1000, '大额订单', '一般订单') AS order_type
FROM orders;
```

---

### 5.3.4 IF() 不适合什么场景

如果判断条件开始变多，不建议一直嵌套 `IF()`。

例如：

```sql
SELECT
    IF(score >= 90, 'A',
        IF(score >= 80, 'B',
            IF(score >= 70, 'C', 'D')
        )
    ) AS grade
FROM scores;
```

这段 SQL 可以执行，但可读性较差。更建议改成 `CASE WHEN`：

```sql
SELECT
    CASE
        WHEN score >= 90 THEN 'A'
        WHEN score >= 80 THEN 'B'
        WHEN score >= 70 THEN 'C'
        ELSE 'D'
    END AS grade
FROM scores;
```

### 使用原则

```text
两个分支：可以用 IF()。
三个以上分支：优先考虑 CASE。
```

## 5.4 IFNULL()

`IFNULL()` 专门用来处理 `NULL`。

### 5.4.1 语法

```sql
IFNULL(value, default_value)
```

含义：

```text
如果 value 不是 NULL，返回 value；
如果 value 是 NULL，返回 default_value。
```

---

### 5.4.2 基本示例

```sql
SELECT IFNULL(NULL, 'Hello World') AS result;
```

结果：

```text
Hello World
```

```sql
SELECT IFNULL('MySQL', 'Hello World') AS result;
```

结果：

```text
MySQL
```

---

### 5.4.3 为什么计算时经常需要 IFNULL()

在 SQL 中，只要运算中出现 `NULL`，结果通常也会变成 `NULL`。

例如：

```sql
SELECT 1000 * (1 + NULL) AS result;
```

结果：

```text
NULL
```

如果佣金比例 `commission_pct` 可能为 `NULL`，直接计算年薪可能会出问题：

```sql
SELECT
    employee_id,
    12 * salary * (1 + commission_pct) AS annual_salary
FROM employees;
```

只要 `commission_pct` 是 `NULL`，结果就可能变成 `NULL`。

更稳的写法是：

```sql
SELECT
    employee_id,
    12 * salary * (1 + IFNULL(commission_pct, 0)) AS annual_salary
FROM employees;
```

意思是：

```text
如果 commission_pct 是 NULL，就当成 0。
```

---

### 5.4.4 IFNULL() 常见场景

| 场景 | 示例 |
| --- | --- |
| 金额为空改成 0 | `IFNULL(amount, 0)` |
| 佣金为空改成 0 | `IFNULL(commission_pct, 0)` |
| 备注为空改成默认文字 | `IFNULL(remark, '无备注')` |
| 昵称为空改成账号 | `IFNULL(nickname, username)` |
| 聚合结果为空改成 0 | `IFNULL(SUM(amount), 0)` |

示例：

```sql
SELECT
    customer_id,
    IFNULL(nickname, username) AS display_name
FROM users;
```

## 5.5 NULLIF()

`NULLIF()` 的作用和 `IFNULL()` 刚好有点相反。

- `IFNULL()`：把 `NULL` 变成默认值。
- `NULLIF()`：把特定值变成 `NULL`。

### 5.5.1 语法

```sql
NULLIF(value1, value2)
```

含义：

```text
如果 value1 = value2，返回 NULL；
否则返回 value1。
```

---

### 5.5.2 基本示例

```sql
SELECT
    NULLIF('mysql', 'mysql') AS r1,
    NULLIF('mysql', '')      AS r2;
```

结果：

| 表达式 | 结果 |
| --- | --- |
| `NULLIF('mysql', 'mysql')` | `NULL` |
| `NULLIF('mysql', '')` | `'mysql'` |

---

### 5.5.3 实务场景：空字符串转 NULL

很多旧系统资料会同时存在：

```text
NULL
''
'   '
```

如果业务上它们都代表「没有填写」，可以用 `TRIM()` + `NULLIF()` 处理：

```sql
SELECT
    user_id,
    NULLIF(TRIM(nickname), '') AS clean_nickname
FROM users;
```

处理逻辑：

```text
先去掉首尾空格；
如果结果是空字符串，就转成 NULL；
否则保留清理后的值。
```

---

### 5.5.4 实务场景：避免除以零

如果分母可能为 `0`，可以用 `NULLIF()` 把 `0` 转成 `NULL`，避免直接除以零。

```sql
SELECT
    product_id,
    sales_amount / NULLIF(sales_count, 0) AS avg_price
FROM product_sales;
```

意思是：

```text
如果 sales_count = 0，就变成 sales_amount / NULL，结果为 NULL；
如果 sales_count 不等于 0，就正常计算平均价格。
```

## 5.6 先分清两种 CASE 写法

`CASE` 是 SQL 中最重要的条件判断写法，尤其适合多分支逻辑。

MySQL 中常见的 `CASE` 写法有两种：

1. `CASE WHEN 条件 THEN 结果`
2. `CASE expr WHEN 值 THEN 结果`

这两种写法看起来像，但思路不同。

---

### 5.6.1 CASE WHEN 条件 THEN 结果

语法：

```sql
CASE
    WHEN 条件1 THEN 结果1
    WHEN 条件2 THEN 结果2
    ELSE 结果n
END
```

重点：

```text
每个 WHEN 后面都是一个完整条件。
```

适合：

- 区间判断。
- 多个不同条件。
- 条件中包含大于、小于、范围、`IS NULL`、多个字段比较。

例如：

```sql
CASE
    WHEN salary >= 15000 THEN '高薪'
    WHEN salary >= 10000 THEN '中高薪'
    ELSE '一般'
END
```

---

### 5.6.2 CASE expr WHEN 值 THEN 结果

语法：

```sql
CASE expr
    WHEN 值1 THEN 结果1
    WHEN 值2 THEN 结果2
    ELSE 结果n
END
```

重点：

```text
CASE 后面先放一个表达式，后面的 WHEN 只写要比较的固定值。
```

适合：

- 状态码映射。
- 部门编号映射。
- 等级代码映射。
- 固定枚举值转换成文字。

例如：

```sql
CASE status
    WHEN 1 THEN '未付款'
    WHEN 2 THEN '已付款'
    WHEN 3 THEN '已发货'
    ELSE '未知状态'
END
```

---

### 5.6.3 两种 CASE 对比

| 比较项目 | `CASE WHEN 条件 THEN` | `CASE expr WHEN 值 THEN` |
| --- | --- | --- |
| 判断方式 | 每个 `WHEN` 是完整条件 | 同一个表达式比较不同值 |
| 类似程序语言 | `if ... else if ... else` | `switch ... case` |
| 适合场景 | 区间、多字段、复杂条件 | 状态码、固定值映射 |
| 能否判断范围 | 可以 | 不适合 |
| 能否处理 `IS NULL` | 可以 | 不适合直接判断 `NULL` |
| 可读性 | 条件清楚但较长 | 固定映射时很简洁 |

---

## 5.7 CASE WHEN

`CASE WHEN` 适合多个条件分支依次判断。

### 5.7.1 基本示例

```sql
SELECT
    CASE
        WHEN 1 > 0 THEN '1 > 0'
        WHEN 2 > 0 THEN '2 > 0'
        ELSE 'unknown'
    END AS result;
```

结果：

```text
1 > 0
```

虽然 `2 > 0` 也成立，但第一个条件已经命中，所以结果是 `'1 > 0'`。

---

### 5.7.2 条件顺序很重要

看下面这个错误示例：

```sql
SELECT
    employee_id,
    salary,
    CASE
        WHEN salary >= 8000 THEN '一般水平'
        WHEN salary >= 10000 THEN '潜力股'
        WHEN salary >= 15000 THEN '高薪'
        ELSE '草根'
    END AS salary_level
FROM employees;
```

问题在于：

```text
salary >= 15000 的员工也符合 salary >= 8000，
所以会先被分到「一般水平」，后面的条件不会再执行。
```

正确写法应该把范围更严格的条件放前面：

```sql
SELECT
    employee_id,
    salary,
    CASE
        WHEN salary >= 15000 THEN '高薪'
        WHEN salary >= 10000 THEN '潜力股'
        WHEN salary >= 8000 THEN '一般水平'
        ELSE '草根'
    END AS salary_level
FROM employees;
```

---

### 5.7.3 典型场景：按薪资区间打标签

```sql
SELECT
    employee_id,
    salary,
    CASE
        WHEN salary >= 15000 THEN '高薪'
        WHEN salary >= 10000 THEN '潜力股'
        WHEN salary >= 8000 THEN '一般水平'
        ELSE '草根'
    END AS salary_level
FROM employees;
```

这个例子体现了 `CASE WHEN` 的典型用途：

```text
根据不同条件，把连续数值切分成几个等级。
```

---

### 5.7.4 典型场景：处理 NULL

如果要判断字段是否为 `NULL`，应该使用 `IS NULL`。

```sql
SELECT
    employee_id,
    commission_pct,
    CASE
        WHEN commission_pct IS NULL THEN '无佣金'
        WHEN commission_pct > 0 THEN '有佣金'
        ELSE '其他'
    END AS commission_type
FROM employees;
```

不要写成：

```sql
CASE commission_pct
    WHEN NULL THEN '无佣金'
    ELSE '其他'
END
```

因为 `NULL` 不能用普通等号比较来判断，应该使用 `IS NULL`。

---

### 5.7.5 典型场景：多个字段共同判断

```sql
SELECT
    order_id,
    payment_status,
    ship_status,
    CASE
        WHEN payment_status = 'PAID' AND ship_status = 'SHIPPED' THEN '已付款已出货'
        WHEN payment_status = 'PAID' AND ship_status <> 'SHIPPED' THEN '已付款未出货'
        WHEN payment_status <> 'PAID' THEN '未付款'
        ELSE '其他'
    END AS order_stage
FROM orders;
```

这种多个字段共同决定结果的场景，就适合用 `CASE WHEN`。

## 5.8 CASE expr WHEN

`CASE expr WHEN` 适合同一个字段或表达式，对比多个固定值。

### 5.8.1 基本示例

```sql
SELECT
    CASE 1
        WHEN 1 THEN '我是1'
        WHEN 2 THEN '我是2'
        ELSE '你是谁'
    END AS result;
```

结果：

```text
我是1
```

---

### 5.8.2 典型场景：状态码映射

```sql
SELECT
    oid,
    `status`,
    CASE `status`
        WHEN 1 THEN '未付款'
        WHEN 2 THEN '已付款'
        WHEN 3 THEN '已发货'
        WHEN 4 THEN '确认收货'
        ELSE '无效订单'
    END AS order_status
FROM t_order;
```

当你手上是订单状态、部门编号、等级代码这类固定代码值时，这种写法通常最直观。

---

### 5.8.3 典型场景：部门编号映射

```sql
SELECT
    employee_id,
    department_id,
    CASE department_id
        WHEN 10 THEN '行政部'
        WHEN 20 THEN '研发部'
        WHEN 30 THEN '销售部'
        ELSE '其他部门'
    END AS department_name
FROM employees;
```

---

### 5.8.4 CASE expr WHEN 不适合区间判断

下面这种需求不适合写成 `CASE salary WHEN ...`：

```text
salary >= 15000 -> 高薪
salary >= 10000 -> 中高薪
salary >= 8000 -> 一般
```

因为这是范围判断，不是固定值映射。

应该使用：

```sql
CASE
    WHEN salary >= 15000 THEN '高薪'
    WHEN salary >= 10000 THEN '中高薪'
    WHEN salary >= 8000 THEN '一般'
    ELSE '其他'
END
```

## 5.9 典型查询场景

### 5.9.1 计算年工资时避免 NULL 干扰

```sql
SELECT
    employee_id,
    salary,
    commission_pct,
    12 * salary * (1 + IFNULL(commission_pct, 0)) AS annual_salary
FROM employees;
```

这里如果不先把 `commission_pct` 的 `NULL` 转成 `0`，有佣金为空的员工就可能算不出结果。

---

### 5.9.2 按职位调整工资

```sql
SELECT
    last_name,
    job_id,
    salary,
    CASE
        WHEN job_id = 'IT_PROG' THEN 1.10 * salary
        WHEN job_id = 'ST_CLERK' THEN 1.15 * salary
        WHEN job_id = 'SA_REP' THEN 1.20 * salary
        ELSE salary
    END AS revised_salary
FROM employees;
```

这个例子也可以写成 `CASE job_id WHEN ...`：

```sql
SELECT
    last_name,
    job_id,
    salary,
    CASE job_id
        WHEN 'IT_PROG' THEN 1.10 * salary
        WHEN 'ST_CLERK' THEN 1.15 * salary
        WHEN 'SA_REP' THEN 1.20 * salary
        ELSE salary
    END AS revised_salary
FROM employees;
```

因为这里是「同一个字段 `job_id` 对不同固定值做映射」，所以两种写法都可以。  
如果判断条件以后会变复杂，例如同时看职位、部门、年资，就更适合 `CASE WHEN`。

---

### 5.9.3 部门调薪练习

题目：查询部门号为 `10`、`20`、`30` 的员工信息。若部门号为 `10`，则工资乘以 `1.1`；部门号为 `20`，则工资乘以 `1.2`；部门号为 `30`，则工资乘以 `1.3`。

```sql
SELECT
    e.employee_id,
    e.last_name,
    e.salary,
    e.department_id,
    CASE e.department_id
        WHEN 10 THEN e.salary * 1.1
        WHEN 20 THEN e.salary * 1.2
        WHEN 30 THEN e.salary * 1.3
        ELSE e.salary
    END AS new_salary
FROM employees e
WHERE e.department_id IN (10, 20, 30);
```

这个练习适合用来区分两种思路：

- 如果是「同一个字段对不同固定值做映射」，优先想到 `CASE expr WHEN`。
- 如果是「不同条件分别判断」，优先想到 `CASE WHEN`。

---

### 5.9.4 订单状态输出中文说明

```sql
SELECT
    order_id,
    status,
    CASE status
        WHEN 'PENDING' THEN '待付款'
        WHEN 'PAID' THEN '已付款'
        WHEN 'SHIPPED' THEN '已出货'
        WHEN 'FINISHED' THEN '已完成'
        WHEN 'CANCELLED' THEN '已取消'
        ELSE '未知状态'
    END AS status_name
FROM orders;
```

这类写法常用于报表、后台列表、导出 Excel 前的数据转换。

---

### 5.9.5 金额区间分类

```sql
SELECT
    order_id,
    total_amount,
    CASE
        WHEN total_amount >= 100000 THEN '大型订单'
        WHEN total_amount >= 10000 THEN '中型订单'
        WHEN total_amount > 0 THEN '小型订单'
        ELSE '无效金额'
    END AS amount_level
FROM orders;
```

区间分类要注意顺序，通常先写大范围或更严格的条件。

---

### 5.9.6 把空字符串当成 NULL 后再兜底

```sql
SELECT
    user_id,
    IFNULL(NULLIF(TRIM(nickname), ''), username) AS display_name
FROM users;
```

处理逻辑：

```text
1. TRIM(nickname)：去掉首尾空格。
2. NULLIF(..., '')：如果清理后是空字符串，就转成 NULL。
3. IFNULL(..., username)：如果昵称为空，就用 username 当显示名称。
```

这是资料清洗中非常实用的组合。

---

## 5.10 条件聚合统计

流程控制函数不只可以用在单行结果中，也常和聚合函数搭配，用于报表统计。

### 5.10.1 统计不同状态订单数量

```sql
SELECT
    COUNT(*) AS total_count,
    SUM(CASE WHEN status = 'PAID' THEN 1 ELSE 0 END) AS paid_count,
    SUM(CASE WHEN status = 'PENDING' THEN 1 ELSE 0 END) AS pending_count,
    SUM(CASE WHEN status = 'CANCELLED' THEN 1 ELSE 0 END) AS cancelled_count
FROM orders;
```

这段 SQL 的意思是：

```text
每一行根据状态返回 1 或 0，
最后用 SUM() 把 1 加起来，得到符合条件的数量。
```

---

### 5.10.2 统计不同金额区间的订单数量

```sql
SELECT
    SUM(CASE WHEN total_amount >= 100000 THEN 1 ELSE 0 END) AS large_order_count,
    SUM(CASE WHEN total_amount >= 10000 AND total_amount < 100000 THEN 1 ELSE 0 END) AS middle_order_count,
    SUM(CASE WHEN total_amount > 0 AND total_amount < 10000 THEN 1 ELSE 0 END) AS small_order_count
FROM orders;
```

---

### 5.10.3 计算条件金额合计

```sql
SELECT
    customer_id,
    SUM(CASE WHEN status = 'PAID' THEN total_amount ELSE 0 END) AS paid_amount,
    SUM(CASE WHEN status = 'REFUNDED' THEN total_amount ELSE 0 END) AS refunded_amount
FROM orders
GROUP BY customer_id;
```

这类写法在报表开发中非常常见。

---

## 5.11 流程控制函数可以放在哪里

流程控制函数最常见的位置如下：

| SQL 位置 | 示例 | 说明 |
| --- | --- | --- |
| `SELECT` | `CASE WHEN salary >= 10000 THEN '高薪' END` | 输出计算后的结果 |
| `ORDER BY` | `ORDER BY CASE status WHEN 'PAID' THEN 1 ELSE 2 END` | 自定义排序规则 |
| `GROUP BY` | `GROUP BY CASE WHEN amount >= 10000 THEN '大额' ELSE '一般' END` | 按分类结果分组 |
| `HAVING` | `HAVING SUM(CASE WHEN status = 'PAID' THEN 1 ELSE 0 END) > 0` | 对条件统计结果筛选 |
| `WHERE` | 可用但要谨慎 | 过度使用可能影响索引和可读性 |

---

### 5.11.1 SELECT 中使用

```sql
SELECT
    employee_id,
    salary,
    IF(salary >= 10000, '高薪', '一般') AS salary_type
FROM employees;
```

---

### 5.11.2 ORDER BY 中使用

假设订单状态希望按照业务顺序排序：

```text
待付款 -> 已付款 -> 已出货 -> 已完成 -> 已取消
```

可以写：

```sql
SELECT
    order_id,
    status
FROM orders
ORDER BY
    CASE status
        WHEN 'PENDING' THEN 1
        WHEN 'PAID' THEN 2
        WHEN 'SHIPPED' THEN 3
        WHEN 'FINISHED' THEN 4
        WHEN 'CANCELLED' THEN 5
        ELSE 99
    END;
```

---

### 5.11.3 GROUP BY 中使用

```sql
SELECT
    CASE
        WHEN total_amount >= 100000 THEN '大型订单'
        WHEN total_amount >= 10000 THEN '中型订单'
        ELSE '小型订单'
    END AS amount_level,
    COUNT(*) AS order_count
FROM orders
GROUP BY
    CASE
        WHEN total_amount >= 100000 THEN '大型订单'
        WHEN total_amount >= 10000 THEN '中型订单'
        ELSE '小型订单'
    END;
```

这样可以按照计算出来的分类结果分组统计。

## 5.12 使用提醒

### 5.12.1 IF() 适合简单分支，复杂分支优先用 CASE

如果只有两个结果，`IF()` 很直观：

```sql
IF(score >= 60, '及格', '不及格')
```

但如果条件超过两个，建议使用 `CASE`：

```sql
CASE
    WHEN score >= 90 THEN 'A'
    WHEN score >= 80 THEN 'B'
    WHEN score >= 70 THEN 'C'
    ELSE 'D'
END
```

---

### 5.12.2 IFNULL() 只判断 NULL，不判断空字符串

```sql
SELECT IFNULL('', 'default') AS result;
```

结果通常是空字符串 `''`，不是 `'default'`。

因为空字符串不是 `NULL`。

如果想把空字符串也当成空值，可以用：

```sql
SELECT IFNULL(NULLIF(TRIM(column_name), ''), 'default') AS result
FROM table_name;
```

---

### 5.12.3 CASE 的 ELSE 建议尽量写上

`ELSE` 不是必写，但建议大多数情况下写上。

如果没有 `ELSE`，且所有条件都没命中，结果会返回 `NULL`。

```sql
CASE
    WHEN status = 1 THEN '未付款'
    WHEN status = 2 THEN '已付款'
END
```

如果 `status = 9`，结果就是 `NULL`。

更稳的写法：

```sql
CASE
    WHEN status = 1 THEN '未付款'
    WHEN status = 2 THEN '已付款'
    ELSE '未知状态'
END
```

---

### 5.12.4 CASE WHEN 会按顺序判断，条件顺序要谨慎

区间判断时，通常要把更严格的条件放前面。

```sql
CASE
    WHEN salary >= 15000 THEN '高薪'
    WHEN salary >= 10000 THEN '中高薪'
    WHEN salary >= 8000 THEN '一般'
    ELSE '其他'
END
```

不要先写 `salary >= 8000`，否则更高薪的人会先被归到一般区间。

---

### 5.12.5 判断 NULL 要用 IS NULL

不要用：

```sql
CASE commission_pct
    WHEN NULL THEN '无佣金'
    ELSE '有佣金'
END
```

应该用：

```sql
CASE
    WHEN commission_pct IS NULL THEN '无佣金'
    ELSE '有佣金'
END
```

---

### 5.12.6 注意返回类型尽量一致

`CASE` 的不同分支最好返回相近类型。

不推荐：

```sql
CASE
    WHEN status = 1 THEN '已付款'
    ELSE 0
END
```

因为一个分支返回字符串，一个分支返回数字，后续程序接值、排序或比较时容易混乱。

建议：

```sql
CASE
    WHEN status = 1 THEN '已付款'
    ELSE '未知状态'
END
```

或全部返回数字代码。

---

### 5.12.7 WHERE 中滥用 CASE 可能影响可读性和索引

例如：

```sql
SELECT *
FROM orders
WHERE CASE
        WHEN status = 'PAID' THEN paid_amount
        ELSE 0
      END > 1000;
```

这类写法可读性较差，也可能影响优化器使用索引。

通常可以改写成更直接的条件：

```sql
SELECT *
FROM orders
WHERE status = 'PAID'
  AND paid_amount > 1000;
```

工作中要优先写清楚、可优化的条件。

---

### 5.12.8 不要把过多业务逻辑塞进 SQL

流程控制函数很方便，但不代表所有业务规则都应该写在 SQL 中。

如果规则非常复杂，例如：

- 多层级审批规则。
- 大量业务状态转换。
- 不同产品线各自有特殊计算方式。
- 规则频繁变动。

就要考虑是否应该放到 Java 应用层、规则表、配置表或专门的规则引擎中维护。

SQL 适合做查询、转换、分类、汇总；复杂业务规则要看团队设计。

---

## 5.13 常见混淆点

### 混淆点 1：IFNULL() 不是 IF()

`IFNULL()` 只判断是不是 `NULL`。

```sql
IFNULL(value, default_value)
```

`IF()` 判断的是条件是否成立。

```sql
IF(condition, value_if_true, value_if_false)
```

不要把 `IFNULL()` 当成普通条件判断。

---

### 混淆点 2：NULL 和空字符串不是同一件事

```sql
SELECT IFNULL('', 'default');
```

结果是空字符串，不是 `'default'`。

因为：

```text
'' 不是 NULL。
```

如果要把空字符串也当成空值，需要搭配 `NULLIF()`。

---

### 混淆点 3：CASE WHEN 和 CASE expr WHEN 不一样

| 写法 | 判断对象 |
| --- | --- |
| `CASE WHEN 条件 THEN` | 每个 `WHEN` 是完整条件 |
| `CASE expr WHEN 值 THEN` | 同一个表达式对比多个固定值 |

区间判断、`IS NULL`、多字段判断，优先用 `CASE WHEN`。  
状态码、等级码、部门编号映射，优先用 `CASE expr WHEN`。

---

### 混淆点 4：CASE 没有 ELSE 时，没命中会返回 NULL

```sql
CASE
    WHEN status = 1 THEN '未付款'
END
```

如果 `status` 不是 `1`，结果就是 `NULL`。

所以建议补上：

```sql
ELSE '未知状态'
```

---

### 混淆点 5：CASE WHEN 的顺序会影响结果

```sql
CASE
    WHEN salary >= 8000 THEN '一般'
    WHEN salary >= 15000 THEN '高薪'
END
```

`salary = 20000` 会先命中 `salary >= 8000`，因此不会得到「高薪」。

正确顺序应该先写更严格的条件。

---

### 混淆点 6：CASE expr WHEN NULL 通常不能正确判断 NULL

不要写：

```sql
CASE commission_pct
    WHEN NULL THEN '无佣金'
    ELSE '有佣金'
END
```

应该写：

```sql
CASE
    WHEN commission_pct IS NULL THEN '无佣金'
    ELSE '有佣金'
END
```

---

### 混淆点 7：条件聚合不是只有 COUNT() 才能做

很多报表统计会写成：

```sql
SUM(CASE WHEN status = 'PAID' THEN 1 ELSE 0 END)
```

这表示把符合条件的行算成 `1`，不符合算成 `0`，最后加总。

这是报表 SQL 中非常常见的技巧。

## 5.14 自我检查题

### 问题 1

`IF(score >= 60, '及格', '不及格')` 适合什么场景？

答案：适合简单二选一。条件成立返回「及格」，否则返回「不及格」。

---

### 问题 2

`IFNULL(commission_pct, 0)` 的作用是什么？

答案：如果 `commission_pct` 不是 `NULL`，返回原值；如果是 `NULL`，返回 `0`。常用于避免计算结果因为 `NULL` 而变成 `NULL`。

---

### 问题 3

`IFNULL('', 'default')` 的结果是 `'default'` 吗？为什么？

答案：不是。结果是空字符串 `''`。因为空字符串不是 `NULL`。

---

### 问题 4

什么时候应该用 `CASE WHEN`，什么时候应该用 `CASE expr WHEN`？

答案：如果是区间、多字段、复杂条件，使用 `CASE WHEN`。如果是同一个字段对比多个固定值，例如状态码映射，使用 `CASE expr WHEN`。

---

### 问题 5

下面 SQL 有什么问题？

```sql
CASE
    WHEN salary >= 8000 THEN '一般'
    WHEN salary >= 15000 THEN '高薪'
    ELSE '其他'
END
```

答案：条件顺序有问题。`salary >= 15000` 的资料会先命中 `salary >= 8000`，因此不会得到「高薪」。应该把 `salary >= 15000` 放前面。

---

### 问题 6

下面 SQL 能正确判断 `commission_pct` 为 `NULL` 吗？

```sql
CASE commission_pct
    WHEN NULL THEN '无佣金'
    ELSE '有佣金'
END
```

答案：不能。判断 `NULL` 应该使用 `IS NULL`：

```sql
CASE
    WHEN commission_pct IS NULL THEN '无佣金'
    ELSE '有佣金'
END
```

---

### 问题 7

`SUM(CASE WHEN status = 'PAID' THEN 1 ELSE 0 END)` 的用途是什么？

答案：统计 `status = 'PAID'` 的资料笔数。符合条件的行算 `1`，不符合算 `0`，最后用 `SUM()` 加总。

---

### 问题 8

下面 SQL 可以怎么简化？

```sql
SELECT *
FROM orders
WHERE CASE
        WHEN status = 'PAID' THEN paid_amount
        ELSE 0
      END > 1000;
```

答案：可以改成更直接的条件：

```sql
SELECT *
FROM orders
WHERE status = 'PAID'
  AND paid_amount > 1000;
```

这样可读性更好，也通常更有利于优化。

---

## 5.15 本节总结

本节可以浓缩成下面几条：

1. 流程控制函数用于在 SQL 查询中根据条件返回不同结果。
2. `IF()` 适合简单二选一，条件复杂时优先考虑 `CASE`。
3. `IFNULL()` 用来处理 `NULL` 兜底，常用于金额、佣金、数量等计算。
4. `NULLIF()` 用来把相同值转成 `NULL`，常用于清洗空字符串或避免除以零。
5. `CASE WHEN` 适合多条件、区间、多字段判断。
6. `CASE expr WHEN` 适合同一个字段对比多个固定值，例如状态码映射。
7. `CASE` 条件顺序会影响结果，区间判断通常要把更严格的条件放前面。
8. 判断 `NULL` 要使用 `IS NULL`，不要用 `CASE expr WHEN NULL`。
9. `ELSE` 建议尽量写上，否则没命中任何条件时会返回 `NULL`。
10. `CASE` 搭配聚合函数可以实现条件统计，是报表 SQL 的高频技巧。
11. 在 `WHERE` 中过度使用流程控制函数可能影响可读性与索引使用，应优先写清楚的布尔条件。

如果只记一句话：

> `IF()` 处理简单二选一，`IFNULL()` 处理空值兜底，`CASE` 处理多分支分类与状态映射。
