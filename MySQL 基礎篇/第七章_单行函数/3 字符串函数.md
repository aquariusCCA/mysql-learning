# 3 字符串函数

> 所属章节：[第七章_单行函数](./README.md)
> 关键字：字符串函数、CONCAT、REPLACE、TRIM、SUBSTR、LOCATE、FIELD、NULLIF
> 建议回查情境：忘了字符串拼接、截取、替换、去空格或查找位置的函数写法，或想快速确认某个字符串函数的用途时

## 本节导读

这一节集中整理 MySQL 中常见的字符串函数。它们大多用来处理文本值，例如拼接字符串、替换内容、转换大小写、截取子串、补齐长度、去掉空格，或查找某个字符串在另一段文本中的位置。

第一次阅读时，建议先按“拼接与替换”“大小写与裁剪”“截取与定位”这三个方向建立印象，再把后面的特殊函数如 `FIELD()`、`FIND_IN_SET()`、`NULLIF()` 当成补充工具来记。复习时如果只是想查某个函数怎么用，可以直接看“快速定位”或下方总表。

## 你会在这篇学到什么

- 常见字符串函数分别适合处理什么类型的问题。
- 字符串长度中的“字符数”和“字节数”有什么区别。
- 拼接、替换、补齐、裁剪、截取和定位类函数应该去哪里查。
- 为什么 MySQL 中字符串位置通常从 `1` 开始。
- `FIELD()`、`FIND_IN_SET()`、`NULLIF()` 这类较特殊函数的基本用途。

## 快速定位

- 想看字符串长度、字符编码或大小写转换：看 `ASCII()`、`CHAR_LENGTH()`、`LENGTH()`、`UPPER()`、`LOWER()`。
- 想拼接或替换字符串：看 `CONCAT()`、`CONCAT_WS()`、`INSERT()`、`REPLACE()`。
- 想从左或右截取、填充、去空格：看 `LEFT()`、`RIGHT()`、`LPAD()`、`RPAD()`、`LTRIM()`、`RTRIM()`、`TRIM()`。
- 想查子串位置或从字符串中取一段：看 `SUBSTR()`、`LOCATE()`、`POSITION()`、`INSTR()`。
- 想处理字符串列表或做特殊比较：看 `ELT()`、`FIELD()`、`FIND_IN_SET()`、`STRCMP()`、`NULLIF()`。

## 快速回查表

| 场景 | 优先看什么 | 说明 |
| --- | --- | --- |
| 想拼接多个字段 | `CONCAT()`、`CONCAT_WS()` | `CONCAT_WS()` 会自动在各段之间加分隔符 |
| 想替换部分内容 | `INSERT()`、`REPLACE()` | 一个按位置替换，一个按匹配到的文本替换 |
| 想统一字母大小写 | `UPPER()`、`LOWER()` | 适合做格式统一或比较前预处理 |
| 想截取左右两边字符 | `LEFT()`、`RIGHT()`、`SUBSTR()` | `SUBSTR()` 适合更灵活的区间截取 |
| 想补齐长度或去掉空白 | `LPAD()`、`RPAD()`、`TRIM()` | 常用于格式整理与输出对齐 |
| 想确认某个子串在哪个位置 | `LOCATE()`、`POSITION()`、`INSTR()` | 找不到时通常返回 `0` |
| 想在字符串列表中查位置 | `FIELD()`、`FIND_IN_SET()` | 一个查参数列表，一个查逗号分隔字符串 |

## 建议阅读顺序

- 第一次学习时，建议先读函数总表，优先记住高频的拼接、替换、截取和裁剪函数。
- 如果你现在最常遇到的是“想处理姓名、地址、编码或备注字段”，优先回看拼接、替换、截取和去空格相关函数。
- 如果你已经熟悉基础函数，只是偶尔忘记某个冷门函数的名字，可以直接使用这篇做速查。

## 字符串函数总表

| 函数 | 用法 |
| --- | --- |
| `ASCII(s)` | 返回字符串 `s` 中第一个字符的 ASCII 码值 |
| `CHAR_LENGTH(s)` | 返回字符串 `s` 的字符数；作用与 `CHARACTER_LENGTH(s)` 相同 |
| `LENGTH(s)` | 返回字符串 `s` 的字节数，和字符集有关 |
| `CONCAT(s1,s2,...,sn)` | 连接 `s1,s2,...,sn` 为一个字符串 |
| `CONCAT_WS(x,s1,s2,...,sn)` | 与 `CONCAT()` 类似，但每个字符串之间会加上分隔符 `x` |
| `INSERT(str,idx,len,replacestr)` | 将字符串 `str` 从 `idx` 位置开始、长度为 `len` 的子串替换为 `replacestr` |
| `REPLACE(str,a,b)` | 用字符串 `b` 替换字符串 `str` 中所有出现的字符串 `a` |
| `UPPER(s)` / `UCASE(s)` | 将字符串 `s` 中的字母全部转为大写 |
| `LOWER(s)` / `LCASE(s)` | 将字符串 `s` 中的字母全部转为小写 |
| `LEFT(str,n)` | 返回字符串 `str` 最左边的 `n` 个字符 |
| `RIGHT(str,n)` | 返回字符串 `str` 最右边的 `n` 个字符 |
| `LPAD(str,len,pad)` | 用字符串 `pad` 在左侧填充 `str`，直到长度为 `len` |
| `RPAD(str,len,pad)` | 用字符串 `pad` 在右侧填充 `str`，直到长度为 `len` |
| `LTRIM(s)` | 去掉字符串 `s` 左侧的空格 |
| `RTRIM(s)` | 去掉字符串 `s` 右侧的空格 |
| `TRIM(s)` | 去掉字符串 `s` 开头和结尾的空格 |
| `TRIM(s1 FROM s)` | 去掉字符串 `s` 开头和结尾的 `s1` |
| `TRIM(LEADING s1 FROM s)` | 去掉字符串 `s` 开头的 `s1` |
| `TRIM(TRAILING s1 FROM s)` | 去掉字符串 `s` 结尾的 `s1` |
| `REPEAT(str,n)` | 返回字符串 `str` 重复 `n` 次后的结果 |
| `SPACE(n)` | 返回 `n` 个空格 |
| `STRCMP(s1,s2)` | 比较字符串 `s1` 和 `s2` 的 ASCII 码值大小 |
| `SUBSTR(s,index,len)` | 返回字符串 `s` 从 `index` 位置开始、长度为 `len` 的子串；作用与 `SUBSTRING()`、`MID()` 相同 |
| `LOCATE(substr,str)` | 返回 `substr` 在 `str` 中首次出现的位置；作用与 `POSITION(substr IN str)`、`INSTR(str,substr)` 相同，未找到时返回 `0` |
| `ELT(m,s1,s2,...,sn)` | 返回指定位置的字符串，例如 `m = 1` 返回 `s1` |
| `FIELD(s,s1,s2,...,sn)` | 返回字符串 `s` 在字符串列表中第一次出现的位置 |
| `FIND_IN_SET(s1,s2)` | 返回字符串 `s1` 在字符串 `s2` 中出现的位置，其中 `s2` 是逗号分隔字符串 |
| `REVERSE(s)` | 返回 `s` 反转后的字符串 |
| `NULLIF(value1,value2)` | 比较两个字符串；如果 `value1 = value2`，则返回 `NULL`，否则返回 `value1` |

## 使用提醒

> 注意：MySQL 中，字符串位置通常从 `1` 开始，而不是从 `0` 开始。

- `CHAR_LENGTH()` 统计的是字符数，`LENGTH()` 统计的是字节数；遇到多字节字符集时，两者结果可能不同。
- `TRIM()` 系列不只可以去空格，也可以去掉指定前缀或后缀。
- `LOCATE()`、`POSITION()`、`INSTR()` 找不到目标时通常返回 `0`，不是 `NULL`。
- `FIND_IN_SET()` 适合处理逗号分隔字符串，但这类设计通常只适合临时查询或兼容旧数据，不是理想的规范化存储方式。

## 示例

### `FIELD()` 与 `FIND_IN_SET()`

```sql
SELECT
    FIELD('mm', 'hello', 'msm', 'amma'),
    FIND_IN_SET('mm', 'hello,mm,amma')
FROM DUAL;

+----------------------------------+-----------------------------------+
| FIELD('mm','hello','msm','amma') | FIND_IN_SET('mm','hello,mm,amma') |
+----------------------------------+-----------------------------------+
|                                0 |                                 2 |
+----------------------------------+-----------------------------------+
```

这个例子可以看出：

- `FIELD()` 会在“参数列表”中找目标值，找不到时返回 `0`
- `FIND_IN_SET()` 会在“逗号分隔字符串”中找目标值，并返回位置

### `NULLIF()`

```sql
SELECT
    NULLIF('mysql', 'mysql'),
    NULLIF('mysql', '');

+-------------------------+---------------------+
| NULLIF('mysql','mysql') | NULLIF('mysql', '') |
+-------------------------+---------------------+
| NULL                    | mysql               |
+-------------------------+---------------------+
```

这个函数常用于“两个值相同就当成空值处理”的场景，可以把某些重复或无效值转换成 `NULL`。

## 常见混淆点

- `CHAR_LENGTH()` 看的是字符个数，`LENGTH()` 看的是字节数，中文场景下尤其容易混淆。
- `INSERT()` 是按位置替换，`REPLACE()` 是按匹配文本替换，两者不是同一类操作。
- `TRIM()` 默认处理的是首尾空格，但也可以指定要移除的内容。
- `FIELD()` 和 `FIND_IN_SET()` 都会“找位置”，但它们查找的数据来源完全不同：一个是参数列表，一个是逗号分隔字符串。
