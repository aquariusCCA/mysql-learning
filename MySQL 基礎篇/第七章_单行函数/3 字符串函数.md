## 本节导读

这一节整理 MySQL 中常见的字符串函数。字符串函数主要用来处理文本值，例如姓名、账号、邮箱、手机号、地址、备注、状态码、商品编码、银行交易说明等。

学习字符串函数时，不建议只按函数名死背，而要先按实际需求分类：

* 想知道字符串有多长：看 `CHAR_LENGTH()`、`LENGTH()`。
* 想统一大小写：看 `UPPER()`、`LOWER()`。
* 想拼接多个字段：看 `CONCAT()`、`CONCAT_WS()`。
* 想替换一段内容：看 `INSERT()`、`REPLACE()`。
* 想截取左边、右边或中间一段：看 `LEFT()`、`RIGHT()`、`SUBSTR()`。
* 想补齐长度或去掉空白：看 `LPAD()`、`RPAD()`、`TRIM()`。
* 想确认某个子串出现在哪里：看 `LOCATE()`、`POSITION()`、`INSTR()`。
* 想处理逗号分隔字符串或固定列表：看 `FIELD()`、`FIND_IN_SET()`。
* 想把两个相同值转成 `NULL`：看 `NULLIF()`。

本节重点不是把每个函数都背下来，而是建立一个判断方式：

> 遇到字符串处理需求时，先判断它属于「长度、拼接、替换、截取、补齐、裁剪、定位、比较、清洗」哪一种问题。

## 3.1 字符串函数的分类思路

字符串函数很多，如果直接背函数名，很容易混乱。比较好的方式是按照「要对字符串做什么操作」来分类。

| 分类    | 代表函数                                 | 解决的问题            |
| ----- | ------------------------------------ | ---------------- |
| 长度与编码 | `ASCII()`、`CHAR_LENGTH()`、`LENGTH()` | 看字符数、字节数、字符编码表现  |
| 大小写转换 | `UPPER()`、`LOWER()`                  | 统一英文大小写          |
| 拼接    | `CONCAT()`、`CONCAT_WS()`             | 把多个字段或常量组合成一个字符串 |
| 插入与替换 | `INSERT()`、`REPLACE()`               | 替换指定位置或指定内容      |
| 截取    | `LEFT()`、`RIGHT()`、`SUBSTR()`        | 取出字符串的一部分        |
| 填充    | `LPAD()`、`RPAD()`                    | 补齐固定长度           |
| 裁剪    | `LTRIM()`、`RTRIM()`、`TRIM()`         | 去掉空格或指定首尾内容      |
| 定位    | `LOCATE()`、`POSITION()`、`INSTR()`    | 查找子串出现的位置        |
| 列表查找  | `ELT()`、`FIELD()`、`FIND_IN_SET()`    | 在固定列表或逗号字符串中找值   |
| 特殊处理  | `STRCMP()`、`REVERSE()`、`NULLIF()`    | 比较、反转、相同值转空      |

### 学习主线

可以把字符串处理想成一条流程：

```text
原始字符串
 -> 清理空白 / 替换内容
 -> 截取需要的部分
 -> 统一格式
 -> 拼接成输出结果
 -> 用于查询、报表或程序处理
```

例如员工表中有字段：

```text
first_name、last_name、email、phone、employee_code
```

实际查询时可能需要：

* 去掉姓名前后的空格。
* 把邮箱统一转小写。
* 把员工编号补齐 6 位。
* 把姓和名拼接成完整姓名。
* 判断电话中是否含有特定区码。

这些都是字符串函数的典型使用场景。

## 3.2 长度与字符编码相关函数

这一组函数用来观察字符串的长度或字符表现。

### 3.2.1 函数表

| 函数                    | 用法                         | 说明                              |
| --------------------- | -------------------------- | ------------------------------- |
| `ASCII(s)`            | 返回字符串 `s` 中第一个字符的 ASCII 码值 | 如果第一个字符不是标准 ASCII 字符，结果要结合字符集理解 |
| `CHAR_LENGTH(s)`      | 返回字符串 `s` 的字符数             | 与 `CHARACTER_LENGTH(s)` 作用相同    |
| `CHARACTER_LENGTH(s)` | 返回字符串 `s` 的字符数             | `CHAR_LENGTH()` 的同义写法           |
| `LENGTH(s)`           | 返回字符串 `s` 的字节数             | 和字符集有关                          |

---

### 3.2.2 CHAR_LENGTH() 与 LENGTH() 的差别

这是字符串函数中最重要的基础观念之一。

```text
CHAR_LENGTH()：看有几个字符。
LENGTH()：看占几个字节。
```

例如：

```sql
SELECT
    CHAR_LENGTH('abc') AS char_len_1,
    LENGTH('abc')      AS byte_len_1,
    CHAR_LENGTH('中文') AS char_len_2,
    LENGTH('中文')      AS byte_len_2;
```

概念上：

| 字符串     | `CHAR_LENGTH()` | `LENGTH()`                |
| ------- | --------------- | ------------------------- |
| `'abc'` | `3`             | 通常为 `3`                   |
| `'中文'`  | `2`             | 取决于字符集，`utf8mb4` 下通常为 `6` |

### 为什么这很重要？

因为在中文、日文、韩文、emoji 等多字节字符场景中：

```text
字符数 ≠ 字节数
```

所以：

* 要判断「使用者看到几个字」：优先使用 `CHAR_LENGTH()`。
* 要判断「存储占用多少字节」：使用 `LENGTH()`。

---

### 3.2.3 ASCII() 示例

```sql
SELECT
    ASCII('A') AS ascii_a,
    ASCII('a') AS ascii_lower_a;
```

结果概念：

| 表达式          | 结果   |
| ------------ | ---- |
| `ASCII('A')` | `65` |
| `ASCII('a')` | `97` |

`ASCII()` 在日常业务中不算高频，但理解它有助于知道字符串比较、编码和排序规则背后的概念。

## 3.3 大小写转换函数

大小写转换函数主要用于英文字符格式统一，例如账号、邮箱、代码、英文姓名等。

### 3.3.1 函数表

| 函数                      | 用法                |
| ----------------------- | ----------------- |
| `UPPER(s)` / `UCASE(s)` | 将字符串 `s` 中的字母转成大写 |
| `LOWER(s)` / `LCASE(s)` | 将字符串 `s` 中的字母转成小写 |

---

### 3.3.2 示例

```sql
SELECT
    UPPER('mysql') AS upper_value,
    LOWER('MYSQL') AS lower_value;
```

结果：

| 表达式              | 结果        |
| ---------------- | --------- |
| `UPPER('mysql')` | `'MYSQL'` |
| `LOWER('MYSQL')` | `'mysql'` |

---

### 3.3.3 实务场景：统一邮箱格式

```sql
SELECT
    user_id,
    LOWER(email) AS normalized_email
FROM users;
```

这类写法常用于查询显示或资料清洗。

如果是正式系统，是否要在写入数据库前就统一成小写，要根据业务规则决定。

## 3.4 字符串拼接函数

字符串拼接是最常见的字符串操作之一。常见场景包括：

* 拼接姓名。
* 拼接地址。
* 拼接显示文字。
* 拼接报表栏位。
* 拼接日志说明。

### 3.4.1 函数表

| 函数                               | 用法             | 说明                             |
| -------------------------------- | -------------- | ------------------------------ |
| `CONCAT(s1,s2,...)`              | 将多个字符串连接成一个字符串 | 任一参数为 `NULL` 时，结果通常为 `NULL`    |
| `CONCAT_WS(separator,s1,s2,...)` | 使用分隔符连接多个字符串   | `separator` 是分隔符，会跳过 `NULL` 参数 |

---

### 3.4.2 CONCAT() 示例

```sql
SELECT CONCAT('Hello', ' ', 'MySQL') AS result;
```

结果：

```text
Hello MySQL
```

用在字段上：

```sql
SELECT
    employee_id,
    CONCAT(first_name, ' ', last_name) AS full_name
FROM employees;
```

---

### 3.4.3 CONCAT() 的 NULL 问题

`CONCAT()` 最容易踩到的坑是：只要有一个参数是 `NULL`，结果通常就会变成 `NULL`。

```sql
SELECT CONCAT('Hello', NULL) AS result;
```

结果：

```text
NULL
```

如果字段可能为 `NULL`，可以搭配 `IFNULL()`：

```sql
SELECT
    CONCAT(IFNULL(first_name, ''), ' ', IFNULL(last_name, '')) AS full_name
FROM employees;
```

---

### 3.4.4 CONCAT_WS() 示例

`CONCAT_WS()` 的 `WS` 可以理解为 With Separator，也就是「带分隔符」。

```sql
SELECT CONCAT_WS('-', '2026', '04', '26') AS date_text;
```

结果：

```text
2026-04-26
```

它也很适合拼接地址：

```sql
SELECT
    CONCAT_WS('', city, district, road, address_detail) AS full_address
FROM customers;
```

如果要用空格分隔姓名：

```sql
SELECT
    CONCAT_WS(' ', first_name, last_name) AS full_name
FROM employees;
```

### CONCAT() 与 CONCAT_WS() 对比

| 比较项目         | `CONCAT()`   | `CONCAT_WS()`  |
| ------------ | ------------ | -------------- |
| 是否自动加分隔符     | 不会           | 会              |
| 分隔符位置        | 需要自己写在参数中    | 第一个参数指定分隔符     |
| 遇到 `NULL` 参数 | 结果通常为 `NULL` | 通常跳过 `NULL` 参数 |
| 适合场景         | 灵活拼接         | 固定分隔符拼接        |

## 3.5 插入与替换函数

这一组函数都和「替换字符串」有关，但思路不同。

| 函数          | 核心思路      |
| ----------- | --------- |
| `INSERT()`  | 按位置替换     |
| `REPLACE()` | 按匹配到的内容替换 |

---

### 3.5.1 INSERT()

`INSERT(str,pos,len,newstr)` 会从字符串 `str` 的第 `pos` 个位置开始，替换长度为 `len` 的内容，替换成 `newstr`。

```sql
SELECT INSERT('abcdef', 2, 3, 'XYZ') AS result;
```

处理过程：

```text
原字符串：a b c d e f
位置：    1 2 3 4 5 6
从第 2 位开始，替换 3 个字符：bcd
替换成 XYZ
结果：aXYZef
```

结果：

```text
aXYZef
```

### 适合场景

`INSERT()` 适合知道固定位置的替换，例如：

* 固定格式编号替换中间几位。
* 遮蔽手机号中间几码。
* 修改固定位置的编码片段。

示例：遮蔽手机号中间四码：

```sql
SELECT INSERT('0912345678', 4, 4, '****') AS masked_phone;
```

结果：

```text
091****678
```

---

### 3.5.2 REPLACE()

`REPLACE(str,from_str,to_str)` 会把字符串 `str` 中所有出现的 `from_str` 替换成 `to_str`。

```sql
SELECT REPLACE('2026/04/26', '/', '-') AS result;
```

结果：

```text
2026-04-26
```

### 适合场景

`REPLACE()` 适合按内容替换，例如：

* 把 `/` 换成 `-`。
* 把空白去掉。
* 把某个错误文字修正成正确文字。
* 清除特殊符号。

示例：去掉字符串中的空格：

```sql
SELECT REPLACE('A B C', ' ', '') AS result;
```

结果：

```text
ABC
```

---

### 3.5.3 INSERT() 与 REPLACE() 对比

| 比较项目       | `INSERT()`    | `REPLACE()`  |
| ---------- | ------------- | ------------ |
| 替换依据       | 位置            | 匹配到的文字       |
| 是否需要知道位置   | 需要            | 不需要          |
| 是否替换所有匹配内容 | 否，替换指定位置长度    | 是，通常替换全部匹配内容 |
| 适合场景       | 固定格式资料、遮蔽部分文字 | 清洗符号、统一文本内容  |

---

## 3.6 字符串截取函数

截取函数用于从字符串中取出一部分内容。

### 3.6.1 函数表

| 函数                       | 用法                       | 说明                         |
| ------------------------ | ------------------------ | -------------------------- |
| `LEFT(str,n)`            | 返回字符串 `str` 最左边的 `n` 个字符 | 取前缀                        |
| `RIGHT(str,n)`           | 返回字符串 `str` 最右边的 `n` 个字符 | 取后缀                        |
| `SUBSTR(str,pos,len)`    | 从 `pos` 位置开始取 `len` 个字符  | 与 `SUBSTRING()`、`MID()` 类似 |
| `SUBSTRING(str,pos,len)` | 从 `pos` 位置开始取 `len` 个字符  | `SUBSTR()` 的同义写法           |
| `MID(str,pos,len)`       | 从 `pos` 位置开始取 `len` 个字符  | `SUBSTR()` 的同义写法           |

---

### 3.6.2 LEFT() 与 RIGHT()

```sql
SELECT
    LEFT('ABCDEF', 3)  AS left_value,
    RIGHT('ABCDEF', 3) AS right_value;
```

结果：

| 表达式                  | 结果    |
| -------------------- | ----- |
| `LEFT('ABCDEF', 3)`  | `ABC` |
| `RIGHT('ABCDEF', 3)` | `DEF` |

### 常见场景

取订单编号前缀：

```sql
SELECT LEFT(order_no, 4) AS order_prefix
FROM orders;
```

取手机号后四码：

```sql
SELECT RIGHT(phone, 4) AS phone_last4
FROM customers;
```

---

### 3.6.3 SUBSTR() / SUBSTRING() / MID()

```sql
SELECT SUBSTR('ABCDEF', 2, 3) AS result;
```

处理过程：

```text
字符串：A B C D E F
位置：  1 2 3 4 5 6
从第 2 位开始，取 3 个字符
结果：BCD
```

结果：

```text
BCD
```

### MySQL 字符串位置通常从 1 开始

这是非常重要的观念。

```sql
SELECT SUBSTR('ABCDEF', 1, 3) AS result;
```

结果：

```text
ABC
```

很多程序语言的字符串索引从 `0` 开始，例如 JavaScript、Java 的部分 API；但 MySQL 字符串函数通常从 `1` 开始。

---

### 3.6.4 实务场景：从身份证或编码中取片段

例如编码格式为：

```text
TW-2026-0001
```

想取年份：

```sql
SELECT SUBSTR('TW-2026-0001', 4, 4) AS year_part;
```

结果：

```text
2026
```

## 3.7 填充函数

填充函数用来把字符串补到指定长度。常见场景是编号补零、固定宽度输出、格式对齐。

### 3.7.1 函数表

| 函数                  | 用法                      | 说明        |
| ------------------- | ----------------------- | --------- |
| `LPAD(str,len,pad)` | 在左侧填充 `pad`，直到长度为 `len` | Left Pad  |
| `RPAD(str,len,pad)` | 在右侧填充 `pad`，直到长度为 `len` | Right Pad |

---

### 3.7.2 LPAD() 示例

```sql
SELECT LPAD('123', 6, '0') AS result;
```

结果：

```text
000123
```

### 常见场景：编号补零

```sql
SELECT
    employee_id,
    LPAD(employee_id, 6, '0') AS employee_code
FROM employees;
```

如果 `employee_id = 25`，结果可能是：

```text
000025
```

---

### 3.7.3 RPAD() 示例

```sql
SELECT RPAD('ABC', 6, '*') AS result;
```

结果：

```text
ABC***
```

### 使用提醒

如果原字符串长度已经超过目标长度，`LPAD()` / `RPAD()` 可能会截断结果。因此使用时要确认目标长度是否合理。

## 3.8 裁剪函数 TRIM()

裁剪函数用于去掉字符串左侧、右侧或首尾的空格，也可以去掉指定内容。

### 3.8.1 函数表

| 函数                         | 用法                 | 说明         |
| -------------------------- | ------------------ | ---------- |
| `LTRIM(s)`                 | 去掉字符串 `s` 左侧空格     | Left Trim  |
| `RTRIM(s)`                 | 去掉字符串 `s` 右侧空格     | Right Trim |
| `TRIM(s)`                  | 去掉字符串 `s` 首尾空格     | 最常用        |
| `TRIM(s1 FROM s)`          | 去掉字符串 `s` 首尾的 `s1` | 指定要移除的内容   |
| `TRIM(LEADING s1 FROM s)`  | 去掉字符串 `s` 开头的 `s1` | 移除前缀       |
| `TRIM(TRAILING s1 FROM s)` | 去掉字符串 `s` 结尾的 `s1` | 移除后缀       |

---

### 3.8.2 去空格示例

```sql
SELECT
    LTRIM('   mysql') AS left_trim,
    RTRIM('mysql   ') AS right_trim,
    TRIM('   mysql   ') AS both_trim;
```

结果概念：

| 表达式                   | 结果        |
| --------------------- | --------- |
| `LTRIM('   mysql')`   | `'mysql'` |
| `RTRIM('mysql   ')`   | `'mysql'` |
| `TRIM('   mysql   ')` | `'mysql'` |

---

### 3.8.3 去指定内容

```sql
SELECT TRIM('x' FROM 'xxxmysqlxxx') AS result;
```

结果：

```text
mysql
```

只去开头：

```sql
SELECT TRIM(LEADING 'x' FROM 'xxxmysqlxxx') AS result;
```

结果：

```text
mysqlxxx
```

只去结尾：

```sql
SELECT TRIM(TRAILING 'x' FROM 'xxxmysqlxxx') AS result;
```

结果：

```text
xxxmysql
```

### 实务场景

`TRIM()` 很适合清理使用者输入或旧系统资料，例如：

```sql
SELECT
    customer_id,
    TRIM(customer_name) AS clean_customer_name
FROM customers;
```

## 3.9 定位函数

定位函数用来查找某个子串在字符串中第一次出现的位置。

### 3.9.1 函数表

| 函数                        | 用法                           | 说明                  |
| ------------------------- | ---------------------------- | ------------------- |
| `LOCATE(substr,str)`      | 返回 `substr` 在 `str` 中首次出现的位置 | 找不到返回 `0`           |
| `POSITION(substr IN str)` | 返回 `substr` 在 `str` 中首次出现的位置 | 与 `LOCATE()` 类似     |
| `INSTR(str,substr)`       | 返回 `substr` 在 `str` 中首次出现的位置 | 参数顺序与 `LOCATE()` 不同 |

---

### 3.9.2 LOCATE() 示例

```sql
SELECT LOCATE('SQL', 'MySQL') AS result;
```

结果：

```text
3
```

因为：

```text
M y S Q L
1 2 3 4 5
```

`SQL` 从第 3 个位置开始出现。

---

### 3.9.3 找不到时返回 0

```sql
SELECT LOCATE('Java', 'MySQL') AS result;
```

结果：

```text
0
```

要特别注意：

```text
找不到通常返回 0，不是 NULL。
```

所以条件判断常写成：

```sql
SELECT *
FROM articles
WHERE LOCATE('MySQL', title) > 0;
```

---

### 3.9.4 POSITION() 示例

```sql
SELECT POSITION('SQL' IN 'MySQL') AS result;
```

结果：

```text
3
```

---

### 3.9.5 INSTR() 示例

```sql
SELECT INSTR('MySQL', 'SQL') AS result;
```

结果：

```text
3
```

### LOCATE() 与 INSTR() 的差别

| 函数                   | 参数顺序           |
| -------------------- | -------------- |
| `LOCATE(substr,str)` | 先写要找的子串，再写原字符串 |
| `INSTR(str,substr)`  | 先写原字符串，再写要找的子串 |

---

## 3.10 列表与集合查找函数

这一组函数容易混淆，因为它们都和「找位置」有关，但资料来源不同。

### 3.10.1 ELT()

`ELT(m,s1,s2,...,sn)` 会返回第 `m` 个字符串。

```sql
SELECT ELT(2, 'Java', 'MySQL', 'Vue') AS result;
```

结果：

```text
MySQL
```

因为第 2 个参数值是 `'MySQL'`。

---

### 3.10.2 FIELD()

`FIELD(s,s1,s2,...,sn)` 会返回 `s` 在后面参数列表中第一次出现的位置。

```sql
SELECT FIELD('MySQL', 'Java', 'MySQL', 'Vue') AS result;
```

结果：

```text
2
```

如果找不到：

```sql
SELECT FIELD('Redis', 'Java', 'MySQL', 'Vue') AS result;
```

结果：

```text
0
```

### 典型场景：自定义排序

```sql
SELECT status
FROM orders
ORDER BY FIELD(status, '未付款', '已付款', '已出货', '已完成');
```

这表示不要按字典顺序排序，而是按指定业务顺序排序。

---

### 3.10.3 FIND_IN_SET()

`FIND_IN_SET(s1,s2)` 会在逗号分隔字符串 `s2` 中查找 `s1` 的位置。

```sql
SELECT FIND_IN_SET('MySQL', 'Java,MySQL,Vue') AS result;
```

结果：

```text
2
```

如果找不到：

```sql
SELECT FIND_IN_SET('Redis', 'Java,MySQL,Vue') AS result;
```

结果：

```text
0
```

### 使用提醒

`FIND_IN_SET()` 适合兼容旧资料或临时查询，但不建议长期依赖逗号分隔字符串来存多值。

例如这种设计：

```text
user_id | roles
1       | admin,manager,auditor
```

虽然可以用：

```sql
SELECT *
FROM users
WHERE FIND_IN_SET('admin', roles) > 0;
```

但更规范的做法通常是拆成关联表，例如：

```text
user_role(user_id, role_code)
```

这样更适合查询、索引和维护。

---

### 3.10.4 ELT()、FIELD()、FIND_IN_SET() 对比

| 函数                   | 作用           | 资料来源    | 找不到时             |
| -------------------- | ------------ | ------- | ---------------- |
| `ELT(m,s1,s2,...)`   | 按位置取值        | 参数列表    | 位置不合法通常返回 `NULL` |
| `FIELD(s,s1,s2,...)` | 查值在参数列表中的位置  | 参数列表    | 返回 `0`           |
| `FIND_IN_SET(s1,s2)` | 查值在逗号字符串中的位置 | 逗号分隔字符串 | 返回 `0`           |

---

## 3.11 比较与特殊处理函数

### 3.11.1 STRCMP()

`STRCMP(s1,s2)` 用来比较两个字符串。

```sql
SELECT
    STRCMP('abc', 'abc') AS r1,
    STRCMP('abc', 'abd') AS r2,
    STRCMP('abd', 'abc') AS r3;
```

结果概念：

| 表达式                   | 结果概念 | 说明   |
| --------------------- | ---- | ---- |
| `STRCMP('abc','abc')` | `0`  | 两者相等 |
| `STRCMP('abc','abd')` | 负数   | 前者较小 |
| `STRCMP('abd','abc')` | 正数   | 前者较大 |

注意：字符串比较结果会受到字符集和排序规则影响。一般业务中较少直接用 `STRCMP()`，但理解它有助于认识字符串比较的行为。

---

### 3.11.2 REVERSE()

`REVERSE(s)` 会把字符串反转。

```sql
SELECT REVERSE('abc') AS result;
```

结果：

```text
cba
```

实际业务中使用频率不高，但在练习题或特殊字符串处理里可能出现。

---

### 3.11.3 REPEAT() 与 SPACE()

```sql
SELECT
    REPEAT('ha', 3) AS repeat_value,
    CONCAT('A', SPACE(3), 'B') AS space_value;
```

结果概念：

| 表达式                          | 结果       |
| ---------------------------- | -------- |
| `REPEAT('ha', 3)`            | `hahaha` |
| `CONCAT('A', SPACE(3), 'B')` | `A   B`  |

---

### 3.11.4 NULLIF()

`NULLIF(value1,value2)` 的逻辑是：

```text
如果 value1 = value2，返回 NULL。
否则返回 value1。
```

示例：

```sql
SELECT
    NULLIF('mysql', 'mysql') AS r1,
    NULLIF('mysql', '')      AS r2;
```

结果：

| 表达式                       | 结果        |
| ------------------------- | --------- |
| `NULLIF('mysql','mysql')` | `NULL`    |
| `NULLIF('mysql','')`      | `'mysql'` |

### 实务场景：把空字符串当成 NULL

```sql
SELECT NULLIF(TRIM(nickname), '') AS clean_nickname
FROM users;
```

这段 SQL 的意思是：

1. 先用 `TRIM()` 去掉首尾空格。
2. 如果结果是空字符串 `''`，就转成 `NULL`。
3. 否则保留原值。

这在资料清洗中很常见。

---

## 3.12 字符串函数的实务场景

### 3.12.1 清理姓名前后空格

```sql
SELECT
    customer_id,
    TRIM(customer_name) AS clean_name
FROM customers;
```

适合清理使用者输入、旧系统迁移资料、报表输出资料。

---

### 3.12.2 拼接完整姓名

```sql
SELECT
    employee_id,
    CONCAT_WS(' ', first_name, last_name) AS full_name
FROM employees;
```

如果 `first_name` 或 `last_name` 有可能为 `NULL`，`CONCAT_WS()` 通常比 `CONCAT()` 更方便。

---

### 3.12.3 统一邮箱小写

```sql
SELECT
    user_id,
    LOWER(TRIM(email)) AS normalized_email
FROM users;
```

这个例子同时做了两件事：

1. `TRIM(email)`：去掉首尾空格。
2. `LOWER(...)`：统一转小写。

---

### 3.12.4 手机号遮蔽

```sql
SELECT
    customer_id,
    INSERT(phone, 4, 4, '****') AS masked_phone
FROM customers;
```

如果手机号是 `0912345678`，结果会变成：

```text
091****678
```

---

### 3.12.5 编号补零

```sql
SELECT
    employee_id,
    LPAD(employee_id, 6, '0') AS employee_code
FROM employees;
```

如果员工编号是 `25`，显示成：

```text
000025
```

---

### 3.12.6 取订单编号中的年份

假设订单编号格式为：

```text
OD202604260001
```

其中第 3 到第 6 位是年份。

```sql
SELECT
    order_no,
    SUBSTR(order_no, 3, 4) AS order_year
FROM orders;
```

---

### 3.12.7 判断备注中是否包含关键字

```sql
SELECT *
FROM transactions
WHERE LOCATE('手续费', remark) > 0;
```

这表示查询 `remark` 中包含「手续费」的资料。

如果是大表、正式搜索需求，通常还要评估索引、全文索引或搜索服务，而不是只依赖 `LOCATE()`。

---

### 3.12.8 自定义状态排序

```sql
SELECT
    order_id,
    status
FROM orders
ORDER BY FIELD(status, '未付款', '已付款', '已出货', '已完成', '已取消');
```

这适合让状态按照业务顺序显示，而不是按照字典顺序。

---

### 3.12.9 把空字符串转成 NULL

```sql
SELECT
    user_id,
    NULLIF(TRIM(nickname), '') AS clean_nickname
FROM users;
```

这类写法很适合资料清洗，因为很多系统会同时存在：

```text
NULL
''
'   '
```

如果业务上它们都代表「没有填写」，就可以用 `TRIM()` + `NULLIF()` 统一处理。

## 3.13 使用提醒

### 3.13.1 注意字符串位置通常从 1 开始

MySQL 字符串函数中的位置通常从 `1` 开始。

```sql
SELECT SUBSTR('MySQL', 1, 2);
```

结果：

```text
My
```

不要用 JavaScript 或 Java 中常见的 `0` 起始索引来理解 MySQL 字符串位置。

---

### 3.13.2 注意 CHAR_LENGTH() 与 LENGTH() 的差别

```sql
SELECT
    CHAR_LENGTH('中文'),
    LENGTH('中文');
```

`CHAR_LENGTH()` 看的是字符个数，`LENGTH()` 看的是字节数。中文、emoji、多语言资料中，这两者很容易不同。

---

### 3.13.3 注意 CONCAT() 的 NULL 问题

```sql
SELECT CONCAT('Hello', NULL);
```

结果通常是：

```text
NULL
```

如果字段可能为 `NULL`，常见处理方式有两种：

```sql
SELECT CONCAT(IFNULL(first_name, ''), IFNULL(last_name, ''))
FROM employees;
```

或：

```sql
SELECT CONCAT_WS(' ', first_name, last_name)
FROM employees;
```

---

### 3.13.4 注意 FIND_IN_SET() 不适合滥用

`FIND_IN_SET()` 可以处理逗号分隔字符串，但这通常不是理想的长期资料设计。

不推荐长期依赖这种结构：

```text
roles = 'admin,manager,auditor'
```

更推荐拆成关联表：

```text
user_role(user_id, role_code)
```

这样更利于索引、查询和维护。

---

### 3.13.5 注意在 WHERE 中对字段套函数可能影响索引

例如：

```sql
SELECT *
FROM users
WHERE LOWER(email) = 'test@example.com';
```

如果 `email` 字段上有索引，对字段套 `LOWER()` 可能影响索引使用。

更常见的做法是：

* 写入资料时就统一格式，例如邮箱统一存小写。
* 查询时直接比较已经规范化的字段。
* 必要时使用函数索引或生成列等方案，具体要看 MySQL 版本和系统设计。

同样，下面这种也要谨慎：

```sql
WHERE SUBSTR(order_no, 1, 2) = 'OD'
```

如果只是查前缀，很多时候可以改成：

```sql
WHERE order_no LIKE 'OD%'
```

实际是否能用索引，要以执行计划为准，但工作中要先有这个意识：

> 不要随便在索引字段外层套函数。

---

### 3.13.6 注意字符串比较受字符集与排序规则影响

大小写是否敏感、中文排序结果、特殊符号比较，都可能受到字符集与排序规则影响。

如果发现字符串比较或排序结果和预期不同，可以搭配信息函数查看：

```sql
SELECT CHARSET('ABC'), COLLATION('ABC');
```

---

## 3.14 常见混淆点

### 混淆点 1：CHAR_LENGTH() 和 LENGTH() 不是同一个概念

```text
CHAR_LENGTH()：几个字符。
LENGTH()：几个字节。
```

英文场景下两者可能刚好一样，但中文、多语言、emoji 场景下经常不同。

---

### 混淆点 2：CONCAT() 遇到 NULL 容易整串变 NULL

```sql
SELECT CONCAT('A', NULL, 'B');
```

结果通常是：

```text
NULL
```

如果不希望这样，要搭配 `IFNULL()` 或改用 `CONCAT_WS()`。

---

### 混淆点 3：INSERT() 和 REPLACE() 都能替换，但依据不同

| 函数          | 替换依据  |
| ----------- | ----- |
| `INSERT()`  | 按位置替换 |
| `REPLACE()` | 按内容替换 |

```sql
SELECT INSERT('abcdef', 2, 3, 'XYZ');
-- aXYZef

SELECT REPLACE('abcabc', 'ab', 'XY');
-- XYcXYc
```

---

### 混淆点 4：SUBSTR() 的位置通常从 1 开始

```sql
SELECT SUBSTR('abcdef', 1, 3);
```

结果：

```text
abc
```

不是从第 `0` 位开始。

---

### 混淆点 5：LOCATE()、POSITION()、INSTR() 参数顺序不完全一样

| 函数           | 写法                        |
| ------------ | ------------------------- |
| `LOCATE()`   | `LOCATE(substr,str)`      |
| `POSITION()` | `POSITION(substr IN str)` |
| `INSTR()`    | `INSTR(str,substr)`       |

尤其是 `LOCATE()` 和 `INSTR()`，参数顺序很容易写反。

---

### 混淆点 6：找不到通常返回 0，不是 NULL

```sql
SELECT LOCATE('x', 'abc');
```

结果：

```text
0
```

所以判断是否找到时，常写：

```sql
LOCATE('x', column_name) > 0
```

---

### 混淆点 7：FIELD() 和 FIND_IN_SET() 都是找位置，但资料来源不同

```sql
SELECT FIELD('b', 'a', 'b', 'c');
-- 2

SELECT FIND_IN_SET('b', 'a,b,c');
-- 2
```

虽然结果都可能是 `2`，但意义不同：

| 函数              | 查找对象    |
| --------------- | ------- |
| `FIELD()`       | 参数列表    |
| `FIND_IN_SET()` | 逗号分隔字符串 |

---

### 混淆点 8：TRIM() 不只是去空格

```sql
SELECT TRIM('x' FROM 'xxxmysqlxxx');
```

结果：

```text
mysql
```

`TRIM()` 可以去掉首尾指定内容，不只可以去空格。

## 3.15 自我检查题

### 问题 1

`CHAR_LENGTH('中文')` 和 `LENGTH('中文')` 一定相等吗？为什么？

答案：不一定。`CHAR_LENGTH()` 统计字符数，`LENGTH()` 统计字节数。中文在多字节字符集下通常一个字会占多个字节。

### 问题 2

`CONCAT('A', NULL, 'B')` 的结果通常是什么？如果不想变成 `NULL`，可以怎么写？

答案：结果通常是 `NULL`。可以使用：

```sql
SELECT CONCAT('A', IFNULL(NULL, ''), 'B');
```

或在适合的场景使用：

```sql
SELECT CONCAT_WS('', 'A', NULL, 'B');
```

### 问题 3

`INSERT('abcdef', 2, 3, 'XYZ')` 的结果是什么？

答案：

```text
aXYZef
```

因为从第 2 个位置开始，把 3 个字符 `bcd` 替换成 `XYZ`。

---

### 问题 4

`REPLACE('2026/04/26', '/', '-')` 的结果是什么？

答案：

```text
2026-04-26
```

---

### 问题 5

`SUBSTR('MySQL', 3, 3)` 的结果是什么？

答案：

```text
SQL
```

因为 MySQL 字符串位置通常从 `1` 开始，`S` 是第 3 个字符。

---

### 问题 6

`LOCATE('SQL', 'MySQL')` 和 `INSTR('MySQL', 'SQL')` 的结果分别是什么？

答案：两者结果都是 `3`。但参数顺序不同：

```sql
LOCATE(substr, str)
INSTR(str, substr)
```

---

### 问题 7

`FIELD('b', 'a', 'b', 'c')` 和 `FIND_IN_SET('b', 'a,b,c')` 有什么差别？

答案：`FIELD()` 是在参数列表中查找；`FIND_IN_SET()` 是在逗号分隔字符串中查找。两者结果都可能是 `2`，但资料来源不同。

### 问题 8

下面 SQL 有什么潜在问题？

```sql
SELECT *
FROM users
WHERE LOWER(email) = 'test@example.com';
```

答案：如果 `email` 字段上有索引，对字段套 `LOWER()` 可能影响索引使用。更好的做法通常是写入时就把邮箱规范化为小写，或根据系统设计使用合适的索引方案。

## 3.16 本节总结

本节可以浓缩成下面几条：

1. 字符串函数主要用于文本资料的长度判断、格式统一、拼接、替换、截取、补齐、裁剪、定位、比较与清洗。
2. `CHAR_LENGTH()` 看字符数，`LENGTH()` 看字节数，中文和多语言场景要特别注意。
3. MySQL 字符串位置通常从 `1` 开始，不是从 `0` 开始。
4. `CONCAT()` 遇到 `NULL` 容易整串变 `NULL`，可以搭配 `IFNULL()` 或使用 `CONCAT_WS()`。
5. `INSERT()` 是按位置替换，`REPLACE()` 是按匹配内容替换。
6. `LEFT()`、`RIGHT()`、`SUBSTR()` 是最常用的截取函数。
7. `LPAD()` 常用于编号补零，`TRIM()` 常用于资料清洗。
8. `LOCATE()`、`POSITION()`、`INSTR()` 都能查找子串位置，但参数写法不同，找不到通常返回 `0`。
9. `FIELD()` 适合固定参数列表查找或自定义排序；`FIND_IN_SET()` 适合处理逗号分隔字符串，但不建议滥用。
10. 在 `WHERE` 中对字段套字符串函数可能影响索引，正式查询要特别谨慎。

如果只记一句话：

> 字符串函数的核心不是背名字，而是判断当前需求是在做「清洗、拼接、替换、截取、补齐、定位」哪一种处理。
