# 第七章_单行函数练习题参考答案

这份参考答案对应 [第七章_单行函数练习题](./第七章_单行函数练习题.md)。

- 查询题默认基于 `資料/atguigudb.sql`
- 除题目特别说明外，优先基于 `employees` 表
- 同一题不只一种写法，下面给的是本章语境下较直接的一版

## 基础题

### 1. 函数分类题

答案：

- 单行函数：输入通常是一行中的某个值，对每一行分别计算一次，每行返回一个结果。
- 聚合函数：输入是一组行，对整组数据汇总后返回一个结果。

差异：

- 输入数据范围不同：单行函数面向单行，聚合函数面向多行
- 输出结果个数不同：单行函数通常“每行一个结果”，聚合函数通常“每组一个结果”

### 2. 分类判断题

答案：

- 属于单行函数：`UPPER()`、`ROUND()`、`NOW()`、`IFNULL()`
- 不属于单行函数：`COUNT()`、`AVG()`

### 3. 数值函数识别

答案：

- `ABS()`：求绝对值
- `CEIL()`：向上取整
- `FLOOR()`：向下取整
- `ROUND()`：四舍五入
- `TRUNCATE()`：按指定精度直接截断，不做四舍五入
- `RAND()`：生成随机数

### 4. `ROUND()` 与 `TRUNCATE()` 比较

答案：

- `ROUND(x, y)`：按第 `y` 位做四舍五入
- `TRUNCATE(x, y)`：直接截断到第 `y` 位，不四舍五入

结果：

- `ROUND(12.343, 2)`：`12.34`
- `TRUNCATE(12.343, 2)`：`12.34`
- `ROUND(12.324, -1)`：`10`
- `TRUNCATE(12.324, -1)`：`10`

说明：这一组例子里后两项结果刚好相同，但处理方式不同。

### 5. 随机数题

答案：

- `RAND()`：每次调用都生成一个随机数
- `RAND(x)`：以固定种子 `x` 作为随机序列起点
- 相同种子的 `RAND(x)` 之所以可重复，是因为伪随机算法在相同起点下会生成同样的序列

### 6. 字符串长度题

答案：

- `CHAR_LENGTH()`：按字符个数计算
- `LENGTH()`：按字节数计算

为什么中文场景可能不同：

- 英文常见一个字符占一个字节
- 中文在 `utf8` / `utf8mb4` 下通常占多个字节
- 所以中文字符串的 `LENGTH()` 往往大于 `CHAR_LENGTH()`

### 7. 拼接题

```sql
SELECT CONCAT(last_name, job_id) AS concat_result
FROM employees;
```

```sql
SELECT CONCAT_WS('-', last_name, job_id) AS concat_ws_result
FROM employees;
```

差别：

- `CONCAT()` 直接顺序拼接
- `CONCAT_WS()` 会自动在参数之间加分隔符，这里是 `-`

### 8. 大小写转换题

```sql
SELECT
    last_name,
    UPPER(last_name) AS upper_name,
    LOWER(last_name) AS lower_name
FROM employees;
```

### 9. 截取与定位题

答案：

- `LEFT(str, n)`：取左边前 `n` 个字符
- `RIGHT(str, n)`：取右边前 `n` 个字符
- `SUBSTR(str, pos, len)`：从指定位置开始截取
- `LOCATE(substr, str)`：找子串第一次出现的位置

示例：

```sql
SELECT LEFT('mysql', 2);
SELECT RIGHT('mysql', 2);
SELECT SUBSTR('mysql', 2, 3);
SELECT LOCATE('sq', 'mysql');
```

### 10. 去空格与补齐题

答案：

- `TRIM()`：去两端空格
- `LTRIM()`：去左侧空格
- `RTRIM()`：去右侧空格
- `LPAD()`：左侧补齐到指定长度
- `RPAD()`：右侧补齐到指定长度

## 进阶题

### 11. 字符串替换题

答案：

- `INSERT()`：按位置替换，指定从哪里开始、替换多长
- `REPLACE()`：按子串内容替换，把匹配到的旧子串整体换掉

### 12. `FIELD()` 与 `FIND_IN_SET()` 比较

答案：

- `FIELD(s, s1, s2, ...)`：在“参数列表”里找 `s` 的位置
- `FIND_IN_SET(s1, s2)`：在“逗号分隔字符串”里找 `s1` 的位置

差别：

- `FIELD()` 的查找来源是多个独立参数
- `FIND_IN_SET()` 的查找来源是一整个逗号串

### 13. `NULLIF()` 判断题

答案：

- `NULLIF(value1, value2)`：如果两者相等，返回 `NULL`；否则返回 `value1`

结果：

- `NULLIF('mysql', 'mysql')`：`NULL`
- `NULLIF('mysql', '')`：`mysql`

### 14. 日期时间入口题

答案：

- 获取当前日期时间：解决“现在是什么时间”
- 时间戳和日期互转：解决“系统时间戳与人类可读时间互转”
- 日期字段提取：解决“按年、月、周、日分析”
- 日期加减与时间差：解决“往前推、往后推、计算间隔”
- 日期格式化与解析：解决“展示格式”和“把字符串转回日期”

### 15. 当前时间函数题

答案：

- `CURDATE()`：当前日期
- `CURTIME()`：当前时间
- `NOW()`：当前日期时间
- `SYSDATE()`：执行到该函数时的系统日期时间
- `UTC_DATE()`：当前 UTC 日期
- `UTC_TIME()`：当前 UTC 时间

### 16. 时间戳转换题

答案：

- `UNIX_TIMESTAMP()`：把日期时间转成 Unix 时间戳
- `FROM_UNIXTIME()`：把 Unix 时间戳转回日期时间

转换方向：

- `UNIX_TIMESTAMP()`：日期时间 -> 整数时间戳
- `FROM_UNIXTIME()`：整数时间戳 -> 日期时间

### 17. 日期字段提取题

答案：

- `YEAR()`：年
- `MONTH()`：月
- `DAY()`：日
- `WEEK()`：一年中的第几周
- `DAYOFYEAR()`：一年中的第几天
- `EXTRACT()`：按指定单位提取，更通用

### 18. 日期加减题

答案：

- `DATE_ADD()`：日期加一个时间间隔
- `DATE_SUB()`：日期减一个时间间隔
- `DATEDIFF()`：算两个日期差多少天
- `TIMEDIFF()`：算两个时间或日期时间的差值
- `LAST_DAY()`：返回当月最后一天

### 19. 日期格式化与解析题

答案：

- `DATE_FORMAT()`：把日期时间转成字符串
- `STR_TO_DATE()`：把字符串解析成日期时间

方向差别：

- `DATE_FORMAT()`：日期 -> 字符串
- `STR_TO_DATE()`：字符串 -> 日期

### 20. `IF()` 与 `IFNULL()` 比较

答案：

- `IF(expr, v1, v2)`：根据条件真假二选一
- `IFNULL(v1, v2)`：只在 `v1` 为 `NULL` 时才用 `v2`

适用场景：

- `IF()`：工资高于某阈值时打标签
- `IFNULL()`：空佣金按 `0` 处理

### 21. 两种 `CASE()` 写法比较

答案：

1. `CASE WHEN 条件 THEN 结果 ... END`
   适合多段区间判断、复杂条件判断
2. `CASE expr WHEN 值 THEN 结果 ... END`
   适合把一个固定表达式映射成不同结果

### 22. 空值兜底题

答案：

- 因为 `NULL` 参与算术运算时，结果通常还是 `NULL`
- 如果 `commission_pct` 是 `NULL`，直接算年薪会把整条表达式也算成 `NULL`
- 所以常先用 `IFNULL(commission_pct, 0)` 把空值补成 `0`

### 23. 加密函数分类题

答案：

- 不可逆处理：`PASSWORD()`、`MD5()`、`SHA()`
- 可逆配对函数：`ENCODE()`、`DECODE()`

补充：`PASSWORD()` 属于历史教材里常见函数，现代环境中不建议依赖它做通用业务加密设计。

### 24. `MD5()`、`SHA()` 与 `ENCODE()` 比较

答案：

- `MD5()` / `SHA()` 更接近摘要或哈希，核心特点是不可逆
- `ENCODE()` / `DECODE()` 是成对出现的，可通过同样规则把内容还原
- 所以它们不是同一类用途，最大差别就是“能不能还原原文”

### 25. 信息函数题

答案：

- `VERSION()`：看 MySQL 版本
- `DATABASE()`：看当前默认数据库
- `USER()`：看当前登录用户字符串
- `CURRENT_USER()`：看当前认证用户
- `CHARSET()`：看字符串使用的字符集
- `COLLATION()`：看字符串使用的排序规则

### 26. 工具函数题

答案：

- `FORMAT()`：把数值格式化显示，例如千分位
- `CONV()`：进制转换
- `INET_ATON()`：IP 字符串转整数
- `INET_NTOA()`：整数转回 IP 字符串
- `BENCHMARK()`：重复执行表达式做性能测试
- `CONVERT(value USING char_code)`：字符集转换

## 挑战题

### 27. 数值函数实作题

```sql
SELECT
    ABS(-123) AS abs_val,
    CEIL(32.32) AS ceil_val,
    FLOOR(-43.23) AS floor_val,
    MOD(12, 5) AS mod_val,
    SQRT(9) AS sqrt_val;
```

### 28. 字符串函数实作题

```sql
SELECT
    last_name,
    CHAR_LENGTH(last_name) AS name_len,
    LEFT(last_name, 3) AS first_3_chars,
    LOCATE('a', LOWER(last_name)) AS first_a_pos
FROM employees;
```

### 29. 条件标签题

```sql
SELECT
    employee_id,
    last_name,
    salary,
    CASE
        WHEN salary >= 15000 THEN '高薪'
        WHEN salary >= 10000 THEN '潜力股'
        WHEN salary >= 8000 THEN '一般'
        ELSE '基础'
    END AS salary_tag
FROM employees;
```

### 30. 状态映射题

```sql
SELECT
    3 AS status_code,
    CASE 3
        WHEN 1 THEN '待处理'
        WHEN 2 THEN '处理中'
        WHEN 3 THEN '已完成'
        WHEN 4 THEN '已取消'
        ELSE '未知状态'
    END AS status_text;
```

### 31. 年薪计算题

```sql
SELECT
    employee_id,
    salary,
    commission_pct,
    12 * salary * (1 + IFNULL(commission_pct, 0)) AS annual_salary
FROM employees;
```

### 32. 日期处理综合题

```sql
SELECT DATE_FORMAT(NOW(), '%Y-%m-%d %H:%i:%s') AS formatted_now;
```

### 33. 改错题：把聚合函数当单行函数用

答案：

- 错在把 `AVG()` 说成和 `UPPER()` 一样“对每一行分别处理一次”
- `UPPER()` 是单行函数，按行处理
- `AVG()` 是聚合函数，面向一组行汇总后返回结果

### 34. 改错题：字符串函数误判

答案：

- 不严谨之处在于把 `LENGTH()` 直接等同于“字符个数”
- `LENGTH()` 返回的是字节数，不一定等于字符数
- 中文和英文在字节长度上可能不同

### 35. 改错题：加密函数误解

答案：

- 问题在于把 `MD5()` 当成可逆函数
- `MD5()` 是不可逆摘要，不能像 `DECODE()` 那样把原文解回来

### 36. 综合题：从需求到 SQL

```sql
SELECT
    employee_id,
    last_name,
    salary,
    UPPER(last_name) AS upper_name,
    CASE
        WHEN salary >= 15000 THEN '高薪'
        WHEN salary >= 10000 THEN '潜力股'
        WHEN salary >= 8000 THEN '一般'
        ELSE '基础'
    END AS salary_tag,
    12 * salary * (1 + IFNULL(commission_pct, 0)) AS annual_salary,
    NOW() AS query_time
FROM employees;
```
