# 4.3 获取月份星期天数等信息

> - 所属章节：[第七章_单行函数 / 4 日期和时间函数](./README.md)  
> - 关键字：YEAR、MONTH、DAY、HOUR、MINUTE、SECOND、MONTHNAME、DAYNAME、WEEKDAY、DAYOFWEEK、QUARTER、WEEK、WEEKOFYEAR、DAYOFYEAR、DAYOFMONTH、日期拆解、周数、季度、星期  
> - 建议回查情境：需要从日期中取出年、月、日、时、分、秒；需要判断星期几、季度、第几周、一年中的第几天；做报表分组、统计维度、日期标签输出时。

## 本节导读

这一节关注的是「拆日期」。

当你已经取得一个日期或日期时间值之后，实际工作中经常不会直接原样显示，而是需要拆出其中某一部分，例如：

- 订单是哪一年？
- 注册资料属于哪个月份？
- 今天是星期几？
- 这笔交易发生在哪一季？
- 这个日期是一年中的第几天？
- 这笔资料要归到第几周报表？

这类需求对应的就是本节函数。

## 4.3.1 取得年、月、日

### 函数表

| 函数 | 用法 |
| --- | --- |
| `YEAR(date)` | 返回日期中的年份 |
| `MONTH(date)` | 返回日期中的月份 |
| `DAY(date)` | 返回日期中月份内的第几天 |
| `DAYOFMONTH(date)` | 返回日期中月份内的第几天 |

### 示例

```sql
SELECT
    YEAR('2026-04-26')  AS year_value,
    MONTH('2026-04-26') AS month_value,
    DAY('2026-04-26')   AS day_value;
```

结果概念：

| 表达式 | 结果 |
| --- | --- |
| `YEAR('2026-04-26')` | `2026` |
| `MONTH('2026-04-26')` | `4` |
| `DAY('2026-04-26')` | `26` |

### 常见用途：按年月统计订单

```sql
SELECT
    YEAR(order_time) AS order_year,
    MONTH(order_time) AS order_month,
    COUNT(*) AS order_count
FROM orders
GROUP BY YEAR(order_time), MONTH(order_time);
```

这可以按年份、月份分组统计订单数量。

## 4.3.2 取得时、分、秒

### 函数表

| 函数 | 用法 |
| --- | --- |
| `HOUR(time)` | 返回小时 |
| `MINUTE(time)` | 返回分钟 |
| `SECOND(time)` | 返回秒 |

### 示例

```sql
SELECT
    HOUR('14:30:25')   AS hour_value,
    MINUTE('14:30:25') AS minute_value,
    SECOND('14:30:25') AS second_value;
```

结果：

| 表达式 | 结果 |
| --- | --- |
| `HOUR('14:30:25')` | `14` |
| `MINUTE('14:30:25')` | `30` |
| `SECOND('14:30:25')` | `25` |

### 常见用途：分析每天哪个时段登入最多

```sql
SELECT
    HOUR(login_time) AS login_hour,
    COUNT(*) AS login_count
FROM login_log
GROUP BY HOUR(login_time)
ORDER BY login_hour;
```

## 4.3.3 取得英文月份与星期名称

| 函数 | 用法 |
| --- | --- |
| `MONTHNAME(date)` | 返回英文月份名称 |
| `DAYNAME(date)` | 返回英文星期名称 |

示例：

```sql
SELECT
    MONTHNAME('2026-04-26') AS month_name,
    DAYNAME('2026-04-26')   AS day_name;
```

结果概念：

| 表达式 | 结果 |
| --- | --- |
| `MONTHNAME('2026-04-26')` | `April` |
| `DAYNAME('2026-04-26')` | `Sunday` |

### 使用提醒

这两个函数返回的是英文名称，不是数字。  
如果报表需要中文月份或中文星期，通常要另外使用 `CASE WHEN` 映射，或交给应用层处理。

示例：

```sql
SELECT
    CASE DAYOFWEEK(order_time)
        WHEN 1 THEN '星期日'
        WHEN 2 THEN '星期一'
        WHEN 3 THEN '星期二'
        WHEN 4 THEN '星期三'
        WHEN 5 THEN '星期四'
        WHEN 6 THEN '星期五'
        WHEN 7 THEN '星期六'
    END AS weekday_name
FROM orders;
```

## 4.3.4 WEEKDAY() 与 DAYOFWEEK()

这是本节最容易混淆的地方。

| 函数 | 编号规则 |
| --- | --- |
| `WEEKDAY(date)` | 周一为 `0`，周日为 `6` |
| `DAYOFWEEK(date)` | 周日为 `1`，周六为 `7` |

### 示例

```sql
SELECT
    WEEKDAY('2026-04-26')   AS weekday_value,
    DAYOFWEEK('2026-04-26') AS dayofweek_value;
```

如果 `2026-04-26` 是星期日，则结果概念是：

| 函数 | 结果 |
| --- | --- |
| `WEEKDAY('2026-04-26')` | `6` |
| `DAYOFWEEK('2026-04-26')` | `1` |

### 记忆方式

```text
WEEKDAY()：工作日思维，Monday = 0。
DAYOFWEEK()：日历周思维，Sunday = 1。
```

---

## 4.3.5 取得季度：QUARTER()

`QUARTER(date)` 返回日期所在季度，范围是 `1` 到 `4`。

| 月份 | 季度 |
| --- | --- |
| 1、2、3 月 | 第 1 季 |
| 4、5、6 月 | 第 2 季 |
| 7、8、9 月 | 第 3 季 |
| 10、11、12 月 | 第 4 季 |

示例：

```sql
SELECT QUARTER('2026-04-26') AS quarter_value;
```

结果：

```text
2
```

### 常见用途：按季度统计业绩

```sql
SELECT
    YEAR(order_time) AS order_year,
    QUARTER(order_time) AS order_quarter,
    SUM(amount) AS total_amount
FROM orders
GROUP BY YEAR(order_time), QUARTER(order_time);
```

---

## 4.3.6 取得周数：WEEK() / WEEKOFYEAR()

| 函数 | 用法 |
| --- | --- |
| `WEEK(date)` | 返回日期是一年中的第几周 |
| `WEEKOFYEAR(date)` | 返回日期是一年中的第几周 |

### 使用提醒

`WEEK()` 的结果可能受到周起始日和周数计算模式影响。  
不同系统、不同地区对「一年第一周」的定义可能不同。

因此，如果周报、财务报表、跨系统统计非常依赖周数，必须先确认团队规则：

- 一周从星期日开始，还是星期一开始？
- 一年第一周如何定义？
- 是否要遵循 ISO 周规则？

### 示例

```sql
SELECT
    WEEK('2026-04-26') AS week_value,
    WEEKOFYEAR('2026-04-26') AS weekofyear_value;
```

---

## 4.3.7 取得一年中的第几天、一个月中的第几天

| 函数 | 用法 |
| --- | --- |
| `DAYOFYEAR(date)` | 返回日期是一年中的第几天 |
| `DAYOFMONTH(date)` | 返回日期是该月中的第几天 |

示例：

```sql
SELECT
    DAYOFYEAR('2026-04-26') AS day_of_year,
    DAYOFMONTH('2026-04-26') AS day_of_month;
```

结果概念：

| 函数 | 结果概念 |
| --- | --- |
| `DAYOFYEAR('2026-04-26')` | 一年中的第几天 |
| `DAYOFMONTH('2026-04-26')` | `26` |

### 常见用途

`DAYOFYEAR()` 可用于年度进度、年度累计统计：

```sql
SELECT
    DAYOFYEAR(order_time) AS day_index,
    SUM(amount) AS daily_amount
FROM orders
WHERE YEAR(order_time) = 2026
GROUP BY DAYOFYEAR(order_time);
```

## 4.3.8 实务场景

### 场景 1：按年月统计订单数

```sql
SELECT
    YEAR(order_time) AS y,
    MONTH(order_time) AS m,
    COUNT(*) AS order_count
FROM orders
GROUP BY YEAR(order_time), MONTH(order_time)
ORDER BY y, m;
```

---

### 场景 2：按小时统计登入次数

```sql
SELECT
    HOUR(login_time) AS login_hour,
    COUNT(*) AS login_count
FROM login_log
GROUP BY HOUR(login_time)
ORDER BY login_hour;
```

---

### 场景 3：输出中文星期

```sql
SELECT
    order_id,
    order_time,
    CASE DAYOFWEEK(order_time)
        WHEN 1 THEN '星期日'
        WHEN 2 THEN '星期一'
        WHEN 3 THEN '星期二'
        WHEN 4 THEN '星期三'
        WHEN 5 THEN '星期四'
        WHEN 6 THEN '星期五'
        WHEN 7 THEN '星期六'
    END AS weekday_name
FROM orders;
```

---

### 场景 4：按季度统计金额

```sql
SELECT
    YEAR(order_time) AS y,
    QUARTER(order_time) AS q,
    SUM(amount) AS total_amount
FROM orders
GROUP BY YEAR(order_time), QUARTER(order_time)
ORDER BY y, q;
```

## 4.3.9 使用提醒

### 1. 在 SELECT 或 GROUP BY 中拆日期很常见

例如：

```sql
SELECT YEAR(order_time), MONTH(order_time), COUNT(*)
FROM orders
GROUP BY YEAR(order_time), MONTH(order_time);
```

这种用于报表统计很常见。

---

### 2. 在 WHERE 中对日期栏位套函数要谨慎

不推荐：

```sql
SELECT *
FROM orders
WHERE YEAR(order_time) = 2026;
```

更常见的范围写法：

```sql
SELECT *
FROM orders
WHERE order_time >= '2026-01-01'
  AND order_time <  '2027-01-01';
```

原因是对栏位套 `YEAR()` 可能影响索引使用。

---

### 3. 周数规则要统一

`WEEK()` 相关函数在跨系统报表中容易出问题。  
如果不同系统对第一周定义不同，统计结果可能不一致。

---

### 4. 英文名称不等于本地化显示

`MONTHNAME()`、`DAYNAME()` 返回英文名称。  
如果系统需要中文显示，通常要自己映射或交给应用层处理。

---

## 4.3.10 常见混淆点

### 混淆点 1：WEEKDAY() 和 DAYOFWEEK() 编号不同

| 日期是星期日时 | 结果 |
| --- | --- |
| `WEEKDAY(date)` | `6` |
| `DAYOFWEEK(date)` | `1` |

---

### 混淆点 2：MONTH() 返回数字，MONTHNAME() 返回英文名称

```sql
SELECT MONTH('2026-04-26'), MONTHNAME('2026-04-26');
```

结果概念：

```text
4, April
```

---

### 混淆点 3：DAY() 和 DAYOFMONTH() 很接近

两者都可以取得日期是当月第几天。  
如果只是取日，使用 `DAY()` 或 `DAYOFMONTH()` 都能表达。

---

### 混淆点 4：WEEK() 不是所有系统都一定一样

周数计算与周起始日、第一周规则有关，做正式周报时要先定义清楚。

---

### 混淆点 5：拆日期做筛选可能影响索引

`WHERE MONTH(order_time) = 4` 虽然好懂，但在大表上可能不适合。  
实务中常改用日期范围。

---

## 4.3.11 自我检查题

### 问题 1

`YEAR('2026-04-26')`、`MONTH('2026-04-26')`、`DAY('2026-04-26')` 分别返回什么？

答案：

```text
2026、4、26
```

---

### 问题 2

`WEEKDAY()` 和 `DAYOFWEEK()` 最大的差别是什么？

答案：编号规则不同。`WEEKDAY()` 周一为 `0`，周日为 `6`；`DAYOFWEEK()` 周日为 `1`，周六为 `7`。

---

### 问题 3

`MONTHNAME('2026-04-26')` 返回数字还是英文名称？

答案：英文月份名称，例如 `April`。

---

### 问题 4

下面 SQL 有什么潜在问题？

```sql
SELECT *
FROM orders
WHERE YEAR(order_time) = 2026;
```

答案：如果 `order_time` 有索引，对栏位套 `YEAR()` 可能影响索引使用。较常见做法是改成日期范围。

---

## 4.3.12 本节总结

本节可以浓缩成下面几句话：

1. `YEAR()`、`MONTH()`、`DAY()` 用来拆出日期中的年、月、日。
2. `HOUR()`、`MINUTE()`、`SECOND()` 用来拆出时间中的时、分、秒。
3. `MONTHNAME()`、`DAYNAME()` 返回英文名称，不是数字。
4. `WEEKDAY()` 与 `DAYOFWEEK()` 编号规则不同，最容易混淆。
5. `QUARTER()` 适合季度统计。
6. `WEEK()` 与 `WEEKOFYEAR()` 做周报时要确认周数规则。
7. 在 `WHERE` 中对日期栏位套函数可能影响索引，正式系统常改用范围条件。

如果只记一句话：

> 日期拆解函数适合做报表维度，但用在筛选条件时要注意索引与周数规则。
