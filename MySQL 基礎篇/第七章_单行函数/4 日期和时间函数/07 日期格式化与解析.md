# 4.7 日期格式化与解析

> - 所属章节：[第七章_单行函数 / 4 日期和时间函数](./README.md)  
> - 关键字：DATE_FORMAT、TIME_FORMAT、GET_FORMAT、STR_TO_DATE、格式符、日期转字符串、字符串转日期、日期显示、日期解析、报表格式、输入清洗  
> - 建议回查情境：需要把日期输出成指定格式；需要把字符串解析成日期；处理前端传入日期字符串、报表日期显示、日志日期格式、文件名日期、年月统计标签时。

## 本节导读

这一节处理日期时间函数中最常见的两个方向：

```text
日期时间 -> 字符串显示：DATE_FORMAT() / TIME_FORMAT()
字符串 -> 日期时间值：STR_TO_DATE()
```

这两个方向非常容易混淆。

例如：

- 数据库里有 `2026-04-26 14:30:25`，页面想显示成 `2026/04/26`。
- 使用者输入 `04/26/2026`，系统想转换成真正的日期值。
- 报表要按 `YYYY-MM` 分组显示。
- 文件名要加上 `YYYYMMDD`。
- 前端传来的字符串要先解析成日期再比较。

这些需求都和本节有关。

## 4.7.1 DATE_FORMAT()：日期时间转字符串

`DATE_FORMAT(date, fmt)` 用来把日期时间按照指定格式输出。

### 基本语法

```sql
DATE_FORMAT(date, fmt)
```

| 参数 | 说明 |
| --- | --- |
| `date` | 日期或日期时间值 |
| `fmt` | 格式字符串 |

---

### 示例 1：输出年月日

```sql
SELECT DATE_FORMAT('2026-04-26 14:30:25', '%Y-%m-%d') AS result;
```

结果：

```text
2026-04-26
```

---

### 示例 2：输出年月日时分秒

```sql
SELECT DATE_FORMAT('2026-04-26 14:30:25', '%Y-%m-%d %H:%i:%s') AS result;
```

结果：

```text
2026-04-26 14:30:25
```

---

### 示例 3：输出年月标签

```sql
SELECT DATE_FORMAT('2026-04-26 14:30:25', '%Y-%m') AS year_month;
```

结果：

```text
2026-04
```

---

### 示例 4：输出文件名日期

```sql
SELECT DATE_FORMAT(NOW(), '%Y%m%d_%H%i%s') AS file_time;
```

结果概念：

```text
20260426_143025
```

---

## 4.7.2 TIME_FORMAT()：时间转字符串

`TIME_FORMAT(time, fmt)` 用来格式化时间。

```sql
SELECT TIME_FORMAT('14:30:25', '%H:%i:%s') AS result;
```

结果：

```text
14:30:25
```

也可以只输出时分：

```sql
SELECT TIME_FORMAT('14:30:25', '%H:%i') AS result;
```

结果：

```text
14:30
```

---

## 4.7.3 STR_TO_DATE()：字符串转日期

`STR_TO_DATE(str, fmt)` 和 `DATE_FORMAT()` 方向相反。

| 函数 | 方向 |
| --- | --- |
| `DATE_FORMAT()` | 日期 -> 字符串 |
| `STR_TO_DATE()` | 字符串 -> 日期 |

### 示例 1：解析 `YYYY-MM-DD`

```sql
SELECT STR_TO_DATE('2026-04-26', '%Y-%m-%d') AS result;
```

结果概念：

```text
2026-04-26
```

---

### 示例 2：解析 `MM/DD/YYYY`

```sql
SELECT STR_TO_DATE('04/26/2026', '%m/%d/%Y') AS result;
```

结果概念：

```text
2026-04-26
```

---

### 示例 3：解析 `YYYYMMDDHHMISS`

```sql
SELECT STR_TO_DATE('20260426143025', '%Y%m%d%H%i%s') AS result;
```

结果概念：

```text
2026-04-26 14:30:25
```

---

## 4.7.4 GET_FORMAT()

`GET_FORMAT(date_type, format_type)` 会返回预定义的日期格式。

示例：

```sql
SELECT GET_FORMAT(DATE, 'USA') AS fmt;
```

结果概念可能类似：

```text
%m.%d.%Y
```

然后可以搭配 `DATE_FORMAT()`：

```sql
SELECT DATE_FORMAT(NOW(), GET_FORMAT(DATE, 'USA')) AS result;
```

### 使用建议

`GET_FORMAT()` 了解即可。  
实际工作中，更多时候会直接写明确格式，例如：

```sql
DATE_FORMAT(order_time, '%Y-%m-%d')
```

因为直接写格式更可读，也更符合团队固定输出规范。

---

## 4.7.5 常用格式符

| 格式符 | 说明 | 示例概念 |
| --- | --- | --- |
| `%Y` | 4 位年份 | `2026` |
| `%y` | 2 位年份 | `26` |
| `%m` | 两位月份 | `04` |
| `%c` | 数字月份 | `4` |
| `%M` | 英文月份全名 | `April` |
| `%b` | 英文月份缩写 | `Apr` |
| `%d` | 两位日期 | `06`、`26` |
| `%e` | 数字日期 | `6`、`26` |
| `%H` | 24 小时制，两位小时 | `14` |
| `%k` | 24 小时制，数字小时 | `14` |
| `%h` / `%I` | 12 小时制，两位小时 | `02` |
| `%l` | 12 小时制，数字小时 | `2` |
| `%i` | 两位分钟 | `30` |
| `%S` / `%s` | 两位秒 | `25` |
| `%T` | 24 小时制时间 | `14:30:25` |
| `%r` | 12 小时制时间 | `02:30:25 PM` |
| `%p` | AM 或 PM | `AM` / `PM` |
| `%W` | 星期英文全名 | `Sunday` |
| `%a` | 星期英文缩写 | `Sun` |
| `%w` | 星期数字，周日为 0 | `0` 到 `6` |
| `%j` | 一年中的第几天，三位数 | `001` 到 `366` |
| `%U` | 一年中的第几周，周日为第一天 | 周数 |
| `%u` | 一年中的第几周，周一为第一天 | 周数 |
| `%%` | 百分号本身 | `%` |

### 最常用格式符

实际工作中最常用的是：

```text
%Y 年
%m 月
%d 日
%H 小时
%i 分钟
%s 秒
```

记住这六个，基本就能处理大多数日期格式化需求。

## 4.7.6 DATE_FORMAT() 实务场景

### 场景 1：报表显示日期

```sql
SELECT
    order_id,
    DATE_FORMAT(order_time, '%Y-%m-%d') AS order_date
FROM orders;
```

---

### 场景 2：按年月分组统计

```sql
SELECT
    DATE_FORMAT(order_time, '%Y-%m') AS order_month,
    COUNT(*) AS order_count
FROM orders
GROUP BY DATE_FORMAT(order_time, '%Y-%m')
ORDER BY order_month;
```

如果资料量大，也可以考虑用范围条件或预先维护年月栏位，视系统设计而定。

---

### 场景 3：产生导出文件名时间

```sql
SELECT DATE_FORMAT(NOW(), '%Y%m%d_%H%i%s') AS export_time;
```

结果概念：

```text
20260426_143025
```

---

### 场景 4：显示中文日期格式

```sql
SELECT DATE_FORMAT(NOW(), '%Y年%m月%d日') AS display_date;
```

结果概念：

```text
2026年04月26日
```

---

## 4.7.7 STR_TO_DATE() 实务场景

### 场景 1：解析前端传来的日期字符串

前端传入：

```text
04/26/2026
```

SQL 解析：

```sql
SELECT STR_TO_DATE('04/26/2026', '%m/%d/%Y') AS parsed_date;
```

---

### 场景 2：解析批次汇入的日期文字

原始资料：

```text
20260426143025
```

解析：

```sql
SELECT STR_TO_DATE('20260426143025', '%Y%m%d%H%i%s') AS parsed_datetime;
```

---

### 场景 3：把字符串解析后用于查询条件

```sql
SELECT *
FROM orders
WHERE order_time >= STR_TO_DATE('2026-04-01', '%Y-%m-%d')
  AND order_time <  STR_TO_DATE('2026-05-01', '%Y-%m-%d');
```

---

## 4.7.8 DATE_FORMAT() 与 STR_TO_DATE() 对比

| 比较项目 | `DATE_FORMAT()` | `STR_TO_DATE()` |
| --- | --- | --- |
| 方向 | 日期时间 -> 字符串 | 字符串 -> 日期时间 |
| 常见用途 | 显示、报表、文件名、标签 | 输入解析、资料清洗、汇入转换 |
| 参数 1 | 日期时间值 | 字符串 |
| 参数 2 | 输出格式 | 输入字符串对应格式 |
| 结果 | 字符串 | 日期或日期时间值 |

### 记忆方式

```text
DATE_FORMAT：日期拿来 format，变成显示文字。
STR_TO_DATE：string to date，字符串变日期。
```

---

## 4.7.9 使用提醒

### 1. 格式必须和原字符串匹配

```sql
SELECT STR_TO_DATE('2026-04-26', '%m/%d/%Y');
```

这就不匹配，因为字符串是 `YYYY-MM-DD`，格式却写成 `MM/DD/YYYY`。  
结果可能是 `NULL`、错误解析或 warning。

正确：

```sql
SELECT STR_TO_DATE('2026-04-26', '%Y-%m-%d');
```

---

### 2. `%i` 是分钟，不是 `%m`

这是非常常见的错误。

| 格式符 | 意义 |
| --- | --- |
| `%m` | 月份 |
| `%i` | 分钟 |

时间格式应该写：

```sql
'%H:%i:%s'
```

不是：

```sql
'%H:%m:%s'
```

---

### 3. DATE_FORMAT() 返回的是字符串

```sql
DATE_FORMAT(order_time, '%Y-%m')
```

结果是字符串，不是日期类型。  
如果后续要做日期计算，要确认是否需要保留原日期值。

---

### 4. 在 WHERE 中对栏位使用 DATE_FORMAT() 要谨慎

不推荐大表中随意写：

```sql
SELECT *
FROM orders
WHERE DATE_FORMAT(order_time, '%Y-%m') = '2026-04';
```

更常见的范围写法：

```sql
SELECT *
FROM orders
WHERE order_time >= '2026-04-01'
  AND order_time <  '2026-05-01';
```

原因是对栏位套 `DATE_FORMAT()` 可能影响索引使用。

---

### 5. 显示格式和储存格式要分开思考

数据库通常应该存标准日期时间类型，例如 `DATE`、`DATETIME`、`TIMESTAMP`。  
至于要显示成 `2026/04/26`、`2026年04月26日`，可以在查询或应用层格式化。

不要因为页面要显示某种格式，就把数据库栏位设计成字符串。

---

## 4.7.10 常见混淆点

### 混淆点 1：DATE_FORMAT() 和 STR_TO_DATE() 方向相反

```text
DATE_FORMAT：日期 -> 字符串
STR_TO_DATE：字符串 -> 日期
```

---

### 混淆点 2：%m 是月份，%i 是分钟

```sql
DATE_FORMAT(NOW(), '%Y-%m-%d %H:%i:%s')
```

这里：

```text
%m = 月
%i = 分
%s = 秒
```

---

### 混淆点 3：DATE_FORMAT() 结果不是日期类型

它输出的是格式化后的字符串。  
如果只是显示没问题；如果要继续做日期运算，要小心类型。

---

### 混淆点 4：STR_TO_DATE() 格式不匹配可能解析失败

```sql
STR_TO_DATE('04/26/2026', '%Y-%m-%d')
```

格式和原字串不匹配，容易得到错误结果、`NULL` 或 warning。

---

### 混淆点 5：GET_FORMAT() 不是必须掌握的高频函数

`GET_FORMAT()` 可以取得预设格式，但实务中更常直接写明确格式符。

## 4.7.11 自我检查题

### 问题 1

把 `2026-04-26 14:30:25` 显示成 `2026-04-26`，应该怎么写？

答案：

```sql
SELECT DATE_FORMAT('2026-04-26 14:30:25', '%Y-%m-%d');
```

---

### 问题 2

把字符串 `04/26/2026` 解析成日期，应该怎么写？

答案：

```sql
SELECT STR_TO_DATE('04/26/2026', '%m/%d/%Y');
```

---

### 问题 3

`%m` 和 `%i` 分别代表什么？

答案：`%m` 是月份，`%i` 是分钟。

---

### 问题 4

下面 SQL 有什么潜在问题？

```sql
SELECT *
FROM orders
WHERE DATE_FORMAT(order_time, '%Y-%m') = '2026-04';
```

答案：对 `order_time` 套 `DATE_FORMAT()` 可能影响索引使用。更常见写法是使用日期范围。

```sql
WHERE order_time >= '2026-04-01'
  AND order_time <  '2026-05-01'
```

---

### 问题 5

`DATE_FORMAT()` 的结果是日期类型还是字符串？

答案：字符串。

## 4.7.12 本节总结

本节可以浓缩成下面几句话：

1. `DATE_FORMAT()` 用来把日期时间格式化成字符串。
2. `TIME_FORMAT()` 用来把时间格式化成字符串。
3. `STR_TO_DATE()` 用来把字符串解析成日期时间。
4. `DATE_FORMAT()` 和 `STR_TO_DATE()` 方向相反，最容易混淆。
5. 最常用格式符是 `%Y`、`%m`、`%d`、`%H`、`%i`、`%s`。
6. `%m` 是月份，`%i` 是分钟。
7. `STR_TO_DATE()` 的格式必须和原字符串匹配。
8. 在 `WHERE` 中对日期栏位使用 `DATE_FORMAT()` 可能影响索引，正式查询常改用范围条件。
9. 数据库存标准日期时间类型，显示格式可以在查询或应用层处理。

如果只记一句话：

> `DATE_FORMAT()` 是日期变文字，`STR_TO_DATE()` 是文字变日期；格式符中 `%m` 是月，`%i` 才是分钟。
