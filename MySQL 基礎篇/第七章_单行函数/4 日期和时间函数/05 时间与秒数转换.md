# 4.5 时间与秒数转换

> - 所属章节：[第七章_单行函数 / 4 日期和时间函数](./README.md)  
> - 关键字：TIME_TO_SEC、SEC_TO_TIME、时间转秒、秒数转时间、HH:MM:SS、时间长度、时长计算、时间比较  
> - 建议回查情境：需要把时间换算成总秒数；需要把秒数还原成 `HH:MM:SS`；计算工时、在线时长、处理耗时、等待时间、通话时间或时间差显示时。

## 本节导读

这一节只有两个函数，但非常实用：

| 方向 | 函数 |
| --- | --- |
| 时间 -> 秒数 | `TIME_TO_SEC(time)` |
| 秒数 -> 时间 | `SEC_TO_TIME(seconds)` |

它们适合处理「时长」问题。

例如：

- `01:30:00` 等于多少秒？
- `5400` 秒要显示成几个小时几分钟？
- 两个时间差要不要先转成秒比较？
- 报表中要把处理耗时显示成 `HH:MM:SS`。

学习这一节时，要分清：

```text
时间点：某个具体发生时间，例如 2026-04-26 14:30:00。
时间长度：经过多久，例如 1 小时 30 分钟。
```

`TIME_TO_SEC()` 和 `SEC_TO_TIME()` 更常用于时间长度或一天中的时间值处理。

---

## 你会在这篇学到什么

- `TIME_TO_SEC(time)` 如何把时间转换成总秒数。
- `SEC_TO_TIME(seconds)` 如何把秒数转换成可读时间。
- 为什么做时间长度比较时，秒数通常更方便。
- 如何计算工时、在线时长、处理耗时。
- `TIME_TO_SEC()` 与 `TIMESTAMPDIFF()` 的使用场景差异。
- 使用 `NOW()`、`TIMEDIFF()`、`TIME_TO_SEC()` 时要注意什么。
- 为什么显示结果和计算用结果最好分开处理。

## 快速定位

| 想解决的问题 | 优先使用 |
| --- | --- |
| 把 `HH:MM:SS` 转成秒 | `TIME_TO_SEC(time)` |
| 把秒数转回 `HH:MM:SS` | `SEC_TO_TIME(seconds)` |
| 比较两个时间长度大小 | 先转秒数再比较 |
| 显示处理耗时 | `SEC_TO_TIME(seconds)` |
| 计算两个时间点差几秒 | `TIMESTAMPDIFF(SECOND, start, end)` 更直接 |
| 把 `TIMEDIFF()` 结果转秒 | `TIME_TO_SEC(TIMEDIFF(end, start))` |

---

## 快速回查表

| 函数 | 用法 | 结果概念 | 高频程度 |
| --- | --- | --- | --- |
| `TIME_TO_SEC(time)` | 将时间转换成秒 | `01:00:00` -> `3600` | 高频 |
| `SEC_TO_TIME(seconds)` | 将秒数转换成时间 | `3600` -> `01:00:00` | 高频 |

---

## 4.5.1 时间转秒数：TIME_TO_SEC()

`TIME_TO_SEC(time)` 会把时间转换成秒数。

计算公式可以理解为：

```text
小时 * 3600 + 分钟 * 60 + 秒
```

### 示例

```sql
SELECT TIME_TO_SEC('01:30:25') AS total_seconds;
```

计算过程：

```text
1 小时 = 3600 秒
30 分钟 = 1800 秒
25 秒 = 25 秒
总计 = 3600 + 1800 + 25 = 5425 秒
```

结果：

```text
5425
```

---

## 4.5.2 秒数转时间：SEC_TO_TIME()

`SEC_TO_TIME(seconds)` 会把秒数转换成时间格式。

### 示例

```sql
SELECT SEC_TO_TIME(5425) AS time_value;
```

结果：

```text
01:30:25
```

---

## 4.5.3 两个函数互相搭配

```sql
SELECT SEC_TO_TIME(TIME_TO_SEC('01:30:25')) AS result;
```

结果：

```text
01:30:25
```

概念上：

```text
'01:30:25' -> 5425 秒 -> '01:30:25'
```

---

## 4.5.4 与 TIMEDIFF() 搭配使用

`TIMEDIFF(time1, time2)` 可以计算两个时间的差。  
如果想把这个差值转换成秒，可以搭配 `TIME_TO_SEC()`。

```sql
SELECT TIME_TO_SEC(TIMEDIFF('14:30:00', '13:00:00')) AS diff_seconds;
```

结果概念：

```text
5400
```

因为 `14:30:00 - 13:00:00 = 01:30:00`，也就是 `5400` 秒。

---

## 4.5.5 实务场景

### 场景 1：计算处理耗时秒数

假设表中有开始与结束时间：

```sql
SELECT
    task_id,
    TIME_TO_SEC(TIMEDIFF(end_time, start_time)) AS duration_seconds
FROM task_log;
```

---

### 场景 2：把处理耗时显示成 HH:MM:SS

如果表中已经存了秒数：

```sql
SELECT
    task_id,
    duration_seconds,
    SEC_TO_TIME(duration_seconds) AS duration_text
FROM task_log;
```

---

### 场景 3：筛选处理超过 30 分钟的任务

30 分钟等于 `1800` 秒。

```sql
SELECT *
FROM task_log
WHERE TIME_TO_SEC(TIMEDIFF(end_time, start_time)) > 1800;
```

如果资料量大，建议考虑在写入时就保存 `duration_seconds`，避免每次查询都计算。

---

### 场景 4：统计平均处理时长

```sql
SELECT
    SEC_TO_TIME(AVG(duration_seconds)) AS avg_duration
FROM task_log;
```

如果 `duration_seconds` 是数字栏位，这种统计会比较直观。

---

### 场景 5：计算当天时间点距离零点几秒

```sql
SELECT TIME_TO_SEC(CURTIME()) AS seconds_from_midnight;
```

如果当前时间是 `14:30:25`，结果概念是：

```text
52225
```

---

## 4.5.6 TIME_TO_SEC() 与 TIMESTAMPDIFF() 怎么选

这两个都可能和「秒数」有关，但场景不同。

| 需求 | 推荐方式 |
| --- | --- |
| 把单纯时间值 `HH:MM:SS` 转成秒 | `TIME_TO_SEC()` |
| 把两个完整日期时间相差转换成秒 | `TIMESTAMPDIFF(SECOND, start_time, end_time)` |
| 已经有 `TIMEDIFF()` 结果，想转秒 | `TIME_TO_SEC(TIMEDIFF(...))` |
| 只想把秒数显示成时间格式 | `SEC_TO_TIME()` |

示例：完整日期时间差几秒：

```sql
SELECT TIMESTAMPDIFF(
    SECOND,
    '2026-04-26 14:00:00',
    '2026-04-26 14:30:25'
) AS diff_seconds;
```

结果：

```text
1825
```

## 4.5.7 使用提醒

### 1. TIME_TO_SEC() 更适合处理时间或时长

```sql
SELECT TIME_TO_SEC('01:30:00');
```

很清楚地表示：把 `1 小时 30 分钟` 转成秒。

如果是两个日期时间点之间的差，优先考虑 `TIMESTAMPDIFF()`。

---

### 2. 秒数适合计算，时间格式适合显示

建议区分：

| 用途 | 适合格式 |
| --- | --- |
| 计算、比较、排序 | 秒数，例如 `5400` |
| 展示给人看 | 时间格式，例如 `01:30:00` |

---

### 3. 大量查询时避免重复计算

如果经常需要查处理时长，建议在业务允许时保存一个 `duration_seconds` 栏位。  
这样统计、排序、筛选会更直观，也更有利于优化。

---

### 4. 注意负数时间差

如果结束时间早于开始时间，时间差可能为负。  
正式系统中要确认是否允许这种资料，或先做资料校验。

---

## 4.5.8 常见混淆点

### 混淆点 1：TIME_TO_SEC() 不是日期转时间戳

```sql
TIME_TO_SEC('01:00:00')
```

得到的是：

```text
3600
```

它不是 Unix 时间戳。  
Unix 时间戳要看 `UNIX_TIMESTAMP()`。

---

### 混淆点 2：SEC_TO_TIME() 是秒数转时间，不是转日期

```sql
SELECT SEC_TO_TIME(3600);
```

结果是：

```text
01:00:00
```

不是某个具体日期时间。

---

### 混淆点 3：时间点和时间长度不同

```text
2026-04-26 14:30:00  是时间点。
01:30:00             是时间长度或一天中的时间。
```

`TIME_TO_SEC()`、`SEC_TO_TIME()` 更常用于后者。

---

### 混淆点 4：TIMEDIFF() 的结果还不是秒数

```sql
SELECT TIMEDIFF('14:30:00', '13:00:00');
```

结果概念是：

```text
01:30:00
```

如果要秒数，需要：

```sql
SELECT TIME_TO_SEC(TIMEDIFF('14:30:00', '13:00:00'));
```

---

## 4.5.9 自我检查题

### 问题 1

`TIME_TO_SEC('01:30:00')` 的结果是什么？

答案：`5400`。

---

### 问题 2

`SEC_TO_TIME(5400)` 的结果是什么？

答案：`01:30:00`。

---

### 问题 3

如果要计算两个完整日期时间相差几秒，优先考虑哪个函数？

答案：`TIMESTAMPDIFF(SECOND, start_time, end_time)`。

---

### 问题 4

`TIME_TO_SEC()` 得到的是 Unix 时间戳吗？

答案：不是。它只是把时间值转换成总秒数。

---

## 4.5.10 本节总结

本节可以浓缩成下面几句话：

1. `TIME_TO_SEC(time)` 把时间转换成总秒数。
2. `SEC_TO_TIME(seconds)` 把秒数转换成 `HH:MM:SS` 形式。
3. 秒数适合计算、比较、排序；时间格式适合显示。
4. 两个完整日期时间相差几秒时，`TIMESTAMPDIFF(SECOND, start, end)` 通常更直接。
5. `TIME_TO_SEC()` 不是 Unix 时间戳函数，不要和 `UNIX_TIMESTAMP()` 混淆。

如果只记一句话：

> `TIME_TO_SEC()` 负责把时间变成秒数，`SEC_TO_TIME()` 负责把秒数变回可读时间。
