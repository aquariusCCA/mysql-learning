# 4.6 日期时间的加减与计算

> - 所属章节：[第七章_单行函数 / 4 日期和时间函数](./README.md)  
> - 关键字：DATE_ADD、ADDDATE、DATE_SUB、SUBDATE、INTERVAL、ADDTIME、SUBTIME、DATEDIFF、TIMEDIFF、TIMESTAMPDIFF、FROM_DAYS、TO_DAYS、LAST_DAY、MAKEDATE、MAKETIME、PERIOD_ADD、日期加减、时间差、最近 N 天、月底、工时  
> - 建议回查情境：需要给日期加减天数、月份或年份；计算两个日期相差几天；计算两个时间相差多久；查询最近 N 天资料；取得月底日期；根据年份与第几天生成日期；处理工时、到期日、有效期、报表区间时。

## 本节导读

这一节是日期时间函数中最偏实战的一节。

前面几节主要在「取得」和「拆解」日期时间，例如取得当前时间、取年份、取月份、取星期。  
这一节开始真正做日期时间运算，例如：

- 当前时间往后加 7 天。
- 到期日往前减 1 个月。
- 计算两个日期相差几天。
- 计算两个时间相差多久。
- 查询最近 7 天新增用户。
- 找出某个月的最后一天。
- 根据年份和第几天生成日期。

学习本节时，重点不是只背函数名，而是先分清需求类型：

```text
日期加减：DATE_ADD() / DATE_SUB()
时间加减：ADDTIME() / SUBTIME()
日期差：DATEDIFF()
时间差：TIMEDIFF()
完整时间点差值：TIMESTAMPDIFF()
辅助日期计算：LAST_DAY() / MAKEDATE() / TO_DAYS()
```

## 4.6.1 使用 INTERVAL 做日期时间加减

`DATE_ADD()` 和 `DATE_SUB()` 是本节最重要的函数。

### 基本语法

```sql
DATE_ADD(datetime, INTERVAL expr type)
DATE_SUB(datetime, INTERVAL expr type)
```

其中：

| 部分 | 说明 |
| --- | --- |
| `datetime` | 要计算的日期或日期时间 |
| `expr` | 要加减的数量 |
| `type` | 单位，例如 `DAY`、`MONTH`、`YEAR`、`HOUR` |

---

### 常见 INTERVAL 单位

| 单位 | 说明 | 示例 |
| --- | --- | --- |
| `SECOND` | 秒 | `INTERVAL 10 SECOND` |
| `MINUTE` | 分钟 | `INTERVAL 5 MINUTE` |
| `HOUR` | 小时 | `INTERVAL 2 HOUR` |
| `DAY` | 天 | `INTERVAL 7 DAY` |
| `WEEK` | 周 | `INTERVAL 2 WEEK` |
| `MONTH` | 月 | `INTERVAL 1 MONTH` |
| `QUARTER` | 季 | `INTERVAL 1 QUARTER` |
| `YEAR` | 年 | `INTERVAL 1 YEAR` |
| `YEAR_MONTH` | 年月组合 | `INTERVAL '1-2' YEAR_MONTH` |
| `DAY_HOUR` | 日小时组合 | `INTERVAL '1 2' DAY_HOUR` |
| `MINUTE_SECOND` | 分秒组合 | `INTERVAL '1:30' MINUTE_SECOND` |

---

## 4.6.2 DATE_ADD()：日期时间加法

### 示例 1：加 1 天

```sql
SELECT DATE_ADD('2026-04-26', INTERVAL 1 DAY) AS result;
```

结果：

```text
2026-04-27
```

---

### 示例 2：加 1 个月

```sql
SELECT DATE_ADD('2026-04-26', INTERVAL 1 MONTH) AS result;
```

结果：

```text
2026-05-26
```

---

### 示例 3：加 30 分钟

```sql
SELECT DATE_ADD('2026-04-26 14:30:00', INTERVAL 30 MINUTE) AS result;
```

结果：

```text
2026-04-26 15:00:00
```

---

### 示例 4：使用负数也可以达到减法效果

```sql
SELECT DATE_ADD('2026-04-26', INTERVAL -7 DAY) AS result;
```

结果：

```text
2026-04-19
```

不过为了可读性，减法场景通常使用 `DATE_SUB()` 更直观。

---

## 4.6.3 DATE_SUB()：日期时间减法

### 示例 1：减 7 天

```sql
SELECT DATE_SUB('2026-04-26', INTERVAL 7 DAY) AS result;
```

结果：

```text
2026-04-19
```

---

### 示例 2：减 1 年

```sql
SELECT DATE_SUB('2026-04-26', INTERVAL 1 YEAR) AS result;
```

结果：

```text
2025-04-26
```

---

### 示例 3：减 2 小时

```sql
SELECT DATE_SUB('2026-04-26 14:30:00', INTERVAL 2 HOUR) AS result;
```

结果：

```text
2026-04-26 12:30:00
```

---

## 4.6.4 ADDDATE() 与 SUBDATE()

`ADDDATE()` 和 `SUBDATE()` 可以理解为 `DATE_ADD()`、`DATE_SUB()` 的类似写法。

```sql
SELECT ADDDATE('2026-04-26', INTERVAL 1 DAY);
SELECT SUBDATE('2026-04-26', INTERVAL 1 DAY);
```

实务上建议优先熟悉：

```sql
DATE_ADD()
DATE_SUB()
```

因为语义更清楚，也更容易和 `INTERVAL` 搭配记忆。

---

## 4.6.5 ADDTIME() 与 SUBTIME()

`ADDTIME()` 和 `SUBTIME()` 主要用于时间值加减。

### ADDTIME()

```sql
SELECT ADDTIME('14:30:00', '01:15:00') AS result;
```

结果：

```text
15:45:00
```

也可以加秒数：

```sql
SELECT ADDTIME('14:30:00', 30) AS result;
```

概念上是加 30 秒。

---

### SUBTIME()

```sql
SELECT SUBTIME('14:30:00', '01:15:00') AS result;
```

结果：

```text
13:15:00
```

---

## 4.6.6 日期差：DATEDIFF()

`DATEDIFF(date1, date2)` 返回两个日期相差的天数。

```sql
SELECT DATEDIFF('2026-04-26', '2026-04-20') AS diff_days;
```

结果：

```text
6
```

它的方向是：

```text
date1 - date2
```

所以：

```sql
SELECT DATEDIFF('2026-04-20', '2026-04-26') AS diff_days;
```

结果：

```text
-6
```

### 重点提醒

`DATEDIFF()` 只关心日期差，不关心时分秒。

```sql
SELECT DATEDIFF('2026-04-26 23:59:59', '2026-04-26 00:00:00') AS diff_days;
```

结果概念：

```text
0
```

因为日期都是同一天。

---

## 4.6.7 时间差：TIMEDIFF()

`TIMEDIFF(time1, time2)` 返回两个时间的差。

```sql
SELECT TIMEDIFF('14:30:00', '13:00:00') AS diff_time;
```

结果：

```text
01:30:00
```

也可以用于日期时间：

```sql
SELECT TIMEDIFF('2026-04-26 14:30:00', '2026-04-26 13:00:00') AS diff_time;
```

结果：

```text
01:30:00
```

如果要得到秒数，可以搭配：

```sql
SELECT TIME_TO_SEC(TIMEDIFF('14:30:00', '13:00:00')) AS diff_seconds;
```

---

## 4.6.8 更实用的完整差值函数：TIMESTAMPDIFF()

虽然原始教材可能重点列 `DATEDIFF()` 和 `TIMEDIFF()`，但实际工作中 `TIMESTAMPDIFF()` 非常常用。

语法：

```sql
TIMESTAMPDIFF(unit, start_datetime, end_datetime)
```

它的方向是：

```text
end_datetime - start_datetime
```

### 示例：相差几秒

```sql
SELECT TIMESTAMPDIFF(
    SECOND,
    '2026-04-26 14:00:00',
    '2026-04-26 14:30:25'
) AS diff_seconds;
```

结果：

```text
1825
```

### 示例：相差几天

```sql
SELECT TIMESTAMPDIFF(
    DAY,
    '2026-04-20 10:00:00',
    '2026-04-26 09:00:00'
) AS diff_days;
```

### 什么时候用 TIMESTAMPDIFF()

| 需求 | 推荐 |
| --- | --- |
| 精确计算两个时间点差几秒 | `TIMESTAMPDIFF(SECOND, start, end)` |
| 计算差几分钟 | `TIMESTAMPDIFF(MINUTE, start, end)` |
| 计算差几小时 | `TIMESTAMPDIFF(HOUR, start, end)` |
| 计算差几天 | `TIMESTAMPDIFF(DAY, start, end)` |
| 只比较日期天数 | `DATEDIFF()` 也可以 |

## 4.6.9 日期辅助函数

### LAST_DAY()

`LAST_DAY(date)` 返回该日期所在月份的最后一天。

```sql
SELECT LAST_DAY('2026-04-26') AS month_end;
```

结果：

```text
2026-04-30
```

常见场景：月底报表、账单截止日、月底到期日。

---

### MAKEDATE()

`MAKEDATE(year,n)` 返回某一年中第 `n` 天对应的日期。

```sql
SELECT MAKEDATE(2026, 1) AS result;
```

结果：

```text
2026-01-01
```

```sql
SELECT MAKEDATE(2026, 32) AS result;
```

结果：

```text
2026-02-01
```

---

### MAKETIME()

`MAKETIME(hour,minute,second)` 根据时、分、秒组合成时间。

```sql
SELECT MAKETIME(10, 21, 23) AS result;
```

结果：

```text
10:21:23
```

---

### TO_DAYS() 与 FROM_DAYS()

| 函数 | 说明 |
| --- | --- |
| `TO_DAYS(date)` | 把日期转换成从某个基准起算的天数编号 |
| `FROM_DAYS(n)` | 把天数编号转换回日期 |

示例：

```sql
SELECT TO_DAYS('2026-04-26') AS day_number;
```

`TO_DAYS()` 有时可以辅助日期差计算，但现代业务中更常直接使用 `DATEDIFF()` 或范围条件。

---

### PERIOD_ADD()

`PERIOD_ADD(period,n)` 用于对年月周期加上指定月数。  
常见 `period` 格式是 `YYYYMM`。

```sql
SELECT PERIOD_ADD(202604, 1) AS next_period;
```

结果概念：

```text
202605
```

适合处理年月周期，例如账期、报表期别。

## 4.6.10 实务场景

### 场景 1：查询最近 7 天新增用户

推荐写法：

```sql
SELECT COUNT(*) AS user_count
FROM users
WHERE created_at >= NOW() - INTERVAL 7 DAY;
```

不太推荐大表中这样写：

```sql
WHERE TO_DAYS(NOW()) - TO_DAYS(created_at) <= 7
```

因为对栏位 `created_at` 套函数可能影响索引使用。

---

### 场景 2：查询今天资料

```sql
SELECT *
FROM orders
WHERE order_time >= CURDATE()
  AND order_time <  CURDATE() + INTERVAL 1 DAY;
```

---

### 场景 3：计算订单到期日

```sql
SELECT
    order_id,
    order_time,
    DATE_ADD(order_time, INTERVAL 30 DAY) AS due_date
FROM orders;
```

---

### 场景 4：找出已经超过 30 天未登入的用户

```sql
SELECT *
FROM users
WHERE last_login_time < NOW() - INTERVAL 30 DAY;
```

---

### 场景 5：计算任务处理秒数

```sql
SELECT
    task_id,
    TIMESTAMPDIFF(SECOND, start_time, end_time) AS duration_seconds
FROM task_log;
```

---

### 场景 6：取得本月最后一天

```sql
SELECT LAST_DAY(CURDATE()) AS current_month_end;
```

---

## 4.6.11 使用提醒

### 1. `DATEDIFF()` 只看日期，不看时分秒

如果你要精确到秒、分钟、小时，不要只用 `DATEDIFF()`。

---

### 2. 完整时间差优先考虑 `TIMESTAMPDIFF()`

```sql
TIMESTAMPDIFF(SECOND, start_time, end_time)
```

比 `TIME_TO_SEC(TIMEDIFF(...))` 更直接，尤其是跨日期时。

---

### 3. 查询最近 N 天建议使用范围条件

推荐：

```sql
WHERE created_at >= NOW() - INTERVAL 7 DAY
```

谨慎：

```sql
WHERE TO_DAYS(NOW()) - TO_DAYS(created_at) <= 7
```

---

### 4. 对索引栏位套函数要谨慎

不推荐：

```sql
WHERE DATE_ADD(created_at, INTERVAL 7 DAY) >= NOW()
```

更常思考是否能改写成：

```sql
WHERE created_at >= NOW() - INTERVAL 7 DAY
```

---

### 5. 月份加减要注意月底

例如 `2026-01-31 + INTERVAL 1 MONTH` 这类场景，要确认 MySQL 的结果是否符合业务预期。月底、账单日、到期日规则要特别明确。

---

## 4.6.12 常见混淆点

### 混淆点 1：DATE_ADD() 和 DATE_SUB() 不是只能加减天

它们可以加减秒、分钟、小时、天、周、月、季、年等。

---

### 混淆点 2：DATEDIFF() 和 TIMEDIFF() 不是同一个方向

| 函数 | 用途 |
| --- | --- |
| `DATEDIFF(date1,date2)` | 计算日期差天数 |
| `TIMEDIFF(time1,time2)` | 计算时间差 |

---

### 混淆点 3：DATEDIFF() 忽略时分秒

同一天内相差 23 小时，`DATEDIFF()` 仍可能返回 `0`。

---

### 混淆点 4：TIMESTAMPDIFF() 的参数方向是 start 到 end

```sql
TIMESTAMPDIFF(unit, start_time, end_time)
```

方向是：

```text
end_time - start_time
```

---

### 混淆点 5：TO_DAYS() 可以做日期差，但不一定是最佳实务写法

查询最近 N 天资料时，范围条件通常更清楚，也更可能有利于索引。

---

## 4.6.13 自我检查题

### 问题 1

如何取得当前时间 7 天后的日期时间？

答案：

```sql
SELECT DATE_ADD(NOW(), INTERVAL 7 DAY);
```

---

### 问题 2

如何取得当前时间 30 分钟前的日期时间？

答案：

```sql
SELECT DATE_SUB(NOW(), INTERVAL 30 MINUTE);
```

或：

```sql
SELECT NOW() - INTERVAL 30 MINUTE;
```

---

### 问题 3

`DATEDIFF('2026-04-26 23:59:59', '2026-04-26 00:00:00')` 结果概念是什么？

答案：`0`，因为 `DATEDIFF()` 只看日期，不看时分秒。

---

### 问题 4

要计算两个完整日期时间相差几秒，推荐什么函数？

答案：

```sql
TIMESTAMPDIFF(SECOND, start_time, end_time)
```

---

### 问题 5

下面 SQL 有什么潜在问题？

```sql
SELECT COUNT(*)
FROM users
WHERE TO_DAYS(NOW()) - TO_DAYS(created_at) <= 7;
```

答案：对 `created_at` 套 `TO_DAYS()` 可能影响索引使用。较常见写法是：

```sql
WHERE created_at >= NOW() - INTERVAL 7 DAY
```

---

## 4.6.14 本节总结

本节可以浓缩成下面几句话：

1. `DATE_ADD()` 和 `DATE_SUB()` 是日期时间加减的核心函数。
2. `INTERVAL` 决定加减的单位，例如 `DAY`、`MONTH`、`YEAR`、`HOUR`。
3. `DATEDIFF()` 计算日期差，只返回天数，并且忽略时分秒。
4. `TIMEDIFF()` 计算时间差，结果是时间格式。
5. `TIMESTAMPDIFF()` 可以指定单位，实务上很适合计算两个完整时间点差几秒、几分钟、几天。
6. `LAST_DAY()` 常用于月底、账单、报表场景。
7. 查询最近 N 天资料时，优先考虑范围条件，避免对索引栏位套函数。

如果只记一句话：

> 日期时间加减用 `DATE_ADD()` / `DATE_SUB()`，完整时间差计算优先想到 `TIMESTAMPDIFF()`，筛选区间尽量写成范围条件。
