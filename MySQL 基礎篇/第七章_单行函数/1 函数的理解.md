## 1.1 什么是函数

函数在计算机语言中几乎贯穿始终。它的作用是把经常使用的操作封装起来，需要的时候直接调用即可。这样既能`提高代码效率`，也能`提高可维护性`。

在 SQL 中，我们同样可以使用函数对检索出来的数据进行处理。借助这些函数，可以更高效地完成数据查询、转换、分析与管理工作。

![函数的作用与分类示意图](./images/1554979529525.png)

从函数定义的角度出发，可以把函数分成`内置函数`和`自定义函数`。在 SQL 语言中也是如此：内置函数是数据库系统已经提供好的通用函数，自定义函数则是使用者根据业务需要自行编写的函数。本章及下一章主要讨论的是 SQL 的内置函数。

## 1.2 不同 DBMS 函数的差异

我们在使用 SQL 语言时，实际上并不是直接和“纯 SQL”打交道，而是通过 SQL 去操作不同的数据库管理系统，也就是 DBMS。

不同 DBMS 之间的函数差异通常很大，甚至往往大于同一门语言不同版本之间的差异。实际上，只有少量函数会被多个 DBMS 同时支持。比如，大多数 DBMS 使用 `||` 或 `+` 做字符串拼接，而在 MySQL 中更常见的字符串拼接函数是 `CONCAT()`。

这意味着：一旦 SQL 里大量使用特定数据库的函数，代码的`可移植性通常会比较差`。因此，在学习和使用函数时，不只要知道“怎么用”，也要知道“这个函数是不是某个 DBMS 的特有实现”。

## 1.3 MySQL 的内置函数及分类

MySQL 提供了大量内置函数，这些函数能让数据维护、数据清洗、结果转换和统计分析变得更方便，也能提高开发人员处理数据时的效率。

如果从`功能角度`来分，MySQL 内置函数可以分为数值函数、字符串函数、日期和时间函数、流程控制函数、加密与解密函数、获取 MySQL 信息函数、聚合函数等。

如果从`使用方式`来分，可以先把这些函数归纳为两大类：`单行函数`和`聚合函数（或分组函数）`。

在进入后续细分类别前，先记住下面这条主线：

- 单行函数：对每一行数据分别处理，每一行都会得到一个结果。
- 聚合函数：对多行数据整体计算，最后只返回一个汇总结果。

![单行函数与聚合函数示意图](./images/1555433204337.png)

## 1.3.1 单行函数（Single-Row Functions）

单行函数会对每一行输入分别执行操作，并为每一行返回一个结果。它不会把多行合并成一个值，因此通常用于对字段值做格式转换、数值处理或日期处理。

### 1.3.1.1 常见的单行函数类型

- 字符串函数（String Functions）
  - `UPPER(str)`：将字符串转换为大写
  - `LOWER(str)`：将字符串转换为小写
  - `CONCAT(str1, str2, ...)`：合并字符串
  - `SUBSTRING(str, start, length)`：截取部分字符串
- 数学函数（Mathematical Functions）
  - `ABS(x)`：返回绝对值
  - `CEIL(x)` / `FLOOR(x)`：向上 / 向下取整
  - `ROUND(x, d)`：四舍五入到 `d` 位小数
- 日期和时间函数（Date & Time Functions）
  - `NOW()`：返回当前日期和时间
  - `DATE_FORMAT(date, format)`：格式化日期
  - `DATEDIFF(date1, date2)`：计算两个日期之间的天数差
- 转换函数（Conversion Functions）
  - `CAST(value AS type)`：转换数据类型
  - `CONVERT(value, type)`：转换数据类型

### 1.3.1.2 示例

```sql
SELECT UPPER('mysql');         -- 结果：'MYSQL'
SELECT ROUND(3.14159, 2);      -- 结果：3.14
SELECT NOW();                  -- 例如：'2026-04-11 20:30:00'
```

## 1.3.2 多行函数（Multiple-Row Functions / 聚合函数）

多行函数也叫`聚合函数`，它会对多行数据执行统计或汇总计算，并最终返回`单个结果`。

### 1.3.2.1 常见的多行函数（聚合函数）

- 计数（Counting）
  - `COUNT(*)`：计算行数
  - `COUNT(column_name)`：计算指定列中非 `NULL` 的行数
- 总和与平均（Sum & Average）
  - `SUM(column_name)`：计算总和
  - `AVG(column_name)`：计算平均值
- 最小值与最大值（Min & Max）
  - `MIN(column_name)`：返回最小值
  - `MAX(column_name)`：返回最大值

### 1.3.2.2 示例

```sql
SELECT COUNT(*) FROM employees;                                   -- 返回表中的总行数
SELECT AVG(salary) FROM employees;                                -- 计算员工的平均薪资
SELECT MAX(age) FROM customers;                                   -- 查询客户的最大年龄
SELECT SUM(sales) FROM orders WHERE order_date >= '2025-01-01';  -- 统计指定日期之后的销售总额
```

### 1.3.2.3 `GROUP BY` 搭配聚合函数

多行函数经常搭配 `GROUP BY` 使用，用来对不同组别的数据分别做汇总计算。

```sql
SELECT department, AVG(salary)
FROM employees
GROUP BY department; -- 按部门计算平均薪资
```

## 1.3.3 总结

| 函数类型 | 说明 | 示例 |
| --- | --- | --- |
| 单行函数 | 针对每一行输入，返回一个结果 | `UPPER(name)`、`ROUND(price, 2)`、`NOW()` |
| 多行函数（聚合函数） | 针对多行输入，返回一个汇总结果 | `COUNT(*)`、`SUM(salary)`、`AVG(age)` |

## 常见混淆点

- 单行函数不是“只对一条记录可用”，而是“对结果集中的每一行分别处理一次”。
- 聚合函数不是“可以处理很多函数”，而是“把多行数据聚合后输出一个结果”。
- 同样是 MySQL 函数，按功能可以继续细分为字符串、数值、日期等类型；按使用方式则可以先分成单行函数和聚合函数。这是两种不同的分类角度。
