# 第七章_单行函数

这一章开始把学习重点从“查询语句怎么写”推进到“查出来的值还能怎么处理”。核心不是死记所有函数名称，而是先建立函数的整体框架：什么是函数、为什么 SQL 需要函数，以及单行函数和聚合函数有什么区别。

目前这一章已经整理出 `8` 个小节，其中日期和时间函数拆成一组子笔记，其余主题保持单篇速查结构，适合先顺读建立分类框架，再按函数类型回查。

## 本章在整体学习地图中的位置

- 前六章已经建立了数据库基础概念、MySQL 环境、基础 `SELECT`、运算符、排序分页和多表查询。
- 这一章开始进入“结果值加工与转换”的阶段，是后续学习字符串函数、数值函数、日期函数与统计函数的前置基础。
- 如果这一章的分类框架没有建立好，后面会容易把“对每一行做处理”和“对多行做汇总”混在一起。

## 本章小节

- [1 函数的理解](./1%20函数的理解.md)：先建立函数的基本概念，理解内置函数与自定义函数、单行函数与聚合函数的区别，并为后续各类函数学习搭好框架。
- [2 数值函数](./2%20数值函数.md)：整理最常见的数值函数，包括取整、随机数、角度弧度互换、三角函数、指数对数与进制转换。
- [3 字符串函数](./3%20字符串函数.md)：整理字符串处理常用函数，包括拼接、替换、大小写转换、截取、裁剪、定位与列表查找。
- [4 日期和时间函数](./4%20日期和时间函数/README.md)：拆成一组子笔记，分别整理当前时间获取、时间戳转换、日期拆解、`EXTRACT`、秒数换算、日期计算和格式化解析。
- [5 流程控制函数](./5%20流程控制函数.md)：整理 `IF()`、`IFNULL()` 和两种 `CASE()` 写法，补上条件判断、空值兜底、状态映射与分类输出的常见场景。
- [6 加密与解密函数](./6%20加密与解密函数.md)：整理 `PASSWORD()`、`MD5()`、`SHA()`、`ENCODE()`、`DECODE()` 的基本用途，重点区分不可逆处理与可逆配对函数。
- [7 MySQL 信息函数](./7%20MySQL%20信息函数.md)：整理 `VERSION()`、`DATABASE()`、`USER()`、`CHARSET()`、`COLLATION()` 等函数，帮助快速确认当前连接环境、数据库上下文与字符串规则。
- [8 其他函数](./8%20其他函数.md)：整理 `FORMAT()`、`CONV()`、`INET_ATON()`、`INET_NTOA()`、`BENCHMARK()`、`CONVERT()` 等较难归类但常用的工具函数。

## 本章适合快速回查的主题

- 忘了 SQL 里的函数到底在做什么：看 [1 函数的理解](./1%20函数的理解.md)
- 分不清单行函数和聚合函数：看 [1 函数的理解](./1%20函数的理解.md) 中的 `1.3.1 单行函数` 和 `1.3.2 多行函数`
- 想先建立 MySQL 内置函数的大致分类框架：看 [1 函数的理解](./1%20函数的理解.md) 中的 `1.3 MySQL 的内置函数及分类`
- 想快速确认取整、四舍五入、截断或随机数函数：看 [2 数值函数](./2%20数值函数.md) 中的 `2.1 基本函数`
- 想回查三角函数、对数函数或进制转换：看 [2 数值函数](./2%20数值函数.md)
- 想回查拼接、替换、大小写转换、去空格或子串定位：看 [3 字符串函数](./3%20字符串函数.md)
- 想处理当前时间、时间戳、日期加减、最近 N 天统计或日期格式化：看 [4 日期和时间函数](./4%20日期和时间函数/README.md)
- 想确认 `IF()`、`IFNULL()`、`CASE WHEN`、状态码映射或薪资分档写法：看 [5 流程控制函数](./5%20流程控制函数.md)
- 想分清 `MD5()`、`SHA()`、`PASSWORD()` 和 `ENCODE()`、`DECODE()` 的区别：看 [6 加密与解密函数](./6%20加密与解密函数.md)
- 想确认当前数据库、连接用户、MySQL 版本、字符集或排序规则：看 [7 MySQL 信息函数](./7%20MySQL%20信息函数.md)
- 想回查数字格式化、进制转换、IP 转换、性能测试或编码转换：看 [8 其他函数](./8%20其他函数.md)

## 哪几篇最常当速查页

- [3 字符串函数](./3%20字符串函数.md)：拼接、截取、替换、定位与去空格最常回查。
- [4 日期和时间函数](./4%20日期和时间函数/README.md)：日期加减、时间戳转换、格式化与最近 N 天统计很常用。
- [5 流程控制函数](./5%20流程控制函数.md)：`IF()`、`IFNULL()`、`CASE()` 在查询结果加工里出现频率很高。
- [6 加密与解密函数](./6%20加密与解密函数.md)：适合快速确认单向处理与可逆配对函数的区别。
- [7 MySQL 信息函数](./7%20MySQL%20信息函数.md) 与 [8 其他函数](./8%20其他函数.md)：适合排查连接环境、编码、IP 转换与性能测试这类零散但常见的问题。
