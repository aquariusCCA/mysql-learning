这一节整理 MySQL 中常见的数值函数。数值函数主要用来处理数字，例如取绝对值、判断正负、取整、四舍五入、截断小数、取余数、产生随机数、做指数对数运算，以及查看不同进制下的表示方式。

* 想查看二进制、八进制、十六进制表示：看 `BIN()`、`OCT()`、`HEX()`、`CONV()`。

本节重点不是把每个函数都背下来，而是知道：

## 2.1 基本数值函数

基本数值函数是最常用的一组函数，主要用来处理：

* 绝对值
* 正负号
* 圆周率
* 最大值与最小值
* 取整
* 取余
* 小数位
* 平方根
* 随机数

### 2.1.1 函数表

| 函数                       | 用法                    | 说明                         |
| ------------------------ | --------------------- | -------------------------- |
| `ABS(x)`                 | 返回 `x` 的绝对值           | 负数转正数，正数维持不变               |
| `SIGN(x)`                | 返回 `x` 的符号            | 正数回 `1`，负数回 `-1`，`0` 回 `0` |
| `PI()`                   | 返回圆周率                 | 约等于 `3.141592653589793`    |
| `CEIL(x)` / `CEILING(x)` | 返回大于或等于 `x` 的最小整数     | 向上取整                       |
| `FLOOR(x)`               | 返回小于或等于 `x` 的最大整数     | 向下取整                       |
| `LEAST(e1,e2,e3,...)`    | 返回列表中的最小值             | 参数可多个                      |
| `GREATEST(e1,e2,e3,...)` | 返回列表中的最大值             | 参数可多个                      |
| `MOD(x,y)`               | 返回 `x` 除以 `y` 后的余数    | 类似 `x % y`                 |
| `RAND()`                 | 返回 `0` 到 `1` 之间的随机值   | 每次结果可能不同                   |
| `RAND(seed)`             | 使用种子产生随机值             | 相同种子通常会产生可重复结果             |
| `ROUND(x)`               | 对 `x` 四舍五入为整数         | 不指定小数位                     |
| `ROUND(x,y)`             | 对 `x` 四舍五入并保留 `y` 位小数 | `y` 可以是正数、零或负数             |
| `TRUNCATE(x,y)`          | 将 `x` 直接截断为 `y` 位小数   | 不四舍五入                      |
| `SQRT(x)`                | 返回 `x` 的平方根           | `x < 0` 时返回 `NULL`         |

---

### 2.1.2 示例：绝对值、符号、取整、取余

```sql
SELECT
    ABS(-123)      AS abs_value,
    SIGN(-23)     AS sign_value,
    PI()          AS pi_value,
    CEIL(32.32)   AS ceil_value,
    FLOOR(32.32)  AS floor_value,
    MOD(12, 5)    AS mod_value
FROM DUAL;
```

结果概念：

| 表达式            | 结果    | 说明             |
| -------------- | ----- | -------------- |
| `ABS(-123)`    | `123` | 取绝对值           |
| `SIGN(-23)`    | `-1`  | 判断为负数          |
| `CEIL(32.32)`  | `33`  | 向上取整           |
| `FLOOR(32.32)` | `32`  | 向下取整           |
| `MOD(12, 5)`   | `2`   | `12 / 5` 余 `2` |

---

### 2.1.3 示例：最大值与最小值

```sql
SELECT
    LEAST(10, 3, 25, 7)    AS min_value,
    GREATEST(10, 3, 25, 7) AS max_value
FROM DUAL;
```

结果：

| 表达式                      | 结果   |
| ------------------------ | ---- |
| `LEAST(10, 3, 25, 7)`    | `3`  |
| `GREATEST(10, 3, 25, 7)` | `25` |

### 使用场景

`LEAST()` 和 `GREATEST()` 很适合做上下限控制。

例如希望折扣率最高不要超过 `0.8`：

```sql
SELECT LEAST(discount_rate, 0.8) AS final_discount_rate
FROM products;
```

意思是：

```text
如果 discount_rate 大于 0.8，就取 0.8；
如果 discount_rate 小于等于 0.8，就取原本的值。
```

## 2.2 小数位处理：ROUND() 与 TRUNCATE()

`ROUND()` 和 `TRUNCATE()` 是数值函数中最容易混淆、也最常用的一组。

它们都可以控制小数位，但处理方式完全不同：

| 函数              | 核心行为       | 会不会四舍五入 |
| --------------- | ---------- | ------- |
| `ROUND(x,y)`    | 四舍五入到指定小数位 | 会       |
| `TRUNCATE(x,y)` | 直接截断到指定小数位 | 不会      |

---

### 2.2.1 ROUND() 示例

```sql
SELECT
    ROUND(12.33)       AS r1,
    ROUND(12.343, 2)   AS r2,
    ROUND(12.345, 2)   AS r3,
    ROUND(12.324, -1)  AS r4
FROM DUAL;
```

结果概念：

| 表达式                 | 结果      | 说明          |
| ------------------- | ------- | ----------- |
| `ROUND(12.33)`      | `12`    | 四舍五入到整数     |
| `ROUND(12.343, 2)`  | `12.34` | 保留两位小数      |
| `ROUND(12.345, 2)`  | `12.35` | 第三位为 `5`，进位 |
| `ROUND(12.324, -1)` | `10`    | 保留到十位       |

---

### 2.2.2 TRUNCATE() 示例

```sql
SELECT
    TRUNCATE(12.66, 1)   AS t1,
    TRUNCATE(12.69, 1)   AS t2,
    TRUNCATE(12.66, 0)   AS t3,
    TRUNCATE(12.66, -1)  AS t4
FROM DUAL;
```

结果概念：

| 表达式                   | 结果     | 说明              |
| --------------------- | ------ | --------------- |
| `TRUNCATE(12.66, 1)`  | `12.6` | 保留一位，不进位        |
| `TRUNCATE(12.69, 1)`  | `12.6` | 即使下一位是 `9` 也不进位 |
| `TRUNCATE(12.66, 0)`  | `12`   | 截断小数部分          |
| `TRUNCATE(12.66, -1)` | `10`   | 截断到十位           |

---

### 2.2.3 ROUND() 与 TRUNCATE() 对比

```sql
SELECT
    ROUND(12.69, 1)    AS round_value,
    TRUNCATE(12.69, 1) AS truncate_value
FROM DUAL;
```

| 表达式                  | 结果     | 说明   |
| -------------------- | ------ | ---- |
| `ROUND(12.69, 1)`    | `12.7` | 四舍五入 |
| `TRUNCATE(12.69, 1)` | `12.6` | 直接截断 |

### 记忆方式

```text
ROUND：看下一位，决定要不要进位。
TRUNCATE：不看下一位，直接切掉。
```

### 实务提醒

金额相关场景要特别小心。
如果是正式财务计算，不能只靠「看起来差不多」来决定用 `ROUND()` 还是 `TRUNCATE()`，而要根据业务规则决定：

* 需要四舍五入：使用 `ROUND()`。
* 需要无条件舍去：使用 `TRUNCATE()`。
* 需要无条件进位：通常使用 `CEIL()` 搭配倍率处理。
* 需要无条件舍去到整数：可以使用 `FLOOR()` 或 `TRUNCATE(x, 0)`，但负数场景要特别确认。

## 2.3 负数场景下的 CEIL() 与 FLOOR()

`CEIL()` 和 `FLOOR()` 在正数时很好理解，但在负数时很容易记错。

### 2.3.1 核心定义

| 函数         | 定义                |
| ---------- | ----------------- |
| `CEIL(x)`  | 返回大于或等于 `x` 的最小整数 |
| `FLOOR(x)` | 返回小于或等于 `x` 的最大整数 |

重点是「大于等于」和「小于等于」，不是单纯看起来往数字绝对值变大或变小。

---

### 2.3.2 示例

```sql
SELECT
    CEIL(43.23)    AS c1,
    FLOOR(43.23)   AS f1,
    CEIL(-43.23)   AS c2,
    FLOOR(-43.23)  AS f2
FROM DUAL;
```

结果概念：

| 表达式             | 结果    | 说明                  |
| --------------- | ----- | ------------------- |
| `CEIL(43.23)`   | `44`  | 大于等于 `43.23` 的最小整数  |
| `FLOOR(43.23)`  | `43`  | 小于等于 `43.23` 的最大整数  |
| `CEIL(-43.23)`  | `-43` | 大于等于 `-43.23` 的最小整数 |
| `FLOOR(-43.23)` | `-44` | 小于等于 `-43.23` 的最大整数 |

### 记忆方式

可以把数字放在数轴上看：

```text
... -45  -44  -43  -42 ...
          ^
        -43.23 在 -44 和 -43 之间
```

* 比 `-43.23` 大的整数有 `-43`、`-42`、`-41`，其中最小的是 `-43`，所以 `CEIL(-43.23) = -43`。
* 比 `-43.23` 小的整数有 `-44`、`-45`，其中最大的是 `-44`，所以 `FLOOR(-43.23) = -44`。

## 2.4 随机数函数 RAND()

`RAND()` 用来产生 `0` 到 `1` 之间的随机数。

### 2.4.1 基本用法

```sql
SELECT
    RAND() AS r1,
    RAND() AS r2,
    RAND() AS r3
FROM DUAL;
```

每次执行时，结果通常都可能不同。

---

### 2.4.2 指定种子 RAND(seed)

```sql
SELECT
    RAND(10) AS r1,
    RAND(10) AS r2,
    RAND(-1) AS r3,
    RAND(-1) AS r4
FROM DUAL;
```

`RAND(seed)` 的特点是：

```text
种子值相同，产生的随机结果通常具有可重复性。
```

这很适合用在教学示例、测试资料或需要可重复验证的场景。

---

### 2.4.3 随机排序

有时会看到这种写法：

```sql
SELECT *
FROM employees
ORDER BY RAND();
```

它的意思是随机排序查询结果。

但要注意：

> 在大表中，`ORDER BY RAND()` 通常不适合直接使用，因为数据库可能需要为大量行产生随机值再排序，成本较高。

如果资料量很小，用于练习或临时抽样可以接受；如果是正式系统的大表随机抽样，要另外设计更合适的方案。

## 2.5 角度与弧度互换函数

很多三角函数使用的是弧度，不是角度。
所以在使用 `SIN()`、`COS()`、`TAN()` 之前，经常需要先确认手上的数值单位。

| 函数           | 用法       |
| ------------ | -------- |
| `RADIANS(x)` | 将角度转换为弧度 |
| `DEGREES(x)` | 将弧度转换为角度 |

---

### 2.5.1 角度转弧度

```sql
SELECT
    RADIANS(30) AS rad_30,
    RADIANS(60) AS rad_60,
    RADIANS(90) AS rad_90
FROM DUAL;
```

概念上：

| 角度     | 弧度      |
| ------ | ------- |
| `30°`  | `π / 6` |
| `60°`  | `π / 3` |
| `90°`  | `π / 2` |
| `180°` | `π`     |

---

### 2.5.2 弧度转角度

```sql
SELECT
    DEGREES(PI())          AS deg_180,
    DEGREES(2 * PI())      AS deg_360,
    DEGREES(RADIANS(90))   AS deg_90
FROM DUAL;
```

结果概念：

| 表达式                    | 结果    |
| ---------------------- | ----- |
| `DEGREES(PI())`        | `180` |
| `DEGREES(2 * PI())`    | `360` |
| `DEGREES(RADIANS(90))` | `90`  |

## 2.6 三角函数

三角函数主要用于处理正弦、余弦、正切及其反函数。一般开发中使用频率不如 `ROUND()`、`TRUNCATE()`、`MOD()` 高，但在坐标、地图、角度、方向、几何计算中会用到。

### 2.6.1 函数表

| 函数           | 用法           | 注意事项                              |
| ------------ | ------------ | --------------------------------- |
| `SIN(x)`     | 返回 `x` 的正弦值  | `x` 通常为弧度                         |
| `COS(x)`     | 返回 `x` 的余弦值  | `x` 通常为弧度                         |
| `TAN(x)`     | 返回 `x` 的正切值  | `x` 通常为弧度                         |
| `ASIN(x)`    | 返回 `x` 的反正弦值 | `x` 必须在 `-1` 到 `1` 之间，否则返回 `NULL` |
| `ACOS(x)`    | 返回 `x` 的反余弦值 | `x` 必须在 `-1` 到 `1` 之间，否则返回 `NULL` |
| `ATAN(x)`    | 返回 `x` 的反正切值 | 回传通常为弧度                           |
| `ATAN2(m,n)` | 返回两个参数的反正切值  | 坐标计算时更稳妥                          |
| `COT(x)`     | 返回 `x` 的余切值  | `x` 通常为弧度                         |

---

### 2.6.2 示例：角度转弧度后再计算

如果想计算 `30°` 的正弦值，不建议直接写：

```sql
SELECT SIN(30);
```

因为 `SIN()` 接收的通常是弧度，`30` 会被当成 `30` 弧度，而不是 `30°`。

正确思路是先用 `RADIANS()` 转换：

```sql
SELECT SIN(RADIANS(30)) AS sin_30;
```

结果概念：

```text
SIN(30°) = 0.5
```

---

### 2.6.3 ATAN2() 为什么值得单独记

`ATAN(x)` 和 `ATAN2(m,n)` 都和反正切有关，但 `ATAN2()` 在处理坐标关系时更实用。

假设有两个点：

```text
point1(x1, y1)
point2(x2, y2)
```

如果使用 `ATAN()`，常见写法是：

```sql
ATAN((y2 - y1) / (x2 - x1))
```

如果 `x2 - x1 = 0`，就可能出现除以 `0` 的问题。

使用 `ATAN2()` 可以写成：

```sql
ATAN2(y2 - y1, x2 - x1)
```

这在处理方向、夹角、坐标差时更稳妥。

---

### 2.6.4 三角函数示例

```sql
SELECT
    SIN(RADIANS(30))       AS sin_30,
    COS(RADIANS(60))       AS cos_60,
    TAN(RADIANS(45))       AS tan_45,
    DEGREES(ASIN(1))       AS asin_1_degree,
    DEGREES(ATAN2(1, 1))   AS atan2_degree
FROM DUAL;
```

结果概念：

| 表达式                    | 结果概念  |
| ---------------------- | ----- |
| `SIN(RADIANS(30))`     | `0.5` |
| `COS(RADIANS(60))`     | `0.5` |
| `TAN(RADIANS(45))`     | `1`   |
| `DEGREES(ASIN(1))`     | `90`  |
| `DEGREES(ATAN2(1, 1))` | `45`  |

---

## 2.7 指数与对数函数

这一组函数主要用来处理幂运算、指数运算和对数运算。

实际开发中不一定天天用，但在统计、模型计算、数学分析、评分公式、成长率计算等场景中可能会出现。

### 2.7.1 函数表

| 函数                        | 用法                   | 注意事项                       |
| ------------------------- | -------------------- | -------------------------- |
| `POW(x,y)` / `POWER(x,y)` | 返回 `x` 的 `y` 次方      | 两者作用相同                     |
| `EXP(x)`                  | 返回 `e` 的 `x` 次方      | `e` 约为 `2.718281828459045` |
| `LN(x)`                   | 返回以 `e` 为底的 `x` 的对数  | `x <= 0` 时返回 `NULL`        |
| `LOG(x)`                  | 返回以 `e` 为底的 `x` 的对数  | 常与 `LN(x)` 等价使用            |
| `LOG10(x)`                | 返回以 `10` 为底的 `x` 的对数 | `x <= 0` 时返回 `NULL`        |
| `LOG2(x)`                 | 返回以 `2` 为底的 `x` 的对数  | `x <= 0` 时返回 `NULL`        |

---

### 2.7.2 示例

```sql
SELECT
    POW(2, 5)     AS pow_value,
    POWER(2, 4)   AS power_value,
    EXP(2)        AS exp_value,
    LN(10)        AS ln_value,
    LOG10(10)     AS log10_value,
    LOG2(4)       AS log2_value
FROM DUAL;
```

结果概念：

| 表达式           | 结果        |
| ------------- | --------- |
| `POW(2, 5)`   | `32`      |
| `POWER(2, 4)` | `16`      |
| `EXP(2)`      | 约 `7.389` |
| `LN(10)`      | 约 `2.302` |
| `LOG10(10)`   | `1`       |
| `LOG2(4)`     | `2`       |

---

### 2.7.3 对数函数的 NULL 行为

对数函数的参数必须大于 `0`。

```sql
SELECT
    LN(0)      AS ln_zero,
    LOG10(-1) AS log10_negative,
    LOG2(NULL) AS log2_null
FROM DUAL;
```

结果通常会是 `NULL`。

所以使用对数函数时，要先确认资料范围是否合理。

例如：

```sql
SELECT
    user_id,
    score,
    CASE
        WHEN score > 0 THEN LOG10(score)
        ELSE NULL
    END AS score_log
FROM user_scores;
```

这样可以避免对 `0` 或负数直接做对数计算。

---

## 2.8 进制间的转换

这一组函数用来查看数值在不同进制下的表示方式，或进行进制转换。

### 2.8.1 函数表

| 函数                            | 用法                                     | 示例概念                      |
| ----------------------------- | -------------------------------------- | ------------------------- |
| `BIN(x)`                      | 返回 `x` 的二进制表示                          | `BIN(10)` -> `1010`       |
| `OCT(x)`                      | 返回 `x` 的八进制表示                          | `OCT(10)` -> `12`         |
| `HEX(x)`                      | 返回 `x` 的十六进制表示                         | `HEX(10)` -> `A`          |
| `CONV(x, from_base, to_base)` | 将 `x` 从 `from_base` 进制转换为 `to_base` 进制 | `CONV('10', 2, 8)` -> `2` |

---

### 2.8.2 示例

```sql
SELECT
    BIN(10)          AS bin_value,
    OCT(10)          AS oct_value,
    HEX(10)          AS hex_value,
    CONV('10', 2, 8) AS conv_value
FROM DUAL;
```

结果概念：

| 表达式                | 结果     | 说明                  |
| ------------------ | ------ | ------------------- |
| `BIN(10)`          | `1010` | 十进制 `10` 的二进制表示     |
| `OCT(10)`          | `12`   | 十进制 `10` 的八进制表示     |
| `HEX(10)`          | `A`    | 十进制 `10` 的十六进制表示    |
| `CONV('10', 2, 8)` | `2`    | 二进制 `10` 转成八进制为 `2` |

---

### 2.8.3 CONV() 的重点

`CONV(x, from_base, to_base)` 的意思是：

```text
把 x 当成 from_base 进制的值，转换成 to_base 进制。
```

例如：

```sql
SELECT CONV('FF', 16, 10) AS decimal_value;
```

意思是：

```text
把十六进制 FF 转成十进制。
```

结果是：

```text
255
```

## 2.9 数值函数的实务场景

### 2.9.1 金额四舍五入

```sql
SELECT
    order_id,
    total_amount,
    ROUND(total_amount, 2) AS display_amount
FROM orders;
```

适合用于报表显示，让金额保留两位小数。

---

### 2.9.2 折扣率上限控制

```sql
SELECT
    product_id,
    discount_rate,
    LEAST(discount_rate, 0.8) AS final_discount_rate
FROM products;
```

意思是折扣率最高只能到 `0.8`。

---

### 2.9.3 分页计算总页数

假设每页 `10` 笔资料，总笔数是 `total_count`。

```sql
SELECT CEIL(total_count / 10) AS total_pages
FROM report_summary;
```

如果总笔数是 `101`，则总页数是 `11`。

---

### 2.9.4 判断奇偶数

```sql
SELECT
    employee_id,
    CASE
        WHEN MOD(employee_id, 2) = 0 THEN '偶数'
        ELSE '奇数'
    END AS number_type
FROM employees;
```

`MOD()` 常用于奇偶判断、分组分流、周期性规则判断。

---

### 2.9.5 计算两个数值的差距

```sql
SELECT
    product_id,
    expected_price,
    actual_price,
    ABS(actual_price - expected_price) AS price_gap
FROM product_price_check;
```

`ABS()` 适合处理差额，因为不管实际价格比较高还是比较低，都可以得到正数差距。

---

### 2.9.6 避免对不合法数值做对数计算

```sql
SELECT
    user_id,
    score,
    CASE
        WHEN score > 0 THEN LOG10(score)
        ELSE NULL
    END AS score_log10
FROM user_scores;
```

因为对数函数的参数必须大于 `0`，所以正式查询中最好先保护资料范围。

---

## 2.10 使用提醒

### 2.10.1 注意 NULL 影响

很多数值函数只要参数中出现 `NULL`，结果就可能变成 `NULL`。

例如：

```sql
SELECT ROUND(NULL, 2);
```

结果是：

```text
NULL
```

如果字段可能为空，可以搭配 `IFNULL()`：

```sql
SELECT ROUND(IFNULL(amount, 0), 2) AS amount_value
FROM orders;
```

---

### 2.10.2 注意财务计算规则

`ROUND()`、`TRUNCATE()`、`CEIL()`、`FLOOR()` 都可能影响金额结果。
在财务、利息、手续费、税额等场景中，必须根据业务规则决定使用哪一个函数。

不要只凭直觉写：

```sql
ROUND(amount, 2)
```

而要确认：

* 是四舍五入？
* 是无条件舍去？
* 是无条件进位？
* 是保留到小数点后几位？
* 负数场景是否存在？

---

### 2.10.3 注意函数可能影响索引

数值函数如果放在 `WHERE` 条件的字段上，可能影响索引使用。

例如：

```sql
SELECT *
FROM orders
WHERE ROUND(total_amount, 0) = 1000;
```

如果 `total_amount` 有索引，这种写法可能让数据库无法直接用范围方式查找，而需要先计算每一行的 `ROUND(total_amount, 0)`。

更常见的思路是改写成范围条件：

```sql
SELECT *
FROM orders
WHERE total_amount >= 999.5
  AND total_amount < 1000.5;
```

实际是否能使用索引，要以执行计划为准，但工作中要养成这个意识：

> 不要随便在索引字段外层套函数。

---

### 2.10.4 注意显示格式与实际数值不同

`ROUND()` 会改变查询结果中的数值，但它不一定代表原始资料真的被修改。

例如：

```sql
SELECT ROUND(price, 2)
FROM products;
```

这只是查询时显示成两位小数，不会改变 `products.price` 原本存储的值。

如果真的要修改资料，需要使用 `UPDATE`。

---

### 2.10.5 注意随机排序的性能

```sql
ORDER BY RAND()
```

写起来很方便，但在大表中通常成本较高。练习和小表可以使用，正式系统的大量资料随机抽样要谨慎。

---

## 2.11 常见混淆点

### 混淆点 1：ROUND() 和 TRUNCATE() 都能控制小数位，但不是同一件事

* `ROUND()` 会四舍五入。
* `TRUNCATE()` 直接截断。

```sql
SELECT
    ROUND(12.69, 1),
    TRUNCATE(12.69, 1);
```

结果概念：

```text
ROUND    -> 12.7
TRUNCATE -> 12.6
```

---

### 混淆点 2：CEIL() 和 FLOOR() 遇到负数时不能只凭感觉

```sql
SELECT
    CEIL(-43.23),
    FLOOR(-43.23);
```

结果概念：

```text
CEIL(-43.23)  -> -43
FLOOR(-43.23) -> -44
```

要用数轴理解，不要用「去掉小数」来理解。

---

### 混淆点 3：三角函数大多吃弧度，不是角度

```sql
SELECT SIN(30);
```

这不是计算 `30°` 的正弦，而是计算 `30` 弧度的正弦。

如果要计算 `30°`，应写：

```sql
SELECT SIN(RADIANS(30));
```

---

### 混淆点 4：ATAN2(m,n) 不只是 ATAN() 的两个参数版

`ATAN2(m,n)` 在坐标计算中更稳妥，因为它不需要先手动计算 `(y2-y1)/(x2-x1)`，可以减少除以 `0` 的问题。

---

### 混淆点 5：LOG()、LN()、LOG10()、LOG2() 的参数必须大于 0

```sql
SELECT LOG10(0), LOG2(-1);
```

这类情况通常会返回 `NULL`，不是一般的数值结果。

---

### 混淆点 6：CONV(10, 2, 8) 的意思容易误解

```sql
SELECT CONV(10, 2, 8);
```

这里的 `10` 会被当成二进制的 `10`，再转换成八进制，所以结果是 `2`。

如果你想把十进制 `10` 转成二进制，要写：

```sql
SELECT CONV(10, 10, 2);
```

结果是：

```text
1010
```

---

## 2.12 自我检查题

### 问题 1

`ROUND(12.69, 1)` 和 `TRUNCATE(12.69, 1)` 的结果分别是什么？为什么？

答案：

```text
ROUND(12.69, 1)    -> 12.7
TRUNCATE(12.69, 1) -> 12.6
```

因为 `ROUND()` 会四舍五入，`TRUNCATE()` 会直接截断。

---

### 问题 2

`CEIL(-8.3)` 和 `FLOOR(-8.3)` 的结果分别是什么？

答案：

```text
CEIL(-8.3)  -> -8
FLOOR(-8.3) -> -9
```

因为 `-8` 是大于等于 `-8.3` 的最小整数；`-9` 是小于等于 `-8.3` 的最大整数。

---

### 问题 3

如果要计算 `45°` 的正切值，应该写 `TAN(45)` 还是 `TAN(RADIANS(45))`？

答案：

```sql
SELECT TAN(RADIANS(45));
```

因为三角函数通常使用弧度值。

---

### 问题 4

为什么大表中不建议随便使用 `ORDER BY RAND()`？

答案：因为数据库可能需要对大量行产生随机值并排序，成本较高。

---

### 问题 5

`CONV('FF', 16, 10)` 的意思是什么？

答案：把十六进制的 `FF` 转换成十进制，结果是 `255`。

---

### 问题 6

下面 SQL 有什么潜在问题？

```sql
SELECT *
FROM orders
WHERE ROUND(total_amount, 0) = 1000;
```

答案：如果 `total_amount` 是索引字段，对它套 `ROUND()` 可能影响索引使用。可以根据业务规则考虑改成范围条件，例如：

```sql
WHERE total_amount >= 999.5
  AND total_amount < 1000.5
```

## 2.13 本节总结

本节可以浓缩成下面几条：

1. 数值函数主要用于处理数字、金额、小数位、随机数、数学运算和进制转换。
2. 高频函数包括 `ABS()`、`CEIL()`、`FLOOR()`、`MOD()`、`ROUND()`、`TRUNCATE()`。
3. `ROUND()` 会四舍五入，`TRUNCATE()` 只截断，这是最重要的区别。
4. `CEIL()` 和 `FLOOR()` 遇到负数时要用数轴理解。
5. `RAND()` 可以产生随机数，但 `ORDER BY RAND()` 在大表中要谨慎。
6. 三角函数大多使用弧度，如果手上是角度，通常要先用 `RADIANS()`。
7. 对数函数的参数必须大于 `0`，否则通常返回 `NULL`。
8. `CONV(x, from_base, to_base)` 的关键是看清楚原始值被当成什么进制。
9. 实务中使用数值函数时，要注意 `NULL`、财务规则、索引影响和查询可读性。

如果只记一句话：

> 数值函数不是只用来算数学题，更常用在金额处理、报表显示、分页计算、条件判断和数据清洗中。
