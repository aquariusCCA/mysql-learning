## 本节导读

这一节主要帮助你认识 MySQL 安装目录里常见的文件和文件夹分别是做什么的，并对源码目录建立一个基础印象。

这篇不是高频操作页，但很适合在你想分清 `bin`、`data`、`my.ini`、安装目录和数据目录时回来看。第一次阅读时，建议先看 `6.1 主要目录结构`，先建立目录角色，再看 `6.2 MySQL 源代码获取`。

如果你正在配置环境变量，可以搭配 [2 MySQL 的下载、安装、配置](./2%20MySQL%20的下载、安装、配置.md) 中的 `Path` 设置一起看；如果你是在排查字符集或配置文件问题，也可以回查 [7 常见问题的解决（课外内容）](./7%20常见问题的解决(课外内容).md)。

## 你会在这篇学到什么

- MySQL 安装目录中常见文件和目录的大致作用。
- `bin` 目录为什么常被加入 `Path` 环境变量。
- `data` 目录和用户创建的数据库文件有什么关系。
- `my.ini` 在 MySQL 配置中的作用。
- 如何在下载页面选择 MySQL 源代码。
- MySQL 源码目录中 `sql`、`libmysql`、`mysql-test`、`mysys` 等子目录的基本含义。

## 关键字

- `bin`：MySQL 可执行文件目录
- `mysql.exe`：常用的 MySQL 命令行客户端程序
- `data`：MySQL 数据文件目录
- `my.ini`：MySQL 主要配置文件
- `ProgramData`：Windows 下常见的数据目录位置
- `Source Code`：下载 MySQL 源代码时需要选择的类型
- `C++`：MySQL 主要使用的开发语言
- `sql`：MySQL 核心代码目录
- `libmysql`：客户端程序 API 相关目录
- `mysql-test`：测试工具目录
- `mysys`：操作系统相关函数和辅助函数目录

## 6.1 主要目录结构

先从最常回查的几个位置开始：

| MySQL 的目录结构 | 说明 |
| --- | --- |
| `bin` 目录 | MySQL 的可执行文件目录，例如 `mysql.exe` |
| `MySQLInstanceConfig.exe` | 数据库配置向导，对应安装和初始化配置时看到的程序 |
| `data` 目录 | 系统数据库所在目录 |
| `my.ini` 文件 | MySQL 的主要配置文件 |
| `C:\ProgramData\MySQL\MySQL Server 8.0\data\` | Windows 下常见的用户数据库文件目录 |

### `bin` 目录

`bin` 目录里放的是 MySQL 的可执行程序，例如常见的：

- `mysql.exe`
- `mysqld.exe`

这也是为什么在前面配置环境变量时，通常会把 `bin` 加到 `Path` 中。因为只有系统能找到这些可执行文件，你才能在命令行里直接输入 `mysql`。

### `data` 目录

`data` 目录主要和数据库文件本身有关。

可以先这样理解：

- MySQL 自带的系统库会放在这里
- 你自己创建的数据库，最终对应的数据文件也会落在相关数据目录下

因此，当你在查“数据库到底存在哪里”“卸载时哪些目录可能还留着旧数据”时，通常都绕不开 `data` 目录或 `ProgramData` 下的数据路径。

### `my.ini` 文件

`my.ini` 是 MySQL 的主要配置文件。

很多常见问题最后都会回到这里，例如：

- 默认字符集
- 服务相关配置
- 部分目录与启动参数

如果你前面遇到过字符集问题，这里和 [4 MySQL 演示使用](./4%20MySQL%20演示使用.md)、[7 常见问题的解决（课外内容）](./7%20常见问题的解决(课外内容).md) 是连在一起看的。

### 安装目录和数据目录要分开理解

初学时最容易混淆的是：安装目录不等于数据库数据目录。

可以先用最简单的方式记住：

- 安装目录：更偏向程序文件和可执行文件
- 数据目录：更偏向数据库实际数据文件

这样在处理环境变量、卸载重装、字符集配置或查看数据文件位置时，就不容易混在一起。

## 6.2 MySQL 源代码获取

如果你想下载 MySQL 源代码，不要在下载页继续使用默认的 `Microsoft Windows` 安装包选项，而是要在下拉栏中选择 `Source Code`。

接着，在下面的平台选项里选择适合的版本，例如 `Windows (Architecture Independent)`，然后再下载压缩包。

下载并解压之后，就可以看到 MySQL 的源码目录。

![MySQL 源码下载入口示意](./images/2402456-20220611182838668-2052351900.png)

### MySQL 主要使用什么语言开发

MySQL 主要使用 `C++` 开发。

所以当你打开源码目录时，看到的本质上就是一组普通的 C++ 工程文件，而不是某种特殊格式。

### 源码目录里常见的子目录

以 `mysql-8.0.22` 为例，常见子目录大致可以这样理解：

![MySQL 源码目录示意](./images/image-20211007154113052.png)

- `sql`：MySQL 核心代码目录
- `libmysql`：客户端程序 API 相关目录
- `mysql-test`：测试工具目录
- `mysys`：操作系统相关函数和辅助函数目录

这些说明的重点不是让你现在就深入读源码，而是让你先建立“源码不是一整团，而是按职责分目录”的基本印象。

### 如何查看源码

源码可以直接用文本编辑器打开查看；如果你已经有 C++ 开发环境，也可以在开发工具中进一步阅读和导航。

![查看 MySQL 源码示意](./images/image-20211007154213156.png)

对初学者来说，这里最重要的不是立刻搞懂所有实现细节，而是知道：

- MySQL 的源码是可读的
- 目录结构是有分工的
- 以后如果想深入理解 MySQL 的实现，可以从这些目录开始追
