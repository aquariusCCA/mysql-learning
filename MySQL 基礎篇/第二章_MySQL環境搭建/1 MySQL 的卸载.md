# 1 MySQL 的卸载

> - 所属章节：MySQL 基础篇 / 第二章_MySQL环境搭建
> - 建议回查情境：安装失败需要重装 MySQL、卸载后仍然残留服务或环境变量、需要确认数据目录是否要保留时
> - 下一节：[2 MySQL 的下载、安装、配置](./2%20MySQL%20的下载、安装、配置.md)

## 本节导读

这一节只解决一个核心问题：**怎样把 MySQL 卸载干净，同时避免误删还要保留的数据。**

对初学者来说，卸载 MySQL 最容易出错的地方不是“不会卸载”，而是：

- 只删了程序，没删残留目录，结果重装继续失败
- 把数据目录一起删掉，导致原有数据库数据丢失
- 看到注册表、环境变量就全部乱删，最后把系统配置弄乱

所以这篇不追求“把所有可能的东西一次删光”，而是强调一个更稳妥的顺序：

**先判断要不要保留数据 → 再停止服务 → 再卸载程序 → 如果重装仍失败，再继续清理残留目录、注册表和环境变量。**

如果你只是普通卸载，通常做到 `步骤1` ~ `步骤3` 就够了；只有在**卸载后重新安装仍失败**时，才继续看 `步骤4` 和 `步骤5`。

## 你会在这篇学到什么

- 卸载 MySQL 前，为什么要先确认是否需要备份数据。
- 为什么“卸载程序”不等于“删除数据库数据”。
- 控制面板卸载和 MySQL Installer 卸载有什么差别。
- 什么情况下只需要标准卸载，什么情况下要继续清理残留目录。
- 为什么注册表和环境变量属于“排查步骤”，而不是一开始就要处理的步骤。

## 一句话先抓核心

**MySQL 卸载的正确顺序是：先停服务，再卸载程序；如果重装仍失败，再按需清理数据目录、残留服务、注册表和环境变量。**

## 快速定位

- `先做判断`：我要保留旧数据吗？
- `步骤1`：停止 MySQL 服务
- `步骤2`：卸载 MySQL 程序
- `步骤3`：清理残留目录
- `步骤4`：清理残留服务 / 注册表（选做）
- `步骤5`：删除环境变量中的 MySQL 路径（选做）
- `常见风险点`：卸载时最容易误操作的地方
- `常见回查问题`：适合重装失败时快速定位

## 关键字

- `MySQL80`：Windows 下常见的 MySQL 服务名
- `服务`：卸载前需要先停止
- `控制面板`：常见的程序卸载入口
- `MySQL Installer`：可执行更完整的卸载
- `Remove the data directory`：是否同步删除数据目录
- `Program Files`：常见安装目录
- `ProgramData`：常见数据目录位置
- `数据备份`：删除目录前必须先确认
- `sc delete`：删除残留服务
- `regedit`：注册表编辑器
- `Path`：可能残留 MySQL `bin` 路径的环境变量

## 建议阅读顺序

- 如果你只是正常卸载：看 `先做判断` → `步骤1` → `步骤2` → `步骤3`
- 如果你已经遇到“重装失败”或“服务名重复”：继续看 `步骤4`
- 如果你遇到命令行仍然识别到旧版 `mysql`：再看 `步骤5`

## 先做判断：我要不要保留旧数据？

在卸载之前，先不要急着点“卸载”，先回答这个问题：

### 情况 1：我只是想重装，但还想保留原本数据库数据

那你应该：

- 先备份数据
- 不要直接删除数据目录
- 在卸载时谨慎处理 `Remove the data directory` 选项

### 情况 2：我就是要彻底删干净，重新开始

那你可以在确认不需要旧数据后：

- 卸载程序
- 删除安装目录
- 删除数据目录
- 必要时再继续清理残留服务、注册表和环境变量

> ⚠️ 记住：
>
> **安装目录** 和 **数据目录** 不是同一个东西。
>
> - 安装目录：偏向程序文件
> - 数据目录：偏向数据库真实数据
>
> 如果你把数据目录删掉，就等于把数据库文件一起删掉。

---

## 步骤1：停止 MySQL 服务

在卸载之前，先停止 MySQL 服务。因为只要服务还在运行，程序文件、数据文件或相关配置就可能仍被系统占用，影响卸载或残留清理。

![在任务管理器中停止 MySQL 服务](./images/image-20211014153604802.png)

### 方式 1：通过图形界面停止服务

可以打开：

- 任务管理器 → `服务`
- 或 `services.msc`

然后找到 MySQL 对应的服务名。课程中常见的是：

```text
MySQL80
```

如果当前状态是“正在运行”，右键选择“停止”。

![在任务管理器中停止 MySQL 服务](./images/image-20211014153604802.png)

### 方式 2：通过命令行停止服务

```bash
net stop MySQL80
```

如果你的服务名不是 `MySQL80`，就改成你自己安装时设置的服务名。

> ✏️ 注意：
>
> * `MySQL80` 只是常见示例，不一定每台机器都一样
> * 如果提示“拒绝访问”或“拒绝服务”，通常需要用**管理员身份**打开命令提示符

---

## 步骤2：卸载 MySQL 程序

停止服务后，再开始卸载程序。

### 方式 1：通过控制面板卸载

进入：

```text
控制面板 → 卸载程序
```

找到 MySQL 相关程序后执行卸载。

![通过控制面板卸载 MySQL](./images/image-20211014153657668.png)

这种方式的特点是：

* 操作简单
* 适合普通卸载
* **通常只删除程序，不一定删除数据目录**

也就是说：**控制面板卸载完成，不代表你的数据库数据已经被删除。**

### 方式 2：通过 MySQL Installer 卸载

如果你当初是通过 MySQL Installer 安装的，也可以再次打开安装器执行卸载。

![在安装器中选择 Remove](./images/image-20211014153722683.png)

基本流程是：

1. 打开 `mysql-installer-community-xxx.msi`
2. 让安装器自动检测已安装的 MySQL 产品
3. 选中 MySQL Server
4. 点击 `Remove`
5. 确认卸载
6. 根据需要决定是否勾选 `Remove the data directory`
7. 执行卸载并完成

这种方式的优点是：

* 更容易看见当前卸载的是哪一个 MySQL 产品
* 能明确控制是否删除数据目录
* 更适合做“较完整的卸载”

### 什么时候要勾选 `Remove the data directory`？

![选择是否删除数据目录](./images/image-20211014154112574.png)

只有在你明确满足下面两个条件时，才建议勾选：

* 你已经确认不需要旧数据
* 你想把环境尽量清干净，重新安装

如果你还不确定数据要不要保留，就**不要勾选**。

---

## 步骤3：清理残留目录

如果你卸载完成后，重新安装 MySQL 仍然失败，那么下一步就不是继续乱点安装器，而是检查残留目录。

常见需要确认的有两类目录：

### 1）安装目录

常见位置例如：

```text
C:\Program Files\MySQL
```

这里主要放的是程序文件。

### 2）数据目录

常见位置例如：

```text
C:\ProgramData\MySQL
```

这里可能保存数据库的数据文件。

> ✏️ `ProgramData` 通常是隐藏目录，看不到时要先确认系统是否显示隐藏项目。

如果你之前有自定义安装目录或数据目录，就不要只盯着默认路径，而要检查你自己当时设定的位置。

### 清理残留目录时的判断原则

* **只想删程序，不想删数据**：不要动数据目录
* **要彻底重装**：安装目录和数据目录都要确认是否删除
* **不确定目录里有没有重要数据**：先备份，再删

> ⚠️ 建议：
>
> 清理完目录后，先重启电脑，再重新安装。
> 这样能减少服务未完全释放、路径缓存未更新等问题。

---

## 步骤4：清理残留服务 / 注册表（选做）

这一部分不是标准卸载的必做步骤，而是：

**只有在你已经卸载程序、删过残留目录，但重新安装仍失败时，再继续处理。**

### 4.1 优先先检查：是不是残留服务没删掉？

有时真正的问题不是注册表，而是 Windows 里还残留着旧的 MySQL 服务。

例如你已经卸载了 MySQL，但系统里仍然保留：

```text
MySQL80
```

这时重新安装可能会提示：

* 同名服务已存在
* 服务注册失败
* 配置阶段失败

可以先尝试用命令删除残留服务：

```bash
sc delete MySQL服务名
```

例如：

```bash
sc delete MySQL80
```

> ✏️ 这是处理“残留服务”的高频方法，通常比一开始就去改注册表更稳妥。

### 4.2 如果仍然失败，再考虑清理注册表

打开注册表编辑器：

```text
regedit
```

然后检查与 MySQL 服务相关的注册表项。

常见位置示意如下：

```text
HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Services\Eventlog\Application\MySQL服务名
HKEY_LOCAL_MACHINE\SYSTEM\ControlSet001\Services\MySQL服务名
HKEY_LOCAL_MACHINE\SYSTEM\ControlSet002\Services\Eventlog\Application\MySQL服务名
HKEY_LOCAL_MACHINE\SYSTEM\ControlSet002\Services\MySQL服务名
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\Eventlog\Application\MySQL服务名
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\MySQL服务名
```

> ⚠️ 注意：
>
> * `ControlSet001`、`ControlSet002` 不一定固定存在，也可能是其他编号
> * 删除前先确认这个服务项确实属于旧的 MySQL
> * 注册表属于系统级配置，操作要谨慎，不要看到相似名字就乱删

---

## 步骤5：删除环境变量中的 MySQL 路径（选做）

如果你已经卸载了 MySQL，但命令行里仍然会识别到旧版 `mysql`，或者你怀疑系统还在指向旧路径，就检查环境变量。

进入系统环境变量后，找到：

```text
Path
```

把其中和 MySQL 相关的路径片段删除，例如：

```text
D:\develop_tools\mysql\MySQLServer8.0.26\bin
```

### 这里最重要的不是“会不会删”，而是“不要删过头”

正确做法是：

* **只删除 MySQL 对应的那一段路径**
* **不要把整个 `Path` 清空**

因为 `Path` 里通常还包含很多其他程序需要的路径。

---

## 常见风险点

### 风险 1：只卸载程序，就以为已经删干净

不是。
控制面板卸载通常只代表程序层面的卸载，不代表数据目录、服务、环境变量一定都已经清干净。

### 风险 2：没备份就删除数据目录

如果你还需要原来的数据库数据，这会直接导致数据丢失。

### 风险 3：一上来就改注册表

注册表不是第一步。
正常顺序应该是：

**停服务 → 卸载程序 → 清理目录 → 删除残留服务 → 最后才考虑注册表**

### 风险 4：把整个 `Path` 都删掉

环境变量只能删 MySQL 对应片段，不能整条删光。

### 风险 5：不知道自己改过安装目录或数据目录

如果你安装时改过路径，就不要只按默认路径找残留，否则可能会漏删真正的问题目录。

---

## 常见回查问题

### 1. 卸载 MySQL 时，为什么要先停服务？

因为服务运行中可能占用程序文件、配置文件或数据目录，导致卸载不完整，或者后续目录删不掉。

### 2. 控制面板卸载和安装器卸载有什么区别？

控制面板更偏向“删除程序”；
安装器卸载更容易控制是否连数据目录一起删除。

### 3. `Remove the data directory` 能不能勾？

只有在你确认不再需要旧数据时才勾。
不确定就不要勾。

### 4. 为什么卸载后重装还是失败？

高频原因通常有 3 个：

* 残留安装目录没删
* 数据目录没清理
* Windows 里还留着旧服务名

### 5. 什么时候才需要清理注册表？

在你已经完成标准卸载、目录清理、残留服务删除之后，重装仍失败时，再处理注册表。

---

## 小结

这一节最重要的，不是记住每一个界面按钮，而是记住下面这条判断链：

1. 先确认旧数据要不要保留
2. 停止 MySQL 服务
3. 卸载 MySQL 程序
4. 必要时清理安装目录和数据目录
5. 如果重装仍失败，再清理残留服务、注册表和环境变量

只要顺序不乱，MySQL 的卸载通常不会太难。
