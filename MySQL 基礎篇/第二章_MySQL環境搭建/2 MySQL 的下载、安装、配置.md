## 本节导读

这一节只解决一个核心问题：

**怎样在 Windows 环境下把 MySQL 下载对、装对、配对，并在出错时知道先查哪里。**

对初学者来说，MySQL 安装最容易乱掉的，不是安装向导本身，而是下面这几类问题：

- 下载了不适合自己的安装包
- 安装时不知道该选哪些组件和目录
- 配置时不清楚端口、授权方式、`root` 密码、服务名分别代表什么
- 安装完成后，命令行仍然找不到 `mysql`
- 以前装过旧版本，结果重装时被残留目录或残留服务卡住

所以这篇不会只是把截图顺着讲一遍，而是按更实用的顺序整理成：

**先分清我要装什么 → 再选择安装包 → 再完成安装与初始化配置 → 再配置环境变量 → 如果失败，再按依赖、目录、服务的顺序排查。**

> ✏️ 说明：
>
> 本节以课程中的 `MySQL 8.0.26` 为示例版本来说明安装流程。  
> 如果你使用的是其他 `8.0.x` 版本，界面细节可能略有不同，但整体思路基本一致。

## 你会在这篇学到什么

- MySQL 常见版本之间的区别，以及为什么初学阶段通常选择社区版。
- Windows 下为什么更推荐使用 `MSI` 安装程序，而不是直接使用 `ZIP` 压缩包。
- `mysql-installer-web-community` 和 `mysql-installer-community` 的区别。
- 安装时为什么建议选择 `Custom`，以及安装目录、数据目录该怎样理解。
- 初始化配置时，`Config Type`、端口号、授权方式、`root` 密码和服务名分别代表什么。
- 为什么要把 MySQL 的 `bin` 目录加入 `Path` 环境变量。
- 安装失败时，应该优先检查哪些依赖、残留目录和残留服务。

## 一句话先抓核心

**MySQL 安装的正确主线是：先选对安装包，再完成安装与初始化配置；如果命令行不能识别 `mysql`，检查 `Path`；如果重装失败，优先检查残留目录和残留服务。**

## 先做判断：我到底要装哪一种？

在下载安装包之前，先不要急着点下载，先判断这两件事。

### 判断 1：我要装 `MySQL 8.0` 还是 `MySQL 5.7`？

对初学阶段来说，通常优先考虑：

- **优先装 `MySQL 8.0`**
  - 更接近当前主流版本
  - 默认配置和认证方式更新
- **只有在课程、项目或旧工具明确要求时，再装 `MySQL 5.7`**

如果你只是跟当前课程学习，而且没有兼容性包袱，通常直接安装 `8.0` 就够了。

### 判断 2：我要下载联网版还是离线版？

Windows 下常见有两种 `MSI` 安装程序：

- `mysql-installer-web-community`
  - 安装包较小
  - 安装过程中需要联网下载组件
- `mysql-installer-community`
  - 安装包较大
  - 安装过程中可直接离线安装
  - **更适合初学者**

如果你的网络不稳定，或者你不想在安装过程中受网络影响，优先选择：

```text
mysql-installer-community
```

## 2.1 先分清 MySQL 常见版本

MySQL 常见版本大致可以这样理解：

* **MySQL Community Server**

  * 社区版
  * 开源免费
  * 适合大多数个人学习和开发环境
* **MySQL Enterprise Edition**

  * 企业版
  * 付费
  * 提供更完整的官方支持
* **MySQL Cluster**

  * 集群版
  * 用于更复杂的集群部署场景
* **MySQL Cluster CGE**

  * 更偏企业级的高级集群版本

对当前这门课来说，最重要的不是背版本名字，而是先记住：

**普通学习场景，默认看社区版就够了。**

另外，官方也提供 `MySQL Workbench` 作为图形化管理工具，但它不等于数据库服务本身。
你现在先把重点放在 **MySQL Server 能否正确安装并启动**，工具类内容放到后面再看会更稳。

## 2.2 下载 MySQL 安装包

### 2.2.1 下载入口

进入 MySQL 官网后，按下面思路进入下载页：

```text
官网首页 → Downloads → MySQL Community (GPL) Downloads → MySQL Community Server
```

### 2.2.2 Windows 下常见两种安装形式

Windows 平台常见有两类安装资源：

* `MSI 安装文件`

  * 带安装向导
  * 更适合初学者
* `ZIP 压缩包`

  * 免安装版
  * 需要自己初始化、配置和启动
  * 不适合作为入门第一选择

所以，初学阶段建议优先下载：

```text
MSI 安装程序
```

### 2.2.3 为什么更推荐 `MSI` 安装程序

因为它有两个优点：

1. 图形化安装向导更直观
2. 能把安装、初始化配置、服务注册串成一条完整流程

这比一开始就自己解压 `ZIP`、手动初始化、手动注册服务更适合入门。

### 2.2.4 两种 `MSI` 安装程序的区别

#### 方式 1：联网版

```text
mysql-installer-web-community
```

特点：

* 安装包较小
* 安装时需要联网下载组件

#### 方式 2：离线版

```text
mysql-installer-community
```

特点：

* 安装包较大
* 安装时不依赖持续联网
* 更稳妥，也更推荐

## 2.3 安装 MySQL 8.0

下载完成后，双击安装文件，按照安装向导完成程序安装。下面以常见的离线安装包为例说明。

### 步骤 1：打开安装向导

双击下载好的 `mysql-installer-community-xxx.msi` 文件，进入安装向导。

### 步骤 2：选择安装类型

![选择 Custom 安装类型](./images/image-20211014170553535.png)

安装器通常会给出几种安装类型，例如：

* `Developer Default`
* `Server only`
* `Client only`
* `Full`
* `Custom`

这里更推荐选择：

```text
Custom
```

### 为什么建议选 `Custom`？

因为它能让你更清楚地控制：

* 要安装哪些产品
* 程序装在哪里
* 数据目录放在哪里

对初学者来说，`Custom` 的意义不是“高级”，而是**更容易建立安装结构的理解**。

### 步骤 3：选择要安装的产品

![选择安装产品](./images/image-20211014170638699.png)

在产品选择界面，把需要的 MySQL Server 加入安装列表。

常见入门场景下，核心就是：

* `MySQL Server 8.0.x`

如果你现在只是先把服务安装起来，不需要一开始就把所有周边工具都装满。

### 步骤 4：确认安装目录和数据目录

![打开高级安装选项](./images/image-20211014170814386.png)

如果你使用默认路径，通常程序和数据会分别落在系统默认位置。
如果你打开 `Advanced Options`，可以进一步自定义目录。

![自定义安装目录与数据目录](./images/image-20211014170857263.png)

这里要先建立一个最重要的认知：

* **安装目录**：偏向程序文件
* **数据目录**：偏向数据库真实数据

所以在看路径时，不要把这两类目录混在一起。

### 自定义路径时的建议

* 尽量不要使用中文目录
* 安装目录和数据目录尽量分开
* 如果你改过目录，后面排错时一定要记得自己改过哪里

### 步骤 5：执行安装

![执行安装](./images/image-20211014170934889.png)

确认产品与路径后，点击 `Next`，再点击 `Execute` 执行安装。

如果安装过程顺利，状态会显示为完成。

## 2.4 配置 MySQL 8.0

程序安装完成后，还要继续做初始化配置。
这一步才是真正把 MySQL 服务器“配起来”的过程。

### 步骤 1：进入产品配置窗口

![进入产品配置窗口](./images/clip_image002-1634203188594.jpg)

安装完成后继续点击 `Next`，进入配置流程。

### 步骤 2：设置服务器类型与端口

![配置服务器类型与端口](./images/clip_image004-1634203188595.jpg)

在这一页通常要关注两个重点：

#### 1）`Config Type`

![Config Type 选项](./images/clip_image006-1634203188595.jpg)

它表示 MySQL 当前按什么场景来配置资源使用。

常见有：

* `Development Machine`

  * 适合个人开发电脑
  * 更保守地使用资源
* `Server Machine`

  * 适合和其他服务一起运行
* `Dedicated Machine`

  * 适合整台机器几乎专门给 MySQL 使用

对个人学习环境来说，通常选：

```text
Development Machine
```

#### 2）端口号

MySQL 默认端口通常是：

```text
3306
```

如果你的机器上没有其他 MySQL 服务占用这个端口，通常保持默认即可。

### 步骤 3：选择授权方式

![设置授权方式](./images/clip_image008-1634203188595.jpg)

MySQL 8.0 默认会提供较新的授权方式，通常使用更现代的密码加密方案；同时也可能提供更旧的兼容模式。

这里先记住一个原则：

* **没有兼容旧工具的需求时，优先使用默认推荐方式**
* **只有当旧版客户端、旧版图形化工具连不上时，再考虑兼容方案**

### 步骤 4：设置 `root` 密码

![设置 root 密码](./images/clip_image010-1634203188595.jpg)

这一步是设置 MySQL 超级管理员账户 `root` 的密码。

注意点：

* 两次输入要一致
* 密码不要太随意
* 这个密码后面登录 MySQL 会直接用到

如果安装向导支持添加其他用户，初学阶段通常可以先不加，等到后面的用户管理章节再学。

### 步骤 5：设置 Windows 服务名

![设置服务名与启动方式](./images/clip_image012-1634203188596.jpg)

MySQL 安装完成后，通常会注册成 Windows 服务。

课程中的服务名是：

```text
MySQL80
```

你可以继续使用这个默认命名，也可以自定义，但要记住你最终用的服务名，因为后面启动、停止服务时会用到。

### 是否勾选开机自启动？

如果你希望电脑启动后自动启动 MySQL，可以勾选：

```text
Start the MySQL Server at System Startup
```

一般学习环境勾不勾都可以，但如果你希望少做一次手动启动，通常可以勾上。

### 运行服务的账户怎么选？

常见有：

* `Standard System Account`
* `Custom User`

入门阶段通常选择前者即可。

### 步骤 6：执行配置

![执行服务器配置](./images/clip_image014-1634203188596.jpg)

确认配置后，点击 `Execute` 执行初始化配置。

### 步骤 7：完成配置

![完成服务器配置](./images/clip_image016.jpg)

全部显示成功后，点击 `Finish` 完成即可。

到这里，MySQL 服务端本体就已经安装并初始化配置完成了。

## 2.5 配置 MySQL 环境变量

安装完成后，如果你想在命令行里直接输入：

```bash
mysql --version
```

或者：

```bash
mysql -u root -p
```

那通常还需要把 MySQL 的 `bin` 目录加入 `Path`。

### 为什么要加 `bin` 目录？

因为 `mysql.exe`、`mysqld.exe` 这类可执行文件就在 `bin` 目录下。
只有系统能找到这些程序，你才能在任意命令行窗口直接输入 `mysql`。

### 配置步骤

1. 打开【此电脑】属性
2. 进入【高级系统设置】
3. 打开【环境变量】
4. 在系统变量中找到 `Path`
5. 把 MySQL 的 `bin` 目录加入进去，例如：

```text
C:\Program Files\MySQL\MySQL Server 8.0\bin
```

### 配置完成后怎么验证？

保存后，**重新打开一个新的命令行窗口**，再执行：

```bash
mysql --version
```

如果能正常显示版本，说明环境变量已经生效。

### 如果还是提示“不是内部或外部命令”

优先检查下面几件事：

* `Path` 里填的是不是 `bin` 目录，而不是上一级目录
* 保存后有没有重新打开命令行窗口
* 你安装时是不是改过安装路径
* 系统里是不是还留着旧版 MySQL 的路径

## 2.6 MySQL 5.7 与 8.0 的差异提醒

如果你安装的是 `MySQL 5.7`，大多数安装步骤和 `8.0` 是相似的，但要特别注意下面几点：

### 1）不要把 `5.7` 和 `8.0` 当成完全一样

虽然安装流程相似，但版本不同，界面、默认配置和认证方式可能会有细微差异。

### 2）如果机器上已经装过 `8.0`，再装 `5.7` 时不要继续用同一个端口

这是最容易忽略的问题之一。

例如：

* 前面已经有一个 MySQL 服务占用了 `3306`
* 你又安装另一个版本，还想继续用 `3306`
* 结果后装的服务无法正常启动

所以如果你要并存安装多个版本，就要明确区分：

* 服务名
* 端口号
* 安装目录
* 数据目录

## 2.7 安装失败时怎么排查

> 🧠 MySQL 安装本身不算复杂，真正让初学者反复卡住的，通常是：
>
> * 系统依赖没装齐
> * 旧版本残留没清干净
> * Windows 里还残留旧服务

所以排查时不要乱试，按顺序来。

### 问题 1：安装包打不开，或者安装过程中直接失败

#### 先检查系统依赖

Windows 安装 MySQL 时，常见要先确认：

* `.NET Framework`
* `Windows Installer`
* `Microsoft Visual C++ 2015-2019` 相关运行库

如果这些依赖不完整，安装器可能无法正常打开，或者在安装过程中失败。

### 问题 2：卸载后重装，还是安装失败

这类问题通常不是“不会安装”，而是**旧版本没有卸载干净**。

优先检查：

#### 1）安装目录是否还在

常见默认位置：

```text
C:\Program Files\MySQL
```

#### 2）数据目录是否还在

常见默认位置：

```text
C:\ProgramData\MySQL
```

> ✏️ `ProgramData` 通常是隐藏目录。
> 如果你安装时改过路径，就不要只看默认目录，要回想自己当时改到了哪里。

如果这些旧目录还在，重新安装时就可能继续读取旧配置或检测到旧残留。

### 问题 3：Windows 里还残留旧的 MySQL 服务

有时你明明已经卸载了程序，但服务列表里仍然还留着旧服务名，例如：

```text
MySQL80
```

这时重新安装可能会提示：

* 同名服务已存在
* 服务注册失败
* 配置阶段执行失败

可以打开命令提示符，执行：

```bash
sc delete MySQL服务名
```

例如：

```bash
sc delete MySQL80
```

执行后，再重新安装会更干净。

## 常见风险点

### 风险 1：下载了 `ZIP` 包，却以为会像安装器一样自动配置

不会。
`ZIP` 版更偏手动配置路线，不适合作为第一套入门安装方式。

### 风险 2：只顾着点下一步，不知道 `Custom` 在控制什么

`Custom` 的重点不是“高级”，而是让你知道程序装在哪里、数据存在哪里、后面排错该去哪里看。

### 风险 3：装好了服务，却没配 `Path`

结果图形化工具能连，命令行却找不到 `mysql`。

### 风险 4：装过旧版本，又直接重装

这时最容易被旧目录、旧数据目录、旧服务名卡住。

### 风险 5：同时安装多个版本，却继续共用端口

这会导致后安装的服务无法正常启动。

## 常见回查问题

### 1. Windows 下推荐下载哪个安装包？

通常优先下载：

```text
mysql-installer-community
```

也就是离线版安装器。

### 2. 为什么不建议初学时直接用 `ZIP` 版？

因为它需要自己处理初始化、启动和更多配置细节，入门阶段更容易乱掉。

### 3. 安装时为什么建议选 `Custom`？

因为它可以明确控制安装产品和目录，更方便你理解安装结构与后续排错。

### 4. `Config Type`、端口、授权方式、`root` 密码、服务名分别在哪一步设置？

都在安装完成后的初始化配置流程里设置。

### 5. 环境变量到底是在做什么？

它让系统能直接找到 `mysql.exe`，所以你才能在命令行直接输入 `mysql`。

### 6. 安装完成后为什么命令行还是找不到 `mysql`？

高频原因通常有 4 个：

* `Path` 没加
* 加错目录
* 没重开命令行窗口
* 旧路径或旧版本干扰

### 7. 卸载后重装失败时，优先检查什么？

按这个顺序查：

1. 依赖有没有问题
2. 安装目录还在不在
3. 数据目录还在不在
4. Windows 服务里有没有残留旧服务名

---

## 小结

这一节最重要的，不是记住每一张安装截图，而是记住下面这条主线：

1. 先确认要装 `8.0` 还是 `5.7`
2. 选择适合自己的安装包，初学阶段优先 `MSI`
3. 用安装器完成程序安装
4. 在初始化配置里看懂端口、授权方式、`root` 密码和服务名
5. 把 `bin` 目录加入 `Path`
6. 如果失败，优先查依赖、残留目录和残留服务

只要这条顺序不乱，MySQL 的安装和配置通常不会太难。
