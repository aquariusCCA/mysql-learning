# 第四章_运算符练习题参考答案

这份参考答案对应 [第四章_运算符练习题](./第四章_运算符练习题.md)。

- 查询题默认基于 `資料/atguigudb.sql`
- 涉及 `employees` 的题目，优先沿用题目要求的写法
- 同一题通常不只一种写法，下面给的是本章语境下较稳妥的一版

## 基础题

### 1. 算术运算符总览

答案：

- `+`：加法
- `-`：减法
- `*`：乘法
- `/`：普通除法
- `DIV`：整数除法，只保留整数部分
- `%`：取模，返回余数
- `MOD`：取模函数，和 `%` 语义一致

### 2. `+` 的语义判断

答案：

- MySQL 里的 `+` 是数值运算符，不是字符串拼接运算符。
- 字符串拼接通常用 `CONCAT()`。
- `+` 两边如果出现非数值，MySQL 会尝试把它们转成数值再计算。
- 转不成正常数值时，通常按 `0` 参与运算，并产生警告。

示例：

```sql
SELECT CONCAT('hello', 'mysql');
```

### 3. 结果类型判断

答案：

- `100 + 50`：整数
- `100 + 35.5`：浮点数
- `100 * 1`：整数
- `100 * 1.0`：浮点数
- `100 / 2`：浮点数
- `100 DIV 3`：整数
- `100 / 0`：`NULL`

### 4. `/` 与 `DIV` 比较题

答案：

- `/` 是普通除法，可以保留小数部分。
- `DIV` 是整数除法，只保留商的整数部分。

结果：

- `100 / 3`：`33.3333...`
- `100 DIV 3`：`33`

### 5. 奇偶筛选题

```sql
SELECT *
FROM employees
WHERE MOD(employee_id, 2) = 0;
```

说明：偶数编号对 `2` 取模结果为 `0`。

### 6. `=` 与 `NULL`

答案：

- `1 = 1`：`1`
- `1 = '1'`：`1`
- `1 = 'abc'`：`0`
- `NULL = NULL`：`NULL`
- `'' = NULL`：`NULL`

说明：

- `=` 会先做常规比较。
- 与 `NULL` 比较时，结果不是 `1` 或 `0`，而是 `NULL`，也就是未知。

### 7. `<=>` 与 `=`

答案：

- `=` 是普通等于比较，遇到 `NULL` 会变成未知。
- `<=>` 是安全等于比较，可以专门处理 `NULL`。

结果：

- `NULL = NULL`：`NULL`
- `NULL <=> NULL`：`1`
- `1 = NULL`：`NULL`
- `1 <=> NULL`：`0`

### 8. 判断空值的标准写法

```sql
SELECT employee_id, last_name, manager_id
FROM employees
WHERE manager_id IS NULL;
```

```sql
SELECT employee_id, last_name, commission_pct
FROM employees
WHERE commission_pct IS NOT NULL;
```

说明：

- 判断空值要用 `IS NULL` / `IS NOT NULL`
- 不建议写 `= NULL` 或 `<> NULL`，因为它们的结果是未知，不会按直觉工作

### 9. 范围筛选题

```sql
SELECT last_name, salary
FROM employees
WHERE salary BETWEEN 2500 AND 3500;
```

说明：`BETWEEN ... AND ...` 默认包含边界值。

### 10. 列表筛选题

```sql
SELECT last_name, employee_id, manager_id
FROM employees
WHERE manager_id IN (100, 101, 201);
```

## 进阶题

### 11. `NOT IN` 风险判断

答案：

- `NOT IN` 可以理解为“和集合里的每一个值都不相等”。
- 如果子查询结果里混入了 `NULL`，其中某一步比较会变成未知。
- 一旦条件链中出现未知，整个判断就可能无法得到 `TRUE`。
- 最终现象通常是：语法没错，但结果比预期少，甚至一行都查不到。

### 12. `LIKE` 基础题

```sql
SELECT *
FROM employees
WHERE first_name LIKE 'S%';
```

### 13. `LIKE` 单字符匹配

```sql
SELECT last_name
FROM employees
WHERE last_name LIKE '_a%';
```

说明：

- `_` 表示恰好一个字符
- `%` 表示任意长度的任意字符序列
- `_a%` 表示第二个字符固定为 `a`，前面一个字符任意，后面长度任意

### 14. `ESCAPE` 思考题

答案：

- 在 `LIKE` 里，`%` 和 `_` 默认是通配符，不再表示字面字符。
- 如果你真想匹配“字符串里真的有 `%`”或“真的有 `_`”，就必须告诉数据库“这个符号现在不是通配符，而是普通字符”。
- `ESCAPE` 的作用就是指定一个转义字符，让你能把 `%`、`_` 当成字面值去匹配。

示例：

```sql
SELECT '100%';
SELECT *
FROM some_table
WHERE some_col LIKE '%\%%' ESCAPE '\';
```

### 15. `REGEXP` 基础题

```sql
SELECT last_name
FROM employees
WHERE last_name REGEXP '^b';
```

说明：

- `^b` 表示以字母 `b` 开头
- 这类简单前缀判断也可以写成 `LIKE 'b%'`
- 但 `REGEXP` 用的是正则表达式语法，表达能力更强

### 16. `LIKE` 与 `REGEXP` 比较

答案：

1. 只判断“是否以某个前缀开头”：优先用 `LIKE`
   原因：写法简单，可读性高。
2. 需要判断“是否匹配某个字符集合”：优先用 `REGEXP`
   原因：字符集合如 `[abc]` 用正则更直接。
3. 需要判断“某字符连续出现至少两次”：优先用 `REGEXP`
   原因：这类重复次数判断是正则更擅长的场景。

### 17. 逻辑运算符判断

答案：

- `NOT`：对条件取反
- `AND`：两个条件都成立才成立
- `OR`：两个条件任意一个成立即可
- `XOR`：两个条件中有且仅有一个成立才成立

示例场景：

- `NOT`：排除 `job_id = 'IT_PROG'` 的员工
- `AND`：工资高且岗位符合的员工
- `OR`：属于部门 `10` 或 `20` 的员工
- `XOR`：满足两个条件中恰好一个的员工

### 18. `AND` 条件题

```sql
SELECT employee_id, last_name, job_id, salary
FROM employees
WHERE salary >= 10000
  AND job_id LIKE '%MAN%';
```

### 19. `OR` 条件题

写法一：

```sql
SELECT employee_id, salary
FROM employees
WHERE salary < 9000
   OR salary > 12000;
```

写法二：

```sql
SELECT employee_id, salary
FROM employees
WHERE salary NOT BETWEEN 9000 AND 12000;
```

说明：两种写法语义等价，第二种更紧凑。

### 20. `XOR` 语义题

条件：

```sql
department_id IN (10, 20) XOR salary > 8000
```

自然语言解释：

- 只保留“两个条件中恰好一个成立”的员工。
- 会被保留：
  - 在 `10` 或 `20` 部门，但工资不大于 `8000`
  - 不在 `10` 或 `20` 部门，但工资大于 `8000`
- 不会被保留：
  - 两个条件都成立
  - 两个条件都不成立

### 21. `NULL` 参与逻辑判断

答案：

- `NOT NULL`：`NULL`
- `1 AND NULL`：`NULL`
- `0 AND NULL`：`0`
- `1 OR NULL`：`1`
- `0 OR NULL`：`NULL`

说明：`NULL` 代表未知，逻辑运算要按三值逻辑判断。

### 22. 改错题：空值比较

错误原因：`= NULL` 不会按“等于空值”来判断。

正确写法：

```sql
SELECT employee_id, commission_pct
FROM employees
WHERE commission_pct IS NULL;
```

### 23. 改错题：优先级误判

问题：`AND` 优先级高于 `OR`，原 SQL 实际会先算 `department_id = 20 AND salary > 8000`。

更稳妥的写法：

```sql
SELECT employee_id, department_id, salary
FROM employees
WHERE department_id IN (10, 20)
  AND salary > 8000;
```

或者：

```sql
SELECT employee_id, department_id, salary
FROM employees
WHERE (department_id = 10 OR department_id = 20)
  AND salary > 8000;
```

## 挑战题

### 24. 位运算基础判断

答案：

- `&`：按位与
- `|`：按位或
- `^`：按位异或
- `~`：按位取反
- `>>`：右移
- `<<`：左移

它们和逻辑运算符的差别：

- 位运算针对二进制位逐位计算
- 逻辑运算针对条件真假计算

### 25. 位运算结果推导

答案：

- `1 & 10`：`0`
- `20 & 30`：`20`
- `1 | 10`：`11`
- `20 | 30`：`30`
- `1 ^ 10`：`11`
- `20 ^ 30`：`10`

说明：先转成二进制逐位算，再换回十进制。

### 26. 位移结果推导

答案：

- `1 >> 2`：`0`
- `4 >> 2`：`1`
- `1 << 2`：`4`
- `4 << 2`：`16`

说明：

- 右移时，右边被移出的位丢弃，左边补 `0`
- 左移时，左边被移出的位丢弃，右边补 `0`

### 27. `~` 的优先级题

答案：

- `~` 是按位取反，不是普通负号。
- 在 `10 & ~1` 里，会先算 `~1`，再和 `10` 做按位与。
- 也就是说它不是 `(10 & 1)` 再取反，而是 `10 & (~1)`。
- 在整数视角下，这个表达式结果是 `10`。

### 28. 正则模式判断题

答案：

- `'^abc'`：以 `abc` 开头
- `'xyz$'`：以 `xyz` 结尾
- `'a.b'`：`a` 和 `b` 中间恰好一个任意字符
- `'[aeiou]'`：任意一个元音字符
- `'[^aeiou]'`：任意一个非元音字符
- `'x{2,}'`：字符 `x` 连续出现至少两次

### 29. 写法选择题

答案：

1. 固定区间：`BETWEEN ... AND ...`
2. 固定值列表：`IN (...)`
3. 某个前缀开头：`LIKE 'xxx%'`
4. 某类字符集合：`REGEXP`
5. 有且仅有一个成立：`XOR`

### 30. 综合题：从需求到 SQL

```sql
SELECT employee_id, last_name, job_id, salary
FROM employees
WHERE salary BETWEEN 5000 AND 12000
  AND job_id NOT IN ('IT_PROG', 'ST_CLERK')
  AND manager_id IS NOT NULL
  AND department_id IN (30, 50, 80);
```

### 31. 综合题：条件改写比较

需求：工资不在 `9000` 到 `12000` 之间。

写法一：

```sql
WHERE NOT (salary >= 9000 AND salary <= 12000)
```

写法二：

```sql
WHERE salary < 9000 OR salary > 12000
```

写法三：

```sql
WHERE salary NOT BETWEEN 9000 AND 12000
```

关系说明：

- 写法一和写法二是德摩根律的等价改写
- `NOT BETWEEN` 本质上也是同一语义的更紧凑写法

### 32. 综合题：优先级与可读性

答案：

1. 只靠记忆优先级不稳妥，因为条件一复杂就容易误判。
2. 显式加括号能把意图写清楚，减少阅读和维护成本。
3. 最容易混用出错的通常是：
   - `AND` / `OR`
   - `NOT`
   - 位运算符和普通比较混在一起
   - `NULL` 判断和普通比较混在一起
