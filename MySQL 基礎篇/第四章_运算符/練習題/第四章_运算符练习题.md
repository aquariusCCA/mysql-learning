# 第四章_运算符练习题

这份题单只基于第四章现有笔记内容设计，重点练习：

- 算术运算符的结果类型与特殊行为
- 比较运算符对普通值与 `NULL` 的处理差异
- 逻辑运算符在复合条件中的真假判断
- 位运算符的基本规则与结果推导
- 运算符优先级与 `REGEXP` 的基础应用

## 使用说明

- 除题目特别说明外，练习数据默认使用仓库中的 `資料/atguigudb.sql`。
- 涉及表查询时，优先基于 `employees` 表完成练习。
- 这一份只放题目，不放答案；建议先独立完成，再回到第四章对应小节对照。
- 涉及“判断结果”或“改错”的题目，建议先写出理由，再写最终 SQL 或最终结果。

## 基础题

### 1. 算术运算符总览

请写出第四章中最基础的算术运算符有哪些，并分别说明它们的作用：

- `+`
- `-`
- `*`
- `/`
- `DIV`
- `%`
- `MOD`

### 2. `+` 的语义判断

说明为什么 MySQL 中的 `+` 不能当作字符串拼接使用，并补充说明：

1. MySQL 中字符串拼接通常该用什么；
2. 当 `+` 两边出现非数值时，MySQL 会怎么处理。

### 3. 结果类型判断

请判断下面表达式的结果是“整数”“浮点数”还是 `NULL`：

- `100 + 50`
- `100 + 35.5`
- `100 * 1`
- `100 * 1.0`
- `100 / 2`
- `100 DIV 3`
- `100 / 0`

### 4. `/` 与 `DIV` 比较题

说明普通除法 `/` 和整数除法 `DIV` 的差别，并分别写出下面两个表达式的结果：

- `100 / 3`
- `100 DIV 3`

### 5. 奇偶筛选题

查询 `employees` 表中 `employee_id` 为偶数的员工全部信息。

要求至少使用一次 `MOD` 或 `%`。

### 6. `=` 与 `NULL`

判断下面表达式的结果分别是什么，并说明原因：

- `1 = 1`
- `1 = '1'`
- `1 = 'abc'`
- `NULL = NULL`
- `'' = NULL`

### 7. `<=>` 与 `=`

说明 `=` 和 `<=>` 的差别，并判断下面表达式的结果：

- `NULL = NULL`
- `NULL <=> NULL`
- `1 = NULL`
- `1 <=> NULL`

### 8. 判断空值的标准写法

请分别写出：

1. 查询 `manager_id` 为空的员工；
2. 查询 `commission_pct` 不为空的员工。

并说明为什么不建议写成 `= NULL` 或 `<> NULL`。

### 9. 范围筛选题

查询工资在 `2500` 到 `3500` 之间的员工姓名与工资。

要求使用 `BETWEEN ... AND ...`。

### 10. 列表筛选题

查询 `manager_id` 为 `100`、`101`、`201` 的员工姓名、员工编号与经理编号。

要求使用 `IN`。

### 12. `LIKE` 基础题

要求使用 `LIKE`。

### 13. `LIKE` 单字符匹配

写一条 SQL，查询某个字段中“第二个字符固定、但前后位置有变化”的记录。

要求你必须实际使用 `_` 通配符，并说明 `_` 和 `%` 的差别。

### 15. `REGEXP` 基础题

写出一条示例 SQL，查询某字段以字母 `b` 开头的记录。

要求：

- 必须使用 `REGEXP`；
- 说明为什么这里也可以用 `LIKE`，但 `REGEXP` 的表达方式是什么。

### 16. `LIKE` 与 `REGEXP` 比较

说明下面几类需求，优先适合用 `LIKE` 还是 `REGEXP`，并说明理由：

1. 只判断“是否以某个前缀开头”；
2. 需要判断“是否匹配某个字符集合”；
3. 需要判断“某字符连续出现至少两次”。

### 17. 逻辑运算符判断

请分别说明下面四个逻辑运算符的语义：

- `NOT`
- `AND`
- `OR`
- `XOR`

并各给出一个适合它的查询场景。

### 18. `AND` 条件题

### 19. `OR` 条件题

查询工资不在 `9000` 到 `12000` 之间的员工编号与工资。

要求：

- 至少写出一版使用 `OR` 的写法；
- 如果你愿意，可以再补一版使用 `NOT BETWEEN` 的写法。

### 20. `XOR` 语义题


### 23. 改错题：优先级误判


## 挑战题

### 24. 位运算基础判断

请分别说明下面位运算符的含义：

- `&`
- `|`
- `^`
- `~`
- `>>`
- `<<`

并说明它们和 `AND`、`OR`、`XOR` 这类逻辑运算符的差别。

### 25. 位运算结果推导

请直接写出下面表达式的结果，并说明你是怎么从二进制推回十进制的：

- `1 & 10`
- `20 & 30`
- `1 | 10`
- `20 | 30`
- `1 ^ 10`
- `20 ^ 30`

### 26. 位移结果推导

判断下面表达式的结果：

- `1 >> 2`
- `4 >> 2`
- `1 << 2`
- `4 << 2`

并说明右移和左移时，哪些位会被丢弃，哪些位置会补 `0`。

### 27. `~` 的优先级题

说明为什么下面这个表达式不能只靠直觉猜结果：

```sql
10 & ~1
```

要求你从“位运算含义 + 运算符优先级”两个角度说明。

### 28. 正则模式判断题

请分别说明下面这些 `REGEXP` 模式各自表示什么：

- `'^abc'`
- `'xyz$'`
- `'a.b'`
- `'[aeiou]'`
- `'[^aeiou]'`
- `'x{2,}'`

### 29. 写法选择题

针对下面几种需求，分别说明你会优先选什么写法，并说明理由：

1. 判断工资是否落在一个固定区间；
2. 判断岗位是否属于几个固定值之一；
3. 判断姓名是否以某个前缀开头；
4. 判断字符串中是否出现某类字符集合；
5. 判断两个条件是否“有且仅有一个成立”。

### 30. 综合题：从需求到 SQL

完成下面需求：

查询 `employees` 表中，满足以下条件的员工编号、姓名、岗位编号与工资：

- `salary` 在 `5000` 到 `12000` 之间；
- `job_id` 不是 `IT_PROG` 和 `ST_CLERK`；

要求：

- 必须同时使用比较运算符与逻辑运算符；
