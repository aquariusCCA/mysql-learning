## 本节导读

这一节讲的是 MySQL 中最常见的比较运算符。

比较运算符最常出现的地方，就是 `WHERE` 条件。你平常写查询时，几乎每次都在做比较：

- 某个字段是否等于某个值
- 是否不等于某个值
- 是否在某个范围内
- 是否属于某个列表
- 是否符合某种字符串模式
- 是否为空

但这章真正容易出错的地方，不是“看不懂符号”，而是：**你以为它会返回真或假，结果它却返回了 `NULL`。**

所以这篇最核心的学习顺序不是先背所有运算符，而是先搞懂一件事：

> **MySQL 中比较运算的结果，不只有 `1` 和 `0`，还可能是 `NULL`。**

只要你把这个前提抓稳，后面的 `=`、`<=>`、`IS NULL`、`IN`、`NOT IN`、`LIKE`、`REGEXP` 都会清楚很多。

## 先记 8 句话

- 比较运算符的结果可能是 `1`、`0` 或 `NULL`。
- `1` 表示条件成立，`0` 表示条件不成立，`NULL` 表示“结果未知”，不是单纯的假。
- `=` 不能正确判断 `NULL` 是否相等。
- 判断空值优先使用 `IS NULL`、`IS NOT NULL`。
- 想在“相等比较”里把 `NULL` 也纳入明确判断时，使用 `<=>`。
- `BETWEEN A AND B` **包含边界**。
- `IN`、`NOT IN` 遇到 `NULL` 时要特别小心，尤其是 `NOT IN`。
- 简单字符串匹配优先用 `LIKE`，复杂模式再用 `REGEXP`。

## 你会在这篇学到什么

- 为什么比较运算符经常出现在 `WHERE` 条件中。
- `1`、`0`、`NULL` 在比较结果中分别代表什么。
- `=`、`<=>`、`<>`、`!=` 对普通值和 `NULL` 的处理差异。
- 如何使用 `IS NULL`、`IS NOT NULL` 判断空值和非空值。
- `LEAST()`、`GREATEST()` 如何返回一组值中的最小值或最大值。
- 如何使用 `BETWEEN AND`、`IN`、`NOT IN` 做范围或列表筛选。
- 如何使用 `LIKE`、`ESCAPE` 和 `REGEXP` 做字符串模式匹配。
- 为什么 `NOT IN` 一旦碰到 `NULL`，结果很容易和直觉不同。

## 关键字

- `比较运算符`：比较左右表达式并返回 `1`、`0` 或 `NULL` 的运算符。
- `1`：比较结果为真。
- `0`：比较结果为假。
- `NULL`：比较结果未知，不等于真，也不等于假。
- `=`：普通相等比较，遇到 `NULL` 时结果通常为 `NULL`。
- `<=>`：安全等于运算符，可以正确比较 `NULL`。
- `<>`、`!=`：不等于运算符。
- `IS NULL`：判断值是否为空。
- `IS NOT NULL`：判断值是否非空。
- `LEAST()`：返回参数中的最小值。
- `GREATEST()`：返回参数中的最大值。
- `BETWEEN AND`：判断值是否落在指定范围内，包含边界。
- `IN`：判断值是否在给定列表中。
- `NOT IN`：判断值是否不在给定列表中。
- `LIKE`：用通配符做简单模式匹配。
- `ESCAPE`：指定转义字符，用来匹配实际的 `%` 或 `_`。
- `REGEXP`：用正则表达式做更复杂的字符串匹配。

## 2.1 先搞懂：比较结果为什么会出现 NULL

在很多编程语言里，你会习惯把比较结果理解成“真 / 假”。

但在 MySQL 中，比较结果还可能出现第三种状态：`NULL`。

你可以把它理解成：

- `1`：成立
- `0`：不成立
- `NULL`：无法确定、未知

### 为什么会出现“未知”

因为 `NULL` 本身表示“没有值”或“值未知”。

既然值本身是未知的，那很多比较自然也无法得出明确的真假。例如：

```sql
SELECT
    1 = 1,
    1 = 2,
    1 = NULL,
    NULL = NULL
FROM DUAL;
```

这里最容易误判的是：

- `1 = NULL` 不是 `0`，而是 `NULL`
- `NULL = NULL` 也不是 `1`，而是 `NULL`

因为你不是在比较“两个确定的值”，而是在比较“未知”和“未知”。

### 这一节最重要的一句话

> **凡是有 `NULL` 参与普通比较，很多时候结果都不是 `1` 或 `0`，而是 `NULL`。**

后面整章的很多规则，其实都是从这里延伸出来的。

### 回查提示

如果你看到条件明明“看起来应该成立”，但实际结果怪怪的，先检查参与比较的值里是不是有 `NULL`。

## 2.2 等号运算符 `=`

等号运算符 `=` 用于判断左右两边的值、字符串或表达式是否相等。

- 相等返回 `1`
- 不相等返回 `0`
- 如果有 `NULL` 参与，结果通常为 `NULL`

### 先记结论

- `=` 是最常用的相等比较写法。
- 两边都是数字时，按数字比较。
- 两边都是字符串时，按字符串比较。
- 一个是数字、一个是字符串时，MySQL 会发生隐式转换。
- 只要有 `NULL` 参与普通等号比较，结果通常就是 `NULL`。

### 基本示例

```sql
SELECT
    1 = 1,
    1 = 0,
    'a' = 'a',
    1 = '1',
    0 = 'abc',
    1 = NULL,
    NULL = NULL
FROM DUAL;
```

### 你应该从这里看懂什么

- `1 = 1` 返回 `1`
- `1 = 0` 返回 `0`
- `'a' = 'a'` 返回 `1`
- `1 = '1'` 会发生类型转换，结果为 `1`
- `0 = 'abc'` 这类写法会触发隐式转换，不适合作为业务 SQL 的好习惯
- `1 = NULL` 与 `NULL = NULL` 都不是普通真假判断，而是 `NULL`

### 类型转换要特别小心

MySQL 在比较时会做隐式转换，这会让一些看起来很奇怪的比较也能返回结果。

例如：

```sql
SELECT 0 = 'abc', 1 = 'abc' FROM DUAL;
```

这里不是因为字符串和数字真的“天然可以比较”，而是因为 MySQL 会尝试进行类型转换。

这类行为虽然要知道，但在实际业务里，**不要依赖这种隐式转换写 SQL**，可读性和稳定性都不好。

### 业务示例

```sql
SELECT
    employee_id,
    salary
FROM employees
WHERE salary = 10000;
```

### 回查提示

如果你看到条件里用了 `=` 却查不出空值，先检查比较对象里是不是有 `NULL`。

## 2.3 安全等于运算符 `<=>`

`<=>` 是 MySQL 中的安全等于运算符。

它和普通 `=` 最大的差别就在于：**它可以明确处理 `NULL`。**

### 先记结论

- `=` 遇到 `NULL` 时，结果通常为 `NULL`。
- `<=>` 遇到 `NULL` 时，仍然可以给出明确的 `1` 或 `0`。
- 两边都为 `NULL` 时，`NULL <=> NULL` 返回 `1`。

### 对照记忆

| 表达式 | 结果 |
| --- | --- |
| `1 = 1` | `1` |
| `1 = NULL` | `NULL` |
| `NULL = NULL` | `NULL` |
| `1 <=> 1` | `1` |
| `1 <=> NULL` | `0` |
| `NULL <=> NULL` | `1` |

### 基本示例

```sql
SELECT
    1 <=> '1',
    1 <=> 0,
    '' <=> NULL,
    NULL <=> NULL
FROM DUAL;
```

### 什么时候该想到 `<=>`

当你的需求不是单纯比较相等，而是：

> **“我想比较相等，而且我不想让 `NULL` 变成未知，我想得到明确结果。”**

这时就该想到 `<=>`。

### 业务示例

```sql
SELECT employee_id, commission_pct
FROM employees
WHERE commission_pct <=> 0.40;
```

如果要比较空值：

```sql
SELECT employee_id, commission_pct
FROM employees
WHERE commission_pct <=> NULL;
```

不过在“查空值”这个场景里，平时还是更推荐直接写 `IS NULL`，因为语义更直观。

### 回查提示

你只要有“既想比较相等，又想把 `NULL` 也纳入明确判断”的需求，就优先想到 `<=>`。

## 2.4 不等于运算符 `<>` / `!=`

`<>` 和 `!=` 都表示“不等于”。

- 不相等返回 `1`
- 相等返回 `0`
- 只要有 `NULL` 参与，结果通常为 `NULL`

### 先记结论

- `<>` 和 `!=` 作用相同。
- 它们可以判断普通值是否不相等。
- 它们不能正确处理 `NULL`。

### 基本示例

```sql
SELECT
    1 <> 1,
    1 != 2,
    'a' != 'b',
    'a' != NULL,
    NULL <> NULL
FROM DUAL;
```

### 你最容易犯的错

很多人会直觉写：

```sql
WHERE commission_pct != NULL
```

这是不对的。

因为 `!= NULL` 不是“判断非空”的标准写法。

判断非空应该写：

```sql
WHERE commission_pct IS NOT NULL
```

### 回查提示

如果条件里用了 `<>` 或 `!=`，但又掺杂了空值，先不要假设它会按普通真假逻辑工作。

## 2.5 空值判断运算符：`IS NULL`、`IS NOT NULL`、`ISNULL()`

这一组是整章最实用、最应该记牢的内容之一。

### 先记结论

- 找空值：优先用 `IS NULL`
- 找非空值：优先用 `IS NOT NULL`
- `ISNULL(expr)` 也能判断空值，但平时阅读性通常不如 `expr IS NULL`
- 不要写 `= NULL`
- 不要写 `!= NULL`

### 判断空值

```sql
SELECT
    NULL IS NULL,
    ISNULL(NULL),
    ISNULL('a'),
    1 IS NULL
FROM DUAL;
```

查询空值记录：

```sql
SELECT employee_id, commission_pct
FROM employees
WHERE commission_pct IS NULL;
```

下面这些写法中，推荐顺序是：

```sql
WHERE commission_pct IS NULL          -- 最推荐
WHERE commission_pct <=> NULL        -- 也可以
WHERE ISNULL(commission_pct)         -- 也可以
WHERE commission_pct = NULL          -- 错误思路
```

### 判断非空值

```sql
SELECT employee_id, commission_pct
FROM employees
WHERE commission_pct IS NOT NULL;
```

也可以写成：

```sql
WHERE NOT commission_pct <=> NULL
WHERE NOT ISNULL(commission_pct)
```

但最直观、最标准、最推荐的还是：

```sql
WHERE commission_pct IS NOT NULL
```

### 一眼看懂

| 需求 | 推荐写法 | 不推荐写法 |
| --- | --- | --- |
| 找空值 | `col IS NULL` | `col = NULL` |
| 找非空值 | `col IS NOT NULL` | `col != NULL` |

### 回查提示

只要需求是“找出空值 / 非空值”，第一反应应该分别是 `IS NULL`、`IS NOT NULL`。

## 2.6 最小值与最大值运算符：`LEAST()` / `GREATEST()`

这两个函数式运算符的作用很直接：

- `LEAST()`：返回一组值中的最小值
- `GREATEST()`：返回一组值中的最大值

### `LEAST()`

```sql
SELECT LEAST(1, 0, 2), LEAST('b', 'a', 'c'), LEAST(1, NULL, 2) FROM DUAL;
```

你应该记住：

- 参数都是数字时，返回最小数字
- 参数都是字符串时，返回最小字符串
- 参数里只要有 `NULL`，结果通常就是 `NULL`

### `GREATEST()`

```sql
SELECT GREATEST(1, 0, 2), GREATEST('b', 'a', 'c'), GREATEST(1, NULL, 2) FROM DUAL;
```

你应该记住：

- 参数都是数字时，返回最大数字
- 参数都是字符串时，返回最大字符串
- 参数里只要有 `NULL`，结果通常就是 `NULL`

### 使用场景

这两个函数适合在“多个值中取最大 / 最小”时快速使用，比手写一长串比较更直接。

### 回查提示

如果你只是想在一组值里取最小值或最大值，先想到 `LEAST()`、`GREATEST()`，再确认参数里有没有 `NULL`。

## 2.7 `BETWEEN AND` 运算符

`BETWEEN AND` 用来判断某个值是否位于某个范围内。

### 最重要的结论

> **`BETWEEN A AND B` 包含 `A` 和 `B` 两个边界。**

### 基本写法

```sql
WHERE salary BETWEEN 2500 AND 3500
```

这表示：

- `salary >= 2500`
- 并且 `salary <= 3500`

### 基本示例

```sql
SELECT
    1 BETWEEN 0 AND 1,
    10 BETWEEN 11 AND 12,
    'b' BETWEEN 'a' AND 'c'
FROM DUAL;
```

### 业务示例

```sql
SELECT last_name, salary
FROM employees
WHERE salary BETWEEN 2500 AND 3500;
```

### 和手写条件的关系

```sql
WHERE salary BETWEEN 2500 AND 3500
```

等价于：

```sql
WHERE salary >= 2500 AND salary <= 3500
```

### 回查提示

如果你要判断“介于两个值之间”，先确认自己是否需要包含边界；`BETWEEN AND` 默认包含两端。

## 2.8 `IN` 与 `NOT IN` 运算符

这一组是业务里非常常用，但也非常容易踩坑的地方。

### 2.8.1 `IN` 运算符

`IN` 用于判断某个值是否在给定列表中。

### 基本示例

```sql
SELECT
    'a' IN ('a', 'b', 'c'),
    1 IN (2, 3),
    NULL IN ('a', 'b'),
    'a' IN ('a', NULL)
FROM DUAL;
```

### 你应该记住什么

- 命中列表中的某个值，返回 `1`
- 不命中时，通常返回 `0`
- 如果比较对象本身是 `NULL`，结果通常为 `NULL`
- 如果列表里出现 `NULL`，结果判断会变复杂，不能只用“在 / 不在”去直觉理解

### 业务示例

```sql
SELECT
    e.employee_id,
    e.last_name,
    e.salary,
    e.manager_id
FROM employees e
WHERE e.manager_id IN (100, 101, 201);
```

### 2.8.2 `NOT IN` 运算符

`NOT IN` 用于判断某个值是否不在给定列表中。

### 基本示例

```sql
SELECT
    'a' NOT IN ('a', 'b', 'c'),
    1 NOT IN (2, 3)
FROM DUAL;
```

### 最容易踩雷的地方

`NOT IN` **一旦遇到 `NULL`，结果很容易不是你以为的那样。**

例如：

```sql
SELECT 1 NOT IN (2, NULL) FROM DUAL;
```

这个结果不是简单的 `1`，而会因为 `NULL` 的参与变成“无法确定”。

### 为什么会这样

因为 `NOT IN` 本质上会展开成多个比较，而只要这些比较里有 `NULL`，整体判断就可能变成 `NULL`。

所以你一定要记住：

> **只要 `NOT IN` 的列表或子查询结果里可能出现 `NULL`，就要特别警惕。**

### 实务建议

- 写 `IN` / `NOT IN` 时，先确认列表或子查询结果里会不会有 `NULL`
- 如果会有 `NULL`，不要只靠直觉判断结果
- 对初学阶段来说，先把这个坑记住，比死背展开规则更重要

### 回查提示

多个离散值的筛选条件，优先想到 `IN`；但只要你用了 `NOT IN`，就必须额外检查里面有没有 `NULL`。

## 2.9 `LIKE` 运算符

`LIKE` 用于做简单的字符串模式匹配。

它比 `=` 灵活，因为它允许你描述“开头是什么”“结尾是什么”“中间某个位置是什么”。

### 先记结论

- `%`：匹配任意长度的字符序列（可以是 `0` 个、1 个或多个）
- `_`：匹配任意单个字符
- `LIKE` 适合简单、直观的字符串匹配
- 如果模式复杂到已经不适合靠 `%` 和 `_` 表达，再切到 `REGEXP`

### 常见写法

| 需求 | 写法 |
| --- | --- |
| 以 `S` 开头 | `LIKE 'S%'` |
| 以 `n` 结尾 | `LIKE '%n'` |
| 包含 `an` | `LIKE '%an%'` |
| 第二个字符是 `a` | `LIKE '_a%'` |
| 恰好 4 个字符 | `LIKE '____'` |

### 业务示例

查询名字以 `S` 开头的员工：

```sql
SELECT first_name
FROM employees
WHERE first_name LIKE 'S%';
```

查询第二个字符是 `o` 的名字：

```sql
SELECT first_name
FROM employees
WHERE first_name LIKE '_o%';
```

### 如何匹配实际的 `%` 或 `_`

有时你不是想把 `%` 和 `_` 当通配符，而是想匹配它们本身。这时就要使用 `ESCAPE`。

例如：

```sql
SELECT *
FROM some_table
WHERE code LIKE 'A#_%' ESCAPE '#';
```

这里 `#` 被指定为转义字符，`#_` 表示“把 `_` 当普通字符看”。

同理，也可以把 `%` 转义成普通字符。

### 回查提示

只要你的需求还是“前缀 / 后缀 / 包含 / 固定某一位”，优先用 `LIKE`；它通常比 `REGEXP` 更直观。

## 2.10 `REGEXP` 运算符

`REGEXP` 用于使用正则表达式进行模式匹配。

它比 `LIKE` 更强大，但也更复杂。

### 先记结论

- `LIKE` 解决简单匹配
- `REGEXP` 解决复杂匹配
- 能用 `LIKE` 讲清楚的情况，通常没必要一上来就用 `REGEXP`

### 常见符号速查

| 符号 | 说明 | 示例 |
| --- | --- | --- |
| `^` | 匹配开头 | `'^S'` |
| `$` | 匹配结尾 | `'n$'` |
| `.` | 匹配任意单个字符 | `'a.c'` |
| `[...]` | 匹配集合中的任意一个字符 | `'[abc]'` |
| `[a-z]` | 匹配范围字符 | `'[0-9]'` |
| `*` | 匹配零个或多个前一字符 | `'a*'` |

### 基本示例

```sql
SELECT
    'shkstart' REGEXP '^s',
    'shkstart' REGEXP 't$',
    'shkstart' REGEXP 'hk'
FROM DUAL;
```

```sql
SELECT
    'atguigu' REGEXP 'gu.gu',
    'atguigu' REGEXP '[ab]'
FROM DUAL;
```

### 什么时候该用 `REGEXP`

当你的需求已经不是简单的：

- 以什么开头
- 以什么结尾
- 是否包含某段文本

而是更复杂的：

- 某个位置必须满足某种字符规则
- 多种字符集合组合
- 更灵活的模式判断

这时再切换到 `REGEXP`。

### 回查提示

当 `LIKE` 已经不够表达你的匹配规则时，再切到 `REGEXP`；否则优先用更直观的 `LIKE`。

## 常见混淆点

- `=` 不能用来判断 `NULL` 是否相等；判断空值应优先使用 `IS NULL` 或 `<=> NULL`。
- `NULL = NULL` 的结果不是 `1`，而是 `NULL`。
- `<=>` 是安全等于运算符，两边都为 `NULL` 时返回 `1`。
- `<>` 和 `!=` 都表示不等于，但它们也不能正常判断 `NULL`。
- `= NULL` 和 `!= NULL` 都不是判断空值 / 非空值的标准写法。
- `BETWEEN A AND B` 包含 `A` 和 `B` 两个边界。
- `IN` 和 `NOT IN` 遇到 `NULL` 时，不要用单纯的“在 / 不在”直觉去理解。
- `NOT IN` 一旦列表里可能有 `NULL`，就必须特别小心。
- `LIKE` 中 `%` 和 `_` 是通配符；要匹配它们本身时需要转义。
- `LIKE` 适合简单模式匹配，`REGEXP` 适合更复杂的正则匹配。

## 常见回查问题

- 比较运算符返回的 `1`、`0`、`NULL` 分别代表什么？
- 为什么 `NULL = NULL` 的结果不是 `1`？
- `<=>` 和 `=` 的差异是什么？
- 查询空值应该用 `IS NULL`、`ISNULL()` 还是 `= NULL`？
- 查询非空值为什么不能写成 `!= NULL`？
- `BETWEEN 2500 AND 3500` 是否包含 `2500` 和 `3500`？
- `IN` 和 `NOT IN` 遇到 `NULL` 时为什么容易出问题？
- `LIKE 'S%'` 和 `LIKE '_o%'` 分别匹配什么？
- `LIKE` 中想匹配实际的 `_` 字符应该怎么写？
- `REGEXP '^s'`、`REGEXP 't$'`、`REGEXP 'hk'` 分别检查什么？

## 一分钟自测

先不要往下看答案，自己判断下面这些表达式会返回什么：

1. `SELECT NULL = NULL FROM DUAL;`
2. `SELECT NULL <=> NULL FROM DUAL;`
3. `SELECT commission_pct = NULL FROM employees;`
4. `SELECT 3000 BETWEEN 2500 AND 3500 FROM DUAL;`
5. `SELECT 1 NOT IN (2, NULL) FROM DUAL;`
6. `SELECT 'Scott' LIKE 'S%' FROM DUAL;`
7. `SELECT 'Scott' REGEXP '^S' FROM DUAL;`

### 参考答案

1. 结果是 `NULL`，不是 `1`。
2. 结果是 `1`，因为 `<=>` 可以安全比较 `NULL`。
3. 这不是判断空值的标准写法，思路错误。
4. 结果是 `1`，因为 `BETWEEN AND` 包含边界范围内的值。
5. 结果不能简单用直觉理解，`NULL` 会让判断变复杂，这就是 `NOT IN` 最容易踩坑的地方。
6. 结果是 `1`，表示以 `S` 开头。
7. 结果是 `1`，表示满足正则“以 `S` 开头”。

## 一句话抓核心

比较运算符这一节真正要掌握的，不只是 `=`、`<>`、`IN`、`LIKE` 会写，而是：**MySQL 的比较结果可能是 `1`、`0` 或 `NULL`；凡是遇到 `NULL`，就必须特别小心 `=`、`!=`、`NOT IN` 这些看起来很直观、实际却最容易误判的地方。**

## 小结

这一节你需要记住：

- 比较运算符的返回结果可能是 `1`、`0` 或 `NULL`。
- `=` 用于普通相等比较，但不能正确判断 `NULL`。
- `<=>` 是安全等于运算符，适合把 `NULL` 也纳入明确比较。
- 判断空值与非空值，标准写法分别是 `IS NULL`、`IS NOT NULL`。
- `<>`、`!=` 都表示不等于，但不能拿来判断空值。
- `LEAST()`、`GREATEST()` 可以快速取多个值中的最小值或最大值，但参数中有 `NULL` 时要特别注意。
- `BETWEEN AND` 用于范围判断，并且包含边界。
- `IN` 与 `NOT IN` 用于列表筛选，其中 `NOT IN` 遇到 `NULL` 时最容易踩坑。
- `LIKE` 适合简单模式匹配，`REGEXP` 适合更复杂的正则匹配。
