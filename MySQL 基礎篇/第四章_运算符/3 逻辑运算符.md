## 本节导读

这一节讲的是 MySQL 中最常见的逻辑运算符：`NOT`、`AND`、`OR`、`XOR`。

它们看起来像是在做“真 / 假判断”，但真正容易出错的地方，不是符号本身，而是下面这三件事：

1. **MySQL 的逻辑结果不只有 `1` 和 `0`，还可能出现 `NULL`。**
2. **MySQL 会把 `0` 当成假，把非 `0` 当成真。**
3. **复合条件里，`AND` 的优先级高于 `OR`，所以不加括号很容易误读。**

也就是说，这一章真正要掌握的，不只是“`AND` 表示并且、`OR` 表示或者”，而是：

> **当 `NULL` 参与判断时，逻辑结果会进入“未知”状态；当多个逻辑运算符混在一起时，优先级会直接影响最终结果。**

第一次学习时，建议先建立“真 / 假 / 未知”的整体概念，再分别理解 `NOT`、`AND`、`OR`、`XOR` 的判断规则。复习时可以重点回查：

- `NOT NULL` 为什么还是 `NULL`
- `0 AND NULL` 为什么是 `0`
- `1 OR NULL` 为什么是 `1`
- 为什么复杂条件里必须加括号
- `XOR` 到底适合什么场景

## 先记 8 句话

- MySQL 的逻辑运算结果可能是 `1`、`0` 或 `NULL`。
- `1` 表示真，`0` 表示假，`NULL` 表示未知。
- 在逻辑上下文中，`0` 会被看作假，非 `0` 会被看作真。
- `NOT` 表示对原有条件取反。
- `AND` 表示多个条件必须同时成立。
- `OR` 表示多个条件只要成立一个即可。
- `XOR` 表示两个条件中有且只能有一个成立。
- 只要 `AND` 和 `OR` 混用，优先想到“要不要加括号”。

## 你会在这篇学到什么

- MySQL 中逻辑运算符为什么会返回 `1`、`0` 或 `NULL`。
- 为什么 `0` 表示假、非 `0` 表示真。
- `NOT` 如何对条件取反，以及 `NULL` 参与时为什么仍然得到 `NULL`。
- `AND` 和 `OR` 如何组合多个筛选条件。
- `XOR` 为什么适合“两个条件只能满足一个”的场景。
- 写复杂 `WHERE` 条件时，如何通过括号避免优先级误判。
- 什么时候应该直接改写条件，而不是依赖自己临时记忆优先级。

## 关键字

- `逻辑运算符`：用于判断表达式真假或组合多个条件的运算符。
- `1`：逻辑判断结果为真。
- `0`：逻辑判断结果为假。
- `NULL`：逻辑判断结果未知，常见于空值参与运算时。
- `NOT`、`!`：对条件取反。
- `AND`、`&&`：多个条件同时满足才为真。
- `OR`：多个条件只要满足一个就为真。
- `XOR`：两个条件中有且只能有一个为真。
- `优先级`：多个运算符同时出现时，MySQL 默认采用的计算顺序。
- `括号 ()`：显式控制逻辑判断顺序的最稳妥方式。

## 快速回查表

| 场景 | 写法 | 结果特征 | 需要注意 |
| --- | --- | --- | --- |
| 对条件取反 | `NOT status = 'ACTIVE'` | 原结果为真则变假，原结果为假则变真 | `NOT NULL` 的结果仍然是 `NULL` |
| 同时满足多个条件 | `salary >= 10000 AND job_id LIKE '%MAN%'` | 两边都真才返回 `1` | 只要有一个条件为 `0`，结果就是 `0` |
| 满足任一条件 | `salary < 9000 OR salary > 12000` | 只要一边为真就返回 `1` | 推荐显式写 `OR`，不要依赖 `||` |
| 只能满足一个条件 | `department_id IN (10, 20) XOR salary > 8000` | 恰好一个条件为真时返回 `1` | 两边都真或都假都返回 `0` |
| 对复合条件取反 | `NOT (salary >= 9000 AND salary <= 12000)` | 先算括号里的整体结果，再取反 | 复杂条件一定要加括号 |
| 混合 `AND` 与 `OR` | `A OR B AND C` | 默认先算 `B AND C` | `AND` 优先级高于 `OR` |

## 3.1 先搞懂：逻辑结果为什么会出现 `NULL`

很多人第一次学逻辑运算时，会下意识把结果理解成“真 / 假”。

但在 MySQL 中，逻辑运算并不只有两种结果，还可能出现第三种状态：`NULL`。

你可以把它理解成：

- `1`：成立
- `0`：不成立
- `NULL`：无法确定、未知

### 为什么会出现“未知”

因为逻辑判断本身，是建立在“已知条件”的基础上。

如果某个参与判断的值本身就是 `NULL`，那就代表这个值未知；既然值未知，很多逻辑判断自然也无法直接落到真或假，而会停在“未知”这个状态。

### 还要再记一件事：`0` 和非 `0`

在 MySQL 的逻辑上下文中：

- `0` 会被当成假
- 非 `0` 会被当成真
- `NULL` 表示未知

所以像下面这种写法：

```sql
SELECT 1 AND -1, 0 AND 1, NOT 0, NOT 2 FROM DUAL;
```

你不要只把它当成数字运算，而要把它看成真假判断：

- `1` 和 `-1` 都属于非 `0`，所以都是真
- `0` 是假
- `NOT 2` 的本质是“对真取反”

### 这一节最重要的一句话

> **逻辑运算不是只在处理 `true / false`，而是在处理 “真 / 假 / 未知”。**

后面整章的很多规则，其实都是从这里延伸出来的。

### 回查提示

如果你看到逻辑条件“看起来应该成立”，但结果却不是 `1` 或 `0`，先检查参与判断的值里是不是有 `NULL`。

## 3.2 逻辑非运算符 `NOT` / `!`

逻辑非（`NOT` 或 `!`）表示：**把原本的判断结果取反。**

### 先记结论

- 原结果为真，`NOT` 后变成假。
- 原结果为假，`NOT` 后变成真。
- 原结果为 `NULL`，`NOT` 后仍然是 `NULL`。

### 基本示例

```sql
SELECT NOT 1, NOT 0, NOT (1 + 1), NOT !1, NOT NULL FROM DUAL;
```

### 你应该从这里看懂什么

- `NOT 1` 返回 `0`，因为 `1` 表示真，取反后变假。
- `NOT 0` 返回 `1`，因为 `0` 表示假，取反后变真。
- `NOT (1 + 1)` 返回 `0`，因为 `1 + 1 = 2`，而 `2` 是非 `0`，会被当成真。
- `NOT NULL` 返回 `NULL`，因为“未知取反”之后仍然是“未知”。

### 常见业务写法

查询职位不是 `IT_PROG`、`ST_CLERK` 或 `SA_REP` 的员工：

```sql
SELECT
    e.last_name,
    e.job_id
FROM employees e
WHERE e.job_id NOT IN ('IT_PROG', 'ST_CLERK', 'SA_REP');
```

你也可以把它理解成：

- 先判断 `e.job_id IN (...)`
- 再对整个结果取反

### 最容易忽略的点

如果被取反的不是单一条件，而是一个复合条件，最好一定要加括号。

例如：

```sql
NOT (salary >= 9000 AND salary <= 12000)
```

这样才是“先判断范围是否成立，再整体取反”。

### 回查提示

当你的需求是“排除某类条件”时，优先想到 `NOT`。如果条件本身比较复杂，记得用括号把原条件包起来。

## 3.3 逻辑与运算符 `AND` / `&&`

逻辑与（`AND` 或 `&&`）表示：**多个条件必须同时满足。**

### 先记结论

- 只要有一个条件为假，结果就是假。
- 所有条件都为真，结果才为真。
- 如果没有出现假，但有 `NULL`，结果通常为 `NULL`。

### 基本示例

```sql
SELECT 1 AND -1, 0 AND 1, 0 AND NULL, 1 AND NULL FROM DUAL;
```

### 一眼看懂

- `1 AND -1` 返回 `1`，因为两个都是非 `0`，都为真。
- `0 AND 1` 返回 `0`，因为只要一边是假，整体就是假。
- `0 AND NULL` 返回 `0`，因为虽然另一边未知，但已经有一边确定是假了，整体不可能为真。
- `1 AND NULL` 返回 `NULL`，因为左边是真，但右边未知，所以无法确定整体是否成立。

### 你可以这样记

> **`AND` 最怕假。只要出现一个假，整体就直接假。**

### 常见业务写法

查询薪资大于等于 `10000`，且职位代号包含 `MAN` 的员工：

```sql
SELECT
    e.employee_id,
    e.last_name,
    e.job_id,
    e.salary
FROM employees e
WHERE e.salary >= 10000
  AND e.job_id LIKE '%MAN%';
```

这里的意思是：

- 薪资条件要成立
- 职位条件也要成立
- 两个缺一不可

### 回查提示

只要你的需求里出现“并且”“同时满足”“两个条件都要成立”，就应该先检查是不是该用 `AND`。

## 3.4 逻辑或运算符 `OR`

逻辑或（`OR`）表示：**多个条件中只要有一个成立即可。**

### 先记结论

- 只要有一个条件为真，结果就是真。
- 所有条件都为假，结果才是假。
- 如果没有出现真，但有 `NULL`，结果通常为 `NULL`。

### 基本示例

```sql
SELECT 1 OR -1, 1 OR 0, 1 OR NULL, 0 OR NULL, NULL OR NULL FROM DUAL;
```

### 一眼看懂

- `1 OR -1` 返回 `1`，因为至少有一个真，事实上两边都真。
- `1 OR 0` 返回 `1`，因为左边已经是真。
- `1 OR NULL` 返回 `1`，因为既然已经有一边确定为真，整体一定为真。
- `0 OR NULL` 返回 `NULL`，因为左边是假，但右边未知，所以无法确定整体是否成立。
- `NULL OR NULL` 返回 `NULL`，因为两边都未知。

### 你可以这样记

> **`OR` 最怕真。只要出现一个真，整体就直接真。**

### 常见业务写法

下面三种写法都表示“查询基本薪资不在 `9000` 到 `12000` 之间的员工编号和基本薪资”：

```sql
SELECT
    e.employee_id,
    e.salary
FROM employees e
WHERE NOT (e.salary >= 9000 AND e.salary <= 12000);
```

```sql
SELECT
    e.employee_id,
    e.salary
FROM employees e
WHERE e.salary < 9000
   OR e.salary > 12000;
```

```sql
SELECT
    e.employee_id,
    e.salary
FROM employees e
WHERE e.salary NOT BETWEEN 9000 AND 12000;
```

### 关于 `||`

虽然某些情况下 `||` 也可能被当作逻辑或使用，但为了避免和 SQL 模式差异混淆，实际写查询时更推荐直接使用 `OR`。

### 回查提示

如果你的需求里出现“或者”“任意一个满足即可”，先考虑 `OR`，然后再检查是否需要配合括号控制优先级。

## 3.5 逻辑异或运算符 `XOR`

逻辑异或（`XOR`）表示：**两个条件中有且只能有一个成立。**

它不是“都满足”，也不是“满足任意一个”，而是：

> **恰好命中一个。**

### 先记结论

- 一个真、一个假，结果为 `1`。
- 两个都真，结果为 `0`。
- 两个都假，结果为 `0`。
- 在常见二元判断里，只要有一边是 `NULL`，结果通常会变成 `NULL`。

### 基本理解

你可以把 `XOR` 理解成“排他性的 OR”。

它和 `OR` 的差别在于：

- `OR`：至少一个成立就行
- `XOR`：只能有一个成立，多一个都不行

### 常见业务写法

```sql
SELECT
    e.department_id,
    e.salary
FROM employees e
WHERE e.department_id IN (10, 20)
XOR e.salary > 8000;
```

这条语句表示：

- 员工要么属于 `10` 或 `20` 号部门
- 要么薪资高于 `8000`
- 但不能同时满足这两个条件

### 什么时候该想到 `XOR`

当你的需求不是：

- “都满足”
- “满足其一即可”

而是：

- “只能命中一个条件”

这时才该想到 `XOR`。

### 回查提示

如果你不是要“都满足”或“满足其一”，而是明确要求“只能命中一个条件”，就该想到 `XOR`。

## 3.6 复合条件中最容易错的地方：优先级与括号

这一小节不是在教新运算符，而是在解决实际写 SQL 时最常犯的错误：

> **条件本身没有写错，但你以为 MySQL 会这样理解，结果它却是按另一种顺序在算。**

### 最重要的结论

> **`AND` 的优先级高于 `OR`。**

所以：

```sql
A OR B AND C
```

默认会被理解成：

```sql
A OR (B AND C)
```

而不是：

```sql
(A OR B) AND C
```

### 一个很典型的误判例子

```sql
WHERE department_id = 10
   OR department_id = 20
  AND salary > 8000
```

很多人直觉会把它理解成：

- 部门是 `10` 或 `20`
- 而且薪资要大于 `8000`

但 MySQL 默认其实会先算：

```sql
WHERE department_id = 10
   OR (department_id = 20 AND salary > 8000)
```

这意味着：

- 只要部门是 `10`，即使薪资不大于 `8000`，也可能被查出来

### 正确写法

如果你真正想表达的是“部门是 `10` 或 `20`，并且薪资要大于 `8000`”，应该写成：

```sql
WHERE (department_id = 10 OR department_id = 20)
  AND salary > 8000
```

### 最稳的写法原则

- 只要 `AND` 和 `OR` 混用，就优先考虑加括号。
- 不要依赖自己临时记忆优先级去赌 SQL 的解释顺序。
- 括号不仅让 MySQL 不会误解，也让未来的你和同事更不容易误读。

### 回查提示

如果一个 `WHERE` 条件里同时出现了 `AND` 和 `OR`，就不要只靠语感判断顺序，优先加括号。

## 常见混淆点

- 逻辑运算符返回的不是程序语言里的 `true` / `false`，而通常是 `1`、`0`、`NULL`。
- 在逻辑上下文中，`0` 表示假，非 `0` 表示真。
- `NOT NULL` 的结果不是 `1` 或 `0`，而仍然是 `NULL`。
- `0 AND NULL` 的结果是 `0`，因为已经有一个条件确定为假。
- `1 OR NULL` 的结果是 `1`，因为已经有一个条件确定为真。
- `AND` 的优先级高于 `OR`。只要条件稍微复杂，就加括号，不要依赖自己临时记忆优先级。
- 为了避免和 SQL 模式差异混淆，实际开发中更推荐写 `OR`，不要把 `||` 当成默认可移植写法。
- `XOR` 很少用，但一旦需求是“只能命中一个条件”，它会比你手写复杂逻辑更直接。

## 一分钟自测

先不要往下看答案，自己判断下面这些表达式会返回什么：

1. `SELECT NOT 0 FROM DUAL;`
2. `SELECT NOT NULL FROM DUAL;`
3. `SELECT 1 AND NULL FROM DUAL;`
4. `SELECT 0 AND NULL FROM DUAL;`
5. `SELECT 1 OR NULL FROM DUAL;`
6. `SELECT 0 OR NULL FROM DUAL;`
7. `SELECT 1 XOR 0 FROM DUAL;`
8. `SELECT 1 XOR 1 FROM DUAL;`
9. `SELECT 0 XOR 0 FROM DUAL;`

### 参考答案

1. 结果是 `1`，因为 `0` 表示假，取反后变真。
2. 结果是 `NULL`，因为未知取反后仍然是未知。
3. 结果是 `NULL`，因为左边为真，右边未知，整体无法确定。
4. 结果是 `0`，因为已经有一个条件确定为假。
5. 结果是 `1`，因为已经有一个条件确定为真。
6. 结果是 `NULL`，因为左边为假，右边未知，整体无法确定。
7. 结果是 `1`，因为恰好只有一个条件为真。
8. 结果是 `0`，因为两个条件都为真，不符合“只能一个成立”。
9. 结果是 `0`，因为两个条件都为假，也不符合“只能一个成立”。

## 一句话抓核心

逻辑运算符这一节真正要掌握的，不只是 `NOT`、`AND`、`OR`、`XOR` 会写，而是：**MySQL 的逻辑判断是在处理“真 / 假 / 未知”；只要 `NULL` 参与，结果就可能变成 `NULL`；只要 `AND` 和 `OR` 混用，就必须警惕优先级并优先加括号。**

## 小结

这一节你需要记住：

- `NOT` 适合对已有条件取反。
- `AND` 适合“多个条件必须同时满足”的场景。
- `OR` 适合“多个条件满足其一即可”的场景。
- `XOR` 适合“两个条件有且只能满足一个”的场景。
- MySQL 的逻辑结果可能是 `1`、`0` 或 `NULL`。
- 在逻辑上下文中，`0` 表示假，非 `0` 表示真。
- 只要 `NULL` 参与逻辑判断，结果就可能变成 `NULL`。
- 写复合逻辑条件时，要特别关注优先级；只要 `AND` 和 `OR` 混用，复杂条件尽量显式加括号。
