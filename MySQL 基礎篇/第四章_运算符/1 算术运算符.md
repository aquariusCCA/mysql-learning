## 本节导读

这一节讲的是 MySQL 中最常见的算术运算符：加法、减法、乘法、除法和求模。

表面上看，这一章只是“会不会算”，但真正容易出错的地方其实不在公式本身，而在 **MySQL 如何处理类型、结果会变成什么、遇到特殊值时会发生什么**。

第一次学习时，建议先抓住 4 个最容易混淆的点：

1. `+` 在 MySQL 中只表示 **数值相加**，**不表示字符串拼接**。
2. 整数和浮点数一起运算时，结果通常会变成浮点数。
3. `/` 是普通除法，重点是“算出真实除法结果”；`DIV` 是整数除法，重点是“只取整数商”。
4. 一个数除以 `0` 时，结果不是正常数值，而是 `NULL`。

复习时，可以重点回查：`+` 的数值转换规则、`/` 与 `DIV` 的差异、除以 `0` 的结果，以及 `%` / `MOD` 的实际用法。

## 先记 5 句话

- MySQL 中的 `+` 不是字符串拼接符。
- 拼接字符串要使用 `CONCAT()`。
- `100 / 3` 和 `100 DIV 3` 的语义不同。
- `%` 和 `MOD` 都表示求余。
- 只要题目里出现“奇数、偶数、整除、每隔几个取一次”，优先想到 `%` 或 `MOD`。

## 你会在这篇学到什么

- 整数、浮点数、字符串参与运算时，结果会如何变化。
- `+` 在 MySQL 中为什么不能拿来拼接字符串。
- `/`、`DIV` 与除以 `0` 时的结果差异。
- 如何用 `%` 或 `MOD` 解决奇偶筛选、整除判断这类常见问题。

## 关键字

- `算术运算符`：对数值或表达式进行数学运算的运算符。
- `+`：加法运算符，在 MySQL 中只表示数值相加。
- `-`：减法运算符。
- `*`：乘法运算符。
- `/`：普通除法运算符，重点是得到除法结果。
- `DIV`：整数除法运算符，只取整数商。
- `%`：求模运算符。
- `MOD`：求余函数式写法，与 `%` 作用相同。
- `CONCAT()`：字符串拼接函数。
- `NULL`：表示空值；在除以 `0` 等场景中常会出现。

## 1.1 运算符总览

算术运算符用于对数值或表达式进行数学计算。MySQL 中这一节最常见的算术运算包括：

- 加法：`+`
- 减法：`-`
- 乘法：`*`
- 除法：`/`
- 整数除法：`DIV`
- 求模：`%`、`MOD`

它们看起来和其他语言很像，但 **语义并不完全一样**。尤其是：

- `+` 在 MySQL 中不是字符串拼接。
- `/` 在 MySQL 中不是“默认取整数”的除法。
- `DIV` 和 `/` 不能混为一谈。

如果你一时记不清这一节到底包含哪些运算，先回到这里看总览，再跳到对应小节。

## 1.2 加法与减法运算符

这一小节最重要的，不是会不会做加减法，而是要搞清楚 **MySQL 对 `+` 的理解和很多编程语言不一样**。

### 先记结论

- `+` 和 `-` 都是数值运算。
- 整数与整数相加减，结果通常还是整数。
- 整数与浮点数相加减，结果通常会变成浮点数。
- `+` 不负责拼接字符串；字符串拼接应该使用 `CONCAT()`。
- 如果参与 `+` 运算的字符串能转成数字，MySQL 会先转成数字再计算；如果不能正常转成数字，通常会按 `0` 参与运算，并产生警告。

### 基本示例

```sql
SELECT
    100 + 50,
    100 - 35.5,
    '10' + 1,
    '3.14' + 2,
    'abc' + 1,
    CONCAT('Hello', ' ', 'MySQL')
FROM DUAL;
```

你应该重点看出下面几件事：

- `100 + 50` 是普通加法，结果为 `150`。
- `100 - 35.5` 中有浮点数参与，结果会带小数。
- `'10' + 1` 不是字符串拼接，而是把 `'10'` 转成数值 `10` 后再相加。
- `'abc' + 1` 也不会变成 `'abc1'`，因为 MySQL 会尝试做数值运算，而不是字符串拼接。
- 真正的字符串拼接方式是 `CONCAT()`。

### 典型示例

```sql
mysql> SELECT 100, 100 + 0, 100 - 0, 100 + 50, 100 + 50 - 30, 100 + 35.5, 100 - 35.5 FROM dual;
+-----+---------+---------+----------+--------------+------------+------------+
| 100 | 100 + 0 | 100 - 0 | 100 + 50 | 100 + 50 -30 | 100 + 35.5 | 100 - 35.5 |
+-----+---------+---------+----------+--------------+------------+------------+
| 100 |     100 |     100 |      150 |          120 |      135.5 |       64.5 |
+-----+---------+---------+----------+--------------+------------+------------+
1 row in set (0.00 sec)
```

### 从结果中你应该记住什么

- 一个整数对整数做加减，结果通常仍是整数。
- 一个整数对浮点数做加减，结果通常会变成浮点数。
- `+` 和 `-` 的优先级相同，表达式会按从左到右的默认顺序计算。
- **最重要的一条：不要把 Java / JavaScript 里“`+` 可以拼接字符串”的习惯带进 MySQL。**

### 易错对照

| 需求 | 错误想法 | 正确写法 |
| --- | --- | --- |
| 把 `10` 和 `20` 相加 | `'10' + '20'` 会得到 `'1020'` | `'10' + '20'` 实际会按数值相加 |
| 拼接姓和名 | `first_name + last_name` | `CONCAT(first_name, last_name)` |
| 字符串不能参与 `+` | `'10' + 1` 会报错 | MySQL 会先尝试转成数字再计算 |

### 回查提示

如果你把其他语言里“`+` 可拼接字符串”的习惯带进 MySQL，最容易在这里出错。记住：**MySQL 的 `+` 只做数值运算。**

## 1.3 乘法与除法运算符

这一小节的重点是：**搞清楚 `*`、`/`、`DIV` 的语义差异**。

### 先记结论

- `*` 是普通乘法。
- `/` 是普通除法，重点是得到除法结果；很多时候你会看到带小数的结果。
- `DIV` 是整数除法，重点是只取整数商。
- 一个数除以 `0`，结果为 `NULL`。
- 如果你本来预期得到整数，却看到了小数，先检查你到底用的是 `/` 还是 `DIV`。

### 典型示例

```sql
mysql> SELECT 100, 100 * 1, 100 * 1.0, 100 / 1.0, 100 / 2, 100 + 2 * 5 / 2, 100 / 3, 100 DIV 0 FROM dual;
+-----+---------+-----------+-----------+---------+-----------------+---------+-----------+
| 100 | 100 * 1 | 100 * 1.0 | 100 / 1.0 | 100 / 2 | 100 + 2 * 5 / 2 | 100 / 3 | 100 DIV 0 |
+-----+---------+-----------+-----------+---------+-----------------+---------+-----------+
| 100 |     100 |     100.0 |  100.0000 | 50.0000 |        105.0000 | 33.3333 |      NULL |
+-----+---------+-----------+-----------+---------+-----------------+---------+-----------+
1 row in set (0.00 sec)
```

### `/` 和 `DIV` 一眼看懂

| 写法 | 含义 | 示例结果 | 适合什么场景 |
| --- | --- | --- | --- |
| `100 / 3` | 普通除法 | `33.3333...` | 需要真实除法结果 |
| `100 DIV 3` | 整数除法 | `33` | 只关心整数商 |

这里你不要死记输出一定显示几位小数，真正要记的是：

- `/` 的核心语义是“做正常除法”。
- `DIV` 的核心语义是“只取整数商”。

### 为什么 `100 / 2` 也可能显示成 `50.0000`？

因为你看到的是 MySQL 对结果的展示形式。这里最重要的不是显示了几个 `0`，而是要知道：**`/` 不是整数除法。**

### 除以 `0` 的特殊情况

```sql
SELECT 100 / 0, 100 DIV 0 FROM DUAL;
```

无论是普通除法还是整数除法，只要除数是 `0`，结果都不是正常数值，而是 `NULL`。

### 业务示例：计算年基本工资

```sql
SELECT
    employee_id,
    salary,
    salary * 12 AS annual_sal
FROM employees;
```

这个写法在业务里很常见：把表中已有字段继续做数学运算，得到派生字段。

### 从结果中你应该记住什么

- 一个数乘以整数 `1`，结果还是原数。
- 一个数乘以浮点数 `1.0`，结果通常会转成浮点数形式。
- 一个数除以另一个数时，如果不能整除，通常会得到带小数的结果。
- `*` 和 `/` 的优先级高于 `+`、`-`；表达式稍微复杂时，建议直接加括号。

### 回查提示

如果你预期得到整数，却看到了小数，先检查自己用的是 `/` 还是 `DIV`。

## 1.4 求模（求余）运算符

求模，也叫求余，表示“一个数除以另一个数之后，余下多少”。

MySQL 中有两种常见写法：

- `%`
- `MOD`

它们的作用相同，只是写法不同。

### 先记结论

- `12 % 5` 和 `12 MOD 5` 的含义完全一致。
- 求模最常见的用途，不是单纯算余数，而是 **做条件判断**。
- 只要题目里出现“能否整除”“奇偶数”“每隔几个取一次”，基本都可以先想到求模。

### 基本示例

```sql
mysql> SELECT 12 % 3, 12 MOD 5 FROM dual;
+--------+----------+
| 12 % 3 | 12 MOD 5 |
+--------+----------+
|      0 |        2 |
+--------+----------+
1 row in set (0.00 sec)
```

这表示：

- `12 % 3 = 0`，说明 `12` 能被 `3` 整除。
- `12 MOD 5 = 2`，说明 `12` 除以 `5` 的余数是 `2`。

### 常见业务写法

#### 1）筛选偶数编号

```sql
SELECT
    *
FROM employees
WHERE employee_id MOD 2 = 0;
```

这里 `employee_id MOD 2 = 0` 的意思是：员工编号除以 `2` 的余数是 `0`，也就是偶数。

#### 2）筛选奇数编号

```sql
SELECT
    *
FROM employees
WHERE employee_id % 2 = 1;
```

#### 3）判断是否能整除

```sql
SELECT
    *
FROM some_table
WHERE amount % 5 = 0;
```

这个条件表示：只保留能够被 `5` 整除的数据。

### 风格建议

`%` 和 `MOD` 都可以使用，但在同一个项目或同一篇笔记里，最好尽量保持一致，阅读时会更顺。

### 回查提示

只要需求里出现“奇数、偶数、每隔几个取一次、能不能整除”，优先想到 `%` 或 `MOD`。

## 常见混淆点

- MySQL 中 `+` 不是字符串拼接运算符；拼接字符串时应使用 `CONCAT()`。
- 字符串参与 `+` 运算时，MySQL 会先尝试做数值转换，而不是直接报“类型不匹配”。
- 整数除以整数时，使用 `/` 也不代表你一定会得到整数商；`/` 和 `DIV` 的语义不同。
- 除以 `0` 时，MySQL 返回的是 `NULL`，这表示结果无效，不是一个正常数值。
- `%` 和 `MOD` 作用相同，真正要记的是“什么时候该用求余”，而不是只背符号。
- 表达式稍微复杂时，不要只靠记忆优先级，直接加括号最稳。

## 一分钟自测

先不要往下看答案，自己判断下面几条语句会发生什么：

1. `SELECT '10' + 2 FROM DUAL;`
2. `SELECT 'ab' + 2 FROM DUAL;`
3. `SELECT 10 / 4, 10 DIV 4 FROM DUAL;`
4. `SELECT 10 % 4, 10 MOD 4 FROM DUAL;`
5. `SELECT CONCAT('A', 'B') FROM DUAL;`

### 参考答案

1. `'10'` 会先转成数值，再和 `2` 相加。
2. `'ab'` 不能正常转成数值，不会变成字符串拼接；MySQL 会按数值转换规则处理。
3. `10 / 4` 关注真实除法结果，`10 DIV 4` 只取整数商。
4. `%` 和 `MOD` 等价，结果相同。
5. `CONCAT()` 才是字符串拼接的标准写法。

## 一句话抓核心

算术运算符这一节真正要掌握的，不只是 `+`、`-`、`*`、`/`、`%` 会算，而是：**在 MySQL 里，`+` 只做数值运算，`/` 和 `DIV` 语义不同，`%` / `MOD` 常用于条件判断，除以 `0` 会得到 `NULL`。**

## 小结

这一节你需要记住：

- `+`、`-`、`*`、`/`、`DIV`、`%`、`MOD` 是最常见的算术运算写法。
- MySQL 中 `+` 只表示数值相加；字符串拼接使用 `CONCAT()`。
- 整数与浮点数混合运算时，结果通常会转成浮点数。
- `/` 表示普通除法，`DIV` 表示整数除法，两者不能混为一谈。
- 一个数除以 `0` 时，结果为 `NULL`。
- `%` 和 `MOD` 都用于求余，常见于奇偶判断、整除判断、周期筛选等场景。
