# Study Notes Rewrite

當使用者提供的筆記，不只是結構差、缺導航、缺導讀，而是**內容本身已不足以支撐正確理解**時，使用這個 skill。

這個 skill 的核心不是做簡單整理，也不是單純加模板。  
它的任務是：**修復內容品質，必要時重寫段落，重建更可理解、可複習、可回查、可教學的版本。**

---

# 這個 skill 的定位

這是一個**內容重寫型 skill**，不是整理型 skill。

## 和整理型 skill 的差異

### 整理型 skill 的重點
- 補 README
- 補章節導航
- 補導讀
- 補關鍵字
- 補回查入口
- 保留正文為主
- 不做大量語意重寫

### 重寫型 skill 的重點
- 修復定義不清
- 修復邏輯斷裂
- 補前提、限制、例外
- 重寫模糊句子
- 重組段落順序
- 必要時重建整節內容
- 讓內容本身變得更能支撐理解與教學

若問題主要是難找、難導航、難回查，應優先使用整理型 skill。  
若問題主要是內容本身難以理解、邏輯不足、表述失真，才使用本 skill。

---

# 核心目標

不要只讓內容變整齊。  
不要只讓內容變短。  
不要只把原文換句話說。  
不要為了好懂而犧牲精度。  
不要為了流暢而刪掉前提、限制、例外與推導。

重寫的目標是：

- 修復內容品質不足的部分
- 讓定義、邏輯、比較、例子與條件更完整
- 讓讀者能更穩定地正確理解
- 讓筆記更適合學習、複習、回查與應用
- 在必要重寫下，盡量保留原意與知識範圍

---

# 何時使用這個 skill

當筆記符合以下任一情況，且已影響理解、判斷、複習或應用時，可使用本 skill：

- 關鍵術語未定義，或定義前後不一致
- 結論缺乏必要前提、原因或推導
- 遺漏限制條件、適用範圍或例外情況
- 容易混淆的概念未被區分
- 段落之間邏輯跳躍嚴重
- 例子不足，導致抽象概念難以落地
- 表述模糊，無法支撐準確理解
- 同一篇中存在前後矛盾
- 原文雖有內容，但已無法作為穩定的教學材料或複習材料

---

# 何時不應使用這個 skill

以下情況通常不應直接使用本 skill，而應優先使用整理型 skill：

- 內容大致正確，只是缺導航
- 結構混亂，但正文仍可理解
- 只是單篇過長，不代表內容品質差
- 只是缺關鍵字、導讀、內部連結
- 只是閱讀順序不明
- 只是倉庫層級設計不佳

若問題主要屬於以上範圍，重寫會過度介入內容本體。

---

# 什麼叫做「重寫」

本 skill 所說的「重寫」，不是只做以下事情：

- 換同義詞
- 改寫口吻
- 縮成摘要
- 把段落拆成條列
- 套模板後重新排版

本 skill 所說的「重寫」，是指對內容本體做受控改造，包括：

- 重寫定義，使邊界更清楚
- 重寫說明，使因果與推導更完整
- 重寫比較，使概念差異更明確
- 重寫例子，使抽象內容更易落地
- 重寫段落順序，使閱讀路徑更合理
- 必要時重建一節，使其能支撐穩定理解

---

# 成功標準

重寫完成後，內容應盡量達成以下效果：

1. 讀者能清楚知道本篇主要在講什麼
2. 讀者能分辨核心概念與相近概念
3. 讀者能看出結論成立所需的前提與條件
4. 讀者能理解主要段落之間的邏輯關係
5. 讀者不會因語句模糊而誤解關鍵內容
6. 讀者可透過例子或比較建立穩定理解
7. 重寫後的重要資訊不比原文更少
8. 定義、限制、例外、推導與比較沒有被不當刪除
9. 若原文可保留的部分仍有價值，重寫後有保留其知識意圖
10. 內容比原文更適合學習、複習、回查與應用

---

# 重寫的核心原則

## 1. 修內容，不只是修表面

這個 skill 的任務是修復內容品質，而不是只優化格式或版面。  
若問題出在內容本身，就不能只補導讀或補標題了事。

## 2. 重寫不等於過度精簡

重寫後可以更清楚，但不應只剩結論。  
若原文中的定義、前提、限制、例外、比較、推導會影響正確理解，應優先保留或重建。

## 3. 理解與精度必須同時成立

不應為了讓句子更順而犧牲技術精度。  
也不應為了保持精度而讓內容完全不可讀。  
本 skill 要求的是：**在可理解的前提下維持表述精度，在維持精度的前提下提升可理解性。**

## 4. 盡量保留原意，不盲目替換知識內容

若原文雖然難懂，但仍可辨識其知識意圖，應優先保留原意並重寫表達。  
不要因為覺得原文差，就任意改變其主張或結論。

## 5. 原文不足時可補，但補充必須受控

若原文缺少必要定義、前提、限制、例外、比較或例子，可補上這些內容。  
但補充必須服務於修復理解，不可無限制擴寫成另一篇新教材。

## 6. 保留可救部分，重寫不可救部分

若原文某些段落本身品質尚可，應保留並小修。  
若某些段落已經嚴重影響理解，才重寫該段。  
不要把所有內容一律推倒重來。

---

# 表述精度保護規格

## 不可在重寫中優先刪除的內容

若原文包含以下內容，且它們對正確理解有作用，則應保留或重建：

- 關鍵定義
- 前提條件
- 限制條件
- 適用範圍
- 例外情況
- 易混淆比較
- 因果關係
- 推導過程
- 關鍵例子
- 對結論成立有影響的補充說明

## 不可只保留結論

若原文或重寫後只剩結論，而讀者無法理解：
- 為什麼成立
- 在什麼條件下成立
- 何時不成立
- 它與相近概念差在哪

則視為重寫失敗。

## 不可把精度問題誤當冗餘

有些內容看似重複，實際上承擔不同功能，例如：

- 一段在下定義
- 一段在講原因
- 一段在舉例
- 一段在比較
- 一段在提醒誤區

這些內容即使語意接近，也不應機械刪除。

---

# 可接受的重寫層級

本 skill 允許分級重寫，不必每次都全文重建。

## Level 1：局部修句
適用情況：
- 原文大致可用
- 問題主要在句子模糊、順序不順、指涉不清

可做：
- 重寫句子
- 明確主詞、條件與結論
- 消除歧義
- 調整局部段落語序

## Level 2：局部內容修補
適用情況：
- 缺少定義
- 缺少比較
- 缺少例子
- 缺少前提或限制

可做：
- 補定義
- 補前提
- 補限制
- 補例子
- 補比較
- 補過渡說明

## Level 3：段落重組
適用情況：
- 段落之間邏輯混亂
- 順序不利於理解
- 先講結論後講基礎，造成閱讀障礙

可做：
- 重新排序
- 拆段
- 合併重複段
- 重建段落之間的銜接

## Level 4：節級重寫
適用情況：
- 某節內容已無法作為教學材料
- 定義混亂、前提缺失、邏輯斷裂嚴重
- 即使修句也無法救回理解性

可做：
- 保留知識意圖
- 以新的教學結構重寫整節
- 重建定義、比較、推導與例子
- 視需要重新命名小節

## Level 5：整篇重建
適用情況：
- 原文整體品質極差
- 前後矛盾大量存在
- 主題混雜無法僅靠局部修復完成

可做：
- 重新規劃章節
- 重新安排知識順序
- 重寫主要內容
- 將可保留資訊融入新版本

注意：只有在原文已明顯無法支撐正確理解時，才應使用這一層級。

---

# 重寫時的判斷問題

在開始重寫前，應先判斷以下問題：

1. 這篇的問題主要是結構問題，還是內容問題？
2. 哪些段落仍可保留？
3. 哪些段落只是難讀，哪些段落已經會誤導？
4. 哪些定義、條件、例外是不能丟的？
5. 哪些部分需要補例子？
6. 哪些地方需要比較？
7. 需要的只是局部修補，還是必須整段重建？

不要一看到內容差就直接全文重寫。  
先分辨問題類型與重寫層級。

---

# 工作流程

1. 讀完整篇，判斷問題是結構型還是內容型
2. 標記內容品質不足的區塊
3. 區分可保留段落與需重寫段落
4. 抽出本篇的核心知識意圖
5. 列出不可遺失的定義、條件、限制、例外與比較
6. 決定重寫層級
7. 先修核心理解斷點，再修局部表述
8. 視需要補例子、比較與過渡說明
9. 檢查重寫後是否失真、失焦或過度精簡
10. 最後才補輔助性結構，如導讀、關鍵字或回查入口

---

# 重寫輸出的理想結構

重寫後的筆記，建議盡量包含以下元素：

1. 清楚標題
2. 本節導讀
3. 核心定義
4. 概念說明
5. 條件 / 前提 / 限制
6. 易混淆比較
7. 例子
8. 小結或一句話抓核心
9. 必要的延伸閱讀或前後文關聯

不是每篇都要完全一樣，但應讓讀者更容易完成以下任務：

- 第一次學習
- 複習喚醒記憶
- 快速回查
- 應用判斷

---

# 補充內容的原則

若原文不足以支撐理解，可以補內容。  
但補充時應遵守以下原則：

## 1. 補足必要，不無限擴寫
補充應以修復理解為目的，而不是把原筆記擴成整本書。

## 2. 補的是支撐理解的關鍵內容
優先補：
- 定義
- 條件
- 例外
- 比較
- 例子
- 過渡說明

## 3. 補充後仍應保持主題集中
不要因為能補，就把內容拉離原本主題。

---

# 當原文存在錯誤或矛盾時

若原文本身存在明顯錯誤、前後矛盾或定義衝突，應優先處理這些問題。

處理原則：

- 若可判定其原意，應在保留知識意圖下修復
- 若無法判定原意，應避免硬性編造完整版本
- 若同一篇出現兩種互斥說法，應消除矛盾並保留較完整、較一致的版本
- 若無法安全確定哪個版本正確，應保守重寫，避免做出未受支持的斷言

---

# 不應做的事

- 不把重寫等同於摘要化
- 不把所有段落都壓成條列
- 不為了看起來乾淨而刪掉必要前提
- 不為了流暢而移除限制條件與例外
- 不因為原文差，就完全忽略原本可保留的知識意圖
- 不無限制擴寫，導致偏離原主題
- 不把補充內容寫成另一套不相干的教材
- 不讓新版本比原文更模糊
- 不讓新版本比原文更武斷
- 不讓新版本失去回查與複習價值

---

# 驗收清單

重寫完成後，檢查以下問題：

## 理解
- 讀者是否能快速知道這篇在講什麼？
- 是否能分辨核心概念與相近概念？
- 是否能理解主要結論成立的原因？

## 精度
- 定義是否清楚？
- 前提、限制、例外是否仍在？
- 推導或因果關係是否完整？
- 是否有因簡化而失真？

## 教學性
- 是否比原文更適合第一次學？
- 是否比原文更適合複習與回查？
- 是否有必要的例子或比較支撐理解？

## 保真
- 是否仍保留原文的知識意圖？
- 是否只在必要範圍內重寫？
- 是否避免無根據擴寫？

## 完整性
- 原文中的重要訊息是否仍存在？
- 新版本是否有新的斷裂、跳躍或模糊處？

---

# 最終原則

這個 skill 的目的，不是把筆記寫得漂亮。  
不是把內容壓短。  
不是把原文全面翻新成另一種文風。

它的目的，是在原始筆記已不足以支撐正確理解時，  
透過受控重寫，修復內容品質，重建更穩定、更完整、更可教學的版本。

若原文還能救，就保留可救部分。  
若原文已失去教學價值，就重建必要部分。  
若重寫會傷害精度，就寧可多保留必要說明。  
重寫的成功，不在於看起來多簡潔，而在於是否更能支持正確理解。
